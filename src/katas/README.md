# Cómo Utilizar las Katas de RaiSE

## La Documentación Precede al Código... y el Plan Precede a la Acción

Las Katas en este directorio son plantillas de procesos estandarizados diseñadas para guiar tareas complejas de ingeniería de software. Cada Kata describe el "qué" y el "porqué" de un proceso, pero no el "cómo" específico para cada contexto de proyecto.

### No Ejecutes una Kata Directamente

Intentar seguir los pasos de una Kata (como la `L2-02-Analisis-Agnostico-Codigo-Fuente.md`) sin una planificación previa puede llevar a resultados limitados o incompletos. El contexto de cada repositorio, historia de usuario o tarea es único.

### El Proceso Correcto: Planificar Antes de Ejecutar

Para asegurar que la aplicación de una Kata sea exitosa y genere el máximo valor, siempre debes crear un **Plan de Implementación** específico para la tarea en cuestión.

El proceso recomendado es el siguiente:

1.  **Identifica la Kata de Proceso**: Selecciona la Kata que describe el proceso que necesitas realizar (por ejemplo, `L2-02` para análisis de código).
2.  **Crea un Plan de Implementación**: Utiliza la meta-kata **`L1-04-generacion-plan-implementacion-hu.md`** como guía para crear un plan de acción detallado. Este plan adaptará los pasos genéricos de la Kata de proceso al contexto específico de tu tarea (ej. "Plan de Implementación para analizar el servicio `templateService.js` usando L2-02").
3.  **Ejecuta el Plan**: Sigue los pasos detallados en tu plan de implementación, usando la Kata de proceso original como referencia conceptual.

Este enfoque asegura que consideres las particularidades de tu proyecto, asignes tareas concretas y definas criterios de aceptación claros antes de comenzar el trabajo.
