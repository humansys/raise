# AI Rules Reasoning for Zambezi Concierge

This document outlines the reasoning behind the AI-powered coding standards and rules established for the `zambezi-concierge` repository. The goal is to ensure consistency, maintainability, and quality in the code generated by AI assistants.

## Initial Repository Analysis

The `zambezi-concierge` repository is a multi-component, polyglot system. An initial analysis has identified the following key architectural components and technologies:

### 1. Admin Dashboard (`admin/`)

*   **Technology**: PHP 8.2+ / Laravel 12+
*   **Key Frameworks/Libraries**:
    *   [Filament](https://filamentphp.com/): For rapid admin panel development.
    *   [Livewire](https://laravel-livewire.com/): Used by Filament for dynamic interfaces.
    *   [Tailwind CSS](https://tailwindcss.com/): For styling.
    *   [Vite](https://vitejs.dev/): For frontend asset bundling.
*   **Purpose**: Provides administrative functionalities for the Zambezi Concierge system.
*   **Coding Standards**: Should adhere to PSR-12 and general Laravel best practices. Given the use of `laravel/pint`, this is already enforced.

### 2. Python Backend (`backend/`)

*   **Technology**: Python
*   **Key Frameworks/Libraries**:
    *   [FastAPI](https://fastapi.tiangolo.com/): A modern, high-performance web framework for building APIs.
    *   [SQLAlchemy](https://www.sqlalchemy.org/): For ORM and database interaction.
    *   [Alembic](https://alembic.sqlalchemy.org/): For database migrations.
    *   [PostgreSQL](https://www.postgresql.org/): The primary database, with the `psycopg2-binary` driver.
    *   [pgvector](https://github.com/pgvector/pgvector): Suggests use of vector embeddings for AI/ML features.
    *   [OpenAI SDK](https://github.com/openai/openai-python): Confirms integration with OpenAI services.
*   **Purpose**: Likely serves as the core API for the application, handling business logic and AI-powered features.
*   **Coding Standards**: Should follow PEP 8 standards.

### 3. Svelte Frontend (`frontend/`)

*   **Technology**: JavaScript / Svelte 5+
*   **Key Frameworks/Libraries**:
    *   [Svelte](https://svelte.dev/): A component framework for building reactive user interfaces.
    *   [Vite](https://vitejs.dev/): As the build and development tool.
    *   [Tailwind CSS](https://tailwindcss.com/): For styling.
*   **Purpose**: Appears to be a user-facing component, possibly a web widget, given its name in `package.json`.

## Strategy for Rules

Based on this analysis, the strategy for creating Cursor rules will be to establish a set of foundational rules and then create specific rules for each component of the application.

1.  **Global Rules**: General coding standards that apply to all parts of the repository.
2.  **Laravel/Filament Rules**: Specific guidelines for the `admin` dashboard.
3.  **Python/FastAPI Rules**: Best practices for the `backend` API.
4.  **Svelte Rules**: Conventions for the `frontend` Svelte application.

This approach will ensure that the AI assistant has the right context for each part of the codebase it is working on.
