# RaiSE Business Model
## Modelo de Negocio

**Versión:** 1.0.0  
**Fecha:** 27 de Diciembre, 2025  
**Propósito:** Documentar cómo RaiSE genera y captura valor.

---

## Modelo: Open Core

RaiSE sigue el modelo **Open Core**:
- **Core open source (MIT):** CLI, reglas base, templates, katas
- **Valor añadido comercial:** Features enterprise, soporte, SLAs

### Rationale
- Construir comunidad y confianza
- Reducir fricción de adopción
- Diferenciación via features enterprise
- Contribuciones community mejoran el core

---

## Tiers y Pricing

| Tier | Precio | Target | Features |
|------|--------|--------|----------|
| **Community** | Gratis | Developers individuales, startups | CLI completo, reglas/katas base, templates, community support |
| **Pro** | $29/user/mes | Equipos pequeños (5-20 devs) | + Katas avanzadas, + Analytics básico, + Priority issues |
| **Enterprise** | Custom | Empresas (50+ devs) | + SSO/SAML, + Audit logging, + SLA, + On-premise, + Custom rules |

### Detalles por Tier

#### Community (Free)
- raise-kit CLI completo
- Reglas y katas base
- Templates core
- Documentación pública
- GitHub Discussions support
- Self-hosted raise-config

#### Pro ($29/user/mes)
- Todo de Community +
- Katas L3 avanzadas
- Dashboard de analytics básico
- Priority en issues
- Actualizaciones early access
- Email support (48h response)

#### Enterprise (Custom)
- Todo de Pro +
- SSO/SAML integration
- Audit logging avanzado
- SLA 99.9% (para componentes hosted)
- On-premise deployment guide
- Custom rule development
- Dedicated support (24h response)
- Training y onboarding
