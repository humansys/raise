# Glosario de Terminología RaiSE

Este documento define la terminología estándar utilizada en los procesos y artefactos de RaiSE, especialmente en la fase de preventa.

---

## 1. Definiciones de Términos Estándar

*   **Capacidad (Capability):** [Definición de Capacidad - e.g., Una habilidad de alto nivel que la organización o el sistema debe poseer.]
*   **Funcionalidad (Feature):** [Definición de Funcionalidad - e.g., Una agrupación lógica de requisitos que proporciona valor a un stakeholder.]
*   **Historia de Usuario (User Story):** [Definición de Historia de Usuario - e.g., Una descripción corta y simple de una funcionalidad contada desde la perspectiva de la persona que desea la nueva capacidad, usualmente un usuario o cliente del sistema.]
*   **Requisito (Requirement):**
    *   **Requisito Funcional (Functional Requirement):** [Definición - e.g., Describe lo que el sistema debe hacer.]
    *   **Requisito No Funcional (Non-Functional Requirement):** [Definición - e.g., Describe cómo el sistema debe realizar una función (rendimiento, seguridad, etc.).]
*   **Stakeholder:** [Definición - e.g., Individuo, grupo u organización que puede afectar, ser afectado por, o percibirse a sí mismo como afectado por una decisión, actividad o resultado de un proyecto.]
*   **Objetivo (Goal/Objective):** [Definición - e.g., Un resultado deseado o estado final que se busca alcanzar.]
*   **Métrica (Metric):** [Definición - e.g., Una medida cuantificable utilizada para rastrear y evaluar el estado de un proceso, producto o proyecto específico.]
*   **Visión de la Solución (Solution Vision):** [Definición - e.g., Describe el futuro estado deseado del producto o sistema.]
*   **Diseño Técnico (Technical Design):** [Definición - e.g., Especifica los detalles técnicos de cómo se implementará la solución.]
*   **Declaración de Trabajo (Statement of Work - SoW):** [Definición - e.g., Documento contractual que detalla el trabajo a realizar, entregables, cronograma y costos.]
*   **Hoja de Ruta / Estimación (Roadmap / Estimation):** [Definición - e.g., Plan visual que muestra la secuencia y tiempo estimados para la entrega de funcionalidades o hitos.]
*   **(Añadir más términos según sea necesario)**

## 3. Jerarquías Comunes

*   **Jerarquía Organizacional/Proyecto:** Stakeholder > Usuario Final (End User) > Persona (si se usa)
*   **Jerarquía de Trabajo (Ejemplo Agile):** Capacidad (Capability) > Funcionalidad (Feature/Epic) > Historia (User Story) > Tarea (Task)
    *   *Nota: Adaptar según la metodología específica del proyecto si difiere.*
*   **Jerarquía de Artefactos (Ejemplo Preventa RaiSE):** Requisitos (PRD) -> Visión -> Priorización -> Diseño Técnico -> Estimación/Roadmap -> SoW

---

## 4. Formato de Referencia a Principios RaiSE

Para referenciar un principio RaiSE dentro de los documentos, usar el siguiente formato:

`[RaiSE: Nombre-Del-Principio]`

**Ejemplos:**

*   `[RaiSE: Human-Centric]`
*   `[RaiSE: Principle-Driven]`
*   `[RaiSE: Verifiable-Results]`
*   `[RaiSE: Explicability]`
*   `[RaiSE: Context-Management]`
*   `[RaiSE: Clarity-Seeking]`
*   `[RaiSE: Continuous-Improvement]`
