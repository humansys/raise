# Guía de Estructura de Monorepo Python  

> **Propósito**: Servir como referencia única para desarrolladores humanos y agentes de IA que generan o modifican código dentro del monorepo. Define convenciones de carpetas, límites de escritura y buenas prácticas relacionadas con Pydantic AI y LlamaIndex.

---

## 1  Principios Rectores  
| Acrónimo | Significado | Aplicación en el repo |
|---|---|---|
| **KISS** | *Keep It Simple, Stupid* | No crear sub‑nombres si no es indispensable. |
| **DRY** | *Don’t Repeat Yourself* | Código y modelos compartidos → `libs/`. |
| **YAGNI** | *You Aren’t Gonna Need It* | Evitar scaffolding prematuro de micro‑servicios. |
| **SRC layout** | Carpeta `src/` por paquete | Impide imports accidentales. |
| **Shared‑Nothing por default** | Cada servicio = paquete aislado | Dependencias y versiones por paquete. |

---

## 2  Árbol de Directorios (canon)

```text
my-monorepo/
├─ README.md
├─ pyproject.toml        # tooling global (pre-commit, ruff, poetic‑versioning)
├─ pants.toml            # o bazel WORKSPACE / hatch.toml
├─ .github/workflows/    # CI matricial
├─ docs/                 # Sphinx/MkDocs, ADRs
├─ scripts/              # CLIs, utils DevEx
├─ infra/                # IaC (Docker, Terraform, k8s)
├─ data/                 # fuentes crudas (NO código)
├─ indexes/              # artefactos LlamaIndex persistidos
│
├─ libs/                 # paquetes compartidos
│   ├─ domain_models/
│   │   ├─ pyproject.toml
│   │   └─ src/myorg/domain_models/
│   └─ rag_utils/
│       └─ src/myorg/rag_utils/
│
└─ packages/             # aplicaciones/servicios
    ├─ service_a/
    │   ├─ pyproject.toml
    │   ├─ src/service_a/
    │   │   ├─ api/
    │   │   ├─ models/
    │   │   └─ retrieval/
    │   └─ tests/
    └─ service_b/
        └─ …
```

### Convenciones específicas
* **`data/`** → solo datasets originales (CSV, PDFs, etc.). Nunca procesados.
* **`indexes/`** → directorios creados por `VectorStoreIndex` o `GraphStoreIndex`. Nombre = `<service>-<store-id>`.
* **`infra/`** → cada subcarpeta contiene un `README.md` con instrucciones para reproducir.
* **`scripts/`** → utilidades a ejecutar desde raíz, nombrar con verbo imperativo (`build_docs.py`).

## 3  Guía para Agentes de IA

1. **Escritura restringida**  
   | Área | Propósito | ¿Puede escribir AI? |
   |------|-----------|---------------------|
   | `packages/<svc>/src/` | Código fuente del servicio | **Sí** (principal) |
   | `packages/<svc>/tests/` | Pruebas unitarias/integración | **Sí** |
   | `libs/` | Utilidades compartidas | Solo con aprobación humana |
   | `docs/` | Documentación técnica | Sí (docstrings y *.md) |
   | `indexes/` | Archivos binarios | **No** (solo lectura) |
   | `infra/` | IaC y Docker | Solo plantillas o cambios aprobados |

2. **Reglas de generación**  
   * Incluir encabezado de archivo `# Generated by <agent-name> YYYY-MM-DD hh:mm`.
   * No crear carpetas fuera del árbol canon sin ticket de arquitectura.
   * Los modelos **Pydantic** se colocan en `models/` o en `libs/domain_models/`.
   * Pipelines **LlamaIndex** viven dentro de `retrieval/` y usan `indexes/<svc>` como `persist_dir`.
   * Actualizar/crear `pyproject.toml` local y correr `poetry lock` al añadir dependencias.

3. **Checks automáticos (CI)**  
   * `ruff --fix` → lint and format.  
   * `pytest` → cobertura mínima 80 %.  
   * `pants tailor --check` → valida grafo de dependencias.

## 4  Versionado y Dependencias
* **SemVer por paquete** (`1.2.0`). Tag en Git: `service_a/v1.2.0`.
* Lockfiles separados: `poetry.lock` dentro de cada paquete.
* Extras opcionales:  
  ```toml
  [project.optional-dependencies]
  rag = ["llama-index>=0.11", "chromadb"]
  ```

---

## 5  Flujo de Trabajo Sugerido
1. Crea rama `feat/<svc>/<ticket>`.
2. Agente genera código en `packages/<svc>/src/` + tests.
3. Push; CI ejecuta lint/tests sólo para ese paquete (Pants target roots).
4. Merge‑Request auto‑asigna reviewers de dominio.
5. Publicación automático a registro interno si `pyproject.toml` marca nueva versión.

---

## 6  Preguntas Frecuentes
* **¿Dónde viven los notebooks?** → Repositorio aparte o carpetas `research/` ignoradas en CI.  
* **¿Cómo compartir constantes de dominio?** → `libs/domain_models.constants`.
* **¿Puedo usar `__init__.py` vacío?** → Sí, pero preferimos namespace‑packages PEP 420.
