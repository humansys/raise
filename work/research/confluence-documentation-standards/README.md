## Question

What are the established best practices for structuring documentation in Confluence for software engineering teams, based on evidence from organizations with decades of documentation practice?

## Contents

- [Evidence Catalog](sources/evidence-catalog.md) — 22 sources rated by evidence level

## Key Finding

Four independent traditions (DITA, Diátaxis, NASA/INCOSE, ISO 15489) converge on the same principles: **type your documents**, **enforce lifecycle**, **assign ownership**, **use controlled vocabularies**. This is a solved problem — we should adopt, not invent.
