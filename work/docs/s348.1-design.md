# S348.1 Design: Documentation Audit

**Story:** S348.1 — Documentation audit
**Epic:** E348 — Documentation Refresh
**Complexity:** Simple (XS, no code changes)
**Date:** 2026-03-05

---

## Problem

RaiSE v2.2 documentation has grown organically. The Astro doc site covers approximately 40% of CLI commands (many under the deprecated `rai memory` namespace rather than current `rai graph`/`rai pattern`/`rai signal`), has zero developer/extension documentation, no `llms.txt`, and a placeholder `AGENTS.md`. Before writing or restructuring anything, we need a systematic inventory of what exists, what is missing, and what is stale — per audience and Diataxis type.

## Value

A prioritized gap matrix gives every subsequent story (S348.2-S348.6) a concrete, auditable input. Without it, documentation work is driven by guesswork rather than evidence. The matrix also serves as the acceptance artifact for the audit itself.

---

## Approach

### Step 1: Inventory all doc artifacts

Enumerate every documentation artifact in the repo:

- **Astro doc site** (`docs/src/content/docs/docs/`): 10 English pages (index, getting-started, 4 concepts, 2 guides, CLI reference, concepts index)
- **Root files:** README.md, CONTRIBUTING.md, CODE_OF_CONDUCT.md, CHANGELOG.md, LICENSE, AGENTS.md
- **Missing root files:** llms.txt, llms-full.txt
- **Agent context:** CLAUDE.md (project-level), .claude/CLAUDE.md (user-level), SKILL.md per skill
- **Spanish translations:** 10 es/ pages (mirror of en/)

### Step 2: Validate CLI reference completeness

Compare `rai --help` command tree against the CLI reference page (`docs/src/content/docs/docs/cli/index.mdx`).

**Actual CLI command groups (20):** info, init, adapter, artifact, backlog, base, docs, discover, doctor, gate, graph, mcp, memory (deprecated), pattern, profile, publish (deprecated), release, session, signal, skill

**Documented in CLI ref:** init, session (3), memory (12 — all deprecated aliases), discover (4), skill (4), profile (1), base (1), release (1) = ~26 subcommands

**NOT documented:** adapter (4 subcmds), artifact (1), backlog (10), docs (3), doctor (1+report), gate (2), graph (7), mcp (6), pattern (3), signal (3), plus missing: info, skill sync, skill set, release check, release publish = ~40+ subcommands

**Coverage:** ~26 of ~66 subcommands documented = **~39%**. Additionally, documented commands use deprecated `rai memory` namespace instead of current `rai graph`/`rai pattern`/`rai signal`.

### Step 3: Assess each artifact

For every doc page/file: accuracy for v2.2, audience served, Diataxis type, status.

### Step 4: Produce gap matrix

Rows = doc topics. Columns = status, audience, Diataxis type, priority, assigned story.

### Step 5: Summarize findings

Top-level stats and prioritized action list.
