# Story: rai-server Dev Documentation

> **Branch:** `story/standalone/rai-server-docs`
> **Base:** `dev`
> **Size:** M

---

## User Story

**As a** developer joining the RaiSE project (or returning after a while),
**I want** a single, reliable entry point that tells me how to run `rai-server` locally,
**So that** I can develop and test the API without reverse-engineering 5 separate files.

---

## Acceptance Criteria

```gherkin
Feature: rai-server local dev setup

  Scenario: Developer sets up rai-server for the first time
    Given the developer has cloned raise-commons and has Docker + uv installed
    When they follow the README in packages/rai-server/
    Then they can reach http://localhost:8000/docs within 10 minutes
    And they have a working API key to make authenticated requests
    And the graph, memory, and agent endpoints return 200

  Scenario: Developer understands the API contract
    Given the README exists
    When they look at the API reference section
    Then they find all endpoints with method, path, auth, request and response schemas
    And they can identify which schemas live in packages/rai-server/src/rai_server/schemas/

  Scenario: Developer knows the architecture
    Given the README exists
    When they read the architecture section
    Then they understand the three-package structure (rai-core / rai-cli / rai-server)
    And they know where migrations live and how to run them
    And they understand the auth model (API key per org, Bearer token)
```
