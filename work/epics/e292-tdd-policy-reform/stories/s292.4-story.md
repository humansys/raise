# S292.4 — Cleanup: session module (2.1x)

## User Story

**As** a developer maintaining the session module,
**I want** the test suite to contain only tests that assert observable behavior,
**so that** every test in the suite earns its presence by catching real bugs.

## Acceptance Criteria

**Scenario: Muda identified and removed**
- Given the session test suite has a ~2.1x test-to-source ratio
- When each test is evaluated against the 4 heuristics (constant assertions, mock-impl tests, magic counts, tautological tests)
- Then every deleted test fails at least one heuristic — deletion is justified by behavior, not by line count

**Scenario: Ratio is diagnostic, not a gate**
- Given the cleanup is complete
- When the ratio is measured
- Then it is recorded as an outcome — if ratio > 2.0x after honest heuristic application, the module's behavior justifies the higher ratio

**Scenario: Suite still green**
- Given tests are deleted
- When `pytest tests/` runs
- Then all remaining tests pass
