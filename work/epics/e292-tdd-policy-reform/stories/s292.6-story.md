# S292.6 — Cleanup: telemetry module

## User Story

**As a** developer working in the telemetry module,
**I want** the test suite to contain only tests that assert real behavior,
**so that** I can trust the tests catch real regressions without navigating muda.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Heuristic-based muda removal
  Given the telemetry test suite has 968 lines across 2 files
  When I apply the 4 standard heuristics + PAT-E-500 + PAT-E-503
  Then all deleted tests fail at least one heuristic
  And the full test suite remains green after deletion
  And the deletion is documented with rationale

Scenario: No over-deletion
  Given a test asserts real behavior (not covered by heuristics)
  When I review it
  Then I leave it untouched
```

## SbE Examples

- `assert result == result` → delete (constant assertion)
- `assert callable(MagicMock())` → delete (PAT-E-500)
- Test that only constructs a model and asserts field == passed value → delete (tautological constructor)
- Test that duplicates parametrized coverage individually → delete (parametrize duplicate)
- Protocol/schema test that calls a trivial fixture → delete (PAT-E-503, mock-impl)
- Test that verifies real writer/schema behavior → keep
