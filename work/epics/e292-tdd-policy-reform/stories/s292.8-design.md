# Design: S292.8 — Cleanup adapters module

## Complexity Assessment

Simple: 2 files, 11 deletions, no new logic, no architectural decisions.

## Problem & Value

`tests/adapters/test_models.py` has 4 tautological-constructor tests that assert
fields equal the values passed at construction — Pydantic guarantees this.

`tests/adapters/test_protocols.py` has 7 mock-impl tests that exercise stub
implementations hardcoded to return specific values. They test the stub, not the
protocol contract. The valuable tests in this file are the `isinstance` checks and
the `test_incomplete_fails_isinstance` tests — those verify the Protocol machinery.

## Approach

Pure deletion. No replacement tests needed.

- `test_models.py`: 4 tautological-constructor tests deleted. Meaningful tests kept:
  required-field validation (`pytest.raises`), round-trip (`model_validate`),
  `test_construct_with_required_fields` (asserts default `metadata == {}`),
  `test_is_str_subclass`, `test_inherits_base_model` (parametrized guardrail).
- `test_protocols.py`: 7 mock-impl tests deleted. Meaningful tests kept:
  `test_conforming_stub_passes_isinstance` (×4), `test_incomplete_fails_isinstance` (×2),
  `test_is_runtime_checkable` (parametrized).

**Files affected:**
- `tests/adapters/test_models.py` — delete 4 tests
- `tests/adapters/test_protocols.py` — delete 7 tests
