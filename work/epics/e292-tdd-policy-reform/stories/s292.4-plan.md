## Muda Catalog

### test_resolver.py (72 lines)

**Pattern: Exact duplicate test**
- `test_accepts_already_normalized_format` (lines 44-47): passes `"SES-177"` as flag, asserts `"SES-177"`.
  Identical to `test_returns_flag_when_provided` (lines 14-17). Same inputs, same assertion. No unique behavior.

### test_state.py (437 lines)

**Pattern: Tautological constructor tests**
- `test_current_work_valid` (lines 61-67): creates `CurrentWork(epic="E15", story="S15.7", ...)`,
  asserts `work.epic == "E15"` and `work.story == "S15.7"`. Tests dataclass field assignment.
- `test_session_state_full` (lines 109-115): creates full state via `_make_session_state()` fixture,
  asserts specific values from that fixture. Tests the fixture, not the schema. Magic count:
  `assert len(state.pending.next_actions) == 1`.

### test_bundle.py (1377 lines)

**Pattern: Constant assertion ŌĆö `assert callable(mock_X)`**
`@patch` injects a `MagicMock`, which is always callable by construction. These assertions
can never fail. They appear in ~27 test methods (2 per method = ~54 lines):
- Every test in `TestAssembleContextBundle`, `TestFormatDeveloperSection`,
  `TestIdentityPrimesRemoved`, `TestBundleReleaseContext`, `TestFormatNarrative`,
  `TestFormatNextSessionPrompt`, `TestAssembleOrientation`, `TestAssembleSections`,
  `TestCountSectionItems` that uses `@patch`.

**Pattern: Tautological Pydantic constructor test ŌĆö `TestSectionManifest`**
- `test_manifest_creation`: creates `SectionManifest(name="governance", count=14, ...)`,
  asserts those exact values back. Tests the constructor.
- `test_manifest_zero_count`: asserts `m.count == 0` on a model created with `count=0`.

### test_close.py (901 lines)

**Clean.** Per-feature test pairs (with/without field) test real YAML+Pydantic pipeline
behavior. No muda identified.
