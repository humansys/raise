# Design: S292.9 — Cleanup context/test_models.py

## Complexity Assessment

Simple: 1 file, 7 deletions, no new logic.

Constraint: file has a pre-existing ImportError at collection time (RAISE-294 —
`ConceptEdge` not in main repo's `rai_core`). Verification uses `ast.parse` for
syntax + confirms no new collection errors introduced.

## Problem & Value

`tests/context/test_models.py` contains 7 tests that construct a `GraphNode` or
`GraphEdge` with known values and assert those same values back. Pydantic
guarantees field assignment — these tests assert nothing about behavior.

## Approach

Pure deletion. No replacement tests.

**Files affected:**
- `tests/context/test_models.py` — delete 7 tests

## Tests Targeted (tautological-constructor)

| Class | Test | Why muda |
|-------|------|----------|
| `TestConceptNode` | `test_create_pattern_node` | 5 field assertions — all echo passed-in values |
| `TestConceptNode` | `test_create_module_node` | Asserts `node.id`, `node.type`, `node.metadata["depends_on"]`, `node.metadata["components"]` — all passed in |
| `TestConceptEdge` | `test_create_learned_from_edge` | Asserts source, target, type, weight, metadata — all passed in |

| `TestConceptEdge` | `test_any_edge_type_accepted` | Passes `type="jira.blocks"`, asserts `== "jira.blocks"` |
