# S292.2 — Design

## Problem & Value

**Problem:** `/rai-story-implement` tells agents to do RED-GREEN-REFACTOR but gives no guidance on what makes a good test. Without anti-patterns, agents default to writing tests that satisfy coverage metrics rather than tests that catch bugs.

**Value:** After this story, every agent running `/rai-story-implement` gets explicit quality guidance inline where it matters — in the TDD cycle step itself.

## Approach

Two targeted additions to `SKILL.md` Step 2 and Quality Checklist. Mirror to `.agent/` copy.
Anti-patterns from S292.1's guardrails-stack.md §4.7 — use identical wording for coherence.

**Components:**

| File | Change type | What |
|------|------------|------|
| `.claude/skills/rai-story-implement/SKILL.md` | Modify | Step 2 + Quality Checklist |
| `.agent/skills/rai-story-implement/SKILL.md` | Modify | Identical changes |

## Exact Changes

### 1. Step 2 — after "Follow project rules, guardrails, and established patterns."

**Add:**
```markdown
**Test quality heuristic:** Each test must assert observable behavior, not implementation details.

Anti-patterns to avoid (muda):
- Constant assertions: `assert "x" == "x"` — always passes, catches nothing
- Mock-implementation tests: only verify an internal call was made, not the outcome
- Magic-number counts: `assert len(items) == 21` — brittle, not behavioral
- Happy-path-only: missing boundary cases (empty input, one item, many items, error)
```

### 2. Quality Checklist — replace TDD cycle item

**Before:**
```
- [ ] TDD cycle followed for each task (RED → GREEN → REFACTOR)
```
**After:**
```
- [ ] TDD cycle followed for each task (RED → GREEN → REFACTOR) — each test asserts observable behavior
```
