# Design: S292.7 ‚Äî Cleanup schemas module

## Complexity Assessment

Simple: 1 file, 5 deletions, no new logic, no architectural decisions.

## Problem & Value

`tests/schemas/test_session_state.py` contains 5 tests that construct a Pydantic
model with known values and assert those same values back. Pydantic guarantees
field assignment ‚Äî these tests catch nothing, assert nothing, and cost tokens
every time an agent reads or modifies the file.

## Approach

Pure deletion. Remove the 5 tautological-constructor tests. No replacement tests
needed ‚Äî the behavior these "test" is already covered by Pydantic's own guarantees
and by the remaining tests (coercion, round-trip, backward-compat).

**Files affected:**
- `tests/schemas/test_session_state.py` ‚Äî delete 5 tests (no other changes)

## Tests Targeted

| Class | Test | Why muda |
|-------|------|----------|
| `TestCurrentWork` | `test_release_field_accepts_value` | Assigns `release="V3.0"`, asserts `== "V3.0"`. Pydantic does this. |
| `TestEpicProgress` | `test_create_progress` | Constructs with 5 fields, asserts all 5. No behavior. |
| `TestSessionState` | `test_with_progress` | Assigns a progress object, asserts `is not None`. Trivial. |
| `TestSessionState` | `test_with_completed_epics` | Passes list at construction, asserts same list back. |
| `TestSessionNarrative` | `test_narrative_accepts_value` | Assigns string, asserts substring in string. |
