# S494.5: Delete MCP Adapter and Migrate Entry Point — Design

## Complexity: Simple

3 components affected, no algorithmic complexity, no external integrations. Pure deletion + entry point rewiring.

## Problem

The codebase has two Jira adapters: `McpJiraAdapter` (MCP bridge) and `AcliJiraAdapter` (ACLI subprocess). The MCP adapter is superseded — keeping it adds confusion, dead code, and false dependency surface.

## Value

Single Jira integration path. No dead code. Simpler mental model for contributors.

## Approach

**Strategy:** Surgical deletion + entry point migration. Not just "comment out" — trace every reference and remove cleanly.

### Components Affected

| Component | Change | Path |
|-----------|--------|------|
| `McpJiraAdapter` | **DELETE** | `packages/raise-pro/src/rai_pro/adapters/mcp_jira.py` |
| MCP Jira tests | **DELETE** | `tests/adapters/test_mcp_jira.py` |
| Entry point `jira` | **MODIFY** | `packages/raise-pro/pyproject.toml` line 31 |
| `__init__.py` | **VERIFY** | `packages/raise-pro/src/rai_pro/adapters/__init__.py` (empty — no action) |

### What We Do NOT Touch

- `McpBridge` / `raise_cli.mcp.bridge` — still used by `mcp_confluence.py`, declarative adapter, `rai mcp` CLI commands
- `mcp_confluence.py` — different adapter, different protocol
- `acli_jira.py` / `acli_bridge.py` — already complete, no changes needed
- `raise_cli.adapters.mcp_bridge` — re-export shim used by other adapters
- Any `work/` docs mentioning McpJiraAdapter — historical artifacts, not code

### ACLI Binary Missing Error

Already handled in `acli_bridge.py:57-61` — `FileNotFoundError` → `AcliBridgeError` with install URL. No additional work needed.

## Examples

### Entry point before (pyproject.toml)
```toml
[project.entry-points."rai.adapters.pm"]
jira = "rai_pro.adapters.mcp_jira:McpJiraAdapter"
```

### Entry point after
```toml
[project.entry-points."rai.adapters.pm"]
jira = "rai_pro.adapters.acli_jira:AcliJiraAdapter"
```

### Runtime resolution (unchanged behavior)
```python
# raise_cli loads entry point "rai.adapters.pm" → "jira"
# Before: McpJiraAdapter (MCP bridge → mcp-atlassian → Jira API)
# After:  AcliJiraAdapter (subprocess → acli jira → Jira API)
```

### Error when acli missing (already works)
```
AcliBridgeError: ACLI binary 'acli' not found in PATH. Install it from https://developer.atlassian.com/cli
```

## Acceptance Criteria

### MUST
- AC1: `mcp_jira.py` module deleted (not emptied, deleted)
- AC2: `test_mcp_jira.py` test file deleted
- AC3: Entry point `jira` points to `AcliJiraAdapter` in `packages/raise-pro/pyproject.toml`
- AC4: Zero `McpJiraAdapter` references in Python source files (grep verification)
- AC5: All existing tests pass, type checks pass, linting passes

### MUST NOT
- Touch `McpBridge`, `mcp_confluence`, or `raise_cli.mcp` — those are alive
- Modify `acli_jira.py` or `acli_bridge.py` — they're complete
- Delete `work/` documentation that mentions McpJiraAdapter — it's historical

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | Delete files, don't empty them | Empty modules are confusing dead code. Git history preserves the old code |
| D2 | Don't add backward-compat re-exports | No users of McpJiraAdapter outside this repo. Clean break |
| D3 | No changes to ACLI adapter | Already complete and tested in S494.3-4. Scope discipline |
