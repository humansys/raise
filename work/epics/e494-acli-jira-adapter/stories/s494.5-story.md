# S494.5: Delete MCP Adapter and Migrate Entry Point

## User Story

**As a** RaiSE developer,
**I want** the MCP Jira adapter removed and the ACLI adapter registered as the sole `jira` entry point,
**So that** the codebase has a single, simpler Jira integration path without MCP subprocess complexity.

## Acceptance Criteria

```gherkin
Scenario: ACLI adapter is the sole jira entry point
  Given the raise-commons package is installed
  When the jira adapter is resolved via entry points
  Then AcliJiraAdapter is returned
  And McpJiraAdapter does not exist

Scenario: Clear error when ACLI binary is missing
  Given acli is not in PATH
  When a backlog command is executed
  Then an error message explains ACLI is required
  And includes guidance on installation

Scenario: No MCP Jira residue in codebase
  Given the migration is complete
  When searching for McpJiraAdapter references
  Then zero results are found
  And mcp-related Jira test files are removed
```
