# S494.6: Integration Test Suite for ACLI Adapter

## User Story

**As a** RaiSE developer,
**I want** on-demand integration tests that exercise the ACLI Jira adapter against real infrastructure,
**So that** I can validate the full round-trip before releases and catch contract mismatches early.

## Acceptance Criteria

```gherkin
Scenario: Integration tests skip when ACLI unavailable
  Given acli is not in PATH or jira.yaml is missing
  When pytest runs with -m integration
  Then all integration tests are skipped with clear reason

Scenario: Core CRUD operations work end-to-end
  Given ACLI is configured and authenticated
  When integration tests run
  Then search, get, create, transition, update, comment all succeed
  And responses parse into correct model types

Scenario: Multi-instance routing works
  Given jira.yaml has multiple instances configured
  When querying different projects
  Then each query routes to the correct Jira site

Scenario: Tests are isolated and idempotent
  Given a test run creates issues
  When the test suite finishes
  Then created issues are transitioned to Done
  And no orphaned test data remains
```
