# S494.6: Integration Test Suite — Retrospective

## Summary

- **Story:** S494.6 (E494)
- **Date:** 2026-03-20
- **Size:** S (estimated 45m, actual ~60m)
- **Variance:** 1.33x — 3 adapter bugs found and fixed during integration testing

## What Went Well

- Integration tests found 3 real bugs that 61 unit tests missed:
  1. `create_issue` parsed wrong response format (full issue vs envelope)
  2. `link_issues` used `--json` flag on unsupported command
  3. `batch_transition` with fake keys doesn't fail (ACLI behavior)
- Reusable pattern established: precondition skip → session-scoped adapter → test issue lifecycle
- RTEST sandbox project keeps test data isolated from real projects
- 15/15 integration tests passing against real Jira

## What to Improve

- S494.1 spike should have captured exact response format for ALL ACLI commands, not just the ones we looked at
- `uv sync --all-extras` vs `uv pip install --reinstall` confusion cost time in S494.5, carried over here

## Heutagogical Checkpoint

1. **Learned:** Mocks validate assumptions, not contracts. When mock data is hand-crafted, it reflects what you *think* the API returns.
2. **Process:** Capture real API output in spike phase, use as test fixtures. Don't invent response formats.
3. **Framework:** Integration test pattern should be a template for every new adapter.
4. **Growth:** Better at distinguishing "tested" from "validated" — unit tests test logic, integration tests validate contracts.

## Patterns

- PAT-E-042: Integration tests find what unit mocks hide
- PAT-E-043: ACLI command response format map (create→full issue, edit→envelope, link→no --json)

## Bugs Fixed

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| `create_issue` empty key | ACLI create returns full issue, adapter expected envelope | Parse `data["key"]` directly |
| `link_issues` fails | `workitem link create` doesn't support `--json` | `json_output=False` param in bridge |
| `batch_transition` no failure on fake keys | ACLI doesn't validate issue existence | Documented as actual behavior |
