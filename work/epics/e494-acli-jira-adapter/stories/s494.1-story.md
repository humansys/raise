# S494.1: Spike — ACLI JSON Mapping Validation

## User Story (Connextra)

As a **framework developer**,
I want to **validate that ACLI --json output maps to existing adapter models**,
so that **I can proceed with the adapter swap knowing the data contract is compatible**.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Search results map to IssueSummary
  Given ACLI returns JSON for a search query
  When I parse the response using nested Jira API format
  Then each issue maps to IssueSummary(key, summary, status, issue_type, parent_key)

Scenario: Single issue maps to IssueDetail
  Given ACLI returns JSON for a workitem view
  When I parse the response using nested format
  Then it maps to IssueDetail with all fields (assignee, priority, labels, dates)

Scenario: Comments map to Comment model
  Given ACLI returns JSON for comment list
  When I parse the response
  Then each comment maps to Comment(id, body, author, created)

Scenario: Create/edit responses map to IssueRef
  Given ACLI returns JSON for a create or edit operation
  When I parse the response
  Then it maps to IssueRef(key, url)
```
