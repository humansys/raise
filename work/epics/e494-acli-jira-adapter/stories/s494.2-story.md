# S494.2: Core Subprocess Wrapper with Telemetry

## User Story (Connextra)

As a **framework developer**,
I want a **thin subprocess wrapper around `acli jira` with JSON parsing and telemetry**,
so that **all adapter methods can delegate to a single, observable function**.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Successful ACLI command
  Given acli is in PATH and authenticated
  When I call _run_acli(["workitem", "search"], {"--jql": "project = RAI", "--limit": "1"})
  Then it returns a parsed dict from the JSON stdout
  And a logfire span is emitted with command, latency_ms, success=True

Scenario: ACLI binary not found
  Given acli is NOT in PATH
  When I instantiate AcliJiraBridge
  Then it raises AcliBridgeError with install instructions

Scenario: ACLI command fails
  Given acli returns non-zero exit code
  When I call _run_acli with invalid arguments
  Then it raises AcliBridgeError with stderr content
  And a logfire span is emitted with success=False

Scenario: Health check
  Given acli is authenticated
  When I call health()
  Then it returns AdapterHealth(healthy=True, latency_ms=N)
```

## Size

S — one module, ~50-80 lines, focused scope
