# S3.6 Design — Knowledge Query CLI

## Problem
The ScaleUp retrieval engine works but has no CLI interface. The agent operator cannot query domain knowledge during sessions without custom Python scripts.

## Value
Connects the knowledge pipeline to the operator's workflow. First step toward pluggable domain packs — the design must support multiple domains without rewriting.

## Approach

Add `rai knowledge query <domain> <query>` to the existing knowledge CLI. The command dynamically resolves adapter and builder from `domain.yaml`, builds the graph, runs retrieval, and formats output with prompting context.

### Components

| Component | Change | Detail |
|---|---|---|
| `knowledge/models.py` | modify | Add `RetrievalConfig`, `PromptingConfig` to `DomainManifest` |
| `knowledge/domain.py` | modify | Add `_resolve_adapter()`, `_resolve_builder()` dynamic import |
| `knowledge/cli.py` | modify | Add `query` command |
| `knowledge/formatter.py` | modify | Add query result formatters (human/compact/json) |
| `domain.yaml` (scaleup) | modify | Add `retrieval` and `prompting` sections |

### Data Flow

```
CLI args (domain, query, format, limit)
  → load_domain(domain_dir) → DomainManifest
  → _resolve_adapter(manifest) → DomainAdapter instance
  → _resolve_builder(manifest) → GraphBuilder instance
  → builder.build_from_directory(node_dir) → Graph
  → retrieve(graph, query, adapter, top_k) → RetrievalResult
  → format(result, manifest.prompting) → stdout
```

## Decisions

### D1: Adapter registration → SchemaRef pattern in domain.yaml
Same `module + class_name` pattern used for schema validation. Dynamic import via `importlib`.
Alternatives: entry point registry (too heavy), convention-by-name (too rigid).

### D2: Builder registration → SchemaRef pattern in domain.yaml
Each domain defines how to build its graph. Same mechanism as adapter and schema.
Alternatives: generic builder (loses typed nodes), optional with fallback (more code).

### D3: Prompting config → Implement now
`prompting.system_context` and `prompting.response_format` in domain.yaml, included in query output.
Alternatives: reserve field only (delays value), defer to future epic (loses momentum).

### D4: Output format → Replicate rai graph query pattern + prompting
`--format/-f` with human/compact/json. Prompting context as header section.
Consistent with existing `rai graph query` DX.
Alternatives: only human+json (loses compact for AI consumption).

## domain.yaml Target Schema

```yaml
name: scaleup
display_name: "Scaling Up (Verne Harnish)"
schema:
  module: rai_agent.scaleup.validation.models
  class_name: OntologyNode
corpus:
  - scaling_up_llamaparse.md
  - metodologias_parsed/
competency_questions: competency_questions.yaml
thresholds:
  cq_coverage: 80.0
required_types:
  - decision
  - concept
  - tool
  - worksheet
  - stage
  - metric
retrieval:
  adapter:
    module: rai_agent.scaleup.retrieval.adapter
    class_name: ScaleUpAdapter
  builder:
    module: rai_agent.scaleup.graph.builder
    class_name: ScaleUpGraphBuilder
prompting:
  system_context: |
    Eres experto en la metodología Scaling Up de Verne Harnish.
    Conecta siempre las respuestas a las 4 decisiones: People, Strategy, Execution, Cash.
    Referencia herramientas y worksheets específicos cuando sean relevantes.
  response_format: |
    1. Diagnóstico (qué decisión y etapa aplica)
    2. Herramientas relevantes (del framework)
    3. Próximo paso concreto
```

## Acceptance Criteria

### MUST
- AC1: `rai knowledge query scaleup "cash flow"` returns ranked nodes from 447 extracted
- AC2: Adapter and builder resolved dynamically from domain.yaml (no hardcoded imports)
- AC3: Prompting context included in output (human, compact, json formats)
- AC4: `--format` flag with human/compact/json (consistent with `rai graph query`)
- AC5: Integration test validates full chain: YAML → graph → retrieve → formatted output

### SHOULD
- AC6: Domain auto-detection when only one domain is registered
- AC7: `--limit` parameter to control top-K results

### MUST NOT
- AC8: No hardcoded ScaleUpAdapter or ScaleUpGraphBuilder in CLI code
- AC9: No LLM response generation — retrieval + prompting context only
