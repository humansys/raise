# S3.2 Scope: Extraction Pipeline

## In Scope

- **LLM extraction pipeline** — corpus → LLM → OntologyNode → validated YAML output
- **OntologyNode validation integration** — extracted nodes validate against the Pydantic schema from S3.1
- **Configurable extraction prompts** — template-based prompts that can be tuned per domain/decision area
- **Source references** — each extracted node traces back to its corpus location (chapter, section)
- **Reproducibility** — deterministic pipeline with same inputs produces structurally equivalent outputs
- **Coverage verification** — measure and report extraction coverage against known corpus concepts
- **S3.4 compatibility** — extracted nodes must be consumable by ScaleUpGraphBuilder

## Out of Scope

- Graph visualization (S3.5)
- Manual editing UI for extracted nodes
- Multi-corpus support (only scaling_up_llamaparse.md)
- HITL curation workflow (S3.3)
- Graph building/integration (S3.4 — already done)
- Retrieval engine (S3.5)
- Embedding/vector-based extraction — pipeline uses structured LLM extraction

## Done Criteria

- [ ] Extracted nodes validate against OntologyNode Pydantic schema
- [ ] Pipeline is reproducible (deterministic with same inputs and config)
- [ ] Coverage ≥80% of corpus concepts (methodologies + worksheets)
- [ ] Source references present on all extracted nodes
- [ ] Extraction prompts are configurable via templates
- [ ] Extracted nodes are compatible with ScaleUpGraphBuilder (S3.4)
- [ ] All tests pass, pyright strict, ruff clean
