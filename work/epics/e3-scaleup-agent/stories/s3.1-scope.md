# S3.1 Schema Validation — Scope

## In Scope
- Competency questions catalog (≥20 questions from Scaling Up methodology)
- Schema traversal check: each CQ maps to a valid path through ontology
- Corpus coverage check: major sections → node type mapping
- Schema consistency: valid references, no orphans, relationship types correct
- ValidationReport model (Pydantic) with structured results
- CLI-invocable validation (or test suite that serves as validation)

## Out of Scope
- Modifying Eduardo's schema (we report, he decides)
- Extraction pipeline (S3.2)
- Graph integration (S3.4)
- Parsing or re-processing the corpus (use existing markdown)

## Done Criteria
- [ ] ≥20 competency questions defined and checked against schema
- [ ] Corpus coverage ≥80% of major sections mapped to node types
- [ ] Zero invalid relationship references in schema
- [ ] Orphan nodes flagged
- [ ] ValidationReport as Pydantic model with structured output
- [ ] All tests passing (TDD)
- [ ] Type checks pass (pyright strict)
