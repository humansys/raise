### T2: /rai-knowledge-discover skill
**Size:** S | **Dependencies:** T1

**Create:** `.claude/skills/rai-knowledge-discover/SKILL.md`

**Steps in skill:**
1. Accept domain name as argument (default: "scaleup")
2. Resolve domain config from `.raise/knowledge/{domain}/domain.yaml`
3. Find corpus path from domain config
4. Call `discover_schema(corpus_path)` via Python
5. Display results: node types, fields, relationship types
6. Save SchemaSpec to `.raise/knowledge/{domain}/discovered_schema.json`
7. HITL: "Review the discovered schema. Proceed to extraction?"

**Verification:** Run the skill manually against ScaleUp domain.

---

### T3: /rai-knowledge-extract skill
**Size:** M | **Dependencies:** T1

**Create:** `.claude/skills/rai-knowledge-extract/SKILL.md`

**Steps in skill:**
1. Accept domain name as argument
2. Resolve domain config, find corpus
3. Create `CorpusChunker`, split by chapter
4. For each chapter:
   a. Call `_extract_chapter_nodes()` (via Python inline)
   b. Report progress: "Chapter {i}/{n}: {title} ŌĆö {count} nodes ({time}s)"
   c. Handle errors: log warning, continue to next chapter
5. Merge, deduplicate, reconcile
6. Write to `.raise/knowledge/{domain}/extracted/`
7. Display summary: total nodes, by type, by decision, warnings
8. HITL: "Review extraction. Run validation?"

**Key detail:** The skill runs Python code inline (not as a subprocess). This means it can report progress after each chapter naturally. The `CLAUDECODE` bypass is handled by T1.

**Verification:** Run against ScaleUp corpus.

### T5: /rai-knowledge-diff skill
**Size:** S | **Dependencies:** T2 (needs extracted nodes to diff)

**Create:** `.claude/skills/rai-knowledge-diff/SKILL.md`

5. Save reports to `.raise/knowledge/{domain}/`
