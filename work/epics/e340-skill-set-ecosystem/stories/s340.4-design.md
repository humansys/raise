# Design: S340.4 — `rai skill set` CLI Group

## Integration Points

| Component | File | Change |
|-----------|------|--------|
| `skill_set_app` | `src/rai_cli/cli/commands/skill_set.py` | **NEW** — Typer sub-app with create/list/diff |
| `skill_app` | `src/rai_cli/cli/commands/skill.py` | Add `skill_app.add_typer(skill_set_app)` |
| `skillsets` module | `src/rai_cli/skills/skillsets.py` | **NEW** — business logic for set operations |

## Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Separate file `skill_set.py` for CLI | SRP: skill.py is already 250 LOC |
| D2 | Separate `skillsets.py` for business logic | Testable without CLI, reusable by `/rai-skillset-manage` |
| D3 | `create` copies builtins by default, `--empty` for empty set | Most users want to customize builtins, not start from scratch |

## Commands

### `rai skill set create <name> [--empty]`
- Default: copy all builtins from installed package to `.raise/skills/<name>/`
- `--empty`: create directory only
- Uses `_copy_skill_tree(Path|Traversable)` from S340.1

### `rai skill set list`
- Scan `.raise/skills/*/` for directories containing at least one `*/SKILL.md`
- Show: name, skill count, which skills differ from builtins

### `rai skill set diff <name>`
- Compare each skill in `.raise/skills/<name>/` against builtin
- Show: added (not in builtins), modified (differs from builtin), unchanged
- Uses `compute_content_hash` from manifest module
