# Story: S340.1 — Skill Set Overlay Deployment

| Field | Value |
|-------|-------|
| Epic | RAISE-340 |
| Size | S |
| Branch | `story/s340.1/skill-set-overlay` |

## What

Add `--skill-set` flag to `rai init`. After the existing builtin deployment
to `.claude/skills/`, overlay skills from `.raise/skills/{set}/` on top.
Same-name wins (full replacement). Manifest tracks `origin: "project"` for
overlay skills.

## Why

Teams need to customize skills without forking builtins. The overlay model
preserves existing upgrade flow and adds team customization on top.

## Acceptance Criteria

- [ ] `rai init --skill-set my-set` copies overlay skills to `.claude/skills/`
- [ ] Overlay skill with same name as builtin replaces it
- [ ] Overlay skill with new name is added alongside builtins
- [ ] Manifest entries for overlay skills have `origin: "project"`
- [ ] `rai init` without `--skill-set` works exactly as before
- [ ] `_copy_skill_tree` accepts `Path | Traversable` (AR R1)
- [ ] Tests cover: overlay, same-name override, new skill addition, no-overlay
