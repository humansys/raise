# Design: S340.1 — Skill Set Overlay Deployment

## Integration Points

| Component | File | Change |
|-----------|------|--------|
| `scaffold_skills()` | `src/rai_cli/onboarding/skills.py` | Add `skill_set` param, overlay loop after builtin deploy |
| `_copy_skill_tree()` | `src/rai_cli/onboarding/skills.py` | Type `source_dir: Path \| Traversable` (AR R1) |
| `SkillManifest` | `src/rai_cli/onboarding/skill_manifest.py` | Add `skill_set: str \| None = None` field |
| `rai init` CLI | `src/rai_cli/cli/commands/init.py` | Add `--skill-set` option, pass to `scaffold_skills()` |

## Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | `skill_set: str | None = None` (not `str = "default"`) | None = no overlay, backward compat. "default" would imply a default/ directory. |
| D2 | Overlay copies with `overwrite=True` | Same-name-wins means overlay always replaces builtin. No three-hash for overlay. |
| D3 | Plugin transforms applied to overlay skills | Overlay skills need same IDE transforms as builtins when deploying to `.claude/skills/`. |

## Data Flow

```
scaffold_skills(skill_set="my-company")
  │
  ├── [existing code] Deploy builtins → .claude/skills/
  │   (three-hash, unchanged)
  │
  └── [NEW ~20 LOC] If skill_set is not None:
      overlay_dir = project_root / ".raise" / "skills" / skill_set
      if not overlay_dir.is_dir(): warn and skip
      for skill_dir in sorted(overlay_dir.iterdir()):
        if skill_dir.is_dir() and (skill_dir / "SKILL.md").exists():
          _copy_skill_tree(skill_dir, skills_dir / skill_dir.name, overwrite=True)
          manifest.skills[skill_dir.name] = SkillEntry(origin="project")
```
