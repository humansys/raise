# Story: S340.2 — Skill Creation Targets Sets

| Field | Value |
|-------|-------|
| Epic | RAISE-340 |
| Size | S |
| Branch | `story/s340.2/skill-create-sets` |

## What

Update `/rai-skill-create` and `rai skill scaffold` to generate skills
into `.raise/skills/{set}/` instead of `.claude/skills/`. Add "customize
builtin" mode that copies an installed builtin as starting point.

## Why

The skill creation UX must match the new skill set architecture. Skills
should be authored in `.raise/skills/{set}/` (source of truth) and
deployed to `.claude/skills/` via `rai init --skill-set`.

## Acceptance Criteria

- [ ] `/rai-skill-create` generates into `.raise/skills/{set}/` when `--set` provided
- [ ] `rai skill scaffold --set X` creates template in `.raise/skills/X/{name}/`
- [ ] "Customize builtin" copies from `.claude/skills/{name}/` as starting point
- [ ] Default behavior (no --set) unchanged: generates in `.claude/skills/`
