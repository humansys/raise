## Risks

| Risk | Mitigation |
|------|------------|
| `get_status_for_project` returns nested structure by issue type | Parse carefully; extract unique statuses across all issue types |
| `issue_createmeta_issuetypes` pagination needed | Handle start/limit params; most projects have <20 issue types |
| Workflow transitions require separate API call | Use `get_all_workflows(expand="transitions")` or fall back to per-issue `get_transitions` |
| Permissive mocks hide semantic bugs (PAT from S1130.1) | Assert mock call args; verify parameter types match API expectations |
