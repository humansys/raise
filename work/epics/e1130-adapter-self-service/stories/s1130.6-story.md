# S1130.6: /rai-adapter-setup Skill

## User Story

**As a** developer setting up a new RaiSE project,
**I want** to run `/rai-adapter-setup` and answer 3-4 questions,
**So that** I get validated YAML config for Jira and/or Confluence without looking up project keys, workflow states, or transition IDs.

## Acceptance Criteria

```gherkin
Scenario: Full setup from scratch
  Given no adapter config exists
  When the user runs /rai-adapter-setup
  Then Rai detects available backends (Jira, Confluence)
  And asks which to configure
  And runs discovery on selected backends
  And generates config YAML
  And writes validated config to .raise/

Scenario: Jira-only setup
  Given JIRA_API_TOKEN is set
  And no .raise/jira.yaml exists
  When the user runs /rai-adapter-setup
  Then Rai discovers Jira projects and workflows
  And asks which project(s) to include
  And writes .raise/jira.yaml

Scenario: Config already exists
  Given .raise/jira.yaml exists
  When the user runs /rai-adapter-setup
  Then Rai warns about existing config
  And offers to regenerate or skip
```
