# S1130.1: Confluence Discovery Service

## User Story

**As a** RaiSE adapter developer (doctor, config generator),
**I want** a `ConfluenceDiscovery` service that queries a live Confluence instance,
**so that** I can build doctor checks and config generators on structured discovery data
instead of raw API calls.

## Acceptance Criteria

```gherkin
Scenario: Discover all accessible spaces
  Given a ConfluenceClient connected to a live instance
  When I call ConfluenceDiscovery.discover() without a space_key
  Then I receive a ConfluenceSpaceMap with all accessible spaces
  And each space has key, name, url, type

Scenario: Discover a specific space with page tree
  Given a ConfluenceClient connected to a live instance
  When I call ConfluenceDiscovery.discover(space_key="RaiSE1")
  Then I receive a ConfluenceSpaceMap with that space
  And top_level_pages contains the root pages of that space

Scenario: Handle inaccessible space gracefully
  Given a ConfluenceClient connected to a live instance
  When I call ConfluenceDiscovery.discover(space_key="NONEXISTENT")
  Then a ConfluenceNotFoundError is raised with a clear message

Scenario: Handle connection failure
  Given a ConfluenceClient with invalid credentials
  When I call ConfluenceDiscovery.discover()
  Then a ConfluenceAuthError is raised
```
