# S1130.3: Adapter Doctor Check

## User Story

**As a** developer running `rai doctor`,
**I want** adapter health checks for Jira and Confluence,
**So that** I get actionable diagnostics when my adapter config is missing, env vars are unset, or the backend is unreachable.

## Acceptance Criteria

```gherkin
Scenario: Config file missing
  Given no .raise/jira.yaml exists
  When rai doctor runs the adapter check
  Then it reports WARN with fix hint to run /rai-adapter-setup

Scenario: Config exists but env vars missing
  Given .raise/jira.yaml exists with instance "humansys"
  And JIRA_API_TOKEN is not set
  When rai doctor runs the adapter check
  Then it reports ERROR with the specific env var name needed

Scenario: Config + env vars valid, backend reachable (online mode)
  Given valid config and env vars
  And --online flag is set
  When rai doctor runs the adapter check
  Then it reports PASS with instance details

Scenario: Backend unreachable (online mode)
  Given valid config and env vars
  And the Jira backend is unreachable
  When rai doctor runs the adapter check
  Then it reports ERROR with connectivity details

Scenario: Both adapters checked
  Given both jira.yaml and confluence.yaml exist
  When rai doctor runs
  Then both Jira and Confluence adapter checks are reported
```
