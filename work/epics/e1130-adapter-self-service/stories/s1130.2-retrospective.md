# S1130.2 Retrospective

## Summary

Implemented `JiraDiscovery` service, `JiraProjectMap` model, 5 discovery models, and 3 new `JiraClient` methods. 21 tests, all passing. AR/QR found two actionable issues — both fixed before merge.

## What went well

- Parallel structure with S1130.1 (ConfluenceDiscovery) made this faster to implement — pattern was established
- TDD caught the model shapes correctly on first pass
- AR caught a real edge case: dedup by name alone vs (name, status_category)
- QR caught missing test coverage for issue type best-effort failure
- `spec=JiraClient` on mocks applied the lesson from S1130.1 QR (PAT-E-002)
- `cast()` + `isinstance` pattern handled pyright strictness cleanly (PAT-E-004)

## What to improve

- **Pyright overhead estimation**: T2 took longer than expected due to `cast()` gymnastics for untyped atlassian-python-api returns. Should budget ~30% extra for pyright compliance when wrapping untyped libraries.
- **Best-effort coverage**: Had two independent try/except blocks but only tested one for failure. QR caught the gap — should always test each independent error path.

## Patterns

- **PAT-E-003: Jira status dedup by (name, category)** — `get_status_for_project` returns statuses grouped by issue type; same status appears multiple times. Deduplicate by tuple, not name alone.
- **PAT-E-004: cast() + isinstance for untyped libraries** — safer than `type: ignore`, preserves type contract while providing runtime safety.

- Actual: ~40 min
