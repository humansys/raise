# S1130.5: Config Generator — Jira

## User Story

**As a** developer setting up Jira integration,
**I want** a config generator that produces valid Jira YAML from discovery data,
**So that** I never need to look up project keys, workflow states, transition IDs, or issue types manually.

## Acceptance Criteria

```gherkin
Scenario: Generate minimal Jira config from discovery
  Given a JiraProjectMap with project "RAISE" on "humansys.atlassian.net"
  When I call generate_jira_config with selected_project="RAISE"
  Then the output dict passes JiraConfig.model_validate()
  And it contains default_instance, instances, and projects sections

Scenario: Workflow states included from discovery
  Given discovery found workflow states [Backlog, In Progress, Done] for "RAISE"
  When I call generate_jira_config with selected_project="RAISE"
  Then the output dict includes workflow.states with discovered states
  And status_mapping is generated from state names

Scenario: Issue types included from discovery
  Given discovery found issue types [Story, Bug, Task] for "RAISE"
  When I call generate_jira_config with selected_project="RAISE"
  Then the output dict includes issue_types section

Scenario: Invalid project rejected
  Given a JiraProjectMap with projects ["RAISE", "RTEST"]
  When I call generate_jira_config with selected_project="NOPE"
  Then a ValueError is raised listing available projects

Scenario: Multiple projects supported
  Given a JiraProjectMap with projects ["RAISE", "RTEST"]
  When I call generate_jira_config with selected_projects=["RAISE", "RTEST"]
  Then the output contains both projects in the projects section
```
