# S1130.6 Scope: /rai-adapter-setup Skill

## In Scope

- SKILL.md for `/rai-adapter-setup` in skills_base
- Orchestrates: detect backends → discover → select → generate → write YAML
- 3-4 interactive questions max
- Handles both Jira and Confluence
- Warns when config already exists, offers to regenerate
- Uses existing: ConfluenceDiscovery, JiraDiscovery, generate_*_config(), ConfluenceConfig, JiraConfig

## Out of Scope

- CLI command (skill is prompt-based, invoked via /rai-adapter-setup)
- Multi-instance setup (single instance per call, composable)
- Team section in Jira config (requires user input beyond discovery)
- Transition ID mapping (lifecycle_mapping — not yet discoverable)

## Done Criteria

1. SKILL.md exists at `skills_base/rai-adapter-setup/SKILL.md`
2. Skill guides user through backend detection → discovery → selection → generation → write
3. Generated YAML passes config validation (ConfluenceConfig/JiraConfig)
4. Skill registered in skills_base/__init__.py (if required)
5. Manual dogfood: run /rai-adapter-setup in a test context
