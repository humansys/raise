# S1130.5 Scope: Config Generator — Jira

## In Scope

- `generate_jira_config(project_map, selected_projects, instance_name, site) -> dict`
- Pure function — no side effects, no file I/O
- Output matches `JiraConfig.model_validate()` schema (validated in tests)
- Generates `workflow.states` and `status_mapping` from discovered WorkflowStates
- Generates `issue_types` section from discovered IssueTypeInfo
- Generates `instances` and `projects` sections from selections
- Error on invalid selected_project not in discovery map

## Out of Scope

- YAML serialization or file writing (S1130.6 skill handles that)
- Interactive project selection (S1130.6 skill handles that)
- Team section generation (requires user input, not discoverable)
- Transition ID discovery (WorkflowState.transitions is empty from current API — future enhancement)
- Multi-instance generation (single instance per call, composable)

## Done Criteria

1. `generate_jira_config()` returns dict that passes `JiraConfig.model_validate()` validation
2. Generated config includes selected projects, instance, and site
3. Workflow states and status mapping generated from discovery data
4. Issue types section generated from discovery data
5. Error raised for project not in discovery map
6. All tests pass, types check, linting clean
7. Module: `packages/raise-cli/src/raise_cli/adapters/jira_config_gen.py`
