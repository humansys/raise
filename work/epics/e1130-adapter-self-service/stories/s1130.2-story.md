# S1130.2: Jira Discovery Service

## User Story

**As a** RaiSE adapter developer (doctor, config generator),
**I want** a `JiraDiscovery` service that queries a live Jira instance,
**so that** I can build doctor checks and config generators on structured discovery data
instead of raw API calls.

## Acceptance Criteria

```gherkin
Scenario: Discover all accessible projects
  Given a JiraClient connected to a live instance
  When I call JiraDiscovery.discover() without a project_key
  Then I receive a JiraProjectMap with all accessible projects
  And each project has key, name, project_type_key

Scenario: Discover a specific project with workflows and issue types
  Given a JiraClient connected to a live instance
  When I call JiraDiscovery.discover(project_key="RAISE")
  Then I receive a JiraProjectMap with that project
  And workflows contains the workflow states with transition IDs
  And issue_types contains the available issue types

Scenario: Handle inaccessible project gracefully
  Given a JiraClient connected to a live instance
  When I call JiraDiscovery.discover(project_key="NONEXISTENT")
  Then a JiraNotFoundError is raised with a clear message

Scenario: Handle connection failure
  Given a JiraClient with invalid credentials
  When I call JiraDiscovery.discover()
  Then a JiraAuthError is raised
```
