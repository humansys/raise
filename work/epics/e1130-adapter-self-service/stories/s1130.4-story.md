# S1130.4: Config Generator — Confluence

## User Story

**As a** RaiSE user running `/rai-adapter-setup`,
**I want** a pure function that generates valid Confluence config from discovery data,
**so that** I never have to manually look up space keys, URLs, or routing structures.

## Acceptance Criteria

```gherkin
Scenario: Generate minimal config from discovered space
  Given a ConfluenceSpaceMap with space "RaiSE1"
  When I call generate_confluence_config(space_map, "RaiSE1", "https://x.atlassian.net/wiki")
  Then the result is a dict matching ConfluenceConfig schema
  And it has a single "default" instance with the selected space

Scenario: Generate config with custom routing
  Given a ConfluenceSpaceMap with space "RaiSE1"
  When I call generate_confluence_config with routing for "adr" and "developer"
  Then the result includes routing entries for those artifact types

Scenario: Selected space not in discovery map
  Given a ConfluenceSpaceMap without space "MISSING"
  When I call generate_confluence_config with selected_space="MISSING"
  Then a ValueError is raised

Scenario: Generated config passes ConfluenceConfig validation
  Given any valid generate_confluence_config output
  When I pass the dict to ConfluenceConfig.from_dict()
  Then validation succeeds without errors
```
