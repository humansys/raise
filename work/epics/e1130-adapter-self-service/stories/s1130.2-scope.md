# S1130.2 Scope: Jira Discovery Service

## In Scope

- 3 new methods on `JiraClient`: `list_projects()`, `get_project_workflows(project_key)`, `get_issue_types(project_key)`
- `ProjectInfo`, `WorkflowState`, `IssueTypeInfo` Pydantic models in `models/pm.py`
- `JiraProjectMap` frozen Pydantic model: `projects`, `workflows`, `issue_types`
- `JiraDiscovery` class wrapping `JiraClient`
- `discover(project_key=None)` → full or filtered `JiraProjectMap`
- Error propagation: auth errors, not-found, API errors from client
- Unit tests with mocked `JiraClient`

## Out of Scope

- CLI command for discovery (D1: internal service only)
- Custom field discovery (not needed for config generator)
- Sprint/board discovery (not needed for config generator)
- Integration tests against live Jira (deferred to S1130.3 doctor)

## Done Criteria

1. `JiraClient.list_projects()` returns list of project dicts
2. `JiraClient.get_project_workflows(project_key)` returns workflow states with transitions
3. `JiraClient.get_issue_types(project_key)` returns issue type list
4. `JiraDiscovery.discover()` returns `JiraProjectMap` with projects, workflows, issue types
5. `JiraDiscovery.discover(project_key="X")` returns filtered map for one project
6. Error cases covered: auth failure, not found, empty project
7. All tests pass, types check, linting clean
8. Module: `packages/raise-cli/src/raise_cli/adapters/jira_discovery.py`
