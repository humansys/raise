# S478.3 — Design: Clean Pro-Specific CLI Logic

## Problem

`src/raise_cli/cli/commands/adapters.py` contains a hardcoded `rai adapter status` command (lines 258-367) with Jira-specific helpers:
- `_JIRA_ENV_VARS` — hardcoded env var names for Jira
- `_check_jira_config()` — checks Jira-specific config and env vars
- `status_command()` — renders Jira-only status output

This couples the community CLI directly to a specific pro adapter.

## Approach

### D1: Remove `rai adapter status` command

Delete from `adapters.py`:
- `_JIRA_ENV_VARS` constant
- `_check_jira_config()` function
- `status_command()` with its `@adapters_app.command("status")` decorator
- Unused `os` import (only used by `_check_jira_config`)

**Rationale:** Status checking should be a protocol method (`health()`/`status()`) on each adapter, not hardcoded Jira logic in the community CLI. This is a future enhancement.

### D2: Other Jira/Confluence references — assessment

Grep found additional references in raise_cli source. Categorized:

**Acceptable (keep as-is):**
- Docstring examples: "e.g., Jira", "JQL for Jira" in protocols, backlog, bridge
- Help text: "JQL for Jira" in CLI commands
- Parser comments: Jira link format in governance/parsers/backlog.py

**Deferred (config-driven, bigger scope):**
- `hooks/builtin/backlog.py` — `_JiraConfig`, lifecycle sync via jira.yaml
- `doctor/checks/project.py` — checks jira.yaml existence
- `onboarding/instructions.py` — generates CLAUDE.md, gated on jira.yaml
- `tier/context.py` — `JIRA_INTEGRATION` feature flag
- `skills_base/*.md` — workflow instructions

These are configuration-driven (read `.raise/jira.yaml`) rather than hardcoded to Jira internals. Out of scope for this XS story.

## Key Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D-478.3-1 | Remove `rai adapter status` entirely | Hardcoded Jira-specific, not generic |
| D-478.3-2 | Keep docstring/help Jira examples | Adapter-agnostic examples, not coupling |
| D-478.3-3 | Defer hooks/doctor/onboarding references | Config-driven, bigger than XS scope |

## Acceptance Criteria

- Zero Jira-specific logic in adapters.py
- `rai adapter list`, `check`, `validate` still work
- All tests, types, lint pass
