# S478.1 Design — Move Pro Adapters to rai_pro

## Problem

Pro-only adapter code (`mcp_jira.py`, `mcp_confluence.py`, `jira_sync.py`) lives inside `raise_cli`, leaking enterprise code into the open-source package wheel.

## Value

Community users get a clean `raise-cli` install without enterprise adapter code or dependencies they don't need.

## Approach

Pure file-move refactor. No logic changes.

### Files to Move

| From | To |
|------|-----|
| `src/raise_cli/adapters/mcp_jira.py` | `src/rai_pro/adapters/mcp_jira.py` |
| `src/raise_cli/adapters/mcp_confluence.py` | `src/rai_pro/adapters/mcp_confluence.py` |
| `src/raise_cli/hooks/builtin/jira_sync.py` | `src/rai_pro/hooks/jira_sync.py` |

### Tests to Update (import paths only)

| Test File | Import Change |
|-----------|---------------|
| `tests/adapters/test_mcp_jira.py` | `raise_cli.adapters.mcp_jira` -> `rai_pro.adapters.mcp_jira` |
| `tests/adapters/test_mcp_confluence.py` | `raise_cli.adapters.mcp_confluence` -> `rai_pro.adapters.mcp_confluence` |
| `tests/test_hooks/test_jira_sync.py` | `raise_cli.hooks.builtin.jira_sync` -> `rai_pro.hooks.jira_sync` |

### New `__init__.py` Files

- `src/rai_pro/adapters/__init__.py`
- `src/rai_pro/hooks/__init__.py`

### Imports That Stay (shared infrastructure)

The moved files import from these `raise_cli` modules. These imports **do not change** — they reference shared infrastructure that remains in the open-source package:

- `raise_cli.adapters.models` — shared adapter model types (IssueRef, IssueDetail, AdapterHealth, etc.)
- `raise_cli.mcp.bridge` — shared MCP bridge infrastructure (McpBridge, McpBridgeError, McpToolResult)
- `raise_cli.hooks.events` — shared hook event types (HookEvent, HookResult, WorkStartEvent, WorkCloseEvent)
- `raise_cli.cli.commands._resolve` — lazy import in jira_sync.py for adapter resolution

## Before / After Examples

### mcp_jira.py imports (unchanged)

```python
# These stay the same — shared infrastructure
from raise_cli.adapters.models import AdapterHealth, BatchResult, ...
from raise_cli.mcp.bridge import McpBridge, McpBridgeError, McpToolResult
```

### test_mcp_jira.py import (changes)

```python
# Before
from raise_cli.adapters.mcp_jira import McpJiraAdapter

# After
from rai_pro.adapters.mcp_jira import McpJiraAdapter
```

### test_jira_sync.py import (changes)

```python
# Before
from raise_cli.hooks.builtin.jira_sync import JiraSyncHook, _load_lifecycle_mapping, _resolve_status_name

# After
from rai_pro.hooks.jira_sync import JiraSyncHook, _load_lifecycle_mapping, _resolve_status_name
```

## Acceptance Criteria

- **MUST**: All 3 source files moved to rai_pro
- **MUST**: All tests pass with updated import paths
- **MUST**: No `raise_cli.adapters.mcp_jira` or `mcp_confluence` imports remain in raise_cli source
- **MUST**: `__init__.py` files exist in new rai_pro subdirectories
- **MUST NOT**: Change any adapter logic or behavior — pure move
- **MUST NOT**: Move shared infrastructure (models.py, bridge.py, protocols.py, events.py)

## Risks

- Entry point registrations in `pyproject.toml` still point to `raise_cli` paths — handled in a separate story.
