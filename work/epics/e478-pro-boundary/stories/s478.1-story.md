# S478.1 — Move Pro Adapters to rai_pro

## User Story

**As a** maintainer of the raise-commons open-source package,
**I want** pro-only adapter code moved from `raise_cli` to `rai_pro`,
**So that** the open-source `raise-cli` distribution ships zero proprietary adapter code.

## Acceptance Criteria

```gherkin
Feature: Pro adapter isolation

  Scenario: Pro adapter files live in rai_pro
    Given the repository after story completion
    Then src/rai_pro/adapters/mcp_jira.py exists
    And src/rai_pro/adapters/mcp_confluence.py exists
    And src/rai_pro/hooks/jira_sync.py exists
    And src/raise_cli/adapters/mcp_jira.py does not exist
    And src/raise_cli/adapters/mcp_confluence.py does not exist
    And src/raise_cli/hooks/builtin/jira_sync.py does not exist

  Scenario: Tests are co-located with new package
    Given the repository after story completion
    Then tests for mcp_jira import from rai_pro.adapters
    And tests for mcp_confluence import from rai_pro.adapters
    And tests for jira_sync import from rai_pro.hooks

  Scenario: No direct imports from raise_cli to pro adapters
    Given the repository after story completion
    When I search raise_cli source for imports of mcp_jira, mcp_confluence, or jira_sync
    Then no direct imports are found
    And discovery is via entry points only

  Scenario: All tests pass
    Given the moved files with updated imports
    When I run the test suite
    Then all tests pass
```

## Size Estimate

**S** (Small) — file moves + import updates, no logic changes.
