## Heutagogical Reflection

### What did I learn?
Automated gates (lint, type-check, tests) only cover `.py` files. Documentation references to removed CLI commands survive all gates unless explicitly grepped. The QR phase caught a stale reference in `.mdx` docs that the initial `.py`-scoped grep missed.

### What would I do differently?
Include `docs/` in the grep scope from the start when removing any CLI command or public API surface. A removal checklist (code, tests, docs, changelog) would prevent the two-pass pattern.

### What surprised me?
The QR gate earned its keep on an XS story. The assumption that "tiny story = nothing to miss" was wrong — the docs reference was invisible to the automated pipeline.

### What pattern emerged?
When removing CLI commands, grep `docs/` for stale references. Automated gates only check `.py` files, so documentation rot is invisible without explicit search.
