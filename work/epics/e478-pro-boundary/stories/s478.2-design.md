# S478.2 — Design: Clean Entry Points and Dependencies

## Problem

The community `raise-cli` pyproject.toml still declares:
- **Pro entry points:** jira (rai.adapters.pm), confluence (rai.docs.targets), jira-sync (rai.hooks)
- **Pro dependencies:** atlassian-python-api, authlib, cryptography, requests, urllib3, certifi

Anyone installing `raise-cli` pulls 6+ unnecessary packages and registers entry points that point to code not included in the community package.

## Approach

### D1: Create `packages/raise-pro/` workspace package

New `packages/raise-pro/pyproject.toml`:
- **name:** raise-pro
- **dependencies:** atlassian-python-api, authlib, cryptography, requests, urllib3, certifi, raise-cli
- **entry points:**
  - `rai.adapters.pm` → `jira = "rai_pro.adapters.mcp_jira:McpJiraAdapter"`
  - `rai.docs.targets` → `confluence = "rai_pro.adapters.mcp_confluence:McpConfluenceAdapter"`
  - `rai.hooks` → `jira-sync = "rai_pro.hooks.jira_sync:JiraSyncHook"`
- **wheel packages:** `src/rai_pro` (at repo root, not inside packages/)

### D2: Clean raise-cli pyproject.toml

Remove from `pyproject.toml`:
- Entry points: jira, confluence, jira-sync
- Dependencies: atlassian-python-api, authlib, cryptography, requests, urllib3, certifi
- Wheel/sdist: remove `src/rai_pro` from packages and include lists

**Verified:** `requests`, `urllib3`, `certifi` have zero imports in `raise_cli` and `raise_core`. All three are only used in `rai_pro/providers/jira/oauth.py`.

### D3: Update workspace config

- `[tool.uv.workspace] members = ["packages/*"]` already auto-discovers new package
- Add `raise-pro = { workspace = true }` to `[tool.uv.sources]`
- Add `--cov=src/rai_pro` to pytest addopts

## Key Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D-478.2-1 | Move all 6 deps to raise-pro (including requests/urllib3/certifi) | Zero imports in community code confirmed by grep |
| D-478.2-2 | Keep `src/rai_pro/` at repo root | Avoids file moves. Hatch wheel config can reference repo-root paths |
| D-478.2-3 | raise-pro depends on raise-cli (not reverse) | Community package must be standalone |

## Risks

| ID | Risk | Mitigation |
|----|------|------------|
| R-478.2-1 | Hatch may not build wheel with packages outside package dir | Test with `hatch build`. Fallback: move src/rai_pro into packages/ |
| R-478.2-2 | Tests need raise-pro in dev | Workspace `uv sync` handles this. Add to dev deps if needed |

## Acceptance Criteria

- MUST: `pip install raise-cli` has zero atlassian/authlib/cryptography deps
- MUST: `packages/raise-pro/pyproject.toml` exists with correct entry points
- MUST: `uv sync` installs both packages, all tests pass
- MUST: No jira/confluence entry points in raise_cli pyproject.toml
- MUST NOT: Break existing test suite
- MUST NOT: Change adapter behavior
