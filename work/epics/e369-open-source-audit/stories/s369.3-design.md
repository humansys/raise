# S369.3 -- CI Hardening: Design

## Problem

CI only tests Python 3.12, but the project claims 3.13 support (classifier
added in S369.2). The CI badge URL needs verification. The release workflow
needs a Trusted Publishers readiness review.

## Approach

Configuration-only changes. Add Python 3.13 to the CI matrix as non-blocking
(continue-on-error), verify the badge URL, and review release.yml against
the PyPI Trusted Publishers checklist.

## Changes

| File | Action | What |
|------|--------|------|
| `.github/workflows/ci.yml` | modify | Add `"3.13"` to `python-version` matrix with `continue-on-error` for 3.13 |
| `.github/workflows/release.yml` | review | Verify Trusted Publishers compatibility (OIDC, environment, action) |
| `README.md` | verify | Confirm badge URL resolves to valid status page |

## Current State Analysis

### ci.yml

- Matrix: `["3.12"]` only
- Steps: checkout, uv install, sync, pytest, ruff check, ruff format, pyright
- Triggers: push to main, PR to main

### release.yml

Already well-configured for Trusted Publishers:
- `id-token: write` permission (required for OIDC)
- `environment: pypi` configured on publish job
- Uses `pypa/gh-action-pypi-publish@release/v1`

### README badge

```
[![CI](https://github.com/humansys-ai/raise-commons/actions/workflows/ci.yml/badge.svg)]
```
URL matches workflow file name (`ci.yml`). Should resolve correctly.

## Decisions

- **D-1:** Add 3.13 as non-blocking. If tests fail on 3.13, document and fix
  in a follow-up story. This avoids blocking PRs on unknown failures.
- **D-2:** Use `continue-on-error` in the matrix for 3.13 to make it visible
  but non-blocking.
- **D-3:** Document Trusted Publishers requirements in story retrospective,
  not a separate ADR.

## Trusted Publishers Requirements

Workflow-side (already in place):
1. `id-token: write` permission
2. `environment: pypi` on publish job
3. `pypa/gh-action-pypi-publish@release/v1` action

Manual PyPI configuration needed:
1. Go to PyPI project settings for `raise-cli`
2. Add a new "Trusted Publisher"
3. Set: owner=`humansys-ai`, repo=`raise-commons`, workflow=`release.yml`, environment=`pypi`
