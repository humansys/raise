# S369.2: Package Metadata & Deps

## User Story

As a developer evaluating raise-commons on PyPI,
I want to see professional metadata (Homepage, Docs, Repository, Changelog links)
and a clean dependency list with no dead entries,
so I can trust this is a well-maintained package.

## Acceptance Criteria (informal, XS story)

- `[project.urls]` added to pyproject.toml with Homepage, Docs, Repository, Changelog
- Dead `tomli` dependency removed (requires python < 3.11 but project requires >= 3.12)
- Python 3.13 classifier added to pyproject.toml
- Dependency vulnerability scan run (pip-audit / safety) with no critical/high findings
