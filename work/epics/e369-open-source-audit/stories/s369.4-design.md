# S369.4 -- Secrets & Hygiene Scan: Design

## Problem

Before publishing to PyPI, we must ensure no secrets, internal references, or
unintended artifacts leak in the published package. A single exposed credential
or internal URL would destroy the trust we are building.

## Approach

Run grep-based scans for secrets and internal references, review .gitignore
completeness, build sdist/wheel and inspect contents. Fix any issues found.
Document results for audit trail.

**Key decision:** Use grep patterns rather than external tools (trufflehog,
gitleaks). Grep is sufficient for a targeted scan of known patterns. External
scanners can be added to CI in a future story.

## Scan Strategy

### 1. Secrets Scan (src/ and top-level config files)

Patterns to grep for:

| Pattern | What it catches |
|---------|----------------|
| `PRIVATE KEY` | Leaked SSH/PGP keys |
| `sk-[a-zA-Z0-9]` | OpenAI/Stripe-style API keys |
| `password\s*=` | Hardcoded passwords |
| `secret\s*=` | Hardcoded secrets |
| `token\s*=` | Hardcoded tokens |
| `Bearer ` | Auth headers |
| `Basic [A-Za-z0-9+/=]` | Basic auth credentials |

### 2. Internal References Scan (src/ code that ships to PyPI)

| Pattern | Why |
|---------|-----|
| `humansys` | Internal org references (legitimate in LICENSE/docs, flag in code) |
| `@humansys` | Internal email addresses |
| `/home/emilio` | Absolute filesystem paths |
| `192.168.` / `10.0.` | Internal network IPs |
| `localhost` | Local dev references (acceptable in tests, flag in shipped code) |

### 3. .gitignore Review

Verify coverage of:
- `.raise/rai/personal/`
- `.raise/rai/memory/.raise/`
- `work/research/`
- `.env` / `.env.*`
- IDE/editor configs
- Build artifacts (`dist/`, `*.egg-info`)

### 4. Distribution Content Audit

```bash
# Build sdist and wheel
uv build

# Inspect sdist
tar tzf dist/*.tar.gz | head -50

# Inspect wheel
unzip -l dist/*.whl | head -50
```

Verify NONE of these appear in distribution:
- `work/`
- `dev/`
- `.raise/`
- `.github/`
- `tests/` (unless intentionally included)
- Personal config files

## Changes

| File/Area | Action | What |
|-----------|--------|------|
| `.gitignore` | review/modify | Add missing patterns for internal artifacts |
| `pyproject.toml` | review | Verify build include/exclude rules |
| `src/` | scan | Grep for secrets and internal references |
| `dist/` output | inspect | Build and verify sdist/wheel contents |

## Decisions

- **D-1:** Grep-based scanning, not external tools. Sufficient for targeted
  known-pattern detection. External tools scan git history which is valuable
  but out of scope for fixes (history rewriting excluded).
- **D-2:** Build and inspect actual sdist/wheel. The only reliable way to
  verify distribution contents.

## Verification

```bash
# Secrets scan -- should return empty or only false positives
grep -rn "PRIVATE KEY\|sk-[a-zA-Z0-9]\|password\s*=\|secret\s*=\|token\s*=" src/

# Internal refs -- should return empty for shipped code
grep -rn "humansys\|/home/emilio\|192\.168\.\|10\.0\." src/

# Distribution contents -- no work/, .raise/, dev/ directories
tar tzf dist/*.tar.gz | grep -E "^[^/]+/(work|\.raise|dev)/"
```
