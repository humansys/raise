# S369.3: CI Hardening -- Scope

## In Scope

- Add Python 3.13 to CI test matrix (`.github/workflows/`)
- Verify CI badge URL in README resolves correctly
- Review `release.yml` for Trusted Publishers readiness
- Document any Trusted Publishers gaps or requirements

## Out of Scope

- CI/CD redesign or migration
- Adding new CI workflows
- Fixing test failures on Python 3.13 (separate story if needed)
- Coverage improvements
- Changing release process

## Done Criteria

- CI tests run on both Python 3.12 and 3.13
- CI badge URL resolves correctly
- Trusted Publishers requirements documented (in story retrospective or ADR)

## Risks

| Risk | Mitigation |
|------|------------|
| Tests fail on Python 3.13 | Fix if simple; drop 3.13 claim if complex (per epic risk plan) |
| Badge URL mismatch with workflow name | Coordinate with S369.1 changes |
