# S369.4: Secrets & Hygiene Scan — Scope

## In Scope

- Secrets scanning with trufflehog or gitleaks (full git history)
- .gitignore completeness review for internal/personal artifacts
- sdist/wheel content audit (build and inspect)
- Internal references scan (URLs, emails, filesystem paths in published code)
- Fixing any issues found during scans

## Out of Scope

- Git history rewriting (if secrets found in history, document and assess separately)
- CI integration of secrets scanning (future story)
- Code refactoring beyond removing internal references
- Coverage improvements

## Done Criteria

- No secrets found in repo (or documented as false positives)
- .gitignore covers all internal/personal artifacts
- Published package contains only intended files
- No internal URLs, emails, or paths in published code
