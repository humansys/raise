# S369.4: Secrets & Hygiene Scan

## User Story

As a maintainer preparing to publish raise-commons as open source,
I want to ensure no secrets, internal references, or unintended artifacts leak in the published package,
so that the repository and PyPI distribution are safe and professional.

## Acceptance Criteria

1. **Secrets scanning** — Run trufflehog or gitleaks against the full repo history; no true positives found (false positives documented).
2. **.gitignore completeness** — All internal/personal artifacts (.raise/rai/memory/, work/, dev-only configs) are covered by .gitignore.
3. **sdist/wheel content audit** — Build sdist and wheel, inspect contents; only intended files are included (no work/, .raise/, personal configs).
4. **Internal references scan** — No internal URLs, email addresses, or filesystem paths remain in published source code.

## Size

S (Small) — scanning and reviewing, fixes expected to be minor.
