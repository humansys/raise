# S369.4 -- Secrets & Hygiene Scan: Plan

## Tasks

### T1: Secrets scan

**Goal:** Grep tracked files for common secret patterns; document results.

**Steps:**
1. Grep `src/` for secret patterns:
   - `PRIVATE KEY` (leaked SSH/PGP keys)
   - `sk-[a-zA-Z0-9]` (OpenAI/Stripe-style API keys)
   - `password\s*=` (hardcoded passwords)
   - `secret\s*=` (hardcoded secrets)
   - `token\s*=` (hardcoded tokens)
   - `Bearer ` (auth headers)
   - `Basic [A-Za-z0-9+/=]` (basic auth credentials)
2. Grep top-level config files (pyproject.toml, etc.) for same patterns
3. Document findings (clean or list of false positives)

**Commit:** `audit(s369.4): secrets scan -- results documented`

---

### T2: Internal references scan

**Goal:** Scan shipped code for internal URLs, emails, paths, IPs.

**Steps:**
1. Grep `src/` for internal reference patterns:
   - `humansys` (org references -- legitimate in LICENSE/docs, flag in code)
   - `@humansys` (internal email addresses)
   - `/home/emilio` (absolute filesystem paths)
   - `192.168.` / `10.0.` (internal network IPs)
   - `localhost` (local dev references)
2. Also scan `*.md` and config files for internal references that ship
3. Fix any issues found (replace with generic or configurable values)

**Commit:** `audit(s369.4): internal references scan` (or fix commit if changes needed)

### T3: .gitignore review

**Goal:** Verify .gitignore covers all internal/personal artifacts.

**Commit:** `fix(s369.4): gitignore completeness` (if changes needed; skip if already complete)

### T4: sdist/wheel content audit

**Goal:** Build distribution and verify only intended files are included.

**Steps:**
1. Build distribution: `uv build`
2. List sdist contents: `tar tzf dist/*.tar.gz | head -50`
3. List wheel contents: `unzip -l dist/*.whl | head -50`
4. Verify NONE of these appear: `work/`, `dev/`, `.raise/`, `.github/`, personal configs
5. If unwanted files found, fix via `pyproject.toml` exclude rules
6. Create progress log at `s369.4-progress.md` with all scan results from T1-T4

**Commit:** `audit(s369.4): distribution content verified` (or fix commit if changes needed)
