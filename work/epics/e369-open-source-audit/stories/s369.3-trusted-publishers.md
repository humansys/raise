# Trusted Publishers Setup — S369.3

## What's Already Configured in release.yml

The workflow at `.github/workflows/release.yml` is fully configured for Trusted Publishers (OIDC):

- `permissions.id-token: write` — enables OIDC token generation
- `environment: pypi` — uses a named GitHub environment
- `pypa/gh-action-pypi-publish@release/v1` — official PyPI publish action (no API token needed)
- Triggers on `v*` tags

No workflow changes are needed.

## Step-by-Step Instructions

1. Go to https://pypi.org and log in with the project owner account
2. Navigate to the package management page:
   - For raise-cli: https://pypi.org/manage/project/raise-cli/settings/publishing/
3. Under **"Add a new publisher"**, select **GitHub Actions**
4. Fill in:
   - **Owner:** `humansys-ai`
   - **Repository name:** `raise-commons`
   - **Workflow name:** `release.yml`
   - **Environment name:** `pypi`
5. Click **"Add"**
6. On GitHub, ensure the `pypi` environment exists:
   - Go to https://github.com/humansys/raise/settings/environments
   - Create environment `pypi` if it doesn't exist
   - Optionally add protection rules (e.g., required reviewers)
7. Test by creating a tag: `git tag v2.2.1 && git push origin v2.2.1`
