# S616.1: Data Model, Auth, and Plan Enforcement — Design

## Problem

raise-server authenticates by org-level API key (`OrgContext`) with no member
identity, no plan enforcement, and no license awareness. This blocks the entire
licensing MVP — every subsequent feature needs member-level auth and plan gating.

## Value

After this story, every request carries member identity and plan context. Existing
endpoints enforce plan requirements. The data model supports all S616.2 CRUD endpoints.

## Complexity

**Complex** — 5+ components: migration, 3 SQLAlchemy models, Pydantic schemas,
auth refactor (`verify_api_key` → `verify_member`), 2 FastAPI dependencies, 3 router
updates, unit tests.

## Approach

Replace the single-table org-level auth with a member-based auth chain. All changes
are server-side. Zero users exist — clean slate, no data migration.

### Components Affected

| Component | Change | File(s) |
|-----------|--------|---------|
| Alembic migration 003 | **create** | `alembic/versions/003_members_licenses.py` |
| SQLAlchemy models | **modify** | `db/models.py` (add MemberRow, ApiKeyRow replaces ApiKey, LicenseRow) |
| Pydantic schemas | **create** | `schemas/auth.py` (MemberContext) |
| Auth module | **modify** | `auth.py` (verify_member replaces verify_api_key, add requires_plan, requires_role) |
| Graph router | **modify** | `api/v1/graph.py` (OrgContext → MemberContext, add plan dep) |
| Memory router | **modify** | `api/v1/memory.py` (same) |
| Agent router | **modify** | `api/v1/agent.py` (same) |
| Tests | **create** | `tests/test_auth.py`, `tests/test_models.py` |

### Key Decisions for This Story

**D1: Models in single file vs split modules** — Keep all models in `db/models.py`.
3 new models don't justify a package split. Matches existing codebase pattern.

**D2: MemberContext location** — Keep in `auth.py` (matches current `OrgContext` pattern).
Move to `schemas/auth.py` in S616.2 when request/response models justify the file.
OrgContext is removed (zero users, no backwards compat needed).

**D3: last_used_at update strategy** — Fire-and-forget after response (background task
or post-response). Don't block the auth flow for a timestamp write. MVP: update inline
in the same session (simplest, acceptable latency at <10 clients).

**D4: Role hierarchy** — `admin` > `member`. Simple string comparison, not ordinal.
`requires_role("admin")` checks `ctx.role == "admin"`. No hierarchy math needed for 2 roles.

## Examples

### Auth: successful member lookup

```python
# Request
GET /api/v1/graph/query?q=auth HTTP/1.1
Authorization: Bearer rsk_abc123def456...

# Internal flow (pseudocode — implement as joined query, not 4 separate SELECTs)
hash = sha256("rsk_abc123def456...")  # → "a1b2c3..."
# Single query: api_keys JOIN members JOIN organizations, then separate license lookup
api_key, member, org = (
    SELECT api_keys.*, members.*, organizations.*
    FROM api_keys
    JOIN members ON api_keys.member_id = members.id
    JOIN organizations ON members.org_id = organizations.id
    WHERE api_keys.key_hash = 'a1b2c3...'
      AND api_keys.is_active = true
      AND members.is_active = true
      AND members.deleted_at IS NULL
)
license = SELECT * FROM licenses WHERE org_id = org.id AND status = 'active' LIMIT 1

# Returns
MemberContext(
    org_id=UUID("..."),
    org_name="HumanSys",
    member_id=UUID("..."),
    email="emilio@humansys.ai",
    role="admin",
    plan="team",
    features=["jira", "confluence", "odoo", "gitlab"],
)
```

### Auth: no license → community

```python
# Same flow, but license query returns None
MemberContext(

    features=[],
)
```

### Auth: expired license → community

```python
# License exists but status = 'expired' (not matched by WHERE status = 'active')
# Same result as no license
```

### Router update pattern (all 3 routers)

```python
# Before (current)
from raise_server.auth import OrgContext, verify_api_key
AuthCtx = Annotated[OrgContext, Depends(verify_api_key)]

from raise_server.auth import MemberContext, requires_plan
# requires_plan returns MemberContext on success
TeamCtx = Annotated[MemberContext, requires_plan("team")]
```

## Data Structures

### MemberContext (Pydantic — auth.py, move to schemas/auth.py in S616.2)

```python
class MemberContext(BaseModel):
    org_id: uuid.UUID
    org_name: str
    member_id: uuid.UUID
    email: str
    role: str           # "admin" | "member"
    plan: str           # "community" | "pro" | "team" | "enterprise"
    features: list[str] # ["jira", "confluence", ...]
```

### MemberRow (SQLAlchemy — db/models.py)

```
id          UUID PK (gen_random_uuid)
org_id      FK → organizations
email       String(255)
name        String(255)
role        String(20) default "member"
is_active   Boolean default true
deleted_at  DateTime(tz) nullable
created_at  DateTime(tz) server_default=now
updated_at  DateTime(tz) server_default=now, onupdate=now
UNIQUE(org_id, email)
```

### ApiKeyRow (SQLAlchemy — replaces ApiKey)

```
id           UUID PK
member_id    FK → members
org_id       FK → organizations (denormalized)
key_hash     String(128), indexed
key_prefix   String(12)
scopes       JSONB default '["full_access"]'
last_used_at DateTime(tz) nullable
is_active    Boolean default true
created_at   DateTime(tz) server_default=now
```

### LicenseRow (SQLAlchemy)

```
id          UUID PK
org_id      FK → organizations
plan        String(20)         -- "pro" | "team" | "enterprise"
features    JSONB              -- ["jira", "confluence", ...]
seats       Integer
status      String(20)         -- "active" | "expired" | "revoked"
expires_at  DateTime(tz)
created_at  DateTime(tz) server_default=now
updated_at  DateTime(tz) server_default=now, onupdate=now
```

### PLAN_RANK constant

```python
PLAN_RANK: dict[str, int] = {"community": 0, "pro": 1, "team": 2, "enterprise": 3}
```

## Acceptance Criteria

### MUST

- AC1: Migration 003 creates `members`, `api_keys` (new), `licenses` tables and drops old `api_keys`
- AC2: `verify_member()` resolves bearer token → member → org → license → `MemberContext`
- AC3: No active license → `plan="community"`, `features=[]` (not error)
- AC4: Expired/revoked license → same as no license (community)
- AC5: Inactive member or inactive key → 401
- AC6: `requires_plan("team")` returns 403 with structured detail when plan insufficient
- AC7: `requires_role("admin")` returns 403 when role insufficient
- AC8: All 3 existing routers (graph, memory, agent) use new auth with `requires_plan("team")`

### SHOULD

- AC9: `last_used_at` updated on successful auth
- AC10: `key_prefix` available in MemberContext or logs for debugging

### MUST NOT

- AC11: No new REST endpoints (those are S616.2)
- AC12: No seed data or CLI changes
