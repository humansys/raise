# S616.1: Data Model, Auth, and Plan Enforcement

## User Story

As a **raise-server operator**, I want the server to authenticate requests
by member identity (not just org-level API keys) and enforce plan-based
access so that only licensed organizations can use pro/team features.

## Acceptance Criteria

```gherkin
Scenario: Member authenticates with API key
  Given a member with an active API key
  When they send a request with "Authorization: Bearer rsk_xxx"
  Then the server resolves member, org, and license
  And returns MemberContext with plan and features

Scenario: Inactive member is rejected
  Given a member with is_active = false
  When they send a request with their API key
  Then the server returns 401

Scenario: Plan enforcement on endpoint
  Given a member on "pro" plan
  When they hit an endpoint requiring "team" plan
  Then the server returns 403 with plan upgrade message

Scenario: No license defaults to community
  Given an org with no active license
  When a member sends a request
  Then MemberContext.plan = "community" and features = []

Scenario: Expired license treated as community
  Given an org with an expired license
  When a member sends a request
  Then MemberContext.plan = "community"
```

## Size

M — migration + 3 models + schemas + auth refactor + plan/role deps + router updates + tests
