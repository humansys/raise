# S616.1: Data Model, Auth, and Plan Enforcement — Retrospective

**Story:** S616.1 | **Date:** 2026-03-25 | **Size:** M

## Summary

Replaced org-level API key auth with member-based auth chain. Created 3 new
SQLAlchemy models (MemberRow, ApiKeyRow, LicenseRow), MemberContext Pydantic
schema, verify_member + requires_plan + requires_role dependencies, migration 003,
and updated 3 existing routers. 33 unit tests + 6 E2E smoke tests.

## What Went Well

- **Pre-implementation arch review** caught 2 simplifications (MemberContext stays
  in auth.py, pure tests instead of SQLite-faking-PG) and 1 naming decision (*Row
  suffix) before any code was written. Zero rework.
- **TDD worked cleanly.** RED→GREEN→REFACTOR cycle was natural for each task.
  Tests caught the `from __future__ import annotations` issue immediately (422s
  instead of expected 200/403).
- **E2E smoke test with real PG** validated all ACs in minutes. docker compose up,
  migrate, seed, curl — clean and fast.
- **Zero-user advantage** fully leveraged: clean drop+recreate, no migration shims,
  no backwards compat code.

## What to Improve

- **T2 (migration) had no automated tests.** Acceptable for MVP, but future
  migrations with data should have round-trip tests.
- **PAT-E-597 keeps recurring.** Need a systematic way to flag this risk in FastAPI
  projects at design time, not just catch it in tests.

## Heutagogical Checkpoint

1. **Learned:** FastAPI dependency override propagation through nested Depends;
   pure logic + real PG smoke is the optimal test split for auth code.
2. **Process change:** Arch review pre-implementation should be standard for M+ stories.
3. **Framework:** Flag `from __future__ import annotations` risk in story design
   when FastAPI dependencies are involved.
4. **Capability gained:** raise-server test infra from scratch; migration authoring;
   dependency factory patterns.

## Patterns

- PAT-E-001: FastAPI tests with Annotated[] must NOT use future annotations
- PAT-E-002: Pure logic tests + real PG smoke > SQLite-faking-PG
- PAT-E-003: Pre-implementation arch review saves rework
