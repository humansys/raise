# Story: Organization, Member, Key, and License Endpoints + Seed

## User Story
As a raise-server admin,
I want REST endpoints to manage organizations, members, API keys, and licenses,
so that I can onboard clients and control access without direct DB manipulation.

## Acceptance Criteria

### Scenario: Create organization
```gherkin
Given I am authenticated as superadmin
When I POST /api/v1/organizations with {"name": "HumanSys", "slug": "humansys"}
Then I receive 201 with the created organization
And the organization exists in the database
```

### Scenario: Create member with seat enforcement
```gherkin
Given organization "humansys" has a license with seats=2
And 2 active members already exist
When I POST /api/v1/organizations/{id}/members with {"email": "new@humansys.ai", "name": "New"}
Then I receive 409 with "Seat limit reached"
```

### Scenario: Create API key (show-once)
```gherkin
Given I am an admin of organization "humansys"
When I POST /api/v1/organizations/{id}/api-keys
Then I receive 201 with the raw key in the response
And subsequent GET /api/v1/organizations/{id}/api-keys shows only prefix, never raw key
```

### Scenario: Create/replace license
```gherkin
Given I am authenticated as superadmin
When I POST /api/v1/organizations/{id}/license with plan, features, seats, expires_at
Then any existing active license is replaced
And the new license is active
```

### Scenario: Soft delete member
```gherkin
Given member "fernando@humansys.ai" is active
When I DELETE /api/v1/organizations/{id}/members/{mid}
Then member.deleted_at is set and member.is_active = false
And the member no longer authenticates
```

### Scenario: Hard delete API key
```gherkin
Given API key with prefix "rsk_abc123" exists
When I DELETE /api/v1/organizations/{id}/api-keys/{kid}
Then the key row is permanently deleted
```

### Scenario: Role-gated endpoints
```gherkin
Given I am authenticated as role "member" (not admin)
When I try to POST/PATCH/DELETE on member or key management endpoints
Then I receive 403 "Requires 'admin' role"
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| POST org {name: "Acme", slug: "acme"} | Create org | 201, org with UUID |
| GET /organizations | List orgs (superadmin) | 200, array of orgs |
| GET /organizations/{id} | Own org detail | 200, org object |
| POST member {email, name} | Create member | 201, member + API key |
| POST member (seats full) | Create member | 409, seat limit |
| POST api-key | Create key | 201, raw key shown once |
| GET api-keys | List keys | 200, prefix only, no raw key |
| DELETE api-key | Revoke key | 204, row deleted |
| POST license {plan, features, seats} | Create license | 201, license active |
| GET license | Current license | 200, license or 404 |
| Seed script | Run seed | Org + admin + license + key created |
