# Story: E2E Integration Tests for License MVP

## User Story
As a developer maintaining raise-server,
I want a repeatable E2E test suite that validates the full license MVP flow against real PostgreSQL,
so that regressions are caught automatically when touching auth, CRUD, or plan enforcement code.

## Acceptance Criteria

### Scenario: Full onboarding flow
```gherkin
Given a fresh database with migrations applied
When the seed script runs and admin performs full onboarding
Then org creation, member creation with auto-key, license assignment all succeed
And the new member can authenticate with their auto-created key
```

### Scenario: Cross-story integration
```gherkin
Given an authenticated member with team plan
When they access plan-gated endpoints (graph, memory, agent)
Then they receive 500 (service layer, not auth) proving auth+plan passed
And a community member receives 403
```

### Scenario: Admin CRUD lifecycle
```gherkin
Given an admin with a valid API key
When they perform the full CRUD lifecycle (create, read, update, delete)
Then all operations succeed with correct status codes
And soft-deleted members can no longer authenticate
```

## Notes
- Tests run with `@pytest.mark.e2e` marker, separated from unit tests
- Requires real PostgreSQL (docker compose)
- Designed to run in CI when postgres service is available
