# S616.2: Organization, Member, Key, and License Endpoints + Seed — Scope

**Story:** S616.2 | **Epic:** E616 | **Size:** M
**Depends on:** S616.1 (done)

## In Scope

- 14 new REST endpoints (5 org + 4 member + 3 key + 2 license)
- Pydantic request/response schemas for all endpoints
- Superadmin gate (MVP: hardcoded or env-based check)
- Org-scoping: members can only access their own org's resources
- Seat limit enforcement at member creation (409 on limit)
- API key show-once pattern (raw key only in POST response)
- Soft delete for members and orgs (deleted_at + is_active)
- Hard delete for API keys (security — no resurrection)
- Seed script for first client (org + admin + license + API key)
- Unit tests for all endpoint logic
- E2E smoke test against real PG

## Out of Scope

- Admin web console (V2)
- Email invitation flow
- API key rotation endpoint (manual revoke+create)
- Cursor pagination (offset is fine for <10 clients)
- Bulk operations
- Undelete endpoints
- Self-service org creation
