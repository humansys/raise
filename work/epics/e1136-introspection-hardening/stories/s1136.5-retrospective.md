# S1136.5 Retrospective: Dogfood Round 2

## Summary

Analyzed 30 learning records generated across 5 epics (E1051, E1133, E1136, E1248, E1065) on two release branches (2.4.0 and 3.0.0). All E1136 done criteria met.

## Results vs Baseline

| Metric | Baseline (E1133) | Dogfood (E1136) | Change |
|--------|-------------------|-----------------|--------|
| Records produced | 3 | 30 | 10x |
| Epics covered | 2 | 5 | 2.5x |
| Acceptance rate | 33% | 76% | +43pp |
| Schema consistency | Drifting | Consistent core | Fixed |
| Git persistence | None | All committed | Fixed |
| Server persistence | None | Available (rai learn push) | New |

## Gaps Found (filed as RAISE-1275 epic)

1. **RAISE-1276** (Major): Graph index unavailable in worktrees — tier1 queries fail in ~60% of records
2. **RAISE-1277** (Major): Autonomous mode skips LEARN — 58% record loss in E1248
3. **RAISE-1278** (Minor): Work ID casing inconsistency — s1051.6 vs S1051.6 duplicates

## What Worked

- Making PRIME/LEARN mandatory in skill instructions (S1136.1) drove adoption from 0% to ~76% acceptance
- Auto-commit (S1136.3) eliminated persistence problem entirely
- Schema inline in skills (S1136.1) eliminated drift — all 30 records have consistent core fields
- Pattern votes provide real signal: agents discriminate between relevant (vote:1) and irrelevant (vote:0) patterns

## What Didn't Work

- Autonomous mode doesn't reliably execute LEARN — needs enforcement mechanism (RAISE-1277)
- Graph index doesn't propagate to worktrees — tier1 is effectively dead in worktree context (RAISE-1276)
- No normalization on work_id — filesystem casing creates duplicates (RAISE-1278)

## Patterns

- PAT: Dogfood analysis on real multi-session data reveals systemic issues that single-session testing cannot surface
- PAT: Acceptance rate improves dramatically when instructions are mandatory vs optional (33% → 76%)
