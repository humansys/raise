# S1136.3 Retrospective — Auto-commit Learning Records

**Story:** S1136.3
**Epic:** E1136 — Introspection Hardening
**Size:** XS (1 SP)
**Date:** 2026-04-03
**Branch:** story/s1136.3/auto-commit-learnings

## Summary

Added explicit `git add .raise/rai/learnings/` instructions at commit points in three skill/aspect files to prevent learning records from being lost when worktrees are cleaned up. Docs-only change, no code modified.

**Estimated:** 15 min | **Actual:** ~15 min (on track)

## What Went Well

- **Straightforward scope.** XS story with clear deliverable — three file edits, all Markdown. No scope creep.
- **Plan was accurate.** Two tasks (edit + verify), linear order, no surprises.
- **Consistent wording.** The same rationale ("records lost on worktree cleanup") appears in all three locations, reinforcing the mental model.

## What to Improve

- **Missing implement learning record.** The `rai-story-implement` LEARN record was not written during implementation. This is exactly the problem S1136.3 addresses — ironic but instructive. The commit instructions are now in place to prevent this going forward.
- **No automated enforcement.** The fix is instruction-level only. A pre-commit hook or CI check could verify learning records are staged, but that is out of scope (and possibly overkill for now).

## Heutagogical Checkpoint

1. **What did I learn?** Worktree-scoped artifacts need explicit staging instructions at the point of decision (Step 4 in implement, LEARN section in review). Relying on developers/agents to remember is fragile. The fix is "nudge at the right moment."

2. **What would I change about the process?** For docs-only XS stories, the full TDD cycle is unnecessary friction. The plan correctly noted "no test" for RED, which is appropriate. No change needed — the process already adapted.

3. **Are there improvements for the framework?** Yes — a future story could add a pre-commit hook that warns if `.raise/rai/learnings/` has unstaged changes. This would be a belt-and-suspenders complement to the instruction-level fix.

4. **What am I more capable of now?** Better awareness of the worktree lifecycle and its implications for artifact persistence. The learning chain concept (design -> plan -> implement -> review records) is clearer as a concrete data flow, not just a process diagram.

## Patterns

### Added
- **PAT-E-701:** Worktree-scoped artifacts (learning records, session state) must be explicitly staged in skill commit instructions — implicit git-add-all is unreliable and forgetting to stage means data loss on worktree cleanup.
