# S1136.3 Scope

## In Scope

1. Identify where task commits do `git add` and ensure `.raise/rai/learnings/` is included
2. Ensure the `rai-story-implement` skill (or equivalent commit guidance) includes learning records in staged files
3. Verify records survive merge into `release/2.4.0`

## Out of Scope

- Server-side persistence (`rai learn push` — S1136.4)
- Changing the learning record format or schema (S1136.2 handled this)
- Adding new CLI commands

## Done Criteria

- [ ] Learning records in `.raise/rai/learnings/` are staged in task commits
- [ ] A record committed on a story branch is present after merge to release branch
- [ ] No manual step required — inclusion is automatic via skill instructions or tooling
