# S654.3 Design — CLI Adaptation

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Session name UX | Positional argument: `rai session start "gemba research"` | Most natural, most frequent use case |
| D2 | `--name` flag | Unchanged — developer name for first-time setup | Backward compat |
| D3 | Shared index write | On `close`, write `SessionIndexEntry` to shared index | Canonical registry |
| D4 | Personal index | Keep writing (dual-write) — deprecate in S654.5 | Zero breakage for existing consumers |
| D5 | Active pointer | Write on start, clear on close | Track current session |
| D6 | Prefix registration | Auto on first start, using profile prefix | Zero config |
| D7 | `list` command | New command reading shared index | Show named sessions |
