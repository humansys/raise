# S654.4 — Context Bundle & Orientation

## User Story

*As a* RaiSE developer,
*I want* `rai session start --context` to show my session history from the shared registry,
*so that* I see my recent sessions regardless of which environment I'm in.

## Acceptance Criteria

```gherkin
Scenario: Context bundle shows recent sessions from shared index
  Given the shared index contains sessions for prefix "E"
  When I run rai session start --project . --context
  Then the context bundle includes recent session names and IDs from the shared index

Scenario: Context bundle shows session name when active
  Given an active session "S-E-260322-1430" named "gemba research"
  When I run rai session start --project . --context
  Then the bundle shows the session name

Scenario: Context bundle works with empty shared index
  Given no shared index exists
  When I run rai session start --project . --context
  Then the bundle assembles without errors (falls back to existing behavior)
```
