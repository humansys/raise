# S654.3 — CLI Adaptation

## User Story

*As a* RaiSE developer,
*I want* `rai session start/close/list` to work with the new session identity model,
*so that* I can create named sessions with timestamp IDs and see my session history from the shared registry.

## Acceptance Criteria

```gherkin
Scenario: Session start creates new-format session
  Given developer prefix "E"
  When I run rai session start --name "epic design"
  Then a session with ID S-E-{YYMMDD}-{HHMM} is created
  And the active pointer is written
  And the session is added to active_sessions in developer.yaml

Scenario: Session close writes to shared index
  Given an active session S-E-260322-1430
  When I run rai session close --summary "completed design"
  Then an entry is appended to .raise/rai/sessions/E/index.jsonl
  And the active pointer is cleared
  And the session is removed from active_sessions

Scenario: Session list shows named sessions from shared index
  Given the shared index contains 3 sessions
  When I run rai session list
  Then I see session names, IDs, and dates

Scenario: Resolver handles both old and new formats
  Given --session S-E-260322-1430
  Then it resolves correctly
  Given --session SES-177
  Then it resolves correctly (legacy)
```
