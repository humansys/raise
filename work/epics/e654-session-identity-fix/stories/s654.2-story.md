# S654.2 — Shared Session Index

## User Story

*As a* RaiSE developer,
*I want* the session index to live in git (committed, per-developer subdirectory),
*so that* my session history travels with the repo and is visible from any environment.

## Acceptance Criteria

```gherkin
Scenario: Write session entry to shared index
  Given a developer with prefix "E"
  And a session "SES-E-20260322T1430" named "gemba session identity"
  When the session is closed with a summary
  Then an entry is appended to .raise/rai/sessions/E/index.jsonl
  And the entry contains id, name, started, closed, type, summary, branch

Scenario: Read session history from shared index
  Given .raise/rai/sessions/E/index.jsonl contains 3 entries
  When session history is loaded for prefix "E"
  Then all 3 entries are returned in order

Scenario: Active session pointer (local)
  Given a session "SES-E-20260322T1430" is started
  When the active session pointer is written
  Then .raise/rai/personal/active-session contains "SES-E-20260322T1430"
  And the pointer is gitignored

Scenario: Shared index directory created on first write
  Given .raise/rai/sessions/E/ does not exist
  When a session entry is written for prefix "E"
  Then the directory is created automatically

Scenario: Read from empty or nonexistent index
  Given no index.jsonl exists for prefix "E"
  When session history is loaded
  Then an empty list is returned
```
