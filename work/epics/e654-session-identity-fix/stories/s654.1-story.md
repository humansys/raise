# S654.1 — Session ID Model & Prefix Registry

## User Story

*As a* RaiSE developer,
*I want* session IDs that are timestamp-based and scoped by developer prefix,
*so that* sessions never collide across worktrees, branches, or machines.

## Acceptance Criteria

```gherkin
Scenario: Generate timestamp-based session ID
  Given a developer with prefix "E"
  When a new session ID is generated
  Then the ID matches format "SES-E-YYYYMMDDT{HHMM}"
  And the timestamp reflects the current time

Scenario: Prefix registry auto-registration
  Given a developer with prefix "E" and name "Emilio Osorio"
  And no prefixes.yaml exists
  When the developer starts their first session
  Then prefixes.yaml is created with entry E: {name: "Emilio Osorio", registered: "YYYY-MM-DD"}

Scenario: Prefix collision detection
  Given prefixes.yaml contains E: {name: "Eduardo Pérez"}
  And a new developer has prefix "E"
  When prefix registration is attempted
  Then the collision is detected
  And an extended prefix is suggested (e.g., "EO")

Scenario: Path traversal prevention
  Given a session ID containing ".." or "/" or "\"
  When normalization is attempted
  Then a ValueError is raised

Scenario: Backward-compatible normalization
  Given an old-format session ID "SES-177"
  When normalization is attempted
  Then it returns "SES-177" (passthrough, not rejected)
```
