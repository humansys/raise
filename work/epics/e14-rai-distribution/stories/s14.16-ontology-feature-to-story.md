# S14.16: Ontology Rename — Feature to Story

> **Size:** L (5 SP)
> **Epic:** E14 Rai Distribution
> **Dependencies:** None (blocking: all future work uses new terminology)
> **Priority:** Critical — ship clean ontology for F&F release (Feb 9)

## Rationale

Our backlog ontology mixes cultures: Epics (agile) containing Features (pre-agile/SAFe).
Analysis of 38 features across 8 epics shows:
- 47% are S-sized (2-3 SP) — behave like stories, not features
- Decomposition is always Feature → Tasks (never Feature → Sub-features)
- Jira mapping (E8) already says `RaiSE Feature → Jira Story`
- Glossary contradicts itself about the hierarchy

**New ontology:** `Epic → Story → Task` (aligns with Jira's native hierarchy)

## Scope

**In Scope:**
- Rename `feature/` branch prefix to `story/` in conventions and docs
- Rename all `/feature-*` skills to `/story-*` (6 skills)
- Update CLI commands referencing "feature" in workflow context
- Update glossary, constitution, governance docs
- Update epic scope files (feature → story terminology)
- Update CLAUDE.md and CLAUDE.local.md references
- Update memory files (MEMORY.md, patterns)
- Update templates and katas referencing "feature"
- Rename `work/epics/*/features/` dirs to `work/epics/*/stories/`

**Out of Scope:**
- Code-level changes to src/ (feature as code concept stays — e.g. "feature flag")
- Renaming already-completed feature IDs (F14.1 stays F14.1 in history)
- Changing git history or existing branch names
- Updating external documentation or websites
