# Implementation Plan: S14.16 Ontology Rename — Feature to Story

## Overview
- **Story:** S14.16
- **Story Points:** 5 SP
- **Size:** L
- **Created:** 2026-02-06
- **Design:** s14.16-ontology-feature-to-story.md

## Approach: Deterministic Tooling

**Principle:** Gather with tools, think with inference. Mechanical renames done mechanically.

### Tools
- **repren** (v2.0.0) — Batch multi-pattern rename with `--full` (dirs + content), `--preserve-case`, `--dry-run`, backups, undo
- **fastmod** (v0.4.4) — Interactive human review for ambiguous remaining "feature" refs
- **Inference** — Only for edge cases: glossary definition rewrites, tagline, product/historical boundary decisions

### Strategy

```
Step 1: repren --dry-run    → Preview all changes
Step 2: Human reviews diff  → Approve / adjust patterns
Step 3: repren (execute)    → Apply deterministic batch
Step 4: fastmod interactive → Handle ambiguous remainder
Step 5: Inference fixup     → Glossary definition, tagline, edge cases
Step 6: pytest              → Verify green
Step 7: git diff review     → Final human check
Step 8: Commit              → One commit per phase
```

## Key Constraints

1. **Historical IDs preserved:** F14.1 stays F14.1. New work uses S-prefix.
2. **Archive untouched:** archive/ is read-only history.
3. **Product "feature" kept:** "enterprise features", "team features" (roadmap terms) stay.
4. **Green-to-green:** Each phase commit must leave tests passing.
5. **Tagline:** "Raise your craft, one story at a time"
