## Story Scope: S248.2 — Hook Protocol & Registry

**Epic:** E248 (Lifecycle Hooks & Workflow Gates)
**Size:** M
**Depends on:** S248.1 (Event Emitter Infrastructure) ✅

### In Scope

- `LifecycleHook` Protocol defining the hook contract (ADR-039 §1)
- `HookRegistry` with entry point discovery via `importlib.metadata` (`rai.hooks` group)
- Priority-based dispatch (hooks execute in priority order)
- Per-hook timeout (5s default) with configurable override
- Error isolation per hook (exception → log + continue, never crash CLI)
- Integration with `EventEmitter` from S248.1 (register hooks, dispatch on emit)
- `rai hooks list` command for discoverability (SHOULD from scope)

### Out of Scope

- Built-in hooks (S248.3: TelemetryHook)
- Wiring events into CLI commands (S248.4)
- Gate Protocol (S248.5 — separate track)
- Async/concurrent hook execution
- Hook enable/disable CLI (`rai adapter enable` pattern — defer to when needed)

### Done Criteria

- [ ] `LifecycleHook` Protocol defined with type annotations
- [ ] `HookRegistry` discovers hooks via entry points
- [ ] Priority-based dispatch verified
- [ ] Per-hook timeout enforced (5s default)
- [ ] Error isolation: broken hook logs warning, doesn't crash CLI
- [ ] EventEmitter integration: emit() triggers registered hooks
- [ ] Tests pass (>90% coverage on story code)
- [ ] All quality checks pass (ruff, pyright, bandit)
- [ ] TDD cycle followed (red-green-refactor)
