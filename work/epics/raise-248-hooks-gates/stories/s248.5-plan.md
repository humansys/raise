## Tasks

### Task 1: WorkflowGate Protocol + GateContext/GateResult dataclasses
- **Description:** Create `gates/` package with Protocol definition and frozen dataclasses. WorkflowGate uses ClassVar fields (gate_id, description, workflow_point) and `evaluate(context) -> GateResult`. GateContext carries working_dir. GateResult carries passed, gate_id, message, details.
- **Files:**
  - Create `src/rai_cli/gates/__init__.py`
  - Create `src/rai_cli/gates/protocol.py`
  - Create `src/rai_cli/gates/models.py`
  - Create `tests/test_gates/__init__.py`
  - Create `tests/test_gates/test_protocol.py`
- **TDD Cycle:**
  - RED: Test Protocol conformance (valid gate passes isinstance), non-conformant fails. Test GateContext/GateResult frozen, defaults, field values.
  - GREEN: Implement Protocol + dataclasses.
  - REFACTOR: Verify pyright clean.
- **Verification:** `pytest tests/test_gates/test_protocol.py -v && pyright src/rai_cli/gates/`
- **Size:** S
- **Dependencies:** None

### Task 2: GateRegistry with entry point discovery
- **Description:** Create GateRegistry mirroring HookRegistry. Entry point group `rai.gates`. Methods: `discover()`, `register()`, `get_gate(gate_id)`, `get_gates_for_point(workflow_point)`, `gates` property. Same error isolation: skip malformed entry points with warning.
- **Files:**
  - Create `src/rai_cli/gates/registry.py`
  - Create `tests/test_gates/test_registry.py`
- **TDD Cycle:**
  - RED: Test discover() loads from mocked entry points. Test register() accepts conformant, rejects non-conformant. Test get_gate() by ID. Test get_gates_for_point() filtering. Test error isolation (bad entry point skipped).
  - GREEN: Implement GateRegistry.
  - REFACTOR: Extract EP_GATES constant, verify pattern matches HookRegistry.
- **Verification:** `pytest tests/test_gates/test_registry.py -v`
- **Size:** S
- **Dependencies:** Task 1

### Task 3: CLI commands — `rai gate check` and `rai gate list`
- **Description:** Create Typer command group for `rai gate`. `check` subcommand takes optional gate_id argument + `--all` flag. `list` subcommand shows discovered gates. Exit code 0 when all pass, 1 when any fail. Error isolation: gate exception → GateResult(passed=False).
- **Files:**
  - Create `src/rai_cli/cli/commands/gate.py`
  - Modify `src/rai_cli/cli/commands/__init__.py` (register group)
  - Create `tests/test_gates/test_cli.py`
- **TDD Cycle:**
  - RED: Test `rai gate list` shows discovered gates. Test `rai gate check <id>` runs specific gate. Test `rai gate check --all` runs all gates. Test exit codes. Test unknown gate_id error. Test gate exception isolation.
  - GREEN: Implement CLI commands with Typer + Rich output.
  - REFACTOR: Align output format with `rai adapters list` style.
- **Verification:** `pytest tests/test_gates/test_cli.py -v`
- **Size:** M
- **Dependencies:** Task 1, Task 2

### Task 4: E2E integration test + entry point wiring
- **Description:** Add empty `rai.gates` entry point group to pyproject.toml. Create a test-only gate to verify the full pipeline: register gate → discover → check → result. Update `gates/__init__.py` with public API exports. Verify all quality gates pass.
- **Files:**
  - Modify `pyproject.toml` (add `rai.gates` group)
  - Modify `src/rai_cli/gates/__init__.py` (exports)
  - Create `tests/test_gates/test_integration.py`
- **TDD Cycle:**
  - RED: E2E test: register a mock gate, discover via registry, invoke via check, verify GateResult on disk/output.
  - GREEN: Wire everything together.
  - REFACTOR: Verify full quality suite passes.
- **Verification:** `pytest tests/test_gates/ -v && ruff check src/rai_cli/gates/ && pyright src/rai_cli/gates/`
- **Size:** S
- **Dependencies:** Task 1, Task 2, Task 3
