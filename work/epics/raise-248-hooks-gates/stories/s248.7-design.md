# S248.7 Design: Remove ceremony from skills

## Problem

11 of 24 skills contain manual `rai signal emit-work` calls (34 total). These are telemetry side effects now handled by TelemetryHook (S248.3). Each call is ~3 lines of SKILL.md content + ~50 tokens of agent execution overhead.

## Approach

Systematic deletion of emit-work Step blocks from 11 skills. Each skill has a predictable pattern:

```markdown
### Step 0: Emit Start (Telemetry)         ← DELETE
rai signal emit-work story {id} ...        ← DELETE

### Step N: Emit Complete (Telemetry)       ← DELETE
rai signal emit-work story {id} ...        ← DELETE
```

**What stays (AD-4 boundary):**
- `rai graph query` / `rai graph context` — inputs the agent reads
- `rai pattern add` — inputs the agent writes
- `rai signal emit-calibration` — velocity data input, not telemetry side effect
- `rai session close` / `rai session start` — CLI commands, not ceremony

**What goes:**
- All `rai signal emit-work` calls (34 occurrences across 11 files)
- The Step blocks containing them (Step 0 / final Step patterns)
- Step number renumbering after deletion

## Acceptance Criteria

**MUST:**
- Zero `rai signal emit-work` / `rai memory emit-work` in `skills_base/`
- All `emit-calibration` calls preserved (not ceremony)
- Step numbers renumbered consistently after deletions
- All quality checks pass (ruff, pyright, pytest, bandit)

**MUST NOT:**
- Remove `rai graph query`, `rai pattern add`, or other input commands
- Remove `emit-calibration` calls
- Change skill logic or behavior beyond ceremony removal

## Verification Gate

```bash
# Zero emit-work calls
grep -r "signal emit-work\|memory emit-work" src/rai_cli/skills_base/ && echo "FAIL" || echo "PASS"

# Calibration preserved
grep -r "emit-calibration" src/rai_cli/skills_base/ | wc -l  # expect 3
```
