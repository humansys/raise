# S248.6 Scope: Built-in Gates + GateBridgeHook

## In Scope

- 4 built-in WorkflowGate implementations: tests, types, lint, coverage
- GateBridgeHook: a LifecycleHook that listens to `before:` events and runs matching gates
- Entry point registration for all 4 gates + bridge hook in pyproject.toml
- Tests for each gate (mocked subprocess) and bridge hook integration

## Out of Scope

- Modifying existing CLI commands (no new `emit()` calls — S248.4 already wired them)
- Configurable gate thresholds (coverage target, lint rules) — future story
- Custom gate discovery from project-level entry points — works by design, not tested here

## Done Criteria

- [ ] 4 gates: `gate-tests`, `gate-types`, `gate-lint`, `gate-coverage` in `src/rai_cli/gates/builtin/`
- [ ] Each gate runs a subprocess (`pytest`, `pyright`, `ruff check`, `pytest --cov`) and returns GateResult
- [ ] GateBridgeHook in `src/rai_cli/hooks/builtin/gate_bridge.py`
- [ ] Bridge subscribes to `before:release:publish`, `before:session:close` and runs matching gates
- [ ] Bridge returns `HookResult(status="abort")` if any gate fails
- [ ] Entry points registered in pyproject.toml for all 4 gates + bridge hook
- [ ] `rai gate check --all` discovers and runs all 4 built-in gates
- [ ] Tests pass, types pass, lint passes
- [ ] Retrospective complete
