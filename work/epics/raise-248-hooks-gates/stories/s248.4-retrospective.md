# Retrospective: S248.4 — Wire Events into CLI Commands

## Summary

- **Story:** S248.4
- **Size:** M (planned), actual effort aligned with S-M boundary

## What Was Delivered

Wired lifecycle events into 6 CLI command files using `create_emitter()` factory pattern. 9 event types now fire on real CLI usage: `session:start`, `before:session:close`, `session:close`, `graph:build`, `pattern:added`, `discover:scan`, `init:complete`, `adapter:loaded`, `adapter:failed`.

**Stats:** 637 lines added across 11 files, 12 new tests (wiring) + 3 tests (factory), 2717 total tests passing.

## What Went Well

- **Factory pattern eliminated duplication** — `create_emitter()` avoided 3-line registry setup in every command file. Clean, single-responsibility.
- **TDD guided every task** — RED-GREEN-REFACTOR produced clean code with no regressions.
- **Quality review caught a real semantic bug** — R1: `AdapterLoadedEvent` was emitting for non-compliant adapters. Fixed before merge.
- **Error isolation preserved** — Hook failures never crash CLI commands, verified E2E.

## What Could Improve

- **Init test mock depth is high** (14 patches) — inherent complexity of `init_command`, but fragile to refactoring. Future: consider integration test with real filesystem instead.
- **Lazy import patching gotcha** — Functions imported inside `init_command` body need patching at source module, not at `cli.commands.init`. This cost ~2 min debugging. Pattern noted.

## Heutagogical Checkpoint

### What did you learn?
- Lazy imports (inside function bodies) can't be patched at the caller module — must patch at source. This is a Python testing fundamental that's easy to forget.
- Quality review as separate step catches semantic issues that pass all automated gates (R1: adapter loaded/failed semantics).

### What would you change about the process?
- Nothing significant. The 8-task plan with factory-first sequencing was optimal.

### Are there improvements for the framework?
- No framework changes needed. The hook/event infrastructure from S248.1-S248.3 was ready to consume.

### What are you more capable of now?
- Wiring events into existing commands with minimal disruption. The pattern is now mechanical and replicable for future commands.
