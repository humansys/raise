# S248.4: Wire Events into CLI Commands

## What & Why

**Problem:** The event emitter (S248.1), hook registry (S248.2), and TelemetryHook (S248.3) exist but nothing fires events. CLI commands run without emitting — hooks never trigger.

**Value:** Wiring emit() calls activates the entire hook pipeline. TelemetryHook auto-replaces manual `rai signal emit` calls. PRO/Enterprise hooks start receiving real events.

## Approach

Insert `emit()` calls at success points in 6 CLI command files, covering all 8 after-events plus `before:session:close`. Each command creates an `EventEmitter` with a discovered `HookRegistry`, constructs the typed event, and calls `emit()`.

**Shared pattern — emitter factory function:**

A module-level helper avoids repeating registry setup in every command:

```python
# src/rai_cli/hooks/emitter.py (add to existing)
def create_emitter() -> EventEmitter:
    """Create an emitter with discovered hooks."""
    registry = HookRegistry()
    registry.discover()
    return EventEmitter(registry=registry)
```

**Components affected:**

| File | Change | Events |
|------|--------|--------|
| `hooks/emitter.py` | Add `create_emitter()` factory | — |
| `cli/commands/session.py` | Modify | `session:start`, `before:session:close`, `session:close` |
| `cli/commands/graph.py` | Modify | `graph:build` |
| `cli/commands/pattern.py` | Modify | `pattern:added` |
| `cli/commands/discover.py` | Modify | `discover:scan` |
| `cli/commands/init.py` | Modify | `init:complete` |
| `cli/commands/adapters.py` | Modify | `adapter:loaded`, `adapter:failed` |

## Examples

### Wiring pattern (session start)

```python
from rai_cli.hooks import SessionStartEvent, create_emitter

# ... at end of successful start() ...
emitter = create_emitter()
emitter.emit(SessionStartEvent(session_id=session_id, developer=developer_name))
```

### Before-event with abort (session close)

```python
from rai_cli.hooks import BeforeSessionCloseEvent, SessionCloseEvent, create_emitter

emitter = create_emitter()

# Before: can abort
before_result = emitter.emit(BeforeSessionCloseEvent(session_id=sid, outcome=outcome))
if before_result.aborted:
    typer.echo(f"Session close aborted: {before_result.abort_message}")
    raise typer.Exit(1)

# ... do close logic ...

# After: observe
emitter.emit(SessionCloseEvent(session_id=sid, outcome=outcome))
```

### Adapter loaded/failed (per adapter in loop)

```python
emitter = create_emitter()
for ep in entry_points:
    try:
        cls = ep.load()
        emitter.emit(AdapterLoadedEvent(adapter_name=ep.name, group=group, adapter_type=type(cls).__name__))
    except Exception as exc:
        emitter.emit(AdapterFailedEvent(adapter_name=ep.name, group=group, error=str(exc)))
```

## Acceptance Criteria

**MUST:**
- All 8 after-events emit from their respective CLI commands
- `before:session:close` fires before close logic, abort stops execution
- `create_emitter()` factory exists to avoid registry setup duplication
- Existing CLI tests pass without modification (emit is fire-and-forget)
- New tests verify event emission per command (mock emitter, assert event type)

**SHOULD:**
- Emit calls are at most 3 lines per command (create + construct + emit)
- Error in emit never crashes the CLI command (error isolation from S248.1)

**MUST NOT:**
- Add emit() to commands not in the event catalog
- Change the signature or behavior of any existing CLI command
- Import events lazily or conditionally — always import at module level
