## Story Scope: S248.5 — WorkflowGate Protocol and Registry

**Epic:** E248 (Lifecycle Hooks & Workflow Gates)
**Size:** M
**Branch:** `epic/e248/hooks-gates` (direct, no story branch)

### In Scope

- `WorkflowGate` Protocol (runtime_checkable) with `gate_id`, `description`, `evaluate(context) -> GateResult`
- `GateContext` frozen dataclass (working directory, gate_id, extra kwargs)
- `GateResult` frozen dataclass (passed, gate_id, message, details)
- `GateRegistry` with entry point discovery (`rai.gates` group) — reuse HookRegistry pattern
- `rai gate check <gate-id>` CLI command (Typer)
- `rai gate check --all` to run all discovered gates
- `rai gate list` to show discovered gates
- Entry point registration for future built-in gates (S248.6)

### Out of Scope

- Built-in gate implementations (tests, types, lint, coverage) → S248.6
- GateBridgeHook (wiring gates to before: events) → S248.6
- Event emitter integration — gates are standalone (AD-5)
- Gate result caching or incremental checks

### Done Criteria

- [ ] `WorkflowGate` Protocol defined with pyright validation
- [ ] `GateContext` and `GateResult` dataclasses implemented
- [ ] `GateRegistry` discovers gates from `rai.gates` entry points

- [ ] Tests pass (>90% coverage)
