# S248.4: Wire Events into CLI Commands

## In Scope

- Add `emit()` calls to CLI command groups that correspond to the event catalog (ADR-039)
- Events to wire: `session:start`, `session:close`, `graph:build`, `pattern:added`, `discover:scan`, `init:complete`, `adapter:loaded`, `adapter:failed`
- `before:` variants only for `session:close` (per AD-6)
- TelemetryHook (S248.3) auto-fires on these events — validates E2E

## Out of Scope

- `release:publish` event wiring (no release command exists yet)
- New CLI commands or flags
- Modifying skill content (S248.7)
- Hook/gate protocol changes

## Done Criteria

- [ ] All 8 event types emit from their respective CLI commands
- [ ] `before:session:close` fires before close logic
- [ ] TelemetryHook receives and logs all emitted events
- [ ] Existing CLI tests still pass (no regressions)
- [ ] New tests verify event emission per command
- [ ] All quality gates pass (ruff, pyright, bandit, tests)
