## Tasks

### Task 1: Base types + first 2 events
- **Description:** Create `hooks/` package with `HookEvent` base class, `HookResult`, `EmitResult`, and first 2 concrete events (`SessionStartEvent`, `GraphBuildEvent`) to validate the frozen dataclass pattern.
- **Files:**
  - Create `src/rai_cli/hooks/__init__.py`
  - Create `src/rai_cli/hooks/events.py`
  - Create `tests/test_hooks/__init__.py`
  - Create `tests/test_hooks/test_events.py`
- **TDD Cycle:**
  - RED: Test `SessionStartEvent` is frozen, has `event_name="session:start"`, auto-sets `timestamp`
  - RED: Test `GraphBuildEvent` has typed payload fields (`project_path`, `node_count`, `edge_count`)
  - RED: Test `HookResult` supports `ok`/`abort`/`error` status
  - GREEN: Implement base class + 2 events + result types
  - REFACTOR: Ensure clean inheritance, no duplication
- **Verification:** `pytest tests/test_hooks/test_events.py -v`
- **Size:** S
- **Dependencies:** None

### Task 2: EventEmitter core — register + emit + error isolation
- **Description:** Create `EventEmitter` with handler registration, dispatch by event name, and error isolation (handler exceptions caught + logged + skipped).
- **Files:**
  - Create `src/rai_cli/hooks/emitter.py`
  - Create `tests/test_hooks/test_emitter.py`
- **TDD Cycle:**
  - RED: Test `register()` + `emit()` dispatches to matching handler
  - RED: Test handler for different event name is NOT called
  - RED: Test broken handler is caught, logged, and skipped — other handlers still run
  - RED: Test `EmitResult.handler_errors` reports the exception
  - GREEN: Implement EventEmitter with try/except per handler
  - REFACTOR: Clean dispatch logic
- **Verification:** `pytest tests/test_hooks/test_emitter.py -v`
- **Size:** M
- **Dependencies:** Task 1 (needs HookEvent, HookResult, EmitResult)

### Task 3: Before-event abort flow
- **Description:** Add 2 `before:` event classes (`BeforeSessionCloseEvent`, `BeforeReleasePublishEvent`) and abort logic in the emitter — any handler returning `abort` on a `before:` event sets `EmitResult.aborted=True`.
- **Files:**
  - Modify `src/rai_cli/hooks/events.py` (add 2 before classes)
  - Modify `src/rai_cli/hooks/emitter.py` (add abort check for before: events)
  - Modify `tests/test_hooks/test_events.py` (before event tests)
  - Modify `tests/test_hooks/test_emitter.py` (abort flow tests)
- **TDD Cycle:**
  - RED: Test `BeforeReleasePublishEvent` has `event_name="before:release:publish"`, is frozen
  - RED: Test emit of `before:` event with abort handler → `EmitResult(aborted=True, abort_message=...)`
  - RED: Test emit of `before:` event with all-ok handlers → `EmitResult(aborted=False)`
  - RED: Test abort on `after:` event is ignored (abort only valid for before:)
  - GREEN: Implement before classes + abort logic in emitter
  - REFACTOR: Clean up event_name prefix detection
- **Verification:** `pytest tests/test_hooks/ -v`
- **Size:** S
- **Dependencies:** Task 2 (needs working emitter)

## Risks

| Risk | Mitigation |
|------|------------|
| Frozen dataclass with default `timestamp` field ordering | Use `field(default_factory=...)` for timestamp — fields with defaults must come after fields without in dataclass inheritance |
| `event_name` as `Literal` + `init=False` may fight with frozen | Validate in Task 1 RED tests before expanding |
