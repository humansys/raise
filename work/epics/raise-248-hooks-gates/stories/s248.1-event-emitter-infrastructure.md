# S248.1: Event Emitter Infrastructure

## In Scope

- 9 typed frozen dataclass events (AD-1, AD-7):
  - `session:start`, `session:close` (session:close gets `before:`)
  - `graph:build`
  - `pattern:added`
  - `discover:scan`
  - `init:complete`
  - `adapter:loaded`, `adapter:failed`
  - `release:publish` (gets `before:`)
- `EventEmitter` class with `emit()` dispatcher
- `before:` variants only for `release:publish` and `session:close` (AD-6)
- Event base class / protocol for typed payloads
- Sync execution model (AD-1)

## Out of Scope

- Hook Protocol and registry (S248.2)
- Gate Protocol (S248.5)
- Wiring emit() into CLI commands (S248.4)
- Async execution
- `signal:emitted` event (dropped per AD-7)

## Done Criteria

- [ ] 9 event dataclasses defined with typed payloads
- [ ] `before:` variants exist for `release:publish` and `session:close`
- [ ] `EventEmitter.emit()` dispatches to registered handlers
- [ ] Handlers can abort `before:` events (return signal)
- [ ] All events are frozen dataclasses
- [ ] Tests pass with >90% coverage on new code
- [ ] pyright, ruff, bandit pass
- [ ] TDD cycle followed
