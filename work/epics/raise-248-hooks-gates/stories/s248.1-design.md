# S248.1 Design: Event Emitter Infrastructure

## What & Why

**Problem:** No typed event infrastructure exists for lifecycle hooks to subscribe to. Cross-cutting concerns (telemetry, notifications, compliance) are hardcoded in skills as `rai signal emit` calls — fragile, verbose, and ungovernable.

**Value:** Foundation for all E248 work. Every subsequent story (S248.2–S248.7) builds on the event types and emitter defined here. Without this, hooks and gates have nothing to react to.

## Architectural Context

**New package:** `src/rai_cli/hooks/` — sibling to `adapters/`, `telemetry/`, etc.

**Layer:** Infrastructure (leaf) — no domain dependencies, consumed by CLI commands and hooks.

**Follows:** ADR-039 (architecture), ADR-033 (entry point pattern precedent).

**Key constraint:** Events are frozen dataclasses (AD-1), NOT Pydantic models. They are internal infrastructure, not boundary objects. Pydantic stays for telemetry schemas and adapter models.

## Approach

### New files

| File | Purpose |
|------|---------|
| `src/rai_cli/hooks/__init__.py` | Public API exports |
| `src/rai_cli/hooks/events.py` | 9 event dataclasses + `HookEvent` base + `HookResult` |
| `src/rai_cli/hooks/emitter.py` | `EventEmitter` class with `emit()` and handler dispatch |

### Event Catalog (9 events, AD-6/AD-7)

| Event name | Class | `before:` | Key payload fields |
|------------|-------|:---------:|-------------------|
| `session:start` | `SessionStartEvent` | No | `session_id`, `developer` |
| `session:close` | `SessionCloseEvent` | **Yes** | `session_id`, `outcome` |
| `graph:build` | `GraphBuildEvent` | No | `project_path`, `node_count`, `edge_count` |
| `pattern:added` | `PatternAddedEvent` | No | `pattern_id`, `content`, `context` |
| `discover:scan` | `DiscoverScanEvent` | No | `project_path`, `language`, `component_count` |
| `init:complete` | `InitCompleteEvent` | No | `project_path`, `project_name` |
| `adapter:loaded` | `AdapterLoadedEvent` | No | `adapter_name`, `group`, `adapter_type` |
| `adapter:failed` | `AdapterFailedEvent` | No | `adapter_name`, `group`, `error` |
| `release:publish` | `ReleasePublishEvent` | **Yes** | `version`, `project_path` |

### Event Base Class

```python
@dataclass(frozen=True)
class HookEvent:
    """Base for all lifecycle events."""
    event_name: str        # e.g. "session:start", "before:release:publish"
    timestamp: datetime    # UTC, set at creation

@dataclass(frozen=True)
class SessionStartEvent(HookEvent):
    event_name: Literal["session:start"] = field(default="session:start", init=False)
    session_id: str = ""
    developer: str = ""
```

### Before-variant pattern

`before:` events are **separate classes** that inherit from the same base. They carry the same payload as the `after:` version (so hooks can inspect what's about to happen).

```python
@dataclass(frozen=True)
class BeforeSessionCloseEvent(HookEvent):
    event_name: Literal["before:session:close"] = field(default="before:session:close", init=False)
    session_id: str = ""
    outcome: str = ""

@dataclass(frozen=True)
class SessionCloseEvent(HookEvent):
    event_name: Literal["session:close"] = field(default="session:close", init=False)
    session_id: str = ""
    outcome: str = ""
```

Total: 9 `after:` events + 2 `before:` events = 11 event classes.

### HookResult

```python
@dataclass(frozen=True)
class HookResult:
    status: Literal["ok", "abort", "error"]
    message: str = ""
```

- `ok` — handler completed successfully
- `abort` — handler requests operation abort (only valid for `before:` events)
- `error` — handler failed (emitter logs and continues)

### EventEmitter

```python
class EventEmitter:
    """Dispatches lifecycle events to registered handlers."""

    def register(self, event_name: str, handler: Callable[[HookEvent], HookResult]) -> None:
        """Register a handler for an event name."""

    def emit(self, event: HookEvent) -> EmitResult:
        """Dispatch event to all registered handlers.

        For before: events, if any handler returns abort, returns EmitResult(aborted=True).
        For after: events, handler errors are logged and skipped.
        """
```

```python
@dataclass(frozen=True)
class EmitResult:
    aborted: bool = False
    abort_message: str = ""
    handler_errors: list[str] = field(default_factory=list)
```

**IMPORTANT:** The emitter is a plain class, not a singleton. CLI commands instantiate it, register hooks (S248.2 adds entry point discovery), and call `emit()`. For S248.1, registration is manual — the public API is the emitter + events.

### Error isolation (AD-1)

Each handler call is wrapped in try/except. Exceptions are caught, logged via `logging.getLogger(__name__)`, and added to `EmitResult.handler_errors`. The CLI never crashes from a handler failure.

Per-handler timeout (5s default) is **deferred to S248.2** where hooks declare their own timeout preferences. S248.1 uses a simple sequential call without timeout.

## Examples

### Creating and emitting an event

```python
from rai_cli.hooks.events import GraphBuildEvent
from rai_cli.hooks.emitter import EventEmitter

emitter = EventEmitter()
emitter.register("graph:build", my_handler)

event = GraphBuildEvent(
    project_path=Path("/home/user/project"),
    node_count=42,
    edge_count=15,
)
result = emitter.emit(event)
assert not result.aborted
```

### Before-event abort flow

```python
from rai_cli.hooks.events import BeforeReleasePublishEvent, HookResult

def compliance_hook(event: HookEvent) -> HookResult:
    if not checks_pass():
        return HookResult(status="abort", message="Compliance check failed")
    return HookResult(status="ok")

emitter.register("before:release:publish", compliance_hook)

event = BeforeReleasePublishEvent(version="2.1.0", project_path=Path("."))
result = emitter.emit(event)
if result.aborted:
    print(f"Publish blocked: {result.abort_message}")
```

### Handler error isolation

```python
def broken_hook(event: HookEvent) -> HookResult:
    raise RuntimeError("oops")

emitter.register("graph:build", broken_hook)
emitter.register("graph:build", good_hook)

result = emitter.emit(GraphBuildEvent(...))
# broken_hook logged + skipped, good_hook still runs
assert len(result.handler_errors) == 1
```

## Acceptance Criteria

**MUST:**
- 9 event dataclasses + 2 before-variant classes (11 total), all frozen
- `HookEvent` base class with `event_name` and `timestamp`
- `HookResult` with `ok`/`abort`/`error` status
- `EventEmitter.emit()` dispatches to registered handlers by event name
- `before:` events support abort (any handler returns abort → `EmitResult.aborted=True`)
- Handler exceptions caught, logged, and skipped (never crash)

**SHOULD:**
- `EmitResult` reports handler errors for diagnostics
- Events have reasonable defaults for optional fields

**MUST NOT:**
- Use Pydantic for events (frozen dataclasses per AD-1)
- Add singleton pattern to EventEmitter
- Add timeout logic (deferred to S248.2)
- Add entry point discovery (S248.2 scope)
