### Task 2: HookRegistry with Entry Point Discovery
- **Description:** Create `hooks/registry.py` with `HookRegistry` class. `discover()` loads from `rai.hooks` entry points (same pattern as `adapters/registry.py`). Validates Protocol conformance. - **Files:** `src/rai_cli/hooks/registry.py` (create), `tests/test_hooks/test_registry.py` (create)
- **TDD Cycle:**
  - RED: Test discover returns empty when no entry points. Test get_hooks_for_event returns priority-sorted hooks. Test non-conformant entry points are skipped with warning.
  - GREEN: Implement `HookRegistry` using `importlib.metadata.entry_points`
  - REFACTOR: Extract shared patterns if any
- **Verification:** `pytest tests/test_hooks/test_registry.py -v && pyright src/rai_cli/hooks/registry.py`
- **Size:** M
- **Dependencies:** Task 1

### Task 3: EventEmitter Integration тАФ Registry + Timeout
- **Description:** Extend `EventEmitter` to accept optional `HookRegistry`. When provided, auto-register discovered hooks as handlers (priority-sorted). Existing bare `HandlerFn` API must still work (backward compat).
- **Files:** `src/rai_cli/hooks/emitter.py` (modify), `tests/test_hooks/test_emitter.py` (modify)
- **TDD Cycle:**
  - RED: Test emitter with registry auto-registers hooks. Test timeout fires for slow hook. Test bare HandlerFn still works (existing tests pass).
  - GREEN: Add `registry` param to `__init__`, wire hooks, add timeout wrapper
  - REFACTOR: Keep emit() clean тАФ extract timeout logic to helper
- **Verification:** `pytest tests/test_hooks/test_emitter.py -v && pyright src/rai_cli/hooks/emitter.py`
- **Size:** M
- **Dependencies:** Task 1, Task 2
