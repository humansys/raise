# Implementation Plan: S248.4 — Wire Events into CLI Commands

## Overview

- **Story:** S248.4
- **Size:** M
- **Dependencies:** S248.2 (hook protocol), S248.3 (telemetry hook)

## Tasks

### Task 1: Add `create_emitter()` factory to emitter module

- **Description:** Add a module-level factory function to `hooks/emitter.py` that creates an `EventEmitter` with a discovered `HookRegistry`. Export from `hooks/__init__.py`. This avoids 3-line setup duplication in every command.
- **Files:** `src/rai_cli/hooks/emitter.py`, `src/rai_cli/hooks/__init__.py`
- **TDD:** RED: test `create_emitter()` returns an `EventEmitter` with registry. GREEN: implement. REFACTOR: none expected.
- **Verification:** `pytest tests/test_hooks/test_emitter.py -x`
- **Size:** XS
- **Dependencies:** None

### Task 2: Wire `session:start` and `session:close` events

- **Description:** In `cli/commands/session.py`:
  - `start()`: emit `SessionStartEvent` after `save_developer_profile()` (line ~197), with `session_id` and developer name.
  - `close()`: emit `BeforeSessionCloseEvent` before close logic. If aborted, echo message and `raise typer.Exit(1)`. Emit `SessionCloseEvent` after close completes (both legacy and structured paths).
- **Files:** `src/rai_cli/cli/commands/session.py`, `tests/test_hooks/test_event_wiring.py` (new)
- **TDD:** RED: test that `start()` emits `SessionStartEvent`, test that `close()` emits `BeforeSessionCloseEvent` + `SessionCloseEvent`, test abort path. GREEN: implement. REFACTOR: extract common test fixtures if needed.
- **Verification:** `pytest tests/test_hooks/test_event_wiring.py -x`
- **Size:** S
- **Dependencies:** Task 1

- **Files:** `src/rai_cli/cli/commands/graph.py`, `tests/test_hooks/test_event_wiring.py`
- **TDD:** RED: test `build()` emits `GraphBuildEvent` with correct counts. GREEN: implement.

## Execution Order

1. Task 1 (factory — foundation)
2. Tasks 2, 3, 4, 5, 6 (parallel — independent wiring, but sequential for clean commits)

## Testing Strategy

Mock `create_emitter` at the module level in each command file to capture emitted events without triggering real hooks. Assert event type and payload fields. No need to test hook dispatch (already covered in S248.1/S248.2 tests).

## Risks

| Risk | Mitigation |
|------|------------|
| Existing tests break from emit side effects | Mock `create_emitter` in tests; emitter errors are isolated |
| Import cycle from hooks → CLI | `create_emitter` lives in hooks module, CLI imports from hooks — no cycle |
