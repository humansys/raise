## Heutagogical Checkpoint

### What did you learn?
- The `emit()` function routes through `get_personal_dir()` which adds `.raise/rai/personal/telemetry/` prefix — integration tests need to know the real path, not just base_path
- `wraps=` in `patch()` is the right pattern for redirecting real I/O to tmp_path while keeping real behavior

### What would you change about the process?
- E2E integration tests should be explicit in the plan as "write to real disk, verify file content" — not just "manual integration test"

### Are there improvements for the framework?
- PAT-E-450 candidate: E2E tests for I/O hooks should verify actual file output, not just mock calls. "Demo the hook firing" in a plan should mean a test that touches disk.

### What are you more capable of now?
- The hook pipeline (Protocol → Registry → Emitter → Hook) is now fully proven E2E with a real consumer. The pattern is mechanical for future hooks.
