# Story: Research Context & Harness Patterns

## User Story
As a RaiSE framework architect,
I want to benchmark Anthropic's context engineering and harness design patterns against RaiSE's current mechanisms,
so that I can make informed decisions about what to adopt, adapt, or reject for the 2.4.0 and enterprise roadmap.

## Acceptance Criteria

### Scenario: Evidence catalog produced
```gherkin
Given the 4 Anthropic articles covering context and harness patterns (Art.2, Art.3, Art.4, Art.6)
When research is conducted with epistemological rigor
Then an evidence catalog exists with claims triangulated from 3+ sources
And each claim has a confidence level (Very High / High / Medium / Low)
```

### Scenario: Benchmark matrix scored
```gherkin
Given the 4 gaps (G4, G10, G12, G14) mapped in the epic scope
When each gap is evaluated against the RaiSE blueprint
Then a benchmark matrix exists with 4-dimension scoring (Value to Core, Simplicity Cost, Observability Gain, Effort)
And priority is calculated as (Value + Observability) - Simplicity Cost
```

### Scenario: Recommendations actionable
```gherkin
Given the benchmark matrix
When recommendations are formulated
Then each gap has a verdict: Adopt (as-is), Adapt (with RaiSE modifications), or Reject (with principled rationale)
And at least one recommendation includes a concrete implementation story for the backlog
```

### Scenario: Effort scaling tension resolved
```gherkin
Given RaiSE's rule "full skill cycle always" vs Anthropic's "scale effort to task size"
When both positions are evaluated with evidence
Then a clear position is taken with rationale
And if the position changes a RaiSE principle, an ADR is proposed
```

## Examples (Specification by Example)

| Gap | RaiSE Mechanism | Anthropic Pattern | Possible Verdict |
|-----|----------------|-------------------|-----------------|
| G4: Context mgmt | Session bundles (~600 tokens) | Compaction/reset policy per task | Adapt — add policy layer |
| G10: Effort scaling | Full skill cycle always | Parametrize by task size | Evaluate — tension with principles |
| G12: Agent test design | 237 tests, human-first | Grep-friendly, budget-aware | Adapt — low cost improvements |
| G14: Planner ambition | No scope fence | Calibrate scope expansion | Adopt — addresses known problem |
