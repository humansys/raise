## Task Breakdown

### T1: Deep Read of 4 Anthropic Articles (S)

**Description:** Structured read of Art.4, Art.2, Art.3, Art.6 in dependency-first order. Extract concrete claims, evidence levels, and RaiSE mappings per article.

**Reading order (dependency-first):**
1. Art.4 — Context Engineering (foundational for G4)
2. Art.2 — Effective Harnesses (foundational for G10, G14)
3. Art.3 — Harness Design (refines G10, G14)
4. Art.6 — Multi-Agent Research (secondary evidence for G12, G14)

**Output:** Raw claim extractions per article (working notes in research report draft).

**TDD analog:** RED = articles unread, claims unextracted. GREEN = each article has 3-5 extracted claims with evidence levels. REFACTOR = consolidate duplicate claims across articles.

**Verification:**
- Each article has at least 3 extracted claims
- Each claim has: source reference, what Anthropic says, evidence level (data/example/assertion)
- RaiSE mapping noted for each claim

**Dependencies:** None (first task).
**Size:** S (~40 min)

---

### T2: Build Evidence Catalog with Triangulated Claims (S)

**Description:** Consolidate raw claims from T1 into a structured evidence catalog. Triangulate each major claim against 3 sources: (1) the Anthropic article, (2) RaiSE operational data (sessions, patterns, known friction), (3) first-principles reasoning from RaiSE architectural principles.

**Output:** Evidence catalog section in research report with 12-20 entries.

**Catalog entry format per design:**
```
### Claim: [Short title]
- Source: Art.N, section reference
- Anthropic says: [quote or precise paraphrase]
- RaiSE reality: [what we actually do/have]
- Triangulation: [Source 2], [Source 3]
- Confidence: Very High / High / Medium / Low
- Maps to gap: G4 / G10 / G12 / G14
```

**TDD analog:** RED = raw claims without triangulation. GREEN = each claim has 3 sources and confidence level. REFACTOR = merge overlapping claims, sharpen confidence assessments.

**Verification:**
- 12-20 catalog entries exist
- All 4 gaps (G4, G10, G12, G14) have at least 3 entries each
- Each entry has confidence level with explicit rationale
- At least 1 triangulation uses RaiSE operational data (not just principles)

**Dependencies:** T1 (claims extracted).
**Size:** S (~20 min)

---

### T3: Score Benchmark Matrix (4 Gaps x 4 Dimensions) (S)

**Description:** Score each of the 4 gaps on the 4 evaluation dimensions defined in epic scope. Calculate priority using the formula: (Value to Core + Observability Gain) - Simplicity Cost. Include 1-sentence rationale per cell.

**Dimensions:**
| Dimension | Scale | Question |
|-----------|-------|----------|
| Value to Core | 0-3 | Strengthens memory, methodology, or entity? |
| Simplicity Cost | 0-3 | Complexity added? (lower = better) |
| Observability Gain | 0-3 | More traceable/auditable? |
| Adoption Effort | XS-XL | Story points to implement |

**Priority thresholds:**
- 4-6: Do now (2.4.0)
- 1-3: Plan for enterprise
- 0 or negative: Reject or defer

**Output:** Completed benchmark matrix with scores, rationales, and priority ranking.

**TDD analog:** RED = empty matrix. GREEN = all 16 cells scored with rationale. REFACTOR = validate scores against evidence catalog (no score without supporting evidence).

**Verification:**
- All 4 gaps scored on all 4 dimensions
- Each score has 1-sentence rationale
- Priority calculated correctly per formula
- Scores are consistent with evidence catalog confidence levels
- No score asserted without evidence catalog reference

**Dependencies:** T2 (evidence catalog informs scoring).
**Size:** S (~20 min)

---

### T4: Resolve G10 Effort Scaling Tension (M)

**Description:** Follow the 4-step deliberation framework from the design document to resolve the tension between RaiSE's "full skill cycle always" (rule #3) and Anthropic's "scale effort to task size" pattern.

**Deliberation steps:**
1. State both positions clearly (with strongest arguments)
2. Gather evidence for each (session data, patterns, operational friction)
3. Test against RaiSE principles (P1, P5, P7 specifically)
4. Decide and document (ADR proposal if principle changes, rationale if not)

**Output:** G10 resolution section in research report with explicit position, evidence, and next steps.

**TDD analog:** RED = tension unresolved, no position taken. GREEN = explicit position with evidence from both sides. REFACTOR = ensure position is internally consistent with other verdicts.

**Verification:**
- Both positions stated with strongest arguments (no strawman)
- Evidence cited for each position (minimum 2 data points per side)
- Tested against P1, P5, P7 with explicit reasoning
- Clear verdict: keep principle / refine principle / change principle
- If changing principle: ADR proposed with before/after
- If keeping/refining: rationale documented with evidence

**Dependencies:** T2 (evidence catalog), T3 (scoring context). This is the highest-risk task — scheduled after scoring to have full context.
**Size:** M (~20 min, but conceptually the hardest task)

---

### T5: Write Verdicts and Backlog Impact (S)

**Description:** For each gap, produce a final verdict (Adopt / Adapt / Reject) with supporting rationale. Draft at least 1 concrete implementation story for the backlog. Cross-reference downstream stories (RAISE-783 for G4, enterprise roadmap for lower-priority items).

**Output:**
- Verdict section in research report (4 verdicts with rationale)
- Backlog impact section (new stories, reprioritization recommendations)
- Cross-references to RAISE-783 and enterprise roadmap

**TDD analog:** RED = no verdicts, no backlog impact. GREEN = all 4 gaps have verdicts with rationale and at least 1 backlog story drafted. REFACTOR = ensure verdicts are consistent with scores and evidence.

**Verification:**
- Each gap has Adopt / Adapt / Reject verdict
- Each verdict cites evidence catalog entries and benchmark scores
- At least 1 implementation story drafted with title, description, and size estimate
- RAISE-783 cross-reference included for G4 findings
- No "Anthropic says so" as sole rationale for any verdict

**Dependencies:** T3 (benchmark scores), T4 (G10 resolution feeds G10 verdict).
**Size:** S (~20 min)

---

## Execution Order

```
T1 (Deep Read) → T2 (Evidence Catalog) → T3 (Benchmark Matrix) → T4 (G10 Resolution) → T5 (Verdicts & Backlog)
```

**Rationale:** Strictly sequential — each task builds on the previous. No parallelism possible because:
- T2 needs T1's raw claims to triangulate
- T3 needs T2's evidence to score
- T4 needs T2+T3 for full context on the highest-stakes decision
- T5 needs T3+T4 to write coherent verdicts

**Risk-first ordering:** T4 (G10 tension) is the riskiest task (may change a RaiSE principle). It is placed after T3 so it has maximum context, but before T5 so its resolution informs the final verdicts.

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Articles not available or gated | Low | High | Fetch articles at T1 start; if unavailable, use cached summaries from epic scope |
| Evidence insufficient for confident scoring | Medium | Medium | Score with available evidence; mark low-confidence cells explicitly; note as limitation |
| G10 resolution is inconclusive | Medium | High | Default to "refine" (add nuance) rather than binary keep/change. Document the ambiguity honestly |
| Scope creep into implementation details | Medium | Low | This is analysis only — no code. Redirect implementation ideas to backlog stories |
| Over-indexing on Anthropic's authority | Low | High | Triangulation requirement (3 sources) prevents single-source verdicts. "Reject" is a valid outcome |
