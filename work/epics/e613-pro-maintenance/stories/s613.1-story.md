# S613.1: rai doctor ACLI Availability and Auth Check

## User Story

**As a** raise-pro user,
**I want** `rai doctor` to verify that ACLI is installed and authenticated,
**So that** I get a clear diagnostic when Jira commands fail instead of cryptic subprocess errors.

## Acceptance Criteria

```gherkin
Scenario: ACLI installed and authenticated
  Given acli is in PATH and authenticated
  When rai doctor runs
  Then it shows PASS for acli-installed and acli-authenticated

Scenario: ACLI not installed
  Given acli is not in PATH
  When rai doctor runs
  Then it shows ERROR with install instructions

Scenario: ACLI installed but not authenticated
  Given acli is in PATH but not authenticated
  When rai doctor runs
  Then it shows WARN with auth command hint
```
