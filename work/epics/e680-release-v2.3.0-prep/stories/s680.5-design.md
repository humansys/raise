# S680.5: Smoke Test Design

## Summary

Execute 6 end-to-end smoke tests against v2.3.0 features and document results.
This is a document-only story — no bug fixes in scope.

## Approach

Run each verification command, capture output, document pass/fail in results file.

## Verifications

| # | Verification | Command | Pass Criteria |
|---|-------------|---------|---------------|
| V1 | Session roundtrip | `rai session start --project . --context` then `rai session close --summary "smoke test" --type maintenance --project .` | Both complete without error |
| V2 | ACLI adapter | `rai backlog search "RAISE-680" -a jira` | Returns results, no adapter errors |
| V3 | Clean install | `uv venv /tmp/raise-smoke && uv pip install -e . --python /tmp/raise-smoke/bin/python` then `rai --help` | Install succeeds, help shows |
| V4 | rai doctor | `rai doctor` | Runs without crash, shows ACLI status |
| V5 | Pattern add scope | `rai pattern add "smoke test pattern" -c "test" -t process` | Pattern added to project scope (not personal) |
| V6 | CLI extensions | `rai --help` | Extensions listed or no crash |

## Results Format

Results go to: `work/epics/e680-release-v2.3.0-prep/stories/s680.5-results.md`

```markdown

| # | Test | Status | Notes |

| V3 | Clean install | PASS/FAIL | evidence |
| V4 | rai doctor | PASS/FAIL | evidence |
| V5 | Pattern add scope | PASS/FAIL | evidence |

```

## Acceptance Criteria

**MUST:**
- All 6 verifications executed
- Results documented with pass/fail and evidence
- Any failures documented with severity assessment

**SHOULD:**
- Clean install tested in isolated venv

**MUST NOT:**
- Fix bugs found during smoke test (document only)

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | Document only, no fixes | Smoke test validates release readiness; fixes go to separate stories |
| D2 | 6 verifications covering major v2.3.0 features | Covers session (E654), ACLI (E494), doctor (RAISE-614), pattern scope (RAISE-608), install, extensions |
