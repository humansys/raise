# S680.3: User & Dev Docs (Community Only)

## User Story

As a RaiSE Community user upgrading to v2.3.0,
I want updated documentation covering new Community features and breaking changes,
So that I can successfully adopt session identity and CLI extensions.

## Acceptance Criteria

```gherkin
Scenario: Session identity documentation
  GIVEN the session docs
  WHEN a user reads them
  THEN they understand the new session identity model (E654)

Scenario: CLI extensions documentation
  GIVEN the extending guide
  WHEN a user reads them
  THEN they understand CLI extensions via entry points (RAISE-594)

Scenario: Pattern scope default documentation
  GIVEN the pattern command docs
  WHEN a user reads them
  THEN they understand default scope changed to project (RAISE-608)

Scenario: Breaking changes documented
  GIVEN any breaking change
  WHEN a user encounters it
  THEN docs explain what changed and how to adapt
```

## Notes

**REDUCED SCOPE:** This story covers Community docs only. Pro features (Jira/ACLI adapter) are excluded because Pro and Community documentation should not be mixed. See parking lot entry "Docs Audience Tagging + Package-Aware Publish".
