# S680.4 Design: Quality gates & security

## Approach

Run 7 automated verification tools against the dev branch codebase. Document results. Fix any failures before proceeding.

No architecture decisions, no code changes, no new abstractions. Pure verification.

## Tools

| Tool | Purpose | Pass Criteria |
|------|---------|---------------|
| `pytest` | Full test suite | 100% pass rate |
| `pyright` | Type checking (strict) | 0 errors |
| `ruff check` | Linting | 0 violations |
| `rai gate check --all` | Framework quality gates | All gates green |
| `rai skill validate` | Skill structure validation | All 19 skills valid |
| `pip-audit` | Dependency CVE scan | 0 known vulnerabilities |
| `rai graph build` | Knowledge graph freshness | Builds without error |

## Failure Protocol

If any tool reports failures:
1. Assess severity (blocker vs. advisory)
2. Fix blockers in this story before proceeding
3. Log advisory findings for follow-up
4. Re-run failed tool to confirm fix

## Output

Results summary documented as evidence for S680.5 (smoke test) and S680.2 (release notes).
