# S680.3: User & Dev Docs Design — Community Only

## Summary

Update 3 existing doc files to reflect v2.3.0 Community features. No new pages, no Pro features.

## Approach

Read each target file, cross-reference source code and epic docs for accuracy, add/modify sections. Mark breaking changes with admonitions.

## Components

### 1. `docs/cli/session.mdx` — Session Identity (E654)

**Add section after existing content, before "See also":**

- **Session Identity (v2.3.0)** — new ID format `S-{prefix}-{YYMMDD}-{HHMM}`, deterministic and timestamp-based
- Developer prefix registry at `.raise/rai/prefixes.yaml` — auto-registers on first `rai session start`, collision detection with suggested alternatives
- Per-project session data in `.raise/rai/personal/` (gitignored) — active pointer, session index, working state
- Privacy model: only `prefixes.yaml` is committed; session data is local by default, opt-in sharing via `.gitignore`

**Breaking change callout:**
- Session data path moved from `~/.rai/developer.yaml` (global) to `.raise/rai/personal/` (per-project)
- Old `SES-NNN` format sessions remain readable but new sessions use new format exclusively
- No automatic migration

### 2. `docs/guides/extending.mdx` — CLI Extensions (RAISE-594)

**Add new section "CLI Extensions" after Lifecycle Hooks, before Validation:**

- Entry point group: `rai.cli.commands`
- Convention: register a `typer.Typer` app in `pyproject.toml`
- Example pyproject.toml entry
- Protection: built-in command name collision detection, duplicate extension detection
- Validation: extensions appear in `rai --help` when installed

### 3. `docs/cli/pattern.mdx` — Default Scope Change (RAISE-608)

**Modify existing `rai pattern add` table:**
- Change `--scope` default from `personal` to `project`

**Add breaking change callout after the flags table:**
- Before: `rai pattern add "..." → personal scope`
- After: `rai pattern add "..." → project scope`
- Migration: add `--scope personal` explicitly for old behavior

- AC3: Pattern docs note default scope changed from `personal` to `project`
- AC4: Breaking changes clearly marked with admonition/callout
- AC5: Before/after examples for breaking changes

**SHOULD:**
- AC6: Cross-reference between related docs where helpful

**MUST NOT:**
- AC7: Reference Pro features (Jira/ACLI adapter)
- AC8: Break existing doc structure/navigation
