# S680.1 Design — Epic Documentation

## Approach

Run `/rai-epic-docs` skill for each of 3 completed epics (E478, E494, E654).
Per epic, generate 5 documentation sections and publish to Confluence (RaiSE1 space).

## Sections per Epic

1. **Worked example** — step-by-step walkthrough of the feature in action
2. **Extension guide** — how to extend/customize the feature
3. **Data flow** — key data paths through the system
4. **Invariants** — guarantees the implementation maintains
5. **Failure modes** — what can go wrong and how it's handled

## Input Sources per Epic

| Source | Description |
|--------|-------------|
| `scope.md` | Epic scope and objectives |
| `design.md` | Architecture decisions |
| `retrospective.md` | Learnings and patterns |
| Git log | Commit history for the epic's stories |
| Source code | Implementation details |

## Output

- Confluence pages in **RaiSE1** space
- One page per epic with all 5 sections

## Sequencing Decision

Start with E478 (simplest, 3 stories) as warmup to validate the `/rai-epic-docs` workflow,
then E494 (largest, 7 stories), then E654 (4 stories).

**Rationale:** Validates the process on the smallest scope first, applies lessons to the larger epics.

## Risks

- `/rai-epic-docs` output quality may need manual editing before publish
- Confluence publish failures (handled by retry / manual fallback)
