# S680.2 — Changelog & release notes

## User Story

As a **user of raise-cli**,
I want **a structured changelog entry for v2.3.0 and GitHub release notes**,
So that **I can understand what changed, what was fixed, and what to watch out for when upgrading**.

## Objective

Create a CHANGELOG.md entry for v2.3.0 following the Keep a Changelog format, covering Added/Changed/Fixed/Security sections. Draft GitHub release notes with summary, highlights, known issues, and contributor recognition.

## Acceptance Criteria

```gherkin
Scenario: CHANGELOG.md has v2.3.0 entry
  Given the CHANGELOG.md file exists
  When I read the v2.3.0 section
  Then it follows Keep a Changelog format
  And it contains an "Added" section with ACLI adapter, session identity, CLI extensions, ISO 27001, rai doctor
  And it contains a "Changed" section with MCP→ACLI migration, session data move, pattern scope default
  And it contains a "Fixed" section listing individual bugfixes
  And it contains a "Security" section with authlib, PyJWT, and astro/cloudflare/undici CVE fixes
  And the release date is 2026-03-30

Scenario: GitHub release notes draft exists
  Given the release notes draft exists
  When I review its contents
  Then it has a summary paragraph
  And it has categorized highlights (features, fixes, security)
  And it lists known issues (RAISE-539, RAISE-634, RAISE-213)
  And it credits contributors
```

## Scope

- **In scope:** CHANGELOG.md v2.3.0 entry, GitHub release notes draft
- **Out of scope:** Publishing the release (S680.6), code changes, fixing open bugs
