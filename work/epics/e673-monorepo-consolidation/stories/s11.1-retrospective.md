# S11.1 Retrospective

## What was delivered
- 5 STRAT Initiatives (STRAT-24..28) with OKR links
- 10 rai-agent Epics migrated to RAISE with component `rai-agent`
- 5 new stories (RAISE-674..678) for monorepo phases
- Portfolio kanban conventions documented
- Easy Agile TeamRhythm installed for cross-project visualization
- RAGENT project created then emptied (monorepo pivot)
- Old RAI stories obsoleted with redirect references

## What changed mid-story
1. **Multi-repo → Monorepo**: Started with separate RAGENT project + GitLab repo.
   User questioned overhead for 5-dev team. Research confirmed monorepo pattern
   (AutoGen, LangChain). Pivoted to raise-commons monorepo with uv workspaces.
2. **RAGENT → RAISE**: Created RAGENT project, migrated 10 Epics, then realized
   single RAISE project with components is simpler. Recreated in RAISE, deleted RAGENT.
3. **ProductGo → Easy Agile**: ProductGo can't do cross-project grouping.
   Easy Agile TeamRhythm supports it natively.

## Patterns learned
- For small teams (≤10 devs), monorepo + single Jira project beats multi-repo + multi-project
- Cross-project Jira links are a constant friction source — avoid when possible
- "Parent to Child relations" (10478) is the link type ProductGo/Easy Agile recognize
- Think → validate → create (SES-035 lesson still applies — we deleted RAGENT issues)

## Waste

- ~30 minutes on ProductGo cross-project research before discovering limitation
- Multiple link type corrections (10411 vs 10478)

## Time
