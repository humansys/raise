# S11.6: Distribution Packaging

## User Story (Connextra)

As a **rai-agent user**,
I want **a Dockerfile and updated documentation for the monorepo**,
so that **I can self-host rai-agent via Docker and understand the project structure**.

## Acceptance Criteria (Gherkin)

```gherkin
Feature: Distribution Packaging

  Scenario: rai-agent Docker image builds
    Given packages/rai-agent/Dockerfile exists
    When I run `docker build -f packages/rai-agent/Dockerfile .`
    Then the image builds successfully
    And the image contains the rai-agent package

  Scenario: docker-compose includes rai-agent service
    Given docker-compose.yml has a rai-agent service
    When I run `docker compose config`
    Then the config is valid
    And rai-agent service references the correct Dockerfile

  Scenario: rai-agent wheel builds correctly
    Given the monorepo workspace is set up
    When I run `uv build --package rai-agent`
    Then a wheel is produced in dist/
    And it contains only rai_agent/ source

  Scenario: README reflects monorepo structure
    Given README.md exists
    When a new user reads it
    Then they understand the 5-package workspace structure
    And they know how to install and run each package
```
