# S11.4 Design — rai-agent Integration

## Problem & Value

**Problem:** rai-agent code (daemon, knowledge, inference) lives isolated in the personal
repo without shared CI, unified lockfile, or independent publishability.

**Value:** With rai-agent in the monorepo, it shares the lockfile, runs tests in the same
CI, and enables `pip install rai-agent` as a distributable package.

## Approach

Copy 33 product files and 45 test files from the personal repo into `packages/rai-agent/`
per the handoff manifest (`package-manifest.yaml`). Wire as a uv workspace member with
its own pyproject.toml. Update root workspace config and tool paths.

## Decisions

| # | Decision | Rationale |
|---|----------|-----------|
| D1 | Depend on raise-core, not raise-cli | Zero imports of raise_cli verified by grep — only raise_core.graph used |
| D2 | Exclude rai-agent from GitHub mirror | OSS decision pending, reversible — add to sync-github.sh exclusions |
| D3 | Omit CLI entry points | Ontology extractor placement in package distribution TBD |
| D4 | Copy integration tests as-is | They use mocks/local infra, not external services — mark if they fail |

## Components

| Component | Change | Files |
|-----------|--------|-------|
| `packages/rai-agent/` | create | src/rai_agent/ (33 files), tests/ (45 files), pyproject.toml |
| `packages/rai-agent/pyproject.toml` | create | base deps + extras ([telegram], [scheduling], [gchat]) |
| Root `pyproject.toml` | modify | workspace sources, dev deps, pytest/pyright/coverage/ruff paths |
| `scripts/sync-github.sh` | modify | add packages/rai-agent to exclusion list |

## Acceptance Criteria

- AC1: `uv sync` resolves rai-agent as workspace member
- AC2: `uv run pytest packages/rai-agent/tests/` passes
- AC3: `uv run pytest` passes (all packages, no regressions)
- AC4: `uv build --package rai-agent` produces wheel with only rai_agent/
- AC5: Cross-package import `rai_agent.knowledge.retrieval.engine → raise_core.graph` works
- AC6: ruff + pyright clean
- AC7: sync-github.sh excludes rai-agent
