# S11.3 Implementation Plan

## Overview

Convert raise-commons to a proper uv workspace monorepo. Each task leaves the
workspace in a resolvable state. Risk-first ordering: source moves before tests,
core before cli (smaller blast radius first).

## Tasks

### T1: Extract raise-core to packages/ (RISKIEST — do first)
**Size:** S | **Deps:** none | **AC:** AC1, AC4

Create `packages/raise-core/` with own pyproject.toml, move source, update root
hatch config to stop including raise_core in the raise-cli wheel.

**Files:**
- Create: `packages/raise-core/pyproject.toml`
- Move: `src/raise_core/` → `packages/raise-core/src/raise_core/`
- Modify: root `pyproject.toml` (remove raise_core from hatch wheel packages, add raise-core as workspace source)

**Verification:**
```bash
uv sync
uv run python -c "from raise_core.graph.engine import Graph; print('OK')"
```

**Why first:** Smallest package, fewest deps, known risk from S463/S516. If this
breaks, we learn early with minimal blast radius.

---

### T2: Extract raise-cli to packages/ + root becomes workspace-only
**Size:** M | **Deps:** T1 | **AC:** AC2, AC3, AC4

Create `packages/raise-cli/` with own pyproject.toml (inheriting all deps, entry
points, hatch config from current root). Convert root pyproject.toml to
workspace-only (remove [project], [build-system], entry points). Keep all tool
config ([tool.ruff], [tool.pyright], etc.) in root.

**Files:**
- Create: `packages/raise-cli/pyproject.toml`
- Move: `src/raise_cli/` → `packages/raise-cli/src/raise_cli/`
- Modify: root `pyproject.toml` (remove [project], [build-system], entry points; update workspace sources)

**Verification:**
```bash
uv sync
uv run rai --version
uv run python -c "from raise_cli.cli.main import app; print('OK')"
```

**Why second:** Larger package, all entry points. Depends on raise-core being
resolvable as workspace member (T1).

## Execution Order

```
T1 (raise-core extract) → T2 (raise-cli extract + root conversion)
                              │
                              ├→ T3 (move tests) → T4 (tool configs) ─┐
                              │                                        ├→ T6 (final)
                              └→ T5 (sync script) ────────────────────┘
```

**Rationale:** Risk-first. T1 is the known risk (S463/S516 prior art). T2 is the
largest change. T3-T5 are lower risk. T5 is independent of T3-T4 so can be done
in parallel. T6 gates everything.

## Risks

| Risk | Task | Mitigation |
|---|---|---|
| uv sync fails after raise-core extraction | T1 | Verify immediately; revert if broken |
| Entry points don't resolve from packages/raise-cli | T2 | Check `rai --version` immediately |
| pytest can't discover tests in packages/*/tests/ | T3 | Update testpaths before running |
| Import-linter contracts fail with new paths | T4 | May need root_packages update |
| sync-github.sh pyproject patching breaks | T5 | Test with dry-run before push |
