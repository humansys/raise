## Heutagogical Checkpoint

1. **Learned:** claude-agent-sdk spawns the claude binary via SubprocessCLITransport.
   Auth is via env vars, with 6 supported methods including `claude setup-token` for
   subscription-based headless auth.
2. **Process:** Verifying assumptions about SDK internals (how auth works, what gets
   spawned) should happen BEFORE design, not during implementation.
3. **Framework:** Infrastructure stories with unknowns (like "how does the SDK actually
   work?") should include a spike/research task in the plan.
4. **Capability:** Docker multi-stage builds for monorepo workspace subsets with
   proprietary package exclusion.
