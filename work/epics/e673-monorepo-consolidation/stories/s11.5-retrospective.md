## Heutagogical Checkpoint

1. **Learned:** Native GHA path filters are sufficient for small monorepos. The temptation to add external actions or custom scripts is unnecessary when transitive deps are static.
2. **Process:** AR before implementation adds value for infrastructure decisions. Q1 (uv sync strategy) would have been an implicit choice without it.
3. **Framework:** Infrastructure stories (YAML/CI) don't fit TDD literally. Plan documented "structural verification" — this is the correct adaptation, not a framework gap.
4. **Capability:** Reusable pattern for uv workspace CI/CD (reusable+callers). Directly applicable if workspace grows.

## Decisions Made

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Native GHA paths filters | Zero deps, sufficient for 5 packages |
| D2 | Reusable workflow + callers | GHA requires paths in on: trigger, can't parametrize |
| D3 | {package}-v{version} tags | Standard monorepo convention |
| Q1 | `uv sync` plain | Local parity, extras not needed for tests |
| Q2 | `pyright` global | Uses pyrightconfig.json, negligible time difference |
| Q3 | Single `pypi` environment | No benefit to per-package at team size 5 |
