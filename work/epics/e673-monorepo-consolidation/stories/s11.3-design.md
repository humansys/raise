# S11.3 Design: uv Workspace Setup

## Problem & Value

raise-commons bundles raise-core and raise-cli into a single wheel with shared versioning.
This prevents rai-agent from depending on raise-core alone and blocks independent package publishing.

Moving each package to `packages/` with its own pyproject.toml enables granular dependencies,
independent versioning, and unblocks S11.4-S11.6.

## Approach

1. Move `src/raise_core/` → `packages/raise-core/src/raise_core/` with own pyproject.toml
2. Move `src/raise_cli/` → `packages/raise-cli/src/raise_cli/` with own pyproject.toml
3. Move `tests/` → per-package `tests/` directories based on import analysis
4. Convert root pyproject.toml to workspace-only (no [project] section)
5. Update all tool config paths (ruff, pyright, pytest, coverage, import-linter)
6. Update sync-github.sh to selective exclusion (only raise-pro, raise-server)

**Key insight from S463/S516:** Module paths don't change (`from raise_core.graph...` stays the same).
Only physical location moves. Zero import changes needed.

## Components

| Component | Change | Notes |
|---|---|---|
| `src/raise_core/` | move → `packages/raise-core/src/raise_core/` | New pyproject.toml, deps: pydantic, networkx |
| `src/raise_cli/` | move → `packages/raise-cli/src/raise_cli/` | New pyproject.toml, inherits all current deps + entry points |
| `pyproject.toml` (root) | modify | Workspace-only: [tool.uv.workspace] + tool configs, no [project] |
| `packages/raise-pro/pyproject.toml` | verify | Already depends on raise-cli via workspace — should still resolve |
| `tests/` | move | Per-package: raise-core (4), raise-cli (241), raise-pro (31) |
| `scripts/sync-github.sh` | modify | Exclude `packages/raise-pro`, `packages/raise-server` only |
| `uv.lock` | regenerate | `uv sync` after restructure |

## Decisions

| # | Choice | Rationale | Alternatives |
|---|---|---|---|
| D1 | Tests per package | Auto leak prevention — proprietary tests excluded by directory, not manual list | Tests in root with selective sync (fragile) |
| D2 | Root pyproject.toml keeps tool config | One place for ruff/pyright/pytest config, workspace-wide consistency | Per-package tool config (duplication) |
| D3 | raise-core deps: only pydantic + networkx | Verified by grep — only third-party imports in raise_core | Include all deps (over-specification) |
| D4 | Selective sync exclusion (Opción A) | Minimal change to existing script, inverts what's excluded | Separate GitHub repos per package (triple maintenance) |
| D5 | Same version (2.3.0) for both packages initially | Matches current release, independent versioning comes later | Start at 0.1.0 for raise-core (confusing) |

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Global `rai` command breaks (TN-003) | Document workaround: `pip install -e packages/raise-core -e packages/raise-cli` |
| Hatch build misconfigured — wheel includes wrong files | Verify with `uv build` + inspect wheel contents |
| pytest can't find tests after move | Update testpaths in root pyproject.toml |
| raise-pro entry points break after restructure | Verify `rai adapter list` post-move |

## Acceptance Criteria

**MUST:**
1. `packages/raise-core/` exists with own pyproject.toml, `src/raise_core/`, and `tests/`
2. `packages/raise-cli/` exists with own pyproject.toml, `src/raise_cli/`, and `tests/`
3. Root pyproject.toml has no `[project]` section
4. `uv sync` succeeds
5. `uv run pytest` passes (all packages)
6. `uv run ruff check` passes
7. `uv run pyright` passes

**SHOULD:**
1. `sync-github.sh` excludes only proprietary packages
2. Wheel contents verified (`uv build` for raise-core and raise-cli)

**MUST NOT:**
1. Change any `from raise_core.*` or `from raise_cli.*` import
2. Modify raise-pro or raise-server source code
3. Change the `rai` CLI entry point behavior
