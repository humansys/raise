# S11.4 Scope

## In Scope

- Create `packages/rai-agent/` directory structure with `pyproject.toml`
- Copy 33 product files from rai personal repo per manifest (daemon/, knowledge/, inference.py)
- Copy 45 test files from rai personal repo per manifest
- Configure pyproject.toml with base deps + optional extras ([telegram], [scheduling], [gchat])
- Wire workspace dependency on raise-core (not raise-cli — per manifest analysis)
- Add rai-agent to root workspace config and dev dependency group
- Update tool configs (pytest testpaths, pyright include, coverage sources, ruff src)
- Update sync-github.sh if rai-agent should be mirrored to GitHub
- Verify `uv sync`, all tests pass, wheel builds correctly

## Out of Scope

- `scaleup/` directory (private, stays in personal repo)
- CI/CD pipeline changes (that's S11.5)
- Docker/distribution packaging (that's S11.6)
- Removing code from personal repo (that's S11.7)
- Publishing to PyPI
- CLI plugin entry point registration (future story)

## Done Criteria

1. `packages/rai-agent/` exists with `src/rai_agent/` and `tests/`
2. `packages/rai-agent/pyproject.toml` has base deps + extras per manifest
3. `uv sync` resolves all workspace members including rai-agent
4. `uv run pytest packages/rai-agent/tests/` passes
5. `uv run pytest` passes (all packages, no regressions)
6. `uv run ruff check` passes
7. `uv run pyright` passes
8. `uv build --package rai-agent` produces correct wheel (only rai_agent/)
9. Cross-package import `rai_agent.knowledge.retrieval.engine → raise_core.graph` works
