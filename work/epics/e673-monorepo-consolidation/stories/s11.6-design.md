# S11.6 Design: RAISE-677 — Distribution Packaging

## Problem & Value

rai-agent is integrated in the monorepo but has no Docker packaging, no quickstart
for third parties, and the README doesn't reflect the 5-package workspace.

A user should be able to run rai-agent with `docker compose up` — no Python, no uv,
no manual dependency installation.

## Approach

### D1: Multi-stage Dockerfile

Stage 1 (node): Install Claude Code CLI via npm.
Stage 2 (python): Copy CLI from stage 1, install rai-agent via uv.

claude-agent-sdk requires the `claude` binary — it spawns it as a subprocess
via SubprocessCLITransport. No API key needed if using subscription auth.

### D2: Auth-agnostic .env.example

Document all 6 auth scenarios supported by Claude Code CLI:
1. Subscription (Pro/Max) via `claude setup-token` → CLAUDE_CODE_TOKEN
2. Anthropic API key (BYOK) → ANTHROPIC_API_KEY
3. AWS Bedrock → CLAUDE_CODE_USE_BEDROCK + AWS credentials
4. Google Vertex → CLAUDE_CODE_USE_VERTEX + GCP credentials
5. Azure Foundry → CLAUDE_CODE_USE_FOUNDRY + Azure credentials
6. LLM Proxy → ANTHROPIC_AUTH_TOKEN + ANTHROPIC_BASE_URL

The Dockerfile doesn't know or care which auth method is used.

### D3: README update

Add "Project Structure" section showing 5 packages + quickstart Docker instructions.

## Components

| Component | Change |
|-----------|--------|
| `packages/rai-agent/Dockerfile` | **create** |
| `docker-compose.yml` | **modify** — add rai-agent service |
| `.env.example` | **create** |
| `README.md` | **modify** — monorepo structure + Docker quickstart |

## Acceptance Criteria

### MUST
- AC1: `docker build -f packages/rai-agent/Dockerfile .` succeeds
- AC2: Image contains `claude` binary and `rai_agent` package
- AC3: `docker compose config` validates with rai-agent service
- AC4: `.env.example` documents all 6 auth scenarios
- AC5: README shows 5-package structure and Docker quickstart
- AC6: `uv build --package rai-agent` produces correct wheel

### MUST NOT
- AC7: No auth credentials hardcoded in Dockerfile or compose
- AC8: No changes to existing services in docker-compose.yml
