# S11.3 Retrospective

## Summary

Converted raise-commons from a single-package repo to a uv workspace monorepo.
Moved raise-core and raise-cli to packages/ with independent pyproject.toml files,
moved 276 test files per-package, and updated the GitHub sync script.

## What Went Well

- **Risk-first ordering** — extracting raise-core first (smallest, known risk from S463/S516)
  gave confidence before the larger raise-cli extraction
- **Zero import changes** — module paths stayed the same, only physical locations moved
- **Prior art** — S463/S516/TN-003 documented the gotchas, so we avoided them

## What to Improve

- **Test move ordering** — the "move everything else" sweep caught pro tests that should
  have been moved by explicit list first (PAT-E-699)
- **README.md path** — should have caught the hatch relative path issue in design phase,
  not during T6 wheel verification (PAT-E-698)

## Heutagogical Checkpoint

1. **Learned:** pytest `--import-mode=importlib` is essential for multi-package workspaces.
   Without it, `tests/__init__.py` across packages creates namespace collisions.
2. **Process change:** For multi-step file moves, move by explicit list in dependency order,
   not by "move specific, then sweep rest."
3. **Framework improvements:** None — the skill cycle worked well for refactoring.
4. **More capable of:** uv workspace monorepo conversion. This pattern is now proven
   and can be reused for S11.4 (rai-agent integration).

## Patterns Added

- **PAT-E-697:** pytest --import-mode=importlib for monorepo test collection
- **PAT-E-698:** hatch readme relative path breaks wheel builds
- **PAT-E-699:** move tests by explicit list first, sweep remainder last

## Final State

- 6 tasks, all complete
- 3983 tests passing, 0 failures
- Ruff clean, pyright 0 errors
- Both wheels build correctly (no cross-contamination)
