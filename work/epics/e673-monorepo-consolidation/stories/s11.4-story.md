# S11.4: rai-agent Integration

## User Story (Connextra)

As a **RaiSE platform developer**,
I want **the rai-agent product code integrated into raise-commons as packages/rai-agent/**,
so that **daemon, knowledge, and inference modules are developed, tested, and published from the monorepo alongside raise-cli and raise-core**.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: rai-agent package resolves in workspace
  Given packages/rai-agent/ exists with pyproject.toml
  And the root workspace includes rai-agent as a member
  When I run `uv sync`
  Then rai-agent resolves with all base + optional dependencies
  And a single uv.lock is generated

Scenario: Cross-package imports work
  Given rai-agent depends on raise-core as workspace member
  When I run `uv run python -c "from rai_agent.knowledge.retrieval.engine import RetrievalEngine"`
  Then the import resolves raise_core.graph from the workspace

Scenario: rai-agent tests pass
  Given all 45 test files are in packages/rai-agent/tests/
  When I run `uv run pytest packages/rai-agent/tests/ --tb=short`
  Then all tests pass with no import errors

Scenario: Existing packages unaffected
  Given rai-agent is added to the workspace
  When I run `uv run pytest`
  Then all existing tests (raise-core, raise-cli, raise-pro) still pass

Scenario: rai-agent wheel builds correctly
  Given packages/rai-agent/pyproject.toml defines hatch wheel config
  When I run `uv build --package rai-agent`
  Then the wheel contains only rai_agent/ (no raise_cli, raise_core, etc.)

Scenario: Optional extras install correctly
  Given rai-agent defines [telegram], [scheduling] extras
  When I run `uv sync --extra telegram --extra scheduling`
  Then python-telegram-bot, apscheduler, and related deps are installed
```
