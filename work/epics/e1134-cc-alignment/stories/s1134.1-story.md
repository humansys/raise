# Story: Description Optimization for All Skills

## User Story
As a RaiSE developer using Claude Code,
I want skill descriptions optimized for CC's automatic selection system,
so that CC accurately invokes the right skill when relevant and stays within the 8K description budget.

## Acceptance Criteria

### Scenario: Description within budget
```gherkin
Given a skill with an optimized description
When CC loads the skill listing
Then the description is not truncated (under 250 chars)
And the key use case is visible in the first 60 chars
```

### Scenario: Accurate auto-selection
```gherkin
Given a skill with verb-first trigger phrases in its description
When the user's request matches the trigger phrases
Then CC selects the correct skill automatically
```

## Examples (Specification by Example)

| Skill | Before (chars) | After (chars) | Pattern |
|-------|:--------------:|:-------------:|---------|
| rai-session-start | 147 | <100 | Verb-first, trigger phrase |
| rai-debug | 197 | <100 | Verb-first, "Use when..." |
| rai-publish | 153 | <100 | Verb-first, context |

## Notes
- CC truncates at 250 chars, but Anthropic targets ~60 chars for clean display
- Total budget: 1% of context window (~8K chars for 35 skills)
- With 13 skills now manual-only (S1134.3), only 22 descriptions load into context
- Design decision D1 in design.md defines the pattern: verb + object + context
