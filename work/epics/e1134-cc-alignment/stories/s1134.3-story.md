# Story: Invocation Control for Side-Effect Skills

## User Story
As a RaiSE developer using Claude Code,
I want side-effect skills to require manual invocation,
so that Claude doesn't auto-trigger destructive operations like merges, publishes, or branch creation.

## Acceptance Criteria

### Scenario: Side-effect skill not auto-invoked
```gherkin
Given a skill with disable-model-invocation: true (e.g., rai-publish)
When Claude encounters a context that matches the skill description
Then Claude does NOT auto-invoke the skill
And the skill is only available via /rai-publish
```

### Scenario: Non-side-effect skill remains auto-invocable
```gherkin
Given a skill without disable-model-invocation (e.g., rai-debug)
When Claude encounters unexpected errors
Then Claude CAN auto-invoke the skill as before
```

## Examples (Specification by Example)

| Skill | Side Effect | disable-model-invocation |
|-------|-------------|:------------------------:|
| rai-epic-close | Pushes to origin, creates MR | true |
| rai-story-close | Merges branch, updates tracker | true |
| rai-publish | Publishes to PyPI | true |
| rai-epic-start | Creates tracker entries, emits signals | true |
| rai-story-start | Creates branches, commits | true |
| rai-session-close | Writes session state | true |
| rai-framework-sync | Writes multiple locations | true |
| rai-epic-run | Orchestrates full epic lifecycle | true |
| rai-story-run | Orchestrates full story lifecycle | true |
| rai-debug | None (analysis only) | false (default) |
| rai-research | None (analysis only) | false (default) |
