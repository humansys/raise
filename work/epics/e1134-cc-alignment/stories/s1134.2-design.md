# S1134.2: Allowed-Tools Declaration — Design

## Summary

Add `allowed-tools` frontmatter to all 35 skill SKILL.md files. Each skill is classified into a tool tier based on what tools its instructions actually reference, then the appropriate allowed-tools list is declared in frontmatter.

## Complexity: Moderate

35 files to modify. Requires reading each skill body to classify correctly, but the pattern is mechanical once classification is done. D2 design needed extension from 6 to 7 tiers to cover governance-writing skills.

## Approach

1. Read each SKILL.md body to identify tool references (Read, Edit, Write, Bash, git, rai CLI, etc.)
2. Classify each skill into the appropriate tier
3. Add `allowed-tools` YAML list to frontmatter, placed after `disable-model-invocation` (or `description` if absent), before `license`
4. Validate YAML syntax on all 35 files

### Design Decision: D2 Tier Extension

The epic design (D2) defined 6 tiers but only explicitly classified ~22 skills. Reading the remaining 13 skills revealed a gap: skills like `rai-discover`, `rai-project-create`, `rai-epic-design`, and `rai-story-design` write files (Edit/Write) and call rai CLI, but do not use git directly. They don't fit Read+CLI (too restrictive) or Full dev (too permissive).

**Resolution:** Split into 7 tiers, adding T3 (Read + Write + CLI) and renaming the original Read+CLI+Git to T4 (Read + Write + CLI + Git) since those skills also need Edit/Write for scope files and tracking updates.

## Tier Classification (All 35 Skills)

### T1: Read-only
**Tools:** `Read, Grep, Glob`
**Rationale:** Pure analysis skills — read code, read governance, produce conversation output only.

| # | Skill | Evidence |
|---|-------|----------|
| 1 | rai-architecture-review | Reads code + governance, produces review findings in conversation |
| 2 | rai-code-audit | Reads modules against code-standards.md, produces scorecard |
| 3 | rai-doctor | Runs `rai doctor` diagnostics, interprets results conversationally |
| 4 | rai-mcp-status | Runs `rai mcp health/list`, presents status table |
| 5 | rai-problem-shape | Conversation-only skill, produces problem brief through dialogue |
| 6 | rai-quality-review | Reads changed files, produces audit findings |
| 7 | rai-session-start | Runs `rai session start`, interprets context bundle |

**Note on T1 + Bash(rai:*):** doctor, mcp-status, and session-start call `rai` CLI commands. However, these are read-only CLI calls (status checks, context loading). Including them in T1 (without Bash) means the skill body's `rai` command snippets are documentation only — the agent reads CLI output that was pre-loaded via the context bundle or runs it outside the skill. If this proves too restrictive in practice, promote to T2.

### T4: Read + Write + CLI + Git
**Tools:** `Read, Edit, Write, Grep, Glob, Bash(rai:*), Bash(git:*)`
**Rationale:** Skills that create branches, commit, merge, or push. Also need Edit/Write for scope files and tracking updates.

| # | Skill | Evidence |
|---|-------|----------|
| 24 | rai-epic-close | Pushes to origin, creates MR; writes retrospective |

### T5: Full dev
**Tools:** `Read, Edit, Write, Grep, Glob, Bash`
**Rationale:** Skills that run tests, linters, type checkers, and arbitrary development commands. Need unrestricted Bash.

| # | Skill | Evidence |
|---|-------|----------|
| 30 | rai-bugfix | Full TDD cycle: write code, run tests, run linters |
| 31 | rai-debug | Runs arbitrary diagnostic commands, modifies code |
| 32 | rai-story-implement | Full TDD: RED-GREEN-REFACTOR, `uv run pytest`, `uv run pyright` |

### T6: Research
**Tools:** `Read, Grep, Glob, Bash(ddgr:*), WebFetch, WebSearch`
**Rationale:** Skills that search the web and fetch external content for evidence gathering.

| # | Skill | Evidence |
|---|-------|----------|
| 33 | rai-research | Uses ddgr, WebFetch, WebSearch for evidence triangulation |

### T7: Orchestration
**Tools:** `Read, Grep, Glob, Bash, Agent, Skill`
**Rationale:** Skills that chain other skills and spawn subagents. Need Bash for context commands and Skill/Agent for delegation.

| # | Skill | Evidence |
|---|-------|----------|
| 34 | rai-epic-run | Orchestrates epic-start, epic-design, epic-plan, epic-close |
| 35 | rai-story-run | Orchestrates story-start through story-close |

## Acceptance Criteria

- AC1: All 35 skills declare `allowed-tools` in YAML frontmatter
- AC2: Bash patterns use subcommand granularity — never bare `Bash` except T5 and T7
- AC3: Each skill's tier matches its actual tool usage in the skill body
- AC4: Field placement consistent (after `disable-model-invocation` or `description`, before `license`)
- AC5: No skill content (body) changed — frontmatter only
- AC6: YAML valid on all 35 skills (no syntax errors)
