# S353.1 Design: Checkpoint Contract

## Problem

The artifact I/O table in epic design.md was written from memory. Gemba revealed several discrepancies — missing reads (manifest.yaml, story.md), missing writes (progress.md), and a critical finding: AR and QR produce no persistent artifacts (verdicts are inline only).

## Approach

Replace the assumed table with a Gemba-verified contract. Flag the AR/QR inline-only pattern as a design consideration for S353.2 (forked subagents need a way to return verdicts).

## Verified Artifact I/O Contract

### Per-phase contract (Gemba-verified)

| Phase | Classification | Reads from disk | Writes to disk | Returns inline |
|-------|:-------------:|----------------|----------------|----------------|
| start | Light (inline) | `.raise/manifest.yaml`, epic `scope.md` | `s{N}.{M}-story.md`, `s{N}.{M}-scope.md` | branch name |
| design | Heavy (fork) | `s{N}.{M}-story.md`, epic `scope.md` (optional) | `s{N}.{M}-design.md` | — |
| plan | Heavy (fork) | `s{N}.{M}-design.md` (optional), `s{N}.{M}-story.md` | `s{N}.{M}-plan.md` | — |
| implement | Heavy (fork) | `s{N}.{M}-plan.md`, `s{N}.{M}-design.md`, `.raise/manifest.yaml` | source code, tests, commits, `progress.md` | task completion summary |
| AR | Heavy (fork) | `s{N}.{M}-design.md`, `.raise/manifest.yaml`, git diff | **nothing** | verdict (PASS/PASS WITH QUESTIONS/SIMPLIFY) + findings |
| QR | Heavy (fork) | `.raise/manifest.yaml`, git diff | **nothing** | verdict (PASS/PASS WITH RECOMMENDATIONS/FAIL) + findings |
| review | Heavy (fork) | story artifacts, `.raise/manifest.yaml`, behavioral patterns | `s{N}.{M}-retrospective.md` | patterns emitted via CLI |
| close | Light (inline) | `s{N}.{M}-retrospective.md`, `.raise/manifest.yaml`, epic `scope.md` | epic scope update | merge commit |

### Key findings from Gemba

1. **AR and QR are inline-only** — they don't write files. For forked subagents, the verdict and findings must come back as the Agent tool return value. This is the natural mechanism (Agent tool returns the subagent's final output).

2. **`.raise/manifest.yaml` is read by almost every phase** — it provides project config (test command, lint command, language). Subagents will read this from disk naturally (it's a project file, not conversation state).

3. **`progress.md` is created by implement** — tracks task-by-task progress within the phase. Useful for resume but not consumed by subsequent phases.

4. **plan reads story.md too** — not just design.md. Story.md has the AC that plan decomposes into tasks.

5. **review emits patterns via CLI** — `rai pattern add` is a side effect. Works identically from subagent (CLI is stateless).

### Subagent return contract

For phases that write artifacts to disk, the orchestrator confirms success by checking the file exists. For AR/QR (inline-only), the orchestrator reads the verdict from the Agent tool return:

| Phase type | Success signal | Orchestrator reads |
|-----------|---------------|-------------------|
| Artifact-producing (design, plan, implement, review) | File exists on disk | File path confirmation |
| Inline-only (AR, QR) | Agent return contains verdict | Verdict + findings from return value |
