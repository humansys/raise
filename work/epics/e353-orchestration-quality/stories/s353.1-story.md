# Story: Checkpoint Contract

## User Story
As the story-run orchestrator,
I want an explicit artifact I/O contract per phase,
so that forked subagents know exactly what to read from disk and what to write.

## Acceptance Criteria

### Scenario: Phase reads correct prerequisites
```gherkin
Given the artifact I/O contract is documented in design.md
When a phase skill (e.g., plan) is invoked as a subagent
Then it knows to read s{N}.{M}-design.md from disk
And it does not depend on conversation context from prior phases
```

### Scenario: Contract matches current skill behavior
```gherkin
Given each phase skill already produces specific artifacts
When the contract is documented
Then it reflects actual current behavior (not aspirational)
And no new artifact formats are introduced
```

## Notes
- The contract already exists implicitly in the phase detection table (scope.md Step 0)
- This story makes it explicit in design.md for subagent consumption
- No code changes — only documentation of existing artifact I/O
- Feeds S353.2 (story-run fork) and S353.3 (epic-run checkpoint)
