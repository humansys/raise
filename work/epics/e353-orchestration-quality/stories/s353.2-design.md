# S353.2 Design: story-run Checkpoint & Fork

## Problem

rai-story-run executes all phases inline in a single context. By phase 5/8, context is saturated and inference quality degrades 4.6x vs standalone.

## Value

Each heavy phase gets fresh context via Agent tool fork. Quality parity with standalone invocation regardless of chain position.

## Approach

Modify Step 2 (Execute Skill Chain) to classify phases as light (inline) or heavy (fork). Heavy phases spawn an Agent tool subagent with the phase's SKILL.md content as the prompt. The orchestrator stays thin — only phase detection, gates, and banner output.

## Components Affected

- `src/rai_cli/skills_base/rai-story-run/SKILL.md` — modify Step 2 (add fork logic)
- `.claude/skills/rai-story-run/SKILL.md` — mirror

### 2. Add fork execution protocol (new section after chain order)

For each **fork** phase, the orchestrator:

1. Reads the skill's SKILL.md from `src/rai_cli/skills_base/rai-{skill-name}/SKILL.md`
2. Spawns Agent tool with:
   - `subagent_type: "general-purpose"`
   - `prompt` containing:
     - The full SKILL.md content
     - Story context: story_id, epic path, paths to prior artifacts on disk
     - Instruction to read CLAUDE.md for project context
     - Instruction to write artifacts to the correct path
3. Waits for agent completion
4. Verifies output:
   - Artifact-producing phases (design, plan, implement, review): check file exists on disk
   - Inline-only phases (AR, QR): read verdict from agent return value
5. Shows completion banner (in main thread)
6. Applies delegation gate if applicable (in main thread)

For each **inline** phase, behavior is unchanged — execute the skill directly in the current context.

### 3. Agent prompt template

```
Execute the following skill for story {story_id}.

## Skill Instructions
{SKILL.md content}

## Story Context
- Story ID: {story_id}
- Epic: {epic_id}
- Epic path: work/epics/{epic_slug}/
- Stories path: work/epics/{epic_slug}/stories/
- Prior artifacts: {list of files that exist on disk for this story}

## Instructions
1. Read CLAUDE.md for project-level context
2. Read the prior artifacts listed above
3. Execute every step in the skill instructions above
4. Write all output artifacts to the paths specified
5. Return a brief summary of what you did and any verdicts/decisions made

ARGUMENTS: {story_id}
```

### 4. Modify full execution rule

Current rule says "Load the skill's SKILL.md, execute every step." For fork phases, this becomes: "Load the skill's SKILL.md, spawn Agent tool with skill content, verify output."

The subagent inherits the full execution rule — the SKILL.md IS the prompt, so it executes every step naturally in its fresh context.

## Acceptance Criteria

- **MUST:** 6 heavy phases fork via Agent tool (design, plan, implement, AR, QR, review)
- **MUST:** 2 light phases stay inline (start, close)
- **MUST:** Delegation gates apply in main thread after fork returns
- **MUST:** AR/QR SIMPLIFY/FAIL stops execution regardless of delegation
- **MUST:** Phase detection unchanged (artifact reverse-scan)
- **SHOULD:** Completion banners show after fork returns (in main thread)
- **MUST NOT:** Pass conversation context to forked subagent (only disk artifacts + SKILL.md)
