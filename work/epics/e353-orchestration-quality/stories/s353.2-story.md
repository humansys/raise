# Story: story-run Checkpoint & Fork

## User Story
As a developer running /rai-story-run,
I want heavy phases (design, plan, implement, AR, QR, review) to fork to fresh-context subagents,
so that each phase produces quality identical to standalone invocation regardless of position in the chain.

## Acceptance Criteria

### Scenario: Heavy phase forks to subagent
```gherkin
Given I invoke /rai-story-run for a story
When the orchestrator reaches a heavy phase (e.g., design)
Then it spawns an Agent tool subagent with the phase skill's SKILL.md content
And the subagent reads prior artifacts from disk (not conversation context)
And the subagent writes output artifacts to disk
And the orchestrator reads a summary from the agent return
```

### Scenario: Light phases stay inline
```gherkin
Given I invoke /rai-story-run for a story
When the orchestrator reaches start or close
Then it executes the phase inline (no fork)
```

### Scenario: AR/QR verdicts return inline
```gherkin
Given the orchestrator forks AR or QR to a subagent
When the subagent completes
Then the verdict (PASS/SIMPLIFY/FAIL) is in the agent return value
And the orchestrator applies the delegation gate based on that verdict
```

### Scenario: Delegation gates still work
```gherkin
Given a heavy phase completes in a forked subagent
When the orchestrator reads the summary
Then it applies the delegation gate (REVIEW/NOTIFY/AUTO) in the main thread
And SIMPLIFY/FAIL verdicts stop execution regardless of delegation level
```
