# S1051.4: Confluence Discovery Service

## User Story

**As a** RaiSE developer setting up Confluence integration,
**I want** a discovery service that queries spaces, page trees, and labels from a live Confluence instance,
**so that** the doctor and setup skill can validate config and generate it from real backend state.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: List available spaces
  Given a configured Confluence client
  When I call discover_spaces()
  Then I get a list of SpaceInfo with key, name, and homepage_id

Scenario: Get page tree for a space
  Given a configured Confluence client and a space key
  When I call discover_page_tree(space_key, max_depth)
  Then I get a tree of PageNode with id, title, children, and labels

Scenario: Get labels for known pages
  Given a configured Confluence client and a list of page IDs
  When I call discover_labels(page_ids)
  Then I get a mapping of page_id to list of labels

Scenario: Produce structured space map
  Given discovery results for spaces, page trees, and labels
  When I call build_space_map(space_key)
  Then I get a SpaceMap with space info, page tree, and label index

Scenario: Discovery handles API errors gracefully
  Given a Confluence client that returns errors
  When I call any discovery method
  Then errors are normalized and wrapped in DiscoveryError

Scenario: Discovery respects depth limit
  Given a space with deeply nested pages
  When I call discover_page_tree(space_key, max_depth=2)
  Then the tree is truncated at depth 2
```
