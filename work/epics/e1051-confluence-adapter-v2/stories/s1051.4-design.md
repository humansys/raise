# S1051.4: Confluence Discovery Service -- Design

## Problem
The doctor (S1051.5) and config setup (S1051.6) need to query live Confluence
state -- spaces, page trees, labels -- to validate config and auto-generate it.
No service exists to bridge ConfluenceClient's low-level methods into
structured discovery results.

## Value
Discovery gives the doctor and setup skill ground truth from the live backend.
Without it, config validation is syntactic only (field types) and setup
requires the user to manually look up space keys, page IDs, and label names.

## Complexity: Simple
4 Pydantic models, 1 service class, 0 external integrations beyond ConfluenceClient.
Algorithm: recursive tree walk with depth limit. Well-bounded.

## Approach

### Target files
```
packages/raise-cli/src/raise_cli/adapters/
  confluence_discovery.py      -- ConfluenceDiscoveryService + models
packages/raise-cli/tests/adapters/
  test_confluence_discovery.py -- Unit tests with mocked ConfluenceClient
```

### Value preservation gate
**What domain knowledge does this layer provide that a generic pass-through wouldn't?**

1. **Depth-limited recursive tree walk** -- ConfluenceClient.get_page_children is flat (one level). Discovery composes it into a bounded recursive traversal with configurable max_depth.
2. **Label aggregation** -- Builds an inverted index (label -> page_ids) from the tree, which no single client method provides.
3. **SpaceMap composite** -- Joins space info + homepage resolution + page tree + label index into one coherent snapshot that doctor and setup consume as a unit.
4. **Error normalization** -- Translates ConfluenceError subtypes into DiscoveryError with actionable context (which space, which page failed).

### Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Models in same file as service | 4 small models tightly coupled to one service; separate file is overhead for no reuse gain |
| D2 | SpaceInfo reused from models/docs.py | Already has key, name, url, type -- no duplication needed |
| D3 | PageNode is new (not PageSummary) | PageSummary has no children or labels -- tree needs recursive structure |
| D4 | DiscoveryError wraps ConfluenceError | Consumers (doctor, setup) should not depend on client exception hierarchy |
| D5 | Default max_depth=3 | Typical Confluence space has 2-3 levels. Prevents runaway traversal on flat-dump spaces |
| D6 | get_labels per page during tree walk | Batching would require a second pass; per-page during walk is simpler and sufficient for discovery volumes |
| D7 | Service takes ConfluenceClient, not config | Caller (doctor/setup) already has a client; avoids double-constructing |
| D8 | homepage_id on SpaceInfo via client.get_space_homepage_id | SpaceInfo from get_spaces() lacks homepage; discovery enriches it |

### Models

```python
class PageNode(BaseModel):
    """Recursive page tree node with labels."""
    id: str
    title: str
    labels: list[str] = Field(default_factory=list)
    children: list[PageNode] = Field(default_factory=list)

class SpaceMap(BaseModel):
    """Complete structural snapshot of a Confluence space."""
    space: SpaceInfo
    homepage_id: str
    page_tree: PageNode
    label_index: dict[str, list[str]]  # label -> [page_id, ...]

class DiscoveryError(Exception):
    """Discovery-layer error wrapping client errors."""
    def __init__(self, message: str, cause: Exception | None = None) -> None:
        self.message = message
        self.cause = cause
        super().__init__(message)
```

```python
    """Queries live Confluence for structural metadata.

    Consumed by doctor (validation) and setup (config generation).
    All methods raise DiscoveryError on failure.
    """

    DEFAULT_MAX_DEPTH: int = 3

    def __init__(self, client: ConfluenceClient) -> None:
        self._client = client

    def discover_spaces(self) -> list[SpaceInfo]:

    ) -> PageNode:
        """Build recursive page tree from space homepage.

        Resolves homepage via client.get_space_homepage_id(),
        then walks children recursively up to max_depth.
        Labels are fetched per page during the walk.
        """
        ...

    def discover_labels(self, page_ids: list[str]) -> dict[str, list[str]]:
        """Get labels for a list of pages. Returns {page_id: [label, ...]}."""
        ...

    def build_space_map(
        self, space_key: str, max_depth: int = DEFAULT_MAX_DEPTH
    ) -> SpaceMap:
        """Composite: space info + page tree + label index.

        1. Find space in discover_spaces() by key
        2. Resolve homepage ID
        3. Build page tree with labels
        4. Build inverted label index from tree
        """
        ...

    def _walk_children(
        self, page_id: str, depth: int, max_depth: int
    ) -> list[PageNode]:
        """Recursive child page walk with depth limit."""
        ...

    @staticmethod
    def _build_label_index(tree: PageNode) -> dict[str, list[str]]:
        """Build inverted index: label -> [page_id, ...] from tree."""
        ...
```

### discover_page_tree flow

```
1. homepage_id = client.get_space_homepage_id(space_key)
2. If None -> DiscoveryError("Space {space_key} has no homepage")
3. homepage_labels = client.get_labels(homepage_id)
4. root = PageNode(id=homepage_id, title=<from client>, labels=homepage_labels)
5. root.children = _walk_children(homepage_id, depth=1, max_depth=max_depth)
6. Return root
```

### _walk_children flow

```
1. If depth > max_depth -> return []
2. children_summaries = client.get_page_children(page_id)
3. For each child:
   a. labels = client.get_labels(child.id)
   b. grandchildren = _walk_children(child.id, depth+1, max_depth)
   c. node = PageNode(id=child.id, title=child.title, labels=labels, children=grandchildren)
4. Return nodes
```

### build_space_map flow

```
1. spaces = discover_spaces()
2. Find space where key == space_key, else DiscoveryError
3. tree = discover_page_tree(space_key, max_depth)
4. label_index = _build_label_index(tree)
5. homepage_id = tree.id  (root of tree IS the homepage)
6. Return SpaceMap(space=space_info, homepage_id=homepage_id, page_tree=tree, label_index=label_index)
```

### Error normalization

```python
def _wrap_error(self, error: Exception, context: str) -> DiscoveryError:
    """Wrap any exception in DiscoveryError with context."""
    if isinstance(error, DiscoveryError):
        return error
    return DiscoveryError(f"{context}: {error}", cause=error)
```

Each public method uses try/except to wrap ConfluenceError into DiscoveryError.

### Error handling
```python
# Space not found
try:
    service.build_space_map("NONEXISTENT")
except DiscoveryError as e:
    print(e.message)  # "build_space_map: space 'NONEXISTENT' not found"

# API failure during tree walk
try:
    service.discover_page_tree("RaiSE1")
except DiscoveryError as e:
    print(e.message)  # "discover_page_tree(RaiSE1): get_page_children(12345): 503 Service Unavailable"
    print(type(e.cause))  # <class 'ConfluenceApiError'>
```

## Acceptance Criteria

### MUST
- AC1: ConfluenceDiscoveryService produces SpaceMap from mocked ConfluenceClient
- AC2: discover_page_tree respects max_depth parameter (truncates at limit)
- AC3: label_index is correctly built as inverted index from page tree traversal
- AC4: API errors (ConfluenceError subtypes) produce DiscoveryError with actionable message and cause chain
- AC5: All new models (PageNode, SpaceMap, DiscoveryError) are typed and tested

### SHOULD
- S1: discover_labels handles missing/deleted pages gracefully (empty list, no error)
- S2: Default max_depth=3 is sensible for typical spaces

### MUST NOT
- MN1: Must not fetch page content/body (discovery is structural only)
- MN2: Must not cache results (stateless per call)
- MN3: Must not import ConfluenceClient at module level (optional dep guard stays in client)
