# S1051.5: Adapter Doctor — Confluence — Scope

## In Scope

- Extend AdapterDoctorCheck._check_confluence_health with config-vs-backend validation
- Validate space_key from config exists on live instance (via ConfluenceDiscovery)
- Validate routing parent_title pages exist in configured space
- Report available spaces in fix_hint when space is missing
- Report actionable fix_hint for each mismatch type
- All new checks gated behind context.online (no live calls in offline mode)
- Unit tests with mocked ConfluenceClient/ConfluenceDiscovery
- Error isolation: discovery failures produce ERROR results, never crash doctor

## Out of Scope

- Label validation (labels are additive, no meaningful mismatch to detect)
- Config file schema validation (already covered by Pydantic models in S1051.3)
- Env var checks (already implemented in existing AdapterDoctorCheck)
- New CLI commands (doctor already has --online flag)
- Jira doctor enhancements (separate story)
- Config auto-fix or auto-generation (S1051.6)

## Done When

- `rai doctor --online` reports space existence check for each configured instance
- Missing space produces ERROR with list of available spaces
- Missing routing parent page produces ERROR with page title and space
- Discovery API errors produce ERROR with connectivity hint (no crash)
- Offline mode skips all config-vs-backend checks (existing behavior preserved)
- All tests pass, pyright clean, ruff clean
