# S1051.3: Config Schema + Pydantic Models

## User Story

**As a** RaiSE user with Confluence integration,
**I want** a validated YAML config schema for Confluence instances,
**so that** the adapter knows how to connect, which space to target, and how to route artifacts.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Valid multi-instance config loads successfully
  Given a .raise/confluence.yaml with default_instance and instances map
  When I load the config
  Then I get a validated ConfluenceConfig with resolved default instance

Scenario: Minimal config with just space_key works
  Given a .raise/confluence.yaml with only url, username, space_key
  When I load the config
  Then it loads as a single "default" instance

Scenario: Routing maps artifact types to parent pages and labels
  Given a config with routing section per instance
  When I resolve routing for artifact_type "adr"
  Then I get the parent_title and labels for that type

Scenario: Missing required fields fail validation
  Given a config missing url or space_key
  When I load the config
  Then a ValidationError is raised with clear message

Scenario: Config loader reads from .raise/confluence.yaml
  Given a project with .raise/confluence.yaml
  When I call load_confluence_config(project_root)
  Then the file is parsed and validated into ConfluenceConfig
```

## Specification by Example

```yaml
# .raise/confluence.yaml — full multi-instance
default_instance: humansys

instances:
  humansys:
    url: "https://humansys.atlassian.net/wiki"
    username: "emilio@humansys.ai"
    space_key: "RaiSE1"
    routing:
      adr:
        parent_title: "Architecture Decision Records"
        labels: ["adr", "architecture"]
      roadmap:
        parent_title: "Roadmap"
        labels: ["roadmap", "planning"]
```

```yaml
# .raise/confluence.yaml — minimal single instance
url: "https://humansys.atlassian.net/wiki"
username: "emilio@humansys.ai"
space_key: "RaiSE1"
```
