# S1051.6: Config Generator — `/rai-adapter-setup` Confluence — Design

## Problem

The existing `confluence_config_gen.py` (from S1130.4) uses the old `ConfluenceSpaceMap` (v1 discovery) and produces a flat dict with `url`, `space_key`, `routing` — not the multi-instance `ConfluenceConfig` schema from S1051.3. The `/rai-adapter-setup` skill references this generator but the output does not match the current config model. A developer setting up Confluence must hand-wire YAML or fix the generated output.

## Value

After this story, `/rai-adapter-setup` generates a valid `.raise/confluence.yaml` that passes `ConfluenceConfig.from_dict()` on the first try. New users go from `CONFLUENCE_API_TOKEN` set → working config in under 5 minutes, zero manual YAML editing.

## Complexity: Moderate

- 3 components (config builder, YAML writer, skill update)
- 1 external integration (ConfluenceDiscoveryService)
- Trivial algorithm (selection → dict mapping)
- Follows established Jira config gen pattern (S1130.5)

## Approach

**Replace** the existing `confluence_config_gen.py` with a new implementation that:
1. Takes `ConfluenceDiscoveryService` results (v2: `SpaceInfo` list from `discover_spaces()`) + user selections
2. Produces a dict that passes `ConfluenceConfig.from_dict()` (multi-instance format)
3. Includes routing suggestions based on discovered top-level page titles
4. Writes YAML via a separate serializer function

**Separate concerns:**
- `generate_confluence_config()` — pure function: discovery data + selections → dict (testable, no I/O)
- `suggest_routing()` — pure function: page tree → routing suggestions (testable, no I/O)
- `write_confluence_config()` — I/O function: dict → `.raise/confluence.yaml` file
- `/rai-adapter-setup` SKILL.md — already correct in structure; update Confluence imports to use v2 discovery

**Pattern:** Follows Jira config gen (S1130.5) exactly — pure function signature, ValueError for bad input, dict output validated by model.

### Decisions

| ID | Decision | Rationale | Alternatives |
|----|----------|-----------|--------------|
| D1 | Replace existing `confluence_config_gen.py` in-place | Same file, same module path — no import changes needed in skill or tests | New file alongside old (more churn) |
| D2 | Use `discover_spaces()` not `build_space_map()` for space selection | Space selection only needs keys/names; full page tree is expensive and unnecessary for initial config | `build_space_map()` (over-fetches) |
| D3 | Routing suggestions from page tree are optional/separate | Config gen works without them; suggestions are a convenience for the interactive skill flow | Required page tree discovery (blocks on slow API) |
| D4 | Output multi-instance format always | Consistent with ConfluenceConfig.from_dict() and Jira pattern; flat format is handled by from_dict() anyway | Flat format (simpler but inconsistent) |
| D5 | `write_confluence_config()` as separate function | I/O isolation — pure gen is testable without tmp_path; write tested separately | Single function with write flag |

### Components

| File | Change |
|------|--------|
| `packages/raise-cli/src/raise_cli/adapters/confluence_config_gen.py` | MODIFY — replace v1 with v2 |
| `packages/raise-cli/tests/adapters/test_confluence_config_gen.py` | CREATE — unit tests |
| `.claude/skills/rai-adapter-setup/SKILL.md` | MODIFY — update Confluence discovery imports if needed |

## Examples

### generate_confluence_config() — happy path

```python
from raise_cli.adapters.confluence_config_gen import generate_confluence_config
from raise_cli.adapters.confluence_config import ConfluenceConfig, ArtifactRouting

config_dict = generate_confluence_config(
    spaces=[
        SpaceInfo(key="RaiSE1", name="RaiSE Project", url="/wiki/spaces/RaiSE1", type="global"),
        SpaceInfo(key="TEAM", name="Team Space", url="/wiki/spaces/TEAM", type="global"),
    ],
    selected_space="RaiSE1",
    instance_url="https://humansys.atlassian.net/wiki",
    instance_name="humansys",
    routing={
        "adr": ArtifactRouting(parent_title="Architecture Decision Records", labels=["adr"]),
        "roadmap": ArtifactRouting(parent_title="Roadmaps", labels=["roadmap"]),
    },
)

# Output dict:
# {
#     "default_instance": "humansys",
#     "instances": {
#         "humansys": {
#             "url": "https://humansys.atlassian.net/wiki",
#             "space_key": "RaiSE1",
#             "instance_name": "humansys",
#             "routing": {
#                 "adr": {"parent_title": "Architecture Decision Records", "labels": ["adr"]},
#                 "roadmap": {"parent_title": "Roadmaps", "labels": ["roadmap"]},
#             },
#         }
#     },
# }

# Validates:
config = ConfluenceConfig.from_dict(config_dict)
assert config.default_instance == "humansys"
assert config.get_instance("humansys").space_key == "RaiSE1"
```

```python
from raise_cli.adapters.confluence_config_gen import generate_confluence_config

generate_confluence_config(
```

### generate_confluence_config() — default routing

```python
config_dict = generate_confluence_config(
    spaces=[SpaceInfo(key="RaiSE1", name="RaiSE Project", url="", type="global")],
    selected_space="RaiSE1",
    instance_url="https://humansys.atlassian.net/wiki",
    instance_name="humansys",
    # routing=None → uses _DEFAULT_ROUTING
)

config = ConfluenceConfig.from_dict(config_dict)
routing = config.get_instance("humansys").routing
assert "adr" in routing
assert "developer" in routing
```

### suggest_routing() — from page tree

```python
from raise_cli.adapters.confluence_config_gen import suggest_routing
from raise_cli.adapters.confluence_discovery import PageNode

tree = PageNode(
    id="100", title="RaiSE Project", labels=[],
    children=[
        PageNode(id="101", title="Architecture Decision Records", labels=["adr"], children=[]),
        PageNode(id="102", title="Roadmaps", labels=["roadmap"], children=[]),
        PageNode(id="103", title="Developer Docs", labels=[], children=[]),
    ],
)

suggestions = suggest_routing(tree)
# -> {
#     "adr": ArtifactRouting(parent_title="Architecture Decision Records", labels=["adr"]),
#     "roadmap": ArtifactRouting(parent_title="Roadmaps", labels=["roadmap"]),
# }
# Matches top-level page titles against known artifact type keywords
```

## Acceptance Criteria

### MUST
- **AC1:** `generate_confluence_config()` returns a dict that passes `ConfluenceConfig.from_dict()` validation
- **AC2:** ValueError raised when `selected_space` is not in the provided spaces list
- **AC3:** Default routing applied when no routing argument provided
- **AC4:** `write_confluence_config()` writes valid YAML to `.raise/confluence.yaml`
- **AC5:** `write_confluence_config()` raises `FileExistsError` when config exists and `overwrite=False`

### SHOULD
- **S1:** `suggest_routing()` matches top-level page titles against known artifact type keywords (adr, roadmap, developer, retrospective)
- **S2:** Generated YAML includes comments explaining key fields

### MUST NOT
- **MN1:** Must not write secrets (API tokens, passwords) to config file
- **MN2:** Must not import `ConfluenceClient` at module level (optional dependency guard)
- **MN3:** Must not perform any I/O in `generate_confluence_config()` (pure function)
