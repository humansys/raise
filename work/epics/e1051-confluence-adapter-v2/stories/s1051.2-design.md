# S1051.2: PythonApiConfluenceAdapter — Design

## Problem
ConfluenceClient (S1051.1) and config schema (S1051.3) exist, but nothing
wires them into the `DocumentationTarget` protocol that `rai docs` commands consume.

## Value
Walking skeleton: `rai docs publish` works with Confluence via pure Python.

## Complexity: Moderate
1 new file, 5 protocol methods, routing logic, entry point registration.

## Approach

### Target files
```
raise-cli/src/raise_cli/adapters/confluence_adapter.py  — NEW
raise-cli/tests/adapters/test_confluence_adapter.py      — NEW
raise-cli/pyproject.toml                                  — MODIFY (entry point)
```

### Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Sync implementation directly | ConfluenceClient is sync; no need for async + SyncDocsAdapter wrapping |
| D2 | No-arg constructor | Resolver calls `cls()` — adapter loads config from CWD |
| D3 | project_root param with CWD default | Testable without patching CWD |
| D4 | publish: create if not exists, update if exists | Idempotent — lookup by title first |
| D5 | can_publish: True if routing configured OR no routing (publish everything) | Permissive default — routing is optional |
| D6 | Import guard same as client | Optional dep — clear error if not installed |

### PythonApiConfluenceAdapter

```python
class PythonApiConfluenceAdapter:
    """Confluence adapter implementing DocumentationTarget.

    Sync implementation — ConfluenceClient is already sync.
    Loaded via entry point: rai.docs.targets = confluence
    """

    def __init__(self, project_root: Path | None = None) -> None:
        root = project_root or Path.cwd()
        self._config = load_confluence_config(root)
        instance = self._config.get_instance()
        self._client = ConfluenceClient(instance)

    def can_publish(self, doc_type: str, metadata: dict[str, Any]) -> bool:
        """True if routing is configured for doc_type, or if no routing at all."""
        ...

    def publish(
        self, doc_type: str, content: str, metadata: dict[str, Any]
    ) -> PublishResult:
        """Publish doc: resolve routing → find/create page → apply labels."""
        ...

    def get_page(self, identifier: str) -> PageContent:
        """Get page by ID."""
        ...

    def search(self, query: str, limit: int = 10) -> list[PageSummary]:
        """Search via CQL."""
        ...

    def health(self) -> AdapterHealth:
        """Delegate to client health."""
        ...
```

### publish() flow

```
1. Extract title from metadata["title"] (required)
2. Resolve routing: config.resolve_routing(doc_type)
3. If routing exists: parent_title → get_page_by_title → parent_id
4. Look up existing page by title
5. If exists: update_page(page_id, title, content)
6. If not: create_page(title, content, parent_id, space)
7. If routing has labels: set_labels(page_id, labels)
8. Return PublishResult(success=True, url=page.url)
```

### Entry point

```toml
[project.entry-points."rai.docs.targets"]
confluence = "raise_cli.adapters.confluence_adapter:PythonApiConfluenceAdapter"
```

## Acceptance Criteria

- AC1: isinstance(adapter, DocumentationTarget) is True
- AC2: publish creates/updates page with routing (parent + labels)
- AC3: can_publish reflects routing config
- AC4: get_page, search, health delegate to client
- AC5: Entry point registered and discoverable
- AC6: No-arg constructor loads from .raise/confluence.yaml
- AC7: pyright + ruff + pytest all green
