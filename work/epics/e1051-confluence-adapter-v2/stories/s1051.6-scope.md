# S1051.6: Config Generator — `/rai-adapter-setup` Confluence — Scope

## In Scope

- Pure config builder function: takes discovery results + user selections, returns ConfluenceConfig
- YAML serializer: ConfluenceConfig model to .raise/confluence.yaml with comments
- Interactive flow in `/rai-adapter-setup` skill: discovery -> present -> select -> generate
- Space discovery via ConfluenceDiscovery (from S1051.4)
- Routing template suggestions (common artifact types: adr, roadmap, design, retrospective)
- Existing config detection with overwrite/abort prompt
- Error handling for discovery failures (unreachable API, bad credentials)
- Unit tests for pure logic (config builder, YAML serializer)
- Integration test for the skill flow (mocked discovery + mocked prompts)

## Out of Scope

- Jira adapter setup (separate adapter, separate story)
- Multi-instance setup in a single run (one instance per invocation, can re-run to add)
- Page creation (generator writes config only, not Confluence pages)
- Label management or label discovery
- Migration from v1 config format (handled by S1051.3 backwards compat)
- GUI or web-based setup wizard

## Done When

- `build_confluence_config()` produces valid ConfluenceConfig from structured inputs
- `config_to_yaml()` writes valid, human-readable YAML to .raise/confluence.yaml
- `/rai-adapter-setup` skill walks through discovery -> selection -> generation
- Existing config is detected and user is prompted before overwrite
- Discovery failures produce clear error messages (no partial config written)
- All tests pass, pyright clean, ruff clean
