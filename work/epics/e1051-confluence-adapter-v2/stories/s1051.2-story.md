# S1051.2: PythonApiConfluenceAdapter — Story

## User Story

**As a** RaiSE user,
**I want** `rai docs publish/get/search` to work with Confluence via pure Python,
**so that** I don't need the MCP stdio + Node subprocess for documentation publishing.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Adapter satisfies DocumentationTarget protocol
  Given a PythonApiConfluenceAdapter instance
  Then isinstance(adapter, DocumentationTarget) is True

Scenario: Publish routes artifact to correct parent page with labels
  Given config with routing for artifact type "adr"
  When I call publish("adr", content, metadata)
  Then it creates/updates the page under the configured parent
  And applies configured labels

Scenario: can_publish returns True for configured types
  Given config with routing for "adr" but not "unknown"
  Then can_publish("adr", {}) returns True
  And can_publish("unknown", {}) returns False

Scenario: get_page returns PageContent
  When I call get_page("12345")
  Then I get a PageContent with id, title, content, url

Scenario: search returns PageSummary list
  When I call search("my query")
  Then I get a list of PageSummary

Scenario: health checks Confluence connectivity
  When Confluence is reachable
  Then health() returns AdapterHealth(healthy=True)

Scenario: Adapter registered via entry point
  Given raise-cli[confluence] is installed
  Then "confluence" appears in get_doc_targets()

Scenario: No-arg constructor loads config from .raise/confluence.yaml
  Given .raise/confluence.yaml exists
  When PythonApiConfluenceAdapter() is instantiated
  Then it loads config and creates ConfluenceClient
```
