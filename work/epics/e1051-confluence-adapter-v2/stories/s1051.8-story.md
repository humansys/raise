# S1051.8: Story Lifecycle Skills → Docs Adapter

## User Story

**As a** RaiSE developer executing story lifecycle skills,
**I want** every artifact to be published to Confluence automatically,
**so that** documentation is always in sync without manual publish steps.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: rai docs publish accepts --file flag
  Given a markdown file at work/epics/e1051/stories/s1051.8-design.md
  When I run rai docs publish story-design --file work/epics/.../s1051.8-design.md --title "S1051.8 Design"
  Then the content is published via the docs adapter

Scenario: Story skills include publish step
  Given the rai-story-design skill
  When executed, after writing the design file
  Then it also runs rai docs publish story-design --file <path> --title <title>

Scenario: Publish works with composite adapter
  Given both filesystem and confluence targets registered
  When rai docs publish story-design --file <path>
  Then filesystem writes to metadata["path"] and Confluence publishes remotely
```
