# S1051.4: Confluence Discovery Service — Scope

## In Scope

- SpaceInfo, PageNode, SpaceMap, DiscoveryError Pydantic models
- ConfluenceDiscoveryService class using ConfluenceClient from S1051.1
- discover_spaces() — list all accessible spaces
- discover_page_tree(space_key, max_depth) — recursive page tree with labels
- discover_labels(page_ids) — batch label lookup
- build_space_map(space_key) — composite: space info + page tree + label index
- Depth-limited traversal to avoid huge page trees
- Error normalization (API errors → DiscoveryError)
- Unit tests with mocked ConfluenceClient

## Out of Scope

- Config generation from discovery (S1051.6)
- Doctor validation against discovery (S1051.5)
- Content/body fetching (discovery is structural only)
- Caching or persistence of discovery results
- Async API (sync only, matching adapter pattern)
- CLI command for discovery (consumed by doctor/setup internally)

## Done When

- ConfluenceDiscoveryService produces SpaceMap from live-shaped mocks
- Page tree respects max_depth parameter
- Label index is built correctly from page tree traversal
- API errors produce DiscoveryError with actionable messages
- All tests pass, pyright clean, ruff clean
