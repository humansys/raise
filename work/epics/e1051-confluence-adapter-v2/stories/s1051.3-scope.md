# S1051.3: Config Schema + Pydantic Models — Scope

## In Scope

- ConfluenceConfig root model (default_instance + instances map)
- ConfluenceInstanceConfig extension (routing, labels per artifact type)
- ArtifactRouting model (parent_title, labels)
- Minimal-config normalization (flat → single "default" instance)
- load_confluence_config(project_root) loader function
- Validation with clear error messages
- Tests for all models + loader + edge cases

## Out of Scope

- CLI integration (S1051.2 wires adapter)
- Config generator skill (S1051.6)
- Discovery-based config suggestions (S1051.4)
- Multi-space per instance (one space_key per instance)
- Token/auth in config (stays in env vars per S1051.1 design)

## Done When

- ConfluenceConfig validates full and minimal YAML schemas
- ArtifactRouting resolves type → parent_title + labels
- load_confluence_config reads .raise/confluence.yaml
- Flat (minimal) config auto-normalizes to multi-instance structure
- All tests pass, pyright clean, ruff clean
- S1051.1 ConfluenceInstanceConfig updated to reference new schema
