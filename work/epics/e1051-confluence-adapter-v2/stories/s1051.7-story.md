# S1051.7: FilesystemDocsTarget + CompositeDocTarget

## User Story

**As a** RaiSE developer using lifecycle skills,
**I want** `rai docs publish` to write to both Confluence and local filesystem,
**so that** I have remote documentation AND local artifacts for offline work and git history.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: FilesystemDocsTarget writes markdown files
  Given a FilesystemDocsTarget with project_root
  When I call publish("story-design", content, {"title": "S1051.7 Design", "path": "work/epics/e1051/stories/s1051.7-design.md"})
  Then the file is written to the specified path
  And PublishResult.success is True

Scenario: CompositeDocTarget delegates to all targets
  Given a CompositeDocTarget wrapping [filesystem, confluence]
  When I call publish(...)
  Then both targets receive the publish call
  And the result reflects the primary target's outcome

Scenario: Resolver auto-composes when multiple targets found
  Given both filesystem and confluence entry points are registered
  When resolve_docs_target(None) is called
  Then it returns a CompositeDocTarget wrapping both

Scenario: Explicit target flag skips composition
  Given both entry points registered
  When resolve_docs_target("confluence") is called
  Then it returns only the confluence adapter (no composition)

Scenario: FilesystemDocsTarget works standalone (no Confluence)
  Given only filesystem entry point registered
  When resolve_docs_target(None) is called
  Then it returns FilesystemDocsTarget directly

Scenario: All lifecycle artifact types have routing config
  Given confluence.yaml with routing for story and epic artifacts
  When can_publish("story-design", {}) is called
  Then it returns True
```
