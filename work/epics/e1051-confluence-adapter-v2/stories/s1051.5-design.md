# S1051.5: Adapter Doctor -- Confluence -- Design

## Problem

`rai doctor --online` validates Confluence connectivity (health check) but does not verify that config references (space_key, routing parent_title) actually exist on the live instance. A developer can have valid credentials and a reachable backend yet still fail on `rai docs publish` because the configured space or parent page doesn't exist.

## Value

Catch config-vs-backend mismatches at diagnosis time instead of at publish time. Actionable fix_hints (listing available spaces, suggesting page creation) reduce round-trips to the Confluence UI.

## Complexity: Simple

| Criterion | Assessment |
|-----------|------------|
| Components | 2 (adapters.py doctor check + tests) |
| Story points | 3 |
| External integrations | 1 (ConfluenceDiscovery, already built in S1051.4) |
| Algorithm complexity | Trivial (lookup + compare) |

**Result:** Simple -- core sections only.

**Gates:** No risk, UX, or integration gate triggered (no user interaction, not a dogfood story, low risk).

## Approach

Extend `_check_confluence_health` (or add sibling methods called from the same `context.online` block) to perform two additional validations after basic connectivity passes:

1. **Space existence check** -- call `ConfluenceDiscovery.discover(space_key=inst.space_key)`. If `ConfluenceNotFoundError`, emit ERROR with available spaces from the exception message.
2. **Routing parent page check** -- for each `ArtifactRouting` in `inst.routing`, call `client.get_page_by_title(routing.parent_title, space=inst.space_key)`. If None, emit ERROR with page title and space.

Both checks are gated behind `context.online` (existing pattern). Discovery/client failures produce ERROR results with connectivity hints -- never crash doctor.

### Components

| Component | Change | Description |
|-----------|--------|-------------|
| `packages/raise-cli/src/raise_cli/doctor/checks/adapters.py` | modify | Add space + routing parent checks after health check |
| `packages/raise-cli/tests/doctor/checks/test_adapters.py` | modify | Add tests for new checks with mocked client/discovery |

### Decision: D1 -- Inline checks vs. separate ConfluenceDoctorCheck class

**Choice:** Inline new checks as private methods in existing `AdapterDoctorCheck`.

**Rationale:** The class already handles Confluence checks. Adding 2 methods (~40 lines total) maintains cohesion. A separate class would require a new entry point and registration -- overhead not justified for 2 checks.

**Alternatives:** Separate `ConfluenceDoctorCheck` class; delegate to ConfluenceDiscovery.validate() method.

### Decision: D2 -- Use ConfluenceDiscovery vs. direct client calls for space check

**Choice:** Use `ConfluenceDiscovery.discover(space_key=...)` for space existence. Use `client.get_page_by_title()` directly for routing parent pages.

**Rationale:** Discovery already wraps the space lookup with proper error typing (`ConfluenceNotFoundError` includes available spaces). For parent pages, `get_page_by_title` is simpler than building a full page tree -- we only need existence, not structure.

**Alternatives:** Use client.get_spaces() directly and check membership; build full SpaceMap for both checks.

### Decision: D3 -- Check ordering

**Choice:** Run space check first; skip routing checks if space doesn't exist.

**Rationale:** If the space is wrong, routing parent pages can't exist in it. Reporting both would produce confusing cascading errors.

**Alternatives:** Run all checks independently, let user see all errors at once.

## Examples

### CLI invocation

```bash
# Online mode -- runs config-vs-backend checks
rai doctor --online

# Offline mode -- skips live validation (existing behavior)
rai doctor
```

### Check results -- space missing

```python
CheckResult(
    check_id="adapter-confluence-space-exists",
    category="adapters",
    status=CheckStatus.ERROR,
    message="Confluence space 'MISSING' not found on instance",
    fix_hint="Check space_key in .raise/confluence.yaml -- available spaces: RaiSE1, DEMO",
)
```

### Check results -- space exists

```python
CheckResult(
    check_id="adapter-confluence-space-exists",
    category="adapters",
    status=CheckStatus.PASS,
    message="Confluence space 'RaiSE1' exists",
)
```

### Check results -- routing parent missing

```python
CheckResult(
    check_id="adapter-confluence-routing-parent",
    category="adapters",
    status=CheckStatus.ERROR,
    message="Routing parent page 'Nonexistent Page' not found in space 'RaiSE1'",
    fix_hint="Create page 'Nonexistent Page' in space RaiSE1, or update routing in confluence.yaml",
)
```

### Check results -- routing parent exists

```python
CheckResult(
    check_id="adapter-confluence-routing-parent-adr",
    category="adapters",
    status=CheckStatus.PASS,
    message="Routing parent page 'Architecture Decision Records' exists (artifact type: adr)",
)
```

### Check results -- discovery failure

```python
CheckResult(
    check_id="adapter-confluence-space-exists",
    category="adapters",
    status=CheckStatus.ERROR,
    message="Confluence space validation failed: ConnectionError(...)",
    fix_hint="Check Confluence connectivity and credentials",
)
```

### Data flow

```
evaluate()
  -> context.online?
    -> _check_confluence_health() [existing -- connectivity]
    -> _check_confluence_space_exists() [new]
      -> ConfluenceDiscovery(client).discover(space_key=inst.space_key)
      -> ConfluenceNotFoundError? -> ERROR with available spaces
      -> success? -> PASS, then _check_confluence_routing_parents()
    -> _check_confluence_routing_parents() [new]
      -> for each (artifact_type, routing) in inst.routing:
        -> client.get_page_by_title(routing.parent_title, space=inst.space_key)
        -> None? -> ERROR with page title and space
        -> found? -> PASS
```

## Acceptance Criteria

### MUST

- AC1: `rai doctor --online` reports PASS when configured space_key exists on instance
- AC2: `rai doctor --online` reports ERROR with available spaces when space_key is missing
- AC3: `rai doctor --online` reports PASS for each routing parent_title that exists
- AC4: `rai doctor --online` reports ERROR with page title and space for each missing routing parent
- AC5: Discovery/client exceptions produce ERROR results with connectivity hint, never crash doctor

### SHOULD

- AC6: Routing parent checks are skipped when space check fails (avoid cascading errors)
- AC7: Check IDs follow existing `adapter-confluence-*` naming convention

### MUST NOT

- Must not make live API calls in offline mode (existing gate preserved)
- Must not modify existing health check behavior
- Must not add new CLI commands or flags
