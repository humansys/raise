# S1051.3: Config Schema + Pydantic Models — Design

## Problem
Current `confluence.yaml.example` has only `space_key`. S1051.1 created a minimal
`ConfluenceInstanceConfig` (4 fields). The adapter (S1051.2) needs multi-instance
config with artifact-type routing — same pattern as `jira.yaml`.

## Value
Validated config schema enables the adapter to connect, route artifacts to the right
parent page with correct labels, and support multiple Confluence instances.

## Complexity: Simple
3 models, 1 loader function, no external deps, clear pattern from jira.yaml.

## Approach

### Target file
```
raise-cli/src/raise_cli/adapters/confluence_config.py  — MODIFY (extend)
```

Single file — models + loader in one place. No new files needed.

### Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Extend existing `confluence_config.py` | Already has ConfluenceInstanceConfig; add sibling models + loader |
| D2 | Mirror jira.yaml structure | Consistency: default_instance + instances map |
| D3 | Flat config auto-normalizes | `{url, username, space_key}` → single "default" instance (backwards compat) |
| D4 | Routing is optional per instance | Not all instances need artifact routing |
| D5 | Module-level Path constant for test seam | PAT-E-589: `_CONFLUENCE_YAML_PATH` relative, testable via patch |
| D6 | No token in config | Auth stays in env vars (S1051.1 design decision) |

### Models

```python
class ArtifactRouting(BaseModel):
    """Routing config for one artifact type."""
    parent_title: str = Field(..., description="Parent page title to publish under")
    labels: list[str] = Field(default_factory=list, description="Labels to apply")

class ConfluenceInstanceConfig(BaseModel):  # EXTEND existing
    """Single Confluence instance config."""
    url: str
    username: str
    space_key: str
    instance_name: str = "default"
    routing: dict[str, ArtifactRouting] = Field(
        default_factory=dict,
        description="Artifact type → routing config"
    )

class ConfluenceConfig(BaseModel):
    """Root config — multi-instance with default."""
    default_instance: str = Field(default="default")
    instances: dict[str, ConfluenceInstanceConfig]

    def get_instance(self, name: str | None = None) -> ConfluenceInstanceConfig:
        """Get instance by name, or default."""
        ...

    def resolve_routing(
        self, artifact_type: str, instance: str | None = None
    ) -> ArtifactRouting | None:
        """Resolve routing for artifact type on instance."""
        ...
```

### YAML examples

```yaml
# Full multi-instance
default_instance: humansys
instances:
  humansys:
    url: "https://humansys.atlassian.net/wiki"
    username: "emilio@humansys.ai"
    space_key: "RaiSE1"
    routing:
      adr:
        parent_title: "Architecture Decision Records"
        labels: ["adr", "architecture"]
      roadmap:
        parent_title: "Roadmap"
        labels: ["roadmap"]
```

```yaml
# Minimal flat (auto-normalized)
url: "https://humansys.atlassian.net/wiki"
username: "emilio@humansys.ai"
space_key: "RaiSE1"
```

## Acceptance Criteria

- AC1: ConfluenceConfig validates full multi-instance schema
- AC2: Flat config auto-normalizes to ConfluenceConfig with "default" instance
- AC3: get_instance() returns default or named instance
- AC4: resolve_routing() returns ArtifactRouting or None
- AC5: load_confluence_config() reads .raise/confluence.yaml
- AC6: Missing file → FileNotFoundError, invalid YAML → ValidationError
- AC7: ConfluenceInstanceConfig backwards compatible (existing S1051.1 usage unchanged)
- AC8: pyright + ruff + pytest all green
