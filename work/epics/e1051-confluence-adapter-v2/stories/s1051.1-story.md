# Story: Confluence Client Wrapper

## User Story
As a RaiSE adapter developer,
I want a Python client that wraps atlassian-python-api with auth resolution and error normalization,
so that higher-level components (adapter, discovery, doctor) have a reliable, tested foundation.

## Acceptance Criteria

### Scenario: Successful page creation
```gherkin
Given a configured ConfluenceClient with valid credentials
When I call create_page(space, title, body, parent_id)
Then a page is created in Confluence
And a PageContent result is returned with page_id and url
```

### Scenario: Page retrieval by title
```gherkin
Given a ConfluenceClient connected to a space with existing pages
When I call get_page_by_title(space, title)
Then the matching page content is returned
And None is returned if no match
```

### Scenario: Auth resolution with instance-specific token
```gherkin
Given CONFLUENCE_API_TOKEN_HUMANSYS is set
When ConfluenceClient is created for instance "humansys"
Then it uses the instance-specific token
```

### Scenario: Auth fallback to generic token
```gherkin
Given CONFLUENCE_API_TOKEN is set but no instance-specific token
When ConfluenceClient is created for instance "humansys"
Then it uses the generic CONFLUENCE_API_TOKEN
```

### Scenario: Missing credentials
```gherkin
Given no CONFLUENCE_API_TOKEN or instance-specific token is set
When ConfluenceClient is created
Then a clear ConfigurationError is raised with setup instructions
```

### Scenario: API error normalization
```gherkin
Given a ConfluenceClient with valid credentials
When an API call fails (network, 401, 404, 500)
Then a normalized ConfluenceApiError is raised
And the original error details are preserved
```

## Examples (Specification by Example)

| Method | Input | Expected |
|--------|-------|----------|
| `create_page` | space="RaiSE1", title="ADR-015", body="<p>...</p>", parent_id="123" | PageContent(id="456", title="ADR-015", url="https://...") |
| `get_page_by_title` | space="RaiSE1", title="ADR-015" | PageContent or None |
| `get_page_by_id` | page_id="456" | PageContent |
| `update_page` | page_id="456", title="ADR-015 v2", body="<p>new</p>" | PageContent |
| `set_labels` | page_id="456", labels=["adr", "v2"] | None (labels applied) |
| `get_labels` | page_id="456" | ["adr", "v2"] |
| `get_spaces` | (no args) | [SpaceInfo(key="RaiSE1", name="RaiSE", ...)] |
| `get_page_children` | page_id="123" | [PageSummary(id="456", title="ADR-015")] |
| `search` | cql="type=page AND space=RaiSE1", limit=10 | [PageSummary, ...] |
| `health` | (no args) | AdapterHealth(status="healthy") |

## Notes
- This is the risk-first story: validates atlassian-python-api against live Confluence
- 10 methods on concrete class (NOT Protocol — see design.md C1 resolution)
- Auth resolution: CONFLUENCE_API_TOKEN_{INSTANCE_UPPER} → CONFLUENCE_API_TOKEN fallback
- Error normalization: all atlassian lib exceptions → ConfluenceApiError
- Optional dependency: imports guarded, clear error if not installed
- References: design.md § ConfluenceClient, § Auth Resolution
