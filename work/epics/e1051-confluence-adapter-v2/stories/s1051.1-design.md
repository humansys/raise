# S1051.1: Confluence Client Wrapper — Design

## Problem
Confluence integration depends on MCP stdio + Node subprocess (mcp-atlassian).
This adds a Node dependency, limits to 5/11 functions, and makes debugging opaque.

## Value
A pure-Python client validates that atlassian-python-api covers all 10 needed
methods against live Confluence, unblocking adapter (S1051.2), discovery (S1051.4),
and doctor (S1051.5).

## Complexity: Moderate
4 components, 1 external integration, low algorithm complexity, HIGH risk (validates library).

## Approach

### Target files
```
raise-cli/src/raise_cli/adapters/
  confluence_client.py      — ConfluenceClient (10 methods)
  confluence_exceptions.py  — Error hierarchy (3 classes)
  confluence_config.py      — ConfluenceInstanceConfig minimal
```

### Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | raise-cli (open-core) | Lower barrier for Jira/Confluence users |
| D2 | SpaceInfo in models/docs.py | Done in RAISE-1060 |
| D3 | Separate `confluence_exceptions.py` | Doctor/discovery import exceptions without loading client |
| D4 | Minimal Pydantic config now | S1051.3 extends; typed from start; no throwaway code |
| D5 | `backoff_and_retry=True` | Library handles 429 retry with exponential backoff + jitter — no custom rate limiter (DRY) |
| D6 | 3 exception classes, not 4 | Drop ConfluenceRateLimitError — library retries 429 automatically, post-retry failures are ConfluenceApiError (YAGNI) |
| D7 | `isinstance` error mapping | Library has typed exceptions (ApiPermissionError, ApiNotFoundError) — no string-matching (KISS) |
| D8 | Import guard in `__init__`, not module level | Allows type analysis without dep installed |
| D9 | `space` param with config default | Multi-space is MVP — methods accept space override, default to config.space_key |

### ConfluenceClient (concrete class, 10 methods)

```python
class ConfluenceClient:
    """Wraps atlassian.Confluence with auth resolution and error normalization."""

    def __init__(self, config: ConfluenceInstanceConfig) -> None:
        # Import guard — allows module import for type checking
        try:
            from atlassian import Confluence
        except ImportError as exc:
            raise ImportError(
                "atlassian-python-api required. Install: pip install raise-cli[confluence]"
            ) from exc

        token = self._resolve_token(config.instance_name)
        self._config = config
        self._client = Confluence(
            url=config.url,
            username=config.username,
            password=token,
            cloud=True,
            backoff_and_retry=True,
            max_backoff_retries=5,
            backoff_factor=1.0,
        )

    # Publishing
    def create_page(self, title: str, body: str,
                    parent_id: str | None = None,
                    space: str | None = None) -> PageContent: ...
    def update_page(self, page_id: str, title: str, body: str) -> PageContent: ...
    def get_page_by_id(self, page_id: str) -> PageContent: ...
    def get_page_by_title(self, title: str,
                          space: str | None = None) -> PageContent | None: ...

    # Labels
    def set_labels(self, page_id: str, labels: list[str]) -> None: ...
    def get_labels(self, page_id: str) -> list[str]: ...

    # Discovery
    def get_spaces(self) -> list[SpaceInfo]: ...
    def get_page_children(self, page_id: str) -> list[PageSummary]: ...

    # Search & Health
    def search(self, cql: str, limit: int = 10) -> list[PageSummary]: ...
    def health(self) -> AdapterHealth: ...

    # Internal
    @staticmethod
    def _resolve_token(instance_name: str) -> str: ...
    def _map_error(self, error: Exception, context: str) -> ConfluenceError: ...
```

### Error hierarchy (3 classes — YAGNI applied)

```python
# confluence_exceptions.py
class ConfluenceError(Exception):
    """Base for all Confluence client errors."""
    def __init__(self, message: str) -> None:
        self.message = message
        super().__init__(message)

class ConfluenceAuthError(ConfluenceError):
    """401/403 — invalid or missing credentials."""

class ConfluenceNotFoundError(ConfluenceError):
    """404 — page, space, or resource not found."""

class ConfluenceApiError(ConfluenceError):
    """Generic API error (incl. post-retry 429, 5xx, etc)."""
    def __init__(self, message: str, status_code: int | None = None) -> None:
        self.status_code = status_code
        super().__init__(message)
```

### Error mapping (isinstance, not string-matching)

```python
from atlassian.errors import ApiError, ApiNotFoundError, ApiPermissionError

def _map_error(self, error: Exception, context: str) -> ConfluenceError:
    if isinstance(error, ApiPermissionError):
        return ConfluenceAuthError(f"{context}: {error}")
    if isinstance(error, ApiNotFoundError):
        return ConfluenceNotFoundError(f"{context}: {error}")
    if isinstance(error, ApiError):
        return ConfluenceApiError(f"{context}: {error}")
    return ConfluenceApiError(f"{context}: unexpected error: {error}")
```

### Auth resolution

```
1. CONFLUENCE_API_TOKEN_{INSTANCE_NAME_UPPER}  (instance-specific)
2. CONFLUENCE_API_TOKEN                         (fallback)
3. Neither → ConfluenceAuthError with setup instructions
```

### Rate limiting strategy

No custom rate limiter. The library's `backoff_and_retry=True` handles:
- Exponential backoff (1s, 2s, 4s, 8s, 16s)
- Random jitter to avoid thundering herd
- Respects `Retry-After` header from Confluence Cloud
- Retries on 413, 429, 503 automatically
- Max 5 retries before raising ApiError

Research: Atlassian recommends max 5 attempts with exponential backoff.
The JiraClient's manual RateLimiter (sliding window) is unnecessary when the
library provides this natively.
