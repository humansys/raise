# S1001.2 Design: Bug Classification & Process Extraction

## Problem & Value

rai-bugfix no clasifica bugs formalmente — no hay datos para análisis de patrones ni extracción de mejora de proceso. Cada bug se trata en aislamiento.

Con Triage + extracción en Review, cada bug produce datos estructurados (4 dimensiones) + una mejora concreta. Habilita dogfood (S1001.3) y dashboard (S1001.4).

## Approach

Crear skillset `raise-maintainability` vacío, copiar base `rai-bugfix`, modificar SKILL.md con fase Triage y Review ampliado.

| Componente | Cambio | Ubicación |
|-----------|--------|-----------|
| Skillset `raise-maintainability` | Create (empty) | `.raise/skills/raise-maintainability/` |
| rai-bugfix v2.0 overlay | Create (copy + modify) | `.raise/skills/raise-maintainability/rai-bugfix/SKILL.md` |

## Cambios al SKILL.md

### 1. Frontmatter
- version: `1.1.0` → `2.0.0`
- Add overlay metadata for raise-maintainability

### 2. Nueva fase Triage (Step 2 — entre Start y Analyse)

Clasificar el bug en 4 dimensiones antes de investigar:

```markdown
### Step 2: Triage *(new — between Start and Analyse)*

Classify the bug in 4 orthogonal dimensions:

| Dimension | Values |
|-----------|--------|
| **Bug Type** | Functional, Interface, Data, Logic, Configuration, Regression |
| **Severity** | S0-Critical, S1-High, S2-Medium, S3-Low |
| **Origin** | Requirements, Design, Code, Integration, Environment |
| **Qualifier** | Missing, Incorrect, Extraneous |

Append to `work/bugs/RAISE-{N}/scope.md`:

TRIAGE:
  Bug Type:    [selection]
  Severity:    [selection]
  Origin:      [selection]
  Qualifier:   [selection]

Update Jira (custom fields must be set in Jira UI — CLI does not support custom fields yet):

rai backlog transition RAISE-{N} triaged -a jira

Triage gate: classification in all 4 dimensions required before advancing to Analyse.
```

### 3. Review ampliado (Step 6 — was Step 5)

Añadir extracción de process improvement después del heutagogical checkpoint:

```markdown
**Process improvement extraction:**

Answer:
1. What change in process/tooling would prevent this **class** of bug?
2. What classification pattern does this bug represent? (e.g., Type=X + Origin=Y → systemic issue in Z)

Add to `work/bugs/RAISE-{N}/retro.md`:

### Process Improvement
**Prevention:** [specific process/tool change]
**Pattern:** [Bug Type]=[X] + [Origin]=[Y] → [systemic insight]
```

### 4. Status references

| Diseño original | Jira real |
|-----------------|-----------|
| Triage | Triaged |
| Retro | Retrospective |

Todas las referencias a `rai backlog transition` usan nombres reales.

## Workaround: Custom Fields

`rai backlog update` no soporta custom fields (solo summary, labels, priority, assignee). La clasificación en Jira se hace vía UI. Documentar en la skill como nota.

## Acceptance Criteria

- **AC1:** `rai skill set list` muestra raise-maintainability con 1 skill
- **AC2:** Triage phase clasifica 4 dimensiones, gate obligatorio antes de Analyse
- **AC3:** Review incluye extracción de process improvement con template
- **AC4:** `rai skill set diff raise-maintainability` ejecuta sin errores
- **AC5:** Status references usan nombres Jira reales

## MUST NOT
- Modificar base rai-bugfix en `.claude/skills/`
- Modificar overlays de raise-dev / raise-dev-ts
