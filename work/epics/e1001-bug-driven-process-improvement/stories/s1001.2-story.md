# Story: Bug Classification & Process Extraction

## User Story
As a developer fixing bugs,
I want rai-bugfix to guide me through classifying the bug in 4 dimensions and extracting a process improvement,
so that every bug produces structured data for pattern analysis and a concrete improvement.

## Acceptance Criteria

### Scenario: Triage phase classifies bug in 4 dimensions
```gherkin
Given I am running rai-bugfix on a bug
When I reach the Triage phase (after Start)
Then I am guided to classify: Bug Type, Severity, Origin, Qualifier
And classification is required before advancing to Analyse
```

### Scenario: Review phase extracts process improvement
```gherkin
Given I have fixed a bug and reached the Review phase
When I complete the review
Then I document a process improvement that would prevent this class of bug
```

### Scenario: Skillset distribution
```gherkin
Given the raise-maintainability skillset exists
When I run rai skill set list
Then raise-maintainability appears with rai-bugfix as its skill
```

## Examples (Specification by Example)

| Phase | Input | Output |
|-------|-------|--------|
| Triage | Bug observed: CLI crashes on Unicode | Bug Type=Functional, Severity=S1-High, Origin=Code, Qualifier=Missing |
| Review | Fix applied: UTF-8 reconfigure at startup | Process improvement: "Add encoding validation to CLI startup checklist" |

## Notes
- Triage phase goes between Start and Analyse in rai-bugfix workflow
- Skillset created empty via `rai skill set create raise-maintainability --empty`
- Base rai-bugfix is copied into skillset, then modified — base skill untouched
- Field names must match S1001.1 Jira fields exactly: Bug Type, Severity, Origin, Qualifier
- Actual Jira workflow statuses: Open → Triaged → Diagnose → Fix → Verify → Retrospective → Done
