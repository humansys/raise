## Approach

### Execution model

- **Resolved bugs (1-21)**: retroactive classification only — triage + classify with 4 dimensions, root cause analysis (5 Whys), extract process improvement. No fix phase.
- **Open bugs (22-37)**: full `/rai-bugfix` triage phase — classify, determine if still valid, transition to appropriate workflow state.

### Per-bug workflow

```
1. Read bug context (Jira description + git history if resolved)
2. Invoke /rai-bugfix triage phase
3. Classify: Bug Type, Severity, Origin, Qualifier
4. Root cause: 5 Whys (even if already fixed)
5. Extract: process improvement → rai pattern add
6. Update: Jira custom fields
```

### Batching

Process in batches of ~5 bugs per task to keep commits granular and allow HITL review between batches.

### Pattern analysis (final task)

After all bugs processed:
- Cross-dimensional analysis: which Type × Origin × Qualifier combos dominate?
- Top 3+ process improvements
- Summary report

## Acceptance Criteria

### MUST
- AC1: All 37 bugs classified with 4 dimensions (Bug Type, Severity, Origin, Qualifier) in Jira custom fields
- AC2: Process improvement extracted and recorded for each bug
- AC3: Cross-dimensional pattern analysis completed
- AC4: Top 3+ concrete process improvements identified
- AC5: At least one bug processed with real Jira API (integration gate)

### SHOULD
- AC6: Summary report published to Confluence
- AC7: Open bugs transitioned to appropriate workflow state after triage

### MUST NOT
- AC8: Must not fix open bugs — classification only, fixes are separate work
- AC9: Must not modify `/rai-bugfix` skill during dogfood (observe friction, log separately)

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | Retroactive mode for resolved bugs | Already fixed — value is in classification + process improvement extraction, not re-fixing |
| D2 | Batch of ~5 per task | Granular commits, HITL between batches, manageable context |
| D3 | Include open bugs in scope | Validates full workflow (not just retroactive), tests triage on bugs with unknown status |
