# Story: Dogfood — Process All 2.3.x Bugs Through New Workflow

## User Story
As a developer using rai-bugfix,
I want every 2.3.x bug processed through the 7-phase bug workflow with 4-dimension classification,
so that we validate the system end-to-end and extract process improvements from each defect.

## Acceptance Criteria

### Scenario: Bug classified with 4 dimensions
```gherkin
Given a 2.3.x bug issue in Jira
When I invoke /rai-bugfix on it
Then the bug is classified with Bug Type, Severity, Origin, and Qualifier
And each field value comes from the constrained set defined in S1001.1
```

### Scenario: Process improvement extracted
```gherkin
Given a classified bug (resolved or open)
When the triage phase completes root cause analysis
Then at least one process improvement is identified
And it is recorded as a pattern via rai pattern add
```

### Scenario: All 2.3.x bugs processed
```gherkin
Given the full list of 2.3.x bugs in Jira
When all have been processed through /rai-bugfix
Then every bug has 4 custom fields filled
And a summary report with pattern analysis is available
```

## Examples (Specification by Example)

| Bug | Type | Severity | Origin | Qualifier | Improvement |
|-----|------|----------|--------|-----------|-------------|
| RAISE-460 | Functional | S2-Medium | Code | Incorrect | MCP result unwrapping needs defensive check |
| RAISE-598 | Functional | S2-Medium | Code | Incorrect | Extract helper before complexity grows |

## Notes
- Use `/rai-bugfix` skill (v2.0, 7 phases) for each bug
- Both resolved and open bugs get processed — resolved bugs still have process lessons
- Pattern analysis at the end: which Type x Origin x Qualifier combos dominate?
- Top 3+ concrete improvements go into the epic retrospective
