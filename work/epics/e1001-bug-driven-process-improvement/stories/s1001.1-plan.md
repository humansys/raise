# S1001.1 Plan: Jira Infrastructure

## Overview

Configuración manual en Jira admin UI. Cero código — no aplica TDD.
Fer ejecuta los pasos en Jira, Rai verifica vía `rai backlog`.

## Tasks

### T1: Crear Custom Fields (S)

**Qué:** Crear 4 campos custom select en Jira admin.

**Pasos en Jira:**
1. Ir a **Settings → Issues → Custom Fields → Create custom field**
2. Tipo: **Select List (single choice)**
3. Crear cada campo:

| Campo | Opciones (una por línea) |
|-------|--------------------------|
| `Bug Type` | `Functional`, `Interface`, `Data`, `Logic`, `Configuration`, `Regression` |
| `Severity` | `S0-Critical`, `S1-High`, `S2-Medium`, `S3-Low` |
| `Origin` | `Requirements`, `Design`, `Code`, `Integration`, `Environment` |
| `Qualifier` | `Missing`, `Incorrect`, `Extraneous` |

4. En **Screens**: asociar cada campo a la screen del issue type **Bug** (al menos Default Screen o la screen que use Bug)
5. Verificar que los 4 campos aparecen al crear/editar un Bug

**Verificación:** Abrir cualquier Bug existente → los 4 campos son visibles y editables.

**Deps:** Ninguna.

---

### T2: Activar Bug Workflow (S)

**Qué:** Crear y activar workflow de 7 estados para Bug issue type.

**Pasos en Jira:**
1. Ir a **Settings → Issues → Workflows**
2. **Add workflow** → nombre: `Bug Classification Workflow`
3. Crear **7 statuses** (si no existen, crearlos en Settings → Statuses):
   - `Open` (category: To Do)
   - `Triage` (category: In Progress)
   - `Diagnose` (category: In Progress)
   - `Fix` (category: In Progress)
   - `Verify` (category: In Progress)
   - `Retro` (category: In Progress)
   - `Done` (category: Done)
4. Crear **transitions**:
   - `Open → Triage` (name: "Start Triage")
   - `Triage → Diagnose` (name: "Start Diagnosis")
   - `Diagnose → Fix` (name: "Start Fix")
   - `Fix → Verify` (name: "Start Verification")
   - `Verify → Retro` (name: "Start Retrospective")
   - `Retro → Done` (name: "Close")
   - `All Statuses → Open` (name: "Reopen") — global transition
5. **Publish** workflow
6. Ir a **Settings → Issues → Workflow Schemes**
7. Editar el scheme del proyecto RAISE → asignar `Bug Classification Workflow` al issue type **Bug**
8. Publicar scheme (Jira pedirá migrar issues existentes — mapear estados)

**Verificación:** Abrir un Bug → las transiciones disponibles corresponden al workflow.

**Deps:** T1 (campos deben existir antes de testear el workflow completo).

### T3: Validar con Bug de Prueba (XS)

**Qué:** Crear bug test, clasificar, transicionar por todo el workflow.

**Pasos:**
1. Crear issue: type=Bug, summary=`Test E1001 — workflow validation`, project=RAISE
2. Clasificar:
   - Bug Type = `Functional`
   - Severity = `S3-Low`
   - Origin = `Code`
   - Qualifier = `Incorrect`
3. Transicionar paso a paso: Open → Triage → Diagnose → Fix → Verify → Retro → Done
4. En cada transición, verificar que los 4 campos custom **persisten** (no se borran)
5. Al llegar a Done: confirmar que los 4 campos tienen sus valores

**Verificación Rai (después de que Fer complete):**
```bash
rai backlog get {TEST_BUG_KEY} -a jira
```

**Deps:** T1, T2.

## Execution Order

T1 (Custom Fields) → T2 (Workflow) → T3 (Validación)

Lineal — cada paso requiere el anterior.
