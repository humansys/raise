# Story: Jira Infrastructure — Custom Fields & Bug Workflow

## User Story
As a developer using rai-bugfix,
I want bugs classified in 4 dimensions with a formal workflow in Jira,
so that every bug produces structured data for pattern analysis and process improvement.

## Acceptance Criteria

### Scenario: Custom fields exist with constrained values
```gherkin
Given I am viewing a Bug issue in Jira
When I open the issue detail
Then I see 4 select fields: Bug Type, Severity, Origin, Qualifier
And each field has its defined constrained values
```

### Scenario: Bug workflow transitions
```gherkin
Given a Bug issue in status Open
When I transition it through the full workflow
Then it passes through: Open → Triage → Diagnose → Fix → Verify → Retro → Done
And each transition is valid and available
```

### Scenario: Validation with test bug
```gherkin
Given I create a test Bug issue
When I classify it (Bug Type=Functional, Severity=S3-Low, Origin=Code, Qualifier=Incorrect)
And I transition it through the full workflow
Then all fields persist and all transitions succeed
```

## Examples (Specification by Example)

| Field | Type | Values |
|-------|------|--------|
| Bug Type | Select | Functional, Interface, Data, Logic, Configuration, Regression |
| Severity | Select | S0-Critical, S1-High, S2-Medium, S3-Low |
| Origin | Select | Requirements, Design, Code, Integration, Environment |
| Qualifier | Select | Missing, Incorrect, Extraneous |

## Notes
- Fer has Jira admin access — no external blocker
- Workflow replaces simple Open → In Progress → Done for Bug issue type
- Field names and values must match exactly what rai-bugfix v2.0 will reference
- See design.md § Target: Jira Schema for full spec
