# S1001.2 Plan: Bug Classification & Process Extraction

## Overview

Crear skillset raise-maintainability + rai-bugfix v2.0 overlay. Deliverable es markdown (SKILL.md), no código — no aplica TDD. Verificación vía `rai skill set` CLI.

## Tasks

### T1: Crear Skillset + Copiar Base Skill (S)

**Qué:** Crear skillset vacío y copiar base rai-bugfix como punto de partida.

**Pasos:**
1. `rai skill set create raise-maintainability --empty`
2. `rai skill set list` → verificar que aparece con 0 skills
3. Copiar `.claude/skills/rai-bugfix/SKILL.md` → `.raise/skills/raise-maintainability/rai-bugfix/SKILL.md`
4. `rai skill set list` → verificar que aparece con 1 skill

**Verificación:**
```bash
rai skill set list
# raise-maintainability | 1 | .raise/skills/raise-maintainability/
```

**AC ref:** AC1 (skillset exists with 1 skill)
**Deps:** Ninguna.

### T2: Modificar SKILL.md — Triage + Review + Status (M)

**Qué:** Editar el overlay SKILL.md con los 3 cambios del diseño.

**Cambios:**
1. **Frontmatter:** version `1.1.0` → `2.0.0`, description actualizada
2. **Insertar Step 2: Triage** entre Start y Analyse:
   - Tabla de 4 dimensiones con valores
   - Template para scope.md (sección TRIAGE)
   - `rai backlog transition RAISE-{N} triaged -a jira`
   - Nota: custom fields se ponen en Jira UI (CLI no soporta)
   - Gate: 4 dimensiones obligatorias antes de Analyse
3. **Renumerar steps** (Analyse→3, Plan→4, Fix→5, Review→6, Close→7)
4. **Ampliar Review (Step 6):** añadir process improvement extraction con template
5. **Status references:** Triaged, Retrospective en todos los `rai backlog transition`

**Verificación:**
```bash
rai skill set diff raise-maintainability
# 1 added (rai-bugfix), muestra diff vs builtin
```

**AC ref:** AC2 (Triage), AC3 (Review), AC5 (status names)
**Deps:** T1.

---

### T3: Validación End-to-End (XS)

**Qué:** Verificar que el skillset y la skill son válidos.

**Pasos:**
1. `rai skill set list` → raise-maintainability con 1 skill
2. `rai skill set diff raise-maintainability` → sin errores
3. `rai skill validate rai-bugfix` (desde el skillset context)
4. Revisar que la skill menciona los nombres exactos de Jira fields de S1001.1
5. Verificar que `rai backlog transition` usa status names reales (triaged, retrospective)

**Verificación:** Todos los checks pasan.

**AC ref:** AC4 (diff sin errores), AC1-AC5 (revisión completa)
**Deps:** T2.
