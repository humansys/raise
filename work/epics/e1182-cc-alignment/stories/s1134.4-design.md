# S1134.4: Validation & Report — Design

## Summary

Write a Python validation script that audits all 35 skill SKILL.md frontmatter files across 3 dimensions (description, allowed-tools, invocation control) and produces a before/after markdown report with explicit pass/fail for each epic done criterion.

## Complexity: Simple

1 script, 1 report output, no architecture. Pure read + analysis + format.

## Approach

### Script: `work/epics/e1134-cc-alignment/validation/validate.py`

Single-file Python script (no dependencies beyond stdlib + PyYAML):

1. **Discover** all `.claude/skills/*/SKILL.md` files via glob
2. **Parse** YAML frontmatter from each file (split on `---` markers)
3. **Collect** metrics per skill across 3 dimensions
4. **Compare** against hardcoded "before" baseline (from epic design.md Gemba)
5. **Format** markdown report with before/after table + per-dimension detail
6. **Write** report to `validation/report.md`

### Metrics Collected

**Dimension 1: Descriptions (S1134.1)**
- Count with description <100 chars (target: 35/35)
- Count with description <250 chars (hard limit: 35/35)
- Average description length
- Count of verb-first descriptions (starts with uppercase verb)
- Total description budget for auto-invocable skills (target: <2,600 chars)

**Dimension 2: allowed-tools (S1134.2)**
- Count with `allowed-tools` present (target: 35/35)
- Count with bare `Bash` (no glob) — should be limited to T5+T7 tier skills
- Count with `Bash` glob patterns

**Dimension 3: Invocation Control (S1134.3)**
- Count with `disable-model-invocation: true` (target: 13 per S1134.3 actual)
- List of disabled skills
- Count of auto-invocable skills (should be 22 = 35 - 13)

### Before Baseline (from epic design.md Gemba)

| Metric | Before Value |
|--------|-------------|
| `allowed-tools` coverage | 0% (0/35) |
| `disable-model-invocation` coverage | 0% (0/35) |
| Description avg length | ~300 chars |
| Descriptions within 250-char budget | 0% |
| Total description budget | ~10,500 chars |

### Report Format

Summary table:

```markdown
| Metric | Before | After | Target | Status |
|--------|--------|-------|--------|--------|
| Description avg length | ~300 chars | XX chars | <100 | PASS/FAIL |
| ...
```

Followed by per-dimension detail sections and epic done criteria checklist.

### Output Location

`work/epics/e1134-cc-alignment/validation/report.md`

## Acceptance Criteria

- AC1: Validation script runs against current skill files without errors
- AC2: Before/after report covers all 3 dimensions with numeric metrics
- AC3: Each epic done criterion from scope.md has explicit pass/fail
- AC4: Report saved as story artifact in epic stories directory

## Risks

None. Read-only analysis of existing files.
