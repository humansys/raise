# Story: Validation & Before/After Metrics Report

## User Story
As a RaiSE developer completing the CC-alignment epic,
I want a validation script and before/after metrics report,
so that I can verify all alignment changes are complete and quantify the improvement.

## Acceptance Criteria

### Scenario: Validation script audits all skills
```gherkin
Given the validation script runs against all 35 skill SKILL.md files
When it checks each dimension (description, allowed-tools, disable-model-invocation)
Then it reports per-skill pass/fail for each dimension
And it reports aggregate coverage percentages
```

### Scenario: Before/after report shows improvement
```gherkin
Given the before metrics (from epic research gemba audit)
And the after metrics (from validation script output)
When the report is generated
Then it shows improvement across all dimensions:
  | Dimension                  | Before | After Target |
  | description avg length     | 300    | <100 chars   |
  | description verb-first %   | 0%     | 100%         |
  | description within budget  | 0%     | 100%         |
  | allowed-tools coverage     | 0%     | 100%         |
  | bare Bash leaks            | N/A    | 0            |
  | disable-model-invocation   | 0%     | 100% of side-effect skills |
  | total description budget   | ~10500 | <2500 chars  |
```

### Scenario: All epic done criteria verified
```gherkin
Given the epic scope done criteria
When the validation report is complete
Then each criterion is explicitly checked and marked pass/fail
```

## Notes
- This is an analysis story: no deliverable source code ships to the package
- Depends on S1134.1 (descriptions), S1134.2 (allowed-tools), S1134.3 (invocation control)
- Validation artifacts live in the epic directory, not in src/
