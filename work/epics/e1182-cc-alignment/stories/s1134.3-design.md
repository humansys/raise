# S1134.3: Invocation Control — Design

## Summary

Add `disable-model-invocation: true` to 9 side-effect/orchestration skills so Claude cannot auto-invoke them. Frees ~900 chars of description budget as a side benefit.

## Complexity: Simple

1 field, 9 files, no logic, no tests.

## Approach

Add `disable-model-invocation: true` to YAML frontmatter of each target skill, placed consistently after `description` and before `license`/`metadata`.

### Target Skills (from D3 classification)

| Skill | Side Effect Category |
|-------|---------------------|
| rai-epic-start | Creates tracker entries, emits signals |
| rai-epic-close | Pushes to origin, creates MR |
| rai-epic-run | Orchestrates full epic lifecycle |
| rai-story-start | Creates branches, commits |
| rai-story-close | Merges branch, updates tracker |
| rai-story-run | Orchestrates full story lifecycle |
| rai-session-close | Writes session state |
| rai-publish | Publishes to PyPI |
| rai-framework-sync | Writes to multiple locations |

### Example (before → after)

**Before:**
```yaml
---
name: rai-publish
description: >
  Guide the human through a structured release workflow...
license: MIT
```

**After:**
```yaml
---
name: rai-publish
description: >
  Guide the human through a structured release workflow...
disable-model-invocation: true
license: MIT
```

## Acceptance Criteria

- AC1: 9 skills have `disable-model-invocation: true`
- AC2: Field placement consistent (after description, before license)
- AC3: No other frontmatter fields changed
- AC4: No skill content changed
- AC5: YAML valid (no syntax errors)
