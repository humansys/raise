## Heutagogical Checkpoint

### What did I learn?

1. **Design-time tier counts drift from reality.** The D2 epic research defined 6 tiers. Auditing all 35 skill bodies during design revealed 11 skills that write files but don't use git — a gap between "Read+CLI" and "Read+Write+CLI+Git". The fix was simple (add T3), but the lesson is: classify from evidence, not from categories.

2. **Batch metadata stories are fast when the schema is stable.** The plan estimated 30min. Actual implementation was ~5min because the classification was fully resolved during design. The expensive part was reading 35 skill bodies to classify them — that happened in design, not implementation.

3. **Subcommand-granularity Bash patterns are the key security lever.** The difference between `Bash` (unrestricted) and `Bash(rai:*)` (rai CLI only) is the primary blast-radius control. Only 5 of 35 skills get unrestricted Bash (3 full-dev + 2 orchestration).

### What would I do differently?

Nothing significant. The design phase correctly caught the tier gap before implementation began. The story was well-scoped.

### What should the framework learn?

- PAT-E-694: 7-tier classification with the governance-writing gap insight. Future skill additions should reference this pattern for tier selection.
