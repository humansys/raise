# S1134.1: Description Optimization — Design

## Summary

Rewrite all 35 skill descriptions to follow CC's proven pattern: verb-first, <100 chars, trigger phrases included. Budget-aware — only 22 auto-invocable skills load into context after S1134.3.

## Complexity: Simple

1 field, 35 files, pattern-based rewriting. No logic, no tests.

## Approach

Apply D1 pattern from epic design:

```
{Verb} {what} {when/context}. Use when {trigger phrase}.
```

**Constraints:**
- Hard limit: 250 chars (CC truncation)
- Target: <100 chars (Anthropic's own commands average 45 chars)
- Verb-first: Start with action verb (Deploy, Review, Run, Fix, etc.)
- Trigger phrases: Include words users naturally type

**Grouping by lifecycle for consistency:**

| Group | Skills | Verb pattern |
|-------|--------|-------------|
| Session | session-start, session-close | "Load...", "Close..." |
| Epic | epic-start, epic-design, epic-plan, epic-run, epic-close, epic-docs, epic-research | "Initialize...", "Design...", "Sequence...", etc. |
| Story | story-start, story-design, story-plan, story-implement, story-run, story-review, story-close | Same pattern as epic |
| Utility | debug, research, bugfix, publish, framework-sync | Action-specific verbs |
| Discovery | discover, project-create, project-onboard, welcome | "Run...", "Guide...", "Onboard..." |
| Review | quality-review, architecture-review, code-audit | "Review...", "Audit...", "Evaluate..." |
| MCP | mcp-add, mcp-remove, mcp-status | "Register...", "Remove...", "Check..." |
| Meta | skill-create, skillset-manage, docs-update, doctor, sonarqube, adapter-setup, problem-shape | Action-specific |

## Acceptance Criteria

- AC1: All 35 descriptions under 250 chars
- AC2: All 35 descriptions under 100 chars (target)
- AC3: All descriptions verb-first
- AC4: No skill content changed
- AC5: YAML valid on all 35 skills
