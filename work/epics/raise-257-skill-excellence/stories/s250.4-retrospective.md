## What to Improve

- **Run full test suite earlier:** T3 (sync) committed without running full suite — the sync script's `update_distributable_list` has a double-write bug that broke `__init__.py`. Caught in T4 but could have been caught before committing T3.
- **Silent parse failures:** 2 skills (architecture-review, quality-review) were silently skipped by validator because they lacked `raise.version` in metadata. The validator should catch Pydantic `ValidationError` not just `ParseError`.

## Heutagogical Checkpoint

1. **Learned:** Sync script bracket-matching logic is fragile — duplicates list when content matches expected format. Skills that fail parse are invisible to the validator.
2. **Process change:** Integration verification (full test suite) should run before committing the sync, not after.
3. **Framework improvements:** (a) Fix sync script double-write bug (deferred), (b) Validator should surface parse errors as validation errors (deferred).
4. **More capable of:** Compressing complex review skills (250→130 lines) while preserving heuristic tables as the core signal.

## Patterns

- **PAT-E-480:** Sync script integration test — always run full suite after sync
- **PAT-E-481:** Silent parse failures — skills failing Pydantic validation are silently skipped
- **PAT-E-442:** Reinforced (+1) — repetitive extractions compound velocity
