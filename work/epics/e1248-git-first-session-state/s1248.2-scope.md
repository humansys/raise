# S1248.2: SessionRegistry + Lifecycle GC

> **Epic:** E1248 — Git-First Session State
> **Size:** M
> **Dependencies:** S1248.1 (protocols + GitStateDeriver)
> **Branch:** `story/s1248.2/session-registry-lifecycle-gc`
> **FMs Addressed:** FM-2 (stale output), FM-3 (zombie sessions), FM-4 (no cleanup)

## Objective

Implement `LocalSessionRegistry` as a unified facade over the existing scattered session lifecycle code (index.py, active-session pointer, profile.active_sessions). Add `gc()` for zombie reaping and retention policy. Register via entry point.

## What Changes for Users

Nothing visible yet — S1248.2 is internal plumbing. The registry wraps existing behavior behind the `SessionRegistry` protocol. S1248.3 wires it into the actual session start/close commands.

## Acceptance Criteria

1. `LocalSessionRegistry` implements all 4 `SessionRegistry` protocol methods
2. `isinstance(LocalSessionRegistry(...), SessionRegistry)` passes
3. `gc()` reaps zombie sessions older than configurable threshold (default 48h)
4. `gc()` enforces session directory retention (keep max 20 or 30 days)
5. `gc()` cleans stale `session-output.yaml` (>24h)
6. Entry point `rai.session.registry` → `local` registered and discoverable
7. Worktree-aware: resolves personal dir via `git rev-parse --git-common-dir`
8. All existing session index/pointer logic is delegated to, not duplicated

## Design Notes

### Wrapping, Not Replacing

`LocalSessionRegistry` delegates to existing functions:
- `register()` → `write_active_session()` + `start_session(profile, ...)`
- `active()` → `read_active_session()` + `profile.active_sessions`
- `close()` → `clear_active_session()` + `write_session_entry()` + `end_session(profile, ...)`
- `gc()` → NEW (no existing equivalent)

We do NOT rewrite session/index.py or session/close.py. The registry is a facade that composes existing functions.
