# S1248.1: Session Protocols + GitStateDeriver

> **Status:** in-progress
> **Size:** M
> **Branch:** story/s1248.1/session-protocols-git-deriver
> **Epic:** E1248 (Git-First Session State)

## Objective

Define the three session protocol contracts (StateDeriver, SessionRegistry, WorkstreamMonitor) and implement GitStateDeriver — the git-only backend that derives current work context from branch, scope.md, and git log.

## Scope

**Delivers:**
- `session/protocols.py` — Protocol definitions with @runtime_checkable
- `session/derive.py` — GitStateDeriver implementation
- Session models (ActivityEntry, SessionInfo, etc.) in schemas
- Entry point registration in pyproject.toml
- Registry discovery functions
- Worktree-aware path resolution
- Unit tests for all derivation cases

**Does NOT deliver:**
- Bundle integration (S1248.3)
- SessionRegistry implementation (S1248.2)
- WorkstreamMonitor implementation (S1248.4)

## Acceptance Criteria

- [ ] Three protocols defined, @runtime_checkable
- [ ] GitStateDeriver.current_work() returns correct CurrentWork for:
  - Release branch (release/2.4.0)
  - Story branch (story/s1248.1/name)
  - Worktree (detects main repo's state)
  - Detached HEAD (graceful fallback)
  - No scope.md (graceful fallback)
- [ ] GitStateDeriver.recent_activity() returns parsed git log entries
- [ ] Entry points discoverable via registry
- [ ] Worktree path resolution works via git-common-dir
- [ ] All tests pass, pyright strict, ruff clean
