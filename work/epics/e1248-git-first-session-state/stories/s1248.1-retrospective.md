## What Went Well

- **Protocol-first design paid off.** Defining protocols before implementation forced clean contracts. The `StateDeriver` protocol is 10 lines — minimal, testable, swappable.
- **TDD caught transitive dependency.** `_infer_phase` calling `_git_log_lines` on `tmp_path` (not a git repo) was caught immediately by failing tests. Fixed by patching `_infer_phase` in `current_work` tests.
- **Entry point wiring follows existing pattern perfectly.** `registry.py` additions were copy-paste of established pattern (3 constants, 3 functions, same `_discover()` path). Zero architectural debt.
- **Walking skeleton strategy validated.** One real implementation + entry point discovery proves the pattern for S1248.2-S1248.5.

## What to Improve

- **Entry point discovery testing gotcha.** Had to run `uv run --reinstall-package raise-cli` to pick up new entry points. This should be documented (now captured as PAT-E-710).
- **No graph available during implementation.** PRIME was empty for story-implement. Not blocking, but means tier1 patterns weren't available. Graph build should be part of session-start.

## Heutagogical Checkpoint

1. **What did I learn?** Instance methods as test seams for subprocess wrappers (PAT-E-709). Clean, no DI overhead, enables focused patching.
2. **What would I change?** Nothing major — the epic-level design was detailed enough to skip story-level design. This validates the "walking skeleton stories can skip /rai-story-design" heuristic.
3. **Framework improvements?** Two patterns captured (PAT-E-709, PAT-E-710). No framework changes needed.
4. **New capability:** Protocol → implementation → entry point → registry discovery pattern is now proven and repeatable.
