### T1: LocalSessionRegistry — register, active, close (M)

**RED:** Test that `LocalSessionRegistry` satisfies `SessionRegistry` protocol. Test `register()` writes active-session pointer. Test `active()` returns current session. Test `close()` clears pointer and writes index entry.

**GREEN:** Implement `LocalSessionRegistry` in `session/registry.py`:
- Constructor takes `project: Path` and `profile: DeveloperProfile`

- `close()` delegates to `clear_active_session()` + `write_session_entry()` + `end_session()`

### T2: gc() — zombie reaping + retention + stale cleanup (M)

**RED:** Test `gc()` removes sessions older than threshold from profile. Test `gc()` enforces directory retention (keeps 20, deletes older). Test `gc()` removes stale session-output.yaml. Test `gc()` returns list of cleaned IDs.

**GREEN:** Implement `gc()` in `LocalSessionRegistry`:
1. **Zombie reaping:** iterate `profile.active_sessions`, remove entries where `started_at` > `max_age_hours` ago
2. **Dir retention:** scan `.raise/rai/personal/sessions/{prefix}/` dirs, sort by mtime, keep max 20 or within 30 days
3. **Stale output:** check `.raise/rai/personal/session-output.yaml`, delete if >24h old

**REFACTOR:** Ensure gc() is idempotent and safe (no-op on empty state).

## Risks

| Risk | Mitigation |
|------|-----------|
| Coupling to DeveloperProfile internals | Facade delegates, doesn't duplicate — changes in profile propagate |
| Session dir structure varies across versions | gc() uses defensive scanning (glob, not hardcoded paths) |
| Worktree personal dir resolution | Proven pattern from S1248.1 (GitStateDeriver) |
