# S1248.3: Bundle Integration + CLAUDE.local.md Kill

> **Epic:** E1248 — Git-First Session State
> **Size:** M
> **Dependencies:** S1248.1 (protocols + GitStateDeriver)
> **Branch:** `story/s1248.3/bundle-integration-claude-local-kill`
> **FMs Addressed:** FM-5 (CLAUDE.local.md dead), FM-7 (git not primary)

## Objective

Wire `GitStateDeriver` into bundle assembly so `current_work` is derived from git instead of session-state.yaml. Remove CLAUDE.local.md references from documentation. YAML remains as fallback for non-derivable fields (narrative, pending, next_session_prompt).

## Acceptance Criteria

1. `assemble_orientation()` uses `GitStateDeriver.current_work()` when available
2. Falls back to `state.current_work` from YAML if derivation fails
3. `format_work_section()` accepts `CurrentWork` directly (not only via SessionState)
4. CLAUDE.local.md references removed from docs
5. No regression in existing bundle output format
6. Session start `--context` flag produces correct output with git-derived state

## Design Notes

### Integration Point

`assemble_orientation()` is the hook point. Before formatting work section:
1. Try `GitStateDeriver().current_work(project_path)`
2. If successful, use derived `CurrentWork`
3. If fails (RuntimeError from git), fall back to `state.current_work`

### What Changes

- `bundle.py:assemble_orientation()` — add git derivation before format_work_section
- `bundle_formatters.py:format_work_section()` — accept `CurrentWork` directly instead of extracting from SessionState
- Docs: remove CLAUDE.local.md mentions from `dev/context.md`, `dev/happy-path-guide.md`, `dev/happy-path-guide-es.md`

### What Stays the Same

- `session-state.yaml` still written on close (backward compat)
- `narrative`, `pending`, `next_session_prompt` still from YAML (not derivable from git)
- `format_last_session()`, `format_narrative()`, etc. — unchanged

## Out of Scope

- Changes to session close flow
- Server backend integration
- Removing session-state.yaml entirely (backward compat)
