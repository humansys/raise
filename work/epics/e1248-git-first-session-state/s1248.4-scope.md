# S1248.4: WorkstreamMonitor Basics

> **Epic:** E1248 — Git-First Session State
> **Size:** S
> **Dependencies:** S1248.2 (SessionRegistry)
> **Branch:** `story/s1248.4/workstream-monitor-basics`
> **FMs Addressed:** None (kaizen capability)

## Objective

Implement `LocalWorkstreamMonitor` — basic session analysis from git log heuristics. Provides commit velocity, TDD compliance ratio, and revert detection. Entry point registration.

## Acceptance Criteria

1. `LocalWorkstreamMonitor` implements `WorkstreamMonitor` protocol
2. `analyze_session()` returns `SessionInsights` with commit_count, test_commit_ratio, revert_count, duration_minutes
3. `suggest_improvements()` returns heuristics based on session analysis
4. Entry point `rai.session.monitor` → `local` registered and discoverable
5. All analysis from git log — no external dependencies

## Out of Scope

- Wiring into session close flow (future)
- Server backend (v3.0)
- Complex pattern recognition (server-side in 3.0)
