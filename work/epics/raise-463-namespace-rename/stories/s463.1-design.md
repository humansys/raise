# S463.1: Rename raise-core — Design

## Approach

Mechanical rename with checkpoint commits for safe rollback. Each checkpoint
is independently revertible.

## Execution Order

### C1: Directory rename (git mv)
1. `git mv packages/rai-core packages/raise-core`
2. `git mv packages/raise-core/src/rai_core packages/raise-core/src/raise_core`
3. Commit checkpoint

### C2: Internal imports (11 files inside raise_core)
1. `sed -i 's/rai_core/raise_core/g'` on all .py files in `packages/raise-core/src/raise_core/`
2. Update `packages/raise-core/pyproject.toml` (name: raise-core, wheel: raise_core)
3. Verify: `python -c "import ast; ast.parse(open(f).read())"` on each file (syntax check)
4. Commit checkpoint

### C3: External imports (rai_cli source + tests)
1. `sed -i 's/from rai_core/from raise_core/g'` on src/rai_cli/ (~49 files)
2. `sed -i 's/import rai_core/import raise_core/g'` on src/rai_cli/ (if any)
3. Same sed on tests/ (~38 files)
4. Update docstrings/comments referencing `rai_core` namespace (not historical)
5. Commit checkpoint

### C4: Workspace config + lock
1. Update root `pyproject.toml`:
   - `rai-core = { workspace = true }` → `raise-core = { workspace = true }`
   - `"rai-core==2.1.0"` → `"raise-core==2.1.0"` in dependencies
   - `packages/rai-core/src` → `packages/raise-core/src` in tool paths
   - Entry point: `local = "rai_core.graph...` → `local = "raise_core.graph...`
2. `uv cache clean && uv lock`
3. Smoke test: `uv run python -c "from raise_core.graph.models import GraphNode; print('OK')"`
4. Full test: `uv run pytest tests/core/ -q` then `uv run pytest -q`
5. Commit checkpoint

## Rollback Strategy

| Level | When | Command |
|-------|------|---------|
| Uncommitted mess | During any C step | `git checkout -- . && git clean -fd` |
| Revert one checkpoint | C fails after commit | `git revert C{N}` |
| Abandon story | Everything broken | `git checkout dev && git branch -D story/s463.1/rename-raise-core` |
| uv cache corrupt | Lock resolution fails | `uv cache clean && uv lock --refresh` |

## Risks

- **sed over-replacement**: `rai_core` could appear in strings/comments that
  shouldn't change. Mitigation: review sed output with `git diff` before each
  commit.
- **uv workspace confusion**: renaming a workspace member may confuse cached
  resolution. Mitigation: `uv cache clean` before `uv lock`.
- **Mock paths in tests**: tests mock `rai_core.*` paths — these must become
  `raise_core.*`. Mitigation: grep for residual `rai_core` after C3.

## Components Affected

- `packages/raise-core/` (was rai-core) — 11 .py files
- `src/rai_cli/` — ~49 files with `from rai_core` imports
- `tests/` — ~38 files with `from rai_core` imports
- `pyproject.toml` (root) — workspace, deps, tool config
- `uv.lock` — auto-regenerated
