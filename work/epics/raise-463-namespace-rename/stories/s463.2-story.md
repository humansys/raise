# S463.2: Rename raise-server

## User Story

As a developer deploying rai-server,
I want the package namespace to be raise-server/raise_server,
So that it's consistent with raise-core and avoids future namespace collisions.

## Acceptance Criteria

Given the rename is applied
When I run `from raise_server.app import create_app`
Then the import succeeds

Given the Dockerfile is updated
When I run `docker build -f packages/raise-server/Dockerfile .`
Then the build succeeds

Given all tests pass
When I run `uv run pytest tests/rai_server/`
Then all tests pass with zero failures
