# S463.1: Rename raise-core

## User Story (Connextra)

**As a** RaiSE developer,
**I want** the rai-core package renamed to raise-core with namespace raise_core,
**so that** the package name no longer collides with Robotec.AI's rai-core on PyPI.

## Acceptance Criteria (Gherkin)

```gherkin
Feature: raise-core package rename

  Scenario: Directory structure reflects new name
    Given the monorepo workspace
    When I list packages/
    Then I see raise-core/ (not rai-core/)
    And the source directory is raise-core/src/raise_core/

  Scenario: Package metadata updated
    Given packages/raise-core/pyproject.toml
    Then the project name is "raise-core"
    And the wheel build target references raise_core

  Scenario: Internal imports within raise_core are consistent
    Given the 11 .py files inside raise_core/
    When I check import statements
    Then all internal imports use raise_core (not rai_core)

  Scenario: Downstream imports from rai_cli updated
    Given the ~49 files in src/rai_cli/ that import from rai_core
    When I check their import statements
    Then all imports reference raise_core

  Scenario: Test imports updated
    Given the ~38 test files that import from rai_core
    When I check their import statements
    Then all imports reference raise_core

  Scenario: Root workspace references updated
    Given the root pyproject.toml
    Then the workspace member references raise-core (not rai-core)
    And uv.sources maps raise-core correctly

  Scenario: Tests pass after rename
    When I run the test suite
    Then all tests pass
    And no ImportError for rai_core occurs
```

## Size

S (Small) -- mechanical rename, no logic changes.

## Dependencies

- None (first story in RAISE-463 sequence)
