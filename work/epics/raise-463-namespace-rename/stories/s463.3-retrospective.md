## Summary

Renamed rai-cli → raise-cli (197 src + 364 test files, 329 mock paths).

## What was delivered

- src/rai_cli → src/raise_cli (197 files)
- All test imports and mock paths updated
- Root pyproject.toml: name, entry points, scripts, tool paths
- Skills .md files: path references updated

## Dragons encountered

1. **rai_pro cross-import** — src/rai_pro/providers/auth/credentials.py imports
   from rai_cli.compat. Not in the original blast radius map. Fixed by updating
   the import.

2. **Stale entry points** — after renaming package from rai-cli → raise-cli,
   old rai-cli entry points remained installed alongside raise-cli. Required
   `uv pip uninstall rai-cli rai-core rai-server` to clean up.

3. **MCP optional extra** — `uv sync --extra dev` doesn't install the `mcp`
   extra. Need `uv sync --all-extras` for full test suite.

## Patterns

- When renaming packages, always check for cross-package imports outside the
  main blast radius (rai_pro was missed in the gemba scan).
- After PyPI package rename, old packages may persist in venv — explicit
  uninstall required.
