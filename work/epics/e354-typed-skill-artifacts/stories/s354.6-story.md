# Story: S354.6 — `rai artifact validate` CLI Command

> **Epic:** E354 | **Jira:** RAISE-423 | **Size:** S

## User Story

As a developer, I want to run `rai artifact validate` to check that YAML artifacts in `.raise/artifacts/` are structurally valid and pass governance rules, so I get deterministic feedback before committing.

## Acceptance Criteria

- `rai artifact validate` reads all `.yaml` files from `.raise/artifacts/`
- Each file is validated against the type registry (Pydantic schema + validators)
- Exit code 0 if all pass, 1 if any fail
- Human-readable output: file name + result per artifact
- Optional `--file PATH` flag to validate a single file
