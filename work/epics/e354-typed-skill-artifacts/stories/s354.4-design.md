# Design: S354.4 — Doc Generation

> **Story:** S354.4 | **Epic:** E354 | **Size:** S

## Summary

Render typed YAML artifacts to human-readable Markdown using plain Markdown templates with `str.format()`. Templates are human-editable, no Jinja.

## Approach

Three pieces:
1. `.raise/templates/artifacts/story-design.md` — Markdown template (human-readable)
2. `render_artifact(artifact) → str` — reads template, pre-renders repeating blocks, fills placeholders
3. `write_doc(artifact, project_root) → Path` — writes to `work/docs/`

Repeating blocks (AC list, integration points, decisions) are pre-rendered in Python into formatted text, then injected into the template as `{acceptance_criteria}`, `{integration_points}`, `{decisions}`.

### Template: `.raise/templates/artifacts/story-design.md`

```markdown
# {story} Design: {summary}

**Story:** {story} | **Epic:** {epic} | **Complexity:** {complexity}
**Skill:** {skill} | **Created:** {created}

## Summary

{summary}

## Acceptance Criteria

{acceptance_criteria}

## Integration Points

{integration_points}

## Decisions

{decisions}
```

Sections with empty content get the whole section (heading + placeholder) removed in post-processing.

## Components

| Component | Change | File |
|-----------|--------|------|
| Template | new | `.raise/templates/artifacts/story-design.md` |
| `render_artifact()` | new | `src/rai_cli/artifacts/renderer.py` |
| `write_doc()` | new | same file |
| `__init__.py` | modify | Export new functions |

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | Markdown template + `str.format()` | Human-readable, customizable without Python. No dependency (no Jinja). |
| D2 | Omit empty sections in post-processing | Clean output for simple stories without empty headings. |
| D3 | `work/docs/` path, same naming as artifacts | `s354.1-design.md` mirrors `s354.1-design.yaml`. Easy to correlate. |
| D4 | Pre-render repeating blocks in Python | Template stays simple (no loop syntax). Python handles the iteration. |
