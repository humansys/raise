# Design: S354.2 — Story-Design Schema + Governance Validators

> **Story:** S354.2 | **Epic:** E354 | **Size:** S

## Summary

First concrete artifact type on top of S354.1's base model. Typed content for story designs + Pydantic validators encoding governance rules.

## Approach

Subclass pattern: `StoryDesignArtifact` inherits `SkillArtifact` and replaces `content: dict` with `content: StoryDesignContent` (typed). Governance rules are `@field_validator` and `@model_validator` on the content model.

Grounded against 3 real designs: s325.6, s338.5, s354.1 — common sections are: summary/problem, complexity, approach/components, decisions, acceptance criteria.

## Components

### New: `src/rai_cli/artifacts/story_design.py`

```python
class Complexity(StrEnum):
    SIMPLE = "simple"
    MODERATE = "moderate"
    COMPLEX = "complex"

class AcceptanceCriterion(BaseModel):
    id: str                    # "AC1", "AC2"
    description: str
    verifiable: bool = True

class IntegrationPoint(BaseModel):
    module: str                # "rai_cli.artifacts"
    change_type: str           # "new", "modification", "deletion"
    files: list[str]

class Decision(BaseModel):
    id: str                    # "D1"
    choice: str
    rationale: str
    alternatives_considered: list[str] = Field(default_factory=list)

class StoryDesignContent(BaseModel):
    summary: str
    complexity: Complexity
    acceptance_criteria: list[AcceptanceCriterion]
    integration_points: list[IntegrationPoint] = Field(default_factory=list)
    decisions: list[Decision] = Field(default_factory=list)

    # Governance validators:
    # - AC count: 1-10 (flexible, not the rigid 3-5 from gate-design)
    # - All AC must be verifiable (warn if not)
    # - Decisions must have rationale (non-empty string)

class StoryDesignArtifact(SkillArtifact):
    artifact_type: Literal[ArtifactType.STORY_DESIGN] = ArtifactType.STORY_DESIGN
    content: StoryDesignContent
```

### Modified: `src/rai_cli/artifacts/reader.py`

`read_artifact` needs a type registry to dispatch to the correct model:

```python
_ARTIFACT_REGISTRY: dict[str, type[SkillArtifact]] = {
    "story-design": StoryDesignArtifact,
}

def read_artifact(path: Path) -> SkillArtifact:
    data = yaml.safe_load(path.read_text())
    artifact_type = data.get("artifact_type", "")
    model_class = _ARTIFACT_REGISTRY.get(artifact_type, SkillArtifact)
    return model_class.model_validate(data)
```

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | AC range 1-10, not 3-5 | Gate says 3-5 but real designs vary. 1-10 is permissive enough to avoid false negatives. |
| D2 | `Literal[ArtifactType.STORY_DESIGN]` for subclass | Pydantic discriminator pattern — ensures `artifact_type` is locked to the correct value. |
| D3 | Type registry in reader | Simple dict dispatch. No plugin system needed for 1-5 artifact types. |
| D4 | `integration_points` and `decisions` optional (empty list) | Simple stories may have no decisions. Schema should not force them. |
