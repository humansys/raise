# Design: S354.6 ‚Äî `rai artifact validate` CLI Command

> **Story:** S354.6 | **Epic:** E354 | **Size:** S

## Summary

Add `rai artifact validate` CLI command that reads YAML artifacts from `.raise/artifacts/`, validates each against the Pydantic type registry, and exits 0/1.

## Approach

Follow `gate.py` pattern: Typer app with a `validate` command. Reuse `read_artifact` / `read_all_artifacts` from `rai_cli.artifacts.reader`. Validation is already done by Pydantic during `model_validate` ‚Äî we just need to catch `ValidationError` and report.

## Components

| Component | Change | File |
|-----------|--------|------|
| `artifact` CLI command | create | `src/rai_cli/cli/commands/artifact.py` |
| Main CLI registration | modify | `src/rai_cli/cli/main.py` |

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | Single `validate` command, not a full CRUD group | YAGNI ‚Äî only validation is needed now. More subcommands can be added later. |
| D2 | Reuse `read_artifact` from reader module | Validation happens inside Pydantic `model_validate`. No separate validation function needed. |

## Acceptance Criteria

- AC1: `rai artifact validate` validates all `.yaml` in `.raise/artifacts/`, exit 0 all pass, 1 any fail
- AC2: `rai artifact validate --file PATH` validates a single file
- AC3: Human-readable output with per-file pass/fail markers
- AC4: `--format json` outputs structured JSON results
