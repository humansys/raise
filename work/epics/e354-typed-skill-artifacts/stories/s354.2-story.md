# Story: S354.2 — Story-Design Schema + Governance Validators

> **Epic:** E354 — Typed Skill Artifacts
> **Branch:** `story/s354.2/story-design-schema`
> **Size:** S
> **Depends:** S354.1 (base artifact model)

## Summary

Define the `StoryDesignArtifact` Pydantic model with typed content fields and governance validators. This is the first concrete artifact type, proving the base model works for real schemas.

## Acceptance Criteria

1. `StoryDesignContent` model with: summary, complexity, acceptance_criteria, integration_points, decisions
2. `StoryDesignArtifact` subclass of `SkillArtifact` with typed content
3. Pydantic validators as governance rules (AC verifiable, complexity valid, decisions have rationale)
4. Round-trip: write story-design YAML → read back → validates with typed content
5. Validation errors have clear messages for governance violations
