# Design: S354.3 — Graph Ingestion

> **Story:** S354.3 | **Epic:** E354 | **Size:** S

## Summary

Add `load_artifacts()` to `GraphBuilder` following the existing `load_*()` pattern. Each artifact becomes a `GraphNode`, refs become edges via `infer_relationships`.

## Approach

Follow the `load_components()` pattern exactly:
1. Read `.raise/artifacts/` via `read_all_artifacts()`
2. Convert each `SkillArtifact` → `GraphNode`
3. Return nodes list
4. Wire into `build()` method

No new GraphNode subclass needed — use `type="artifact"` with metadata carrying the artifact details.

## Components

| Component | Change | File |
|-----------|--------|------|
| `GraphBuilder.load_artifacts()` | new method | `src/rai_cli/context/builder.py` |
| `GraphBuilder.build()` | add `load_artifacts()` call | same file |

### Conversion logic

```python
def load_artifacts(self) -> list[GraphNode]:
    artifacts_dir = self.project_root / ".raise" / "artifacts"
    artifacts = read_all_artifacts(artifacts_dir)
    nodes = []
    for artifact in artifacts:
        node_id = f"artifact-{artifact.story or artifact.epic or 'unknown'}-{artifact.artifact_type.value}"
        node = GraphNode(
            id=node_id,
            type="artifact",
            content=artifact.content.get("summary", str(artifact.content))
                    if isinstance(artifact.content, dict)
                    else getattr(artifact.content, "summary", str(artifact.content)),
            source_file=f".raise/artifacts/{...}",
            created=artifact.created.isoformat(),
            metadata={
                "artifact_type": artifact.artifact_type.value,
                "skill": artifact.skill,
                "story": artifact.story,
                "epic": artifact.epic,
                "version": artifact.version,
            },
        )
        nodes.append(node)
    return nodes
```

### Edge inference

Refs are stored in metadata. The existing `infer_relationships()` uses keyword matching — artifact metadata fields like `story: S354.1` and `epic: E354` will naturally match story/epic nodes by keyword overlap. No custom edge logic needed.

## Decisions

| ID | Choice | Rationale |
|----|--------|-----------|
| D1 | `type="artifact"`, no GraphNode subclass | Follows load_components pattern. Subclass adds complexity without value for 1 type. |
| D2 | Content = summary field | Summary is the most useful text for graph queries. Full content goes in metadata if needed. |
| D3 | Rely on keyword-based edge inference | Refs in metadata naturally match existing nodes. No custom edge code needed. |
