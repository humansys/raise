# Retrospective: S354.2 — Story-Design Schema + Governance Validators

## What went well
- Subclass pattern worked cleanly — `StoryDesignArtifact` inherits `SkillArtifact` naturally
- Type registry is 5 lines of code, does exactly what's needed
- Governance as Pydantic validators is simple and testable

## What could improve
- Adding the registry broke 4 existing tests (reader + roundtrip used untyped content). Tests should have been written against the typed model from the start, but S354.1 didn't have it yet. Expected consequence of incremental delivery.
- Missing imports in test file caught by runtime, not linter (pytest/ValidationError)

## Patterns
- Literal discriminator pattern: `artifact_type: Literal[ArtifactType.STORY_DESIGN]` locks subclass to its type
- Lazy registry with `global` avoids circular imports while keeping reader simple

## Metrics
- Tests: 37 total (17 new for S354.2)
- Files: 1 new production, 2 new test, 3 modified
