## Summary

## Approach

Follow existing model patterns (`MemoryConcept`, `Concept`): StrEnum for types, BaseModel with Field descriptors, metadata dict for extensibility.

### Key Model

```python
class ArtifactType(StrEnum):
    STORY_DESIGN = "story-design"

class ArtifactRefs(BaseModel):
    backlog_item: str | None = None
    epic_scope: str | None = None
    related_artifacts: list[str] = Field(default_factory=list)

class SkillArtifact(BaseModel):
    artifact_type: ArtifactType
    version: int = Field(default=1, ge=1)
    skill: str
    created: datetime
    story: str | None = None
    epic: str | None = None
    content: dict[str, Any]  # Subclasses override with typed content
    refs: ArtifactRefs = Field(default_factory=ArtifactRefs)
    metadata: dict[str, Any] = Field(default_factory=dict)
```

### YAML Format

```yaml

  summary: "..."

  backlog_item: RAISE-402

```

Flat YAML (no nested `artifact:` key) â€” simpler to read/write, matches Pydantic field names directly.

### File Naming

`{story_or_epic_id}-{artifact_type}.yaml` â†’ e.g., `s354.1-design.yaml`

Writer derives filename from `story` + `artifact_type` fields.

### Storage

Writer creates `.raise/artifacts/` if it doesn't exist. Reader globs `*.yaml`.
