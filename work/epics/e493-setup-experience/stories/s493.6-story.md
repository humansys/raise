# Story S493.6: Fix `raise-cli` pipx install without `--include-deps`

## User Story

**As a** developer installing raise-cli,
**I want** `pipx install .` (or `pipx install rai-cli`) to expose the `rai` command,
**So that** I don't need the `--include-deps` workaround to get a working CLI.

## Context

The `[project.scripts] rai = "rai_cli.cli.main:app"` entry already exists in `pyproject.toml`, so the fix may already be in place. This story investigates the current state and fixes or documents accordingly.

## Acceptance Criteria (Gherkin)

```gherkin
Feature: pipx install exposes rai command

  Scenario: pipx install exposes rai command without --include-deps
    Given a clean environment with no prior rai installation
    When I run `pipx install .` from the raise-commons root
    Then the `rai` command is available on PATH
    And no `--include-deps` flag was needed

  Scenario: rai --version works after pipx install
    Given rai was installed via `pipx install .`
    When I run `rai --version`
    Then it prints a version string without error
```

## Size

S (Small) — investigation + possible metadata fix
