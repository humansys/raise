# S493.4 Design — Add Developer Checks to `rai doctor`

## Complexity: Simple

- 1 new file following established pattern (DoctorCheck protocol)
- 1 line in pyproject.toml entry points
- 4 checks, all trivial (file exists, env var set, binary in PATH)

## Problem

`rai doctor` checks environment (Python, OS) and project (.raise/, manifest, graph) but not developer-level setup. On a new machine, a developer has no way to know their profile, credentials, Claude Code, or MCP servers are missing until they hit errors during work.

## Value

After this, `rai doctor -v` gives a complete picture of machine readiness. The `/rai-doctor` skill can guide fixes conversationally.

## Approach

New check class `DeveloperCheck` following the exact pattern of `EnvironmentCheck` and `ProjectCheck`:

- Implements `DoctorCheck` protocol (ClassVar fields + `evaluate()` method)
- Registered via `rai.doctor.checks` entry point in `pyproject.toml`
- Category: `"developer"`
- `requires_online = False` (all checks are local)

### Components

| Component | Change | Files |
|-----------|--------|-------|
| DeveloperCheck | new | `src/raise_cli/doctor/checks/developer.py` |
| Entry point | modify | `pyproject.toml` |

## The 4 Checks

| # | check_id | What | Pass | Warn |
|---|----------|------|------|------|
| 1 | `dev-profile` | `~/.rai/developer.yaml` exists | exists | missing → "Run /rai-welcome or rai profile import" |
| 2 | `dev-credentials` | Jira/Confluence env vars or `.env` | any token set | none → "Create .env with JIRA_URL, JIRA_USERNAME, JIRA_API_TOKEN" |
| 3 | `dev-claude-code` | `shutil.which("claude")` | found | missing → "Install Claude Code: https://claude.ai/claude-code" |
| 4 | `dev-mcp-servers` | `~/.claude.json` has `mcpServers` with entries | has entries | missing/empty → "Run /rai-mcp-add to configure MCP servers" |

All checks return WARN (not ERROR) because developer setup is advisory — you can still use `rai` CLI commands without a profile or Claude Code.

## Examples

### Output

```
$ rai doctor -v

     environment    Python 3.12.3
     environment    raise-cli 2.2.1
     environment    Linux 6.17.0 (x86_64)
     developer      Developer profile found (Emilio, prefix E)
  !! developer      No Jira/Confluence credentials found
       hint: Create .env with JIRA_URL, JIRA_USERNAME, JIRA_API_TOKEN
     developer      Claude Code available
  !! developer      No MCP servers configured
       hint: Run /rai-mcp-add to configure MCP servers
     project        .raise/ directory exists
     ...
```

### Check Implementation (follows existing pattern exactly)

```python
class DeveloperCheck:
    check_id: ClassVar[str] = "developer"
    category: ClassVar[str] = "developer"
    description: ClassVar[str] = "Developer profile, credentials, Claude Code, MCP servers"
    requires_online: ClassVar[bool] = False

    def evaluate(self, context: DoctorContext) -> list[CheckResult]:
        results: list[CheckResult] = []
        results.append(self._check_profile())
        results.append(self._check_credentials(context.working_dir))
        results.append(self._check_claude_code())
        results.append(self._check_mcp_servers())
        return results
```

### Entry Point Registration

```toml
[project.entry-points."rai.doctor.checks"]
environment = "raise_cli.doctor.checks.environment:EnvironmentCheck"
project = "raise_cli.doctor.checks.project:ProjectCheck"
developer = "raise_cli.doctor.checks.developer:DeveloperCheck"
```

## Acceptance Criteria

- **MUST:** 4 checks implemented following DoctorCheck protocol
- **MUST:** Entry point registered so `rai doctor` discovers it automatically
- **MUST:** All checks return WARN (not ERROR) — developer setup is advisory
- **MUST:** Fix hints are actionable (specific commands/URLs)
- **MUST:** Profile check shows name and prefix when found
- **MUST NOT:** Read credential values — only check existence
