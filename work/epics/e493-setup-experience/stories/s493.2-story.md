# Story S493.2: Profile Export/Import

## User Story

**As a** developer setting up a new machine,
**I want** to export my RaiSE profile from my old machine and import it on the new one,
**So that** I retain my identity, experience level, patterns, and coaching context without manual recreation.

## Context

Developer profile lives at `~/.rai/developer.yaml`. It contains: name, experience level, communication prefs, pattern prefix, coaching context, corrections, deadlines. When moving to a new machine, this must be ported. Currently there's no tooling — manual copy loses machine-local state (active_sessions) that shouldn't transfer.

The epic design (work/epics/e493-setup-experience/design.md) defines the bundle format:
- YAML bundle with _meta (version, exported_at, source_machine) + profile (minus machine-local fields)
- Import validates, strips active_sessions/current_session, adjusts paths

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Export profile to YAML bundle
  Given a developer profile exists at ~/.rai/developer.yaml
  When I run "rai profile export"
  Then a YAML file is written to stdout (or --output path)
  And it contains _meta with version, timestamp, source_machine
  And it contains the profile without active_sessions or current_session

Scenario: Export with --output flag
  Given a developer profile exists
  When I run "rai profile export --output /tmp/profile-bundle.yaml"
  Then the bundle is written to /tmp/profile-bundle.yaml

Scenario: Export fails gracefully without profile
  Given no developer profile exists
  When I run "rai profile export"
  Then it prints an error message and exits with non-zero

Scenario: Import profile from bundle
  Given a valid profile bundle at /tmp/profile-bundle.yaml
  When I run "rai profile import /tmp/profile-bundle.yaml"
  Then ~/.rai/developer.yaml is created/updated
  And active_sessions is empty (machine-local state cleared)
  And current_session is None

Scenario: Import preserves existing sessions if profile exists
  Given a developer profile already exists with active sessions
  When I run "rai profile import /tmp/profile-bundle.yaml"
  Then the user is warned about overwriting
  And must confirm (or use --force)

Scenario: Import rejects invalid bundle
  Given a file that is not a valid profile bundle
  When I run "rai profile import bad-file.yaml"
  Then it prints a validation error and exits non-zero
```

## Size

M (Medium) — new CLI subcommands + export/import logic + tests
