# S493.5 Scope — CLI .env Loading

## In Scope

- Add `python-dotenv` to `pyproject.toml` dependencies
- Load `.env` from CWD at CLI startup (before command dispatch)
- Existing env vars take precedence over `.env` values (`override=False`)

## Out of Scope

- Recursive `.env` search (parent directories)
- `.env.local`, `.env.production`, or other variant files
- Loading `.env` in non-CLI contexts (library usage)
- Encrypting or validating `.env` contents

## Done Criteria

- [ ] `python-dotenv` added as dependency
- [ ] `.env` loaded at CLI entrypoint before command dispatch
- [ ] Shell env vars take precedence over `.env` values
- [ ] No crash when `.env` is absent
- [ ] Tests cover: .env present, .env absent, env var precedence
