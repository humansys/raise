# S493.5 — CLI loads .env for adapter credentials

## User Story (Connextra)

**As a** developer using RaiSE from the terminal,
**I want** `rai backlog` and other adapter commands to load `.env` from the project root,
**So that** I don't need to manually export environment variables every terminal session.

## Context

Currently, `.env` is only loaded by Claude Code's MCP server. When running `rai backlog list`
from a plain terminal, Jira/Confluence credentials are not available unless manually exported.
This is the #7 friction point from the new-machine setup experience.

`python-dotenv` is not currently a dependency but the pattern is standard Python.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: CLI loads .env from project root
  Given a project with a .env file containing JIRA_URL and JIRA_TOKEN
  When I run "rai backlog list" from the project root
  Then the command has access to the environment variables from .env

Scenario: CLI works without .env
  Given a project without a .env file
  When I run "rai backlog list"
  Then the command proceeds normally (no crash, no warning about missing .env)

Scenario: Existing env vars take precedence
  Given JIRA_URL is set in the shell environment
  And a .env file contains a different JIRA_URL
  When I run "rai backlog list"
  Then the shell environment value is used (not the .env value)
```

## Size

S (Small) — add dependency + ~5 lines at CLI entrypoint
