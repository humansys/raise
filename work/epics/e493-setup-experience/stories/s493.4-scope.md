# S493.4 Scope — Add New-Machine Checks to `rai doctor`

## In Scope

- New check class: `src/raise_cli/doctor/checks/developer.py` (DeveloperCheck)
- 4 new checks in "developer" category:
  1. Developer profile exists (~/.rai/developer.yaml)
  2. Credentials available (env vars or .env for Jira/Confluence)
  3. Claude Code available (claude in PATH)
  4. MCP servers configured (~/.claude.json has mcpServers)
- Register via entry point in pyproject.toml
- Tests for all 4 checks

## Out of Scope

- Changing existing environment or project checks
- Adding --new-machine flag (existing --category developer serves same purpose)
- Auto-fix for developer checks (profile/credentials are user decisions)
- MCP server health checks (that's /rai-mcp-status)

## Done When

- `rai doctor -v` shows developer category checks
- All 4 checks implemented with actionable fix hints
- Tests pass for all check functions
- Entry point registered in pyproject.toml
