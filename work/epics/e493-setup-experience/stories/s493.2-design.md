# Story S493.2 Design: Profile Export/Import

## Complexity

Moderate (3-4 components: portability module, CLI subcommands, bundle validation, tests)

## Problem & Value

- **Problem:** No way to port developer identity between machines. Manual copy of `~/.rai/developer.yaml` brings machine-local state (active_sessions, current_session, project paths) that causes confusion on the new machine.
- **Value:** 1-2 commands to transfer identity. Clean separation of portable vs machine-local data.

## Approach

### New file: `src/rai_cli/onboarding/profile_portability.py`

Two pure functions plus a Pydantic model for the bundle:

```python
class ProfileBundle(BaseModel):
    """Portable profile bundle with metadata."""
    _meta: BundleMeta  # version, exported_at, source_machine
    profile: dict      # DeveloperProfile minus machine-local fields

def export_profile(profile: DeveloperProfile) -> ProfileBundle:
    """Strip machine-local fields, add metadata, return bundle."""

def import_profile(bundle: ProfileBundle) -> DeveloperProfile:
    """Validate bundle, construct DeveloperProfile with clean state."""
```

Machine-local fields stripped on export (D3):
- `active_sessions` (list of machine-specific sessions)
- `current_session` (deprecated, machine-specific)
- `projects` (absolute paths, machine-specific)

### Modify: `src/rai_cli/cli/commands/profile.py`

Add two subcommands to existing `profile_app`:

```python
@profile_app.command()
def export(output: Optional[Path] = None) -> None:
    """Export profile to portable YAML bundle."""

@profile_app.command(name="import")  # 'import' is reserved keyword
def import_cmd(path: Path, force: bool = False) -> None:
    """Import profile from YAML bundle."""
```

## Key Decisions

| ID | Decision | Rationale | Alternatives Considered |
|----|----------|-----------|------------------------|
| D1 | Export to stdout by default, `--output` for file | Unix convention, pipeable | Always write to file |
| D2 | Import requires confirmation when profile exists (`--force` to skip) | Prevent accidental data loss | Always overwrite; merge profiles |
| D3 | Strip `active_sessions`, `current_session`, `projects` | Machine-local state that causes confusion on new machine | Strip only `active_sessions` |

## Bundle Format

```yaml
_meta:
  version: 1
  exported_at: "2026-03-06T21:00:00Z"
  source_machine: "macbook-emilio"
profile:
  name: Emilio
  pattern_prefix: E
  experience_level: ha
  communication:
    style: direct
    language: es
    skip_praise: true
    detailed_explanations: false
    redirect_when_dispersing: true
  skills_mastered: [...]
  universal_patterns: [...]
  first_session: "2025-06-15"
  last_session: "2026-03-06"
  coaching:
    strengths: [...]
    growth_edge: "..."
    trust_level: established
    corrections: [...]
    relationship: { quality: strong, trajectory: stable }
  deadlines: [...]
  # active_sessions: EXCLUDED
  # current_session: EXCLUDED
  # projects: EXCLUDED
```

## Examples

### Export

```bash
$ rai profile export
_meta:
  version: 1
  exported_at: "2026-03-06T21:00:00Z"
  source_machine: "macbook-emilio"
profile:
  name: Emilio
  pattern_prefix: E
  experience_level: ha
  ...

$ rai profile export --output ~/profile-bundle.yaml
Profile exported to /Users/emilio/profile-bundle.yaml
```

### Import

```bash
$ rai profile import ~/profile-bundle.yaml
Profile found at ~/.rai/developer.yaml. Overwrite? [y/N]: y
Profile imported from macbook-emilio (exported 2026-03-06).

$ rai profile import ~/profile-bundle.yaml --force
Profile imported from macbook-emilio (exported 2026-03-06).
```

## Acceptance Criteria

| ID | Priority | Criterion |
|----|----------|-----------|
| AC1 | MUST | Export produces valid YAML bundle with `_meta` and `profile` (minus machine-local fields) |
| AC2 | MUST | Import creates/updates `~/.rai/developer.yaml`, strips `active_sessions`/`current_session`/`projects` |
| AC3 | MUST | Import warns on overwrite unless `--force` |
| AC4 | MUST | Export fails gracefully when no profile exists (error message + non-zero exit) |
| AC5 | MUST | Import rejects invalid bundle with clear error message |
| AC6 | SHOULD | Export to stdout by default (pipeable) |
| AC7 | MUST NOT | Export Claude memory or MCP config |

## Test Strategy

- Unit tests for `export_profile()`: verify machine-local fields stripped, `_meta` present
- Unit tests for `import_profile()`: valid bundle round-trip, invalid bundle rejection
- CLI integration tests: stdout output, `--output` file write, `--force` flag, confirmation prompt
- Edge cases: empty profile, missing `_meta`, wrong version number
