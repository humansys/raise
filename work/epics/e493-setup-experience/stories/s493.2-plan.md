# Story S493.2 Plan: Profile Export/Import

## Execution Order

T1 -> T2 -> T3 -> T4 (linear — each builds on the previous)

## Tasks

### T1: Export logic + tests (S)

**Goal:** Pure function to export a DeveloperProfile into a portable ProfileBundle.

**TDD Cycle:**
- RED: Test `export_profile()` returns ProfileBundle with `_meta` (version, exported_at, source_machine) and `profile` dict. Test machine-local fields (`active_sessions`, `current_session`, `projects`) are excluded. Test `serialize_bundle()` produces valid YAML string.
- GREEN: Create `src/rai_cli/onboarding/profile_portability.py` with:
  - `BundleMeta(BaseModel)`: `version: int`, `exported_at: datetime`, `source_machine: str`
  - `ProfileBundle(BaseModel)`: `meta: BundleMeta`, `profile: dict[str, Any]`
  - `MACHINE_LOCAL_FIELDS: frozenset = {"active_sessions", "current_session", "projects"}`
  - `export_profile(profile: DeveloperProfile) -> ProfileBundle` — `model_dump(mode="json")`, pop machine-local fields, add meta
  - `serialize_bundle(bundle: ProfileBundle) -> str` — dump as YAML with `_meta` key mapping
- REFACTOR: Extract constants, ensure clean separation

**Files:**
- New: `src/rai_cli/onboarding/profile_portability.py`
- New: `tests/onboarding/test_profile_portability.py`

**Verification:** `pytest tests/onboarding/test_profile_portability.py -x && ruff check src/rai_cli/onboarding/profile_portability.py && pyright src/rai_cli/onboarding/profile_portability.py`

### T2: Import logic + tests (S)

**Goal:** Parse a YAML bundle and reconstruct a clean DeveloperProfile.

**TDD Cycle:**
- RED: Test `parse_bundle()` validates YAML structure, rejects missing `_meta`, rejects wrong version. Test `import_profile()` returns DeveloperProfile with machine-local fields cleared. Test round-trip: export -> serialize -> parse -> import produces equivalent profile (minus machine-local).
- GREEN: Add to `profile_portability.py`:
  - `parse_bundle(content: str) -> ProfileBundle` — `yaml.safe_load`, validate structure, construct ProfileBundle
  - `import_profile(bundle: ProfileBundle) -> DeveloperProfile` — reconstruct DeveloperProfile from bundle profile dict, ensure `active_sessions=[]`, `current_session=None`, `projects=[]`
- REFACTOR: Error messages, edge cases (empty profile fields)

**Files:**
- Modify: `src/rai_cli/onboarding/profile_portability.py`
- Modify: `tests/onboarding/test_profile_portability.py`

**Verification:** `pytest tests/onboarding/test_profile_portability.py -x && ruff check src/rai_cli/onboarding/profile_portability.py && pyright src/rai_cli/onboarding/profile_portability.py`

### T3: CLI subcommands + tests (S)

**Goal:** Wire export/import logic into `rai profile export` and `rai profile import` CLI commands.

**TDD Cycle:**
- RED: Test `export` subcommand outputs YAML to stdout. Test `export --output path` writes file. Test `export` with no profile exits non-zero. Test `import path` calls import_profile and saves. Test `import` prompts for confirmation when profile exists. Test `import --force` skips confirmation.
- GREEN: Add to `src/rai_cli/cli/commands/profile.py`:
  - `export(output: Optional[Path] = None)` — load profile, export_profile, serialize_bundle, print or write
  - `import_cmd(path: Path, force: bool = False)` — read file, parse_bundle, check existing profile, confirm unless --force, import_profile, save_developer_profile
- REFACTOR: Error handling, user-facing messages

**Files:**
- Modify: `src/rai_cli/cli/commands/profile.py`
- New or modify: `tests/cli/commands/test_profile.py`

**Verification:** `pytest tests/cli/commands/test_profile.py -x && ruff check src/rai_cli/cli/commands/profile.py && pyright src/rai_cli/cli/commands/profile.py`

### T4: Integration test (XS)

**Goal:** Verify full round-trip: export -> import produces equivalent profile.

**Steps:**
- Create a DeveloperProfile with all fields populated (including coaching, corrections, deadlines)
- Export to bundle string
- Parse and import bundle
- Assert imported profile matches original on all portable fields
- Assert machine-local fields are empty/None
- Edge case: export profile with no coaching context, reimport

**Files:**
- Modify: `tests/onboarding/test_profile_portability.py` (add integration test class/section)

**Verification:** `pytest tests/onboarding/test_profile_portability.py -x`

## Risks

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Pydantic `_meta` field name conflicts with private attribute convention | Medium | Use `meta` internally, serialize as `_meta` in YAML output |
| YAML serialization of date/datetime types | Low | Use `mode="json"` in model_dump for consistent serialization |
| Typer `import` reserved keyword collision | Low | Already planned: use `name="import"` + `import_cmd` function name |
