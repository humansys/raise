# Story S493.1: OS-specific Installation Guide

## User Story

**As a** new developer wanting to use RaiSE,
**I want** clear OS-specific installation instructions,
**So that** I can install and run `rai --version` following docs alone on macOS, Linux, or WSL.

## Context

There's no centralized installation guide. The README has partial info. RAISE-492 tracks this.
Real-world friction: asdf/pyenv intercepting pip/pipx, rai-cli vs raise-cli confusion.

## Acceptance Criteria (Gherkin)

Scenario: New user follows macOS guide
  Given a macOS machine with no prior RaiSE installation
  When I follow docs/installation.md macOS section step by step
  Then `rai --version` works

Scenario: New user follows Linux guide
  Given a Linux machine with no prior RaiSE installation
  When I follow docs/installation.md Linux section step by step
  Then `rai --version` works

Scenario: README links to installation guide
  Given I open README.md
  Then I find a link to docs/installation.md for detailed setup

## Size

S (Small) — documentation only, no code changes
