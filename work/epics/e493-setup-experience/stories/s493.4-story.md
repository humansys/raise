# S493.4 — Add New-Machine Checks to `rai doctor`

## User Story

As a **developer setting up a new machine**,
I want `rai doctor` to check developer-level setup (profile, credentials, Claude Code, MCP),
So that I know what's missing beyond just project structure.

## Context

`rai doctor` already exists (E352) with environment checks (Python, OS, extras) and
project checks (.raise/, manifest, graph, skills, gitignore). What's missing are
**developer-level** checks that matter on a new machine but aren't project-specific.

## Acceptance Criteria

```gherkin
Scenario: Missing developer profile detected
  Given no ~/.rai/developer.yaml exists
  When I run `rai doctor`
  Then a "developer" category check shows warn
  And the fix hint says to run /rai-welcome or rai profile import

Scenario: Missing environment variables detected
  Given JIRA_API_TOKEN is not set and no .env exists
  When I run `rai doctor`
  Then a "developer" category check shows warn for missing credentials

Scenario: Claude Code not installed
  Given claude is not in PATH
  When I run `rai doctor`
  Then a "developer" category check shows warn
  And the fix hint references Claude Code installation

Scenario: MCP servers not configured
  Given ~/.claude.json has no mcpServers
  When I run `rai doctor`
  Then a "developer" category check shows warn
```
