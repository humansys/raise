# S350.3: MEMORY.md + CLAUDE.md Regeneration

## User Story (Connextra)

**As a** developer onboarding to a RaiSE project,
**I want** MEMORY.md and CLAUDE.md to be automatically regenerated from canonical sources,
**So that** my Rai experience is always consistent and never drifts from the project's actual state.

## Context

MEMORY.md and CLAUDE.md are derived artifacts that currently drift via hand-edits (M4, M5 from S350.1 inventory). The generators exist but are only called during `rai init` first-time setup. This story wires them into the lifecycle:

- `rai graph build` triggers MEMORY.md regeneration via hook
- `rai init` always regenerates CLAUDE.md from `.raise/` sources
- CLAUDE.md becomes a generated artifact (never hand-edited, per ADR-044 D2)

## Acceptance Criteria (Gherkin)

### AC1: MEMORY.md regeneration on graph build
```gherkin
Given a project with a knowledge graph
When I run `rai graph build`
Then MEMORY.md is regenerated from the graph
And the previous MEMORY.md content is preserved or improved
```

### AC2: CLAUDE.md regeneration on rai init
```gherkin
Given a project with .raise/ directory containing identity, methodology, and manifest
When I run `rai init`
Then CLAUDE.md is regenerated from the canonical sources
And the generated file contains all required sections (Identity, Process Rules, Branch Model, CLI Reference, External Integrations)
```

### AC3: CLAUDE.md is always generated
```gherkin
Given CLAUDE.md exists and has been hand-edited
When I run `rai init`
Then the hand-edited version is overwritten with the generated version
And a warning is shown if local changes would be lost
```

### AC4: Hook integration
```gherkin
Given the builtin hooks are registered
When a GraphBuildEvent is emitted
Then the memory_md_sync hook is invoked
And MEMORY.md at .raise/rai/memory/MEMORY.md is updated
```

### AC5: CLAUDE.local.md for per-developer overrides
```gherkin
Given CLAUDE.local.md exists in the project root
When Claude Code loads project context
Then CLAUDE.local.md supplements CLAUDE.md without modifying it
And CLAUDE.local.md is git-ignored
```
