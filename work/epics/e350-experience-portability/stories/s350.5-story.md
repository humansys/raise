# S350.5: Enhanced rai init -- Idempotent, Regenerate All Derived Artifacts

## User Story (Connextra)

**As a** developer joining or maintaining a RaiSE project,
**I want** `rai init` to be idempotent -- safe to run multiple times, regenerating all derived artifacts without destroying configuration,
**So that** I can onboard with `git clone` + `rai init` and existing developers can update their local setup after pulling changes.

## Context

ADR-044 D4 decided against a separate `rai upgrade` command: `rai init` IS the upgrade path. This story wires together the regeneration capabilities built in S350.3 (CLAUDE.md generation) and S350.4 (base pattern loading) into a single idempotent init flow.

Currently `rai init` handles first-time setup. This story extends it to also handle updates:
- Regenerate CLAUDE.md from `.raise/` sources (S350.3 built the generator)
- Copy/update base patterns from the package to `.raise/rai/memory/` (S350.4 built the 3-level model)
- Create personal directory structure (`.raise/rai/memory/personal/`, etc.)
- Detect existing project and skip destructive operations
- Report what was created vs. what was updated

## Acceptance Criteria (Gherkin)

### AC1: First-time init creates everything
```gherkin
Given a fresh git repository with no .raise/ directory
When I run `rai init`
Then .raise/ directory structure is created
And CLAUDE.md is generated from .raise/ sources
And base patterns are copied from the package
And personal directories are created (git-ignored)
```

### AC2: Re-running init updates derived artifacts
```gherkin
Given a project already initialized with `rai init`
When I run `rai init` again
Then CLAUDE.md is regenerated (reflecting any .raise/ source changes)
And base patterns are refreshed from the package (new patterns added, obsolete removed)
And existing project patterns are preserved
And existing personal patterns are preserved
And no configuration is lost
```

### AC3: Init is safe after pulling upstream changes
```gherkin
Given a developer pulls changes that modify .raise/ sources
When they run `rai init`
Then CLAUDE.md reflects the updated sources
And new base patterns from the upgraded package are available
And their personal patterns remain untouched
```

### AC4: Init reports what it did
```gherkin
Given `rai init` runs (first-time or update)
Then it reports which artifacts were created
And which artifacts were updated
And which artifacts were skipped (already current)
```

### AC5: Skills are synced from builtin to deployment
```gherkin
Given the package includes builtin skills in src/rai_cli/skills_base/
When I run `rai init`
Then skills are copied/updated to .claude/skills/
And stale skills not in the package are flagged (not deleted)
```
