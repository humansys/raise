# S350.1: Inventory & Classify Artifacts

## User Story

**As a** developer onboarding to the RaiSE project,
**I want** a complete inventory of every artifact that constitutes the Rai experience, classified by portability layer,
**So that** I know what ships with the package, what is shared via git, what is per-developer, and what is generated — enabling the team to design the portability solution with clear boundaries.

## Acceptance Criteria (Gherkin)

```gherkin
Feature: Artifact Inventory & Classification

  Scenario: Complete inventory exists
    Given the Rai experience is spread across multiple directories and file types
    When I read the classification document
    Then every file/directory that affects Rai behavior is listed
    And no artifact that affects behavior is missing from the inventory

  Scenario: Each artifact has a classification
    Given the classification document lists all artifacts
    When I look at any artifact entry
    Then it is classified as exactly one of: base, project, personal, or derived
    And the classification includes the current distribution mechanism (git, package, generated, ignored)

  Scenario: Classification covers all portability layers
    Given the four layers are base, project, personal, and derived
    When I review the classification
    Then at least one artifact exists in each layer
    And the layer definitions are documented with clear boundaries

  Scenario: Artifacts without clear classification are flagged
    Given some artifacts may have ambiguous classification
    When the inventory is complete
    Then any artifact with mixed concerns is flagged with a note
    And a recommendation for resolution is provided
```
