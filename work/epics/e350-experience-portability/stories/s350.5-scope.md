# S350.5: Enhanced rai init -- Scope

## Objective

Make `rai init` idempotent: safe for first-time setup AND updates, regenerating all derived artifacts without destroying configuration.

## In Scope

- **Idempotent init flow** -- detect existing project, switch between create and update modes
  - First-time: create `.raise/` structure, generate everything
  - Update: regenerate derived artifacts, preserve configuration
- **CLAUDE.md regeneration** -- call the generator built in S350.3 on every init
  - Always regenerate from `.raise/` canonical sources
  - No manual edits to CLAUDE.md survive (by design, per ADR-044)
- **Base pattern sync** -- copy/refresh base patterns from package
  - Use the 3-level model from S350.4
  - Add new base patterns, update changed ones
  - Never touch project or personal patterns
- **Personal directory creation** -- ensure `.raise/rai/memory/personal/` exists
  - Create if missing, skip if present
  - Verify `.gitignore` coverage
- **Skills sync** -- copy builtin skills to `.claude/skills/`
  - Update existing skills from package
  - Flag stale skills (in deployment but not in package)
- **Reporting** -- clear output of what init did
  - Created / Updated / Skipped / Flagged categories
- **Tests** -- unit tests for init logic, integration test for full flow

## Out of Scope

- `rai upgrade` as separate command (ADR-044 D4: init IS upgrade)
- Diff preview before changes (SHOULD in epic scope, deferred)
- Interactive confirmation prompts
- Migration from legacy `.rai/` directory layouts
- `rai skill sync` as standalone command (S350.6)

## Dependencies

- S350.3 (done): CLAUDE.md generator -- `rai_cli.init.claude_md` or similar
- S350.4 (done): 3-level pattern model -- base pattern loading from package
- Existing `rai init` command -- `src/rai_cli/commands/init.py`
