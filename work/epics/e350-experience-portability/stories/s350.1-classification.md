# S350.1: Artifact Classification

**Created:** 2026-03-04 | **Method:** Filesystem audit + git-status cross-reference

## Layer Definitions

| Layer | Scope | Distribution | Boundary |
|-------|-------|-------------|----------|
| **Base** | Ships with `rai-cli` package; identical across all projects/developers | PyPI package (`src/rai_cli/`) | Universal Rai identity, methodology, governance, base patterns, builtin skills |
| **Project** | Shared across team members for one repository | Git-tracked in `.raise/` or repo root | Project config, project patterns, integrations, templates, katas, gates, backlog |
| **Personal** | Per-developer, never shared | `.gitignore`'d under `.raise/rai/personal/`, `.raise/rai/sync/`, `.claude/settings.local.json` | Sessions, calibration, personal telemetry, sync state |
| **Derived** | Generated from other sources; should be regenerable | Currently git-tracked (target: regenerated on demand) | CLAUDE.md, MEMORY.md, graph HTMLs, index.json, skill projections, manifests |

### Key Findings

1. **Dual-location skills** are the largest derived-layer concern: 31 skills exist in both `skills_base/` (source) and `.claude/skills/` (projection), requiring sync via `rai init`.

2. **Identity and methodology projections** (`.raise/rai/identity/`, `.raise/rai/framework/`) are currently treated as project files but are actually derived from base sources. This creates confusion about source-of-truth.

3. **CLAUDE.md drift** is the most impactful mixed concern: it is generated but then hand-edited, making regeneration risky without a diff/merge strategy.

4. **Legacy `.claude/` files** (RAI.md, RAI-naming.md, rai.archive/) are git-tracked but superseded. They add noise without active value.

5. **Telemetry has three locations**: `.raise/telemetry/` (project structure), `.raise/rai/telemetry/` (legacy, gitignored), `.raise/rai/personal/telemetry/` (current personal). The legacy location should be migrated.

6. **`.claude/worktrees/`** is not gitignored despite being per-developer ephemeral state.

7. **5 deployment-only skills** exist in `.claude/skills/` without builtin counterparts — these are project-layer skills that bypass the normal `skills_base/` → `.claude/skills/` projection.
