# S350.3: MEMORY.md + CLAUDE.md Regeneration — Scope

## In Scope

- **Hook: memory_md_sync** — builtin hook that listens to GraphBuildEvent and calls `generate_memory_md()`
  - Located at `src/rai_cli/hooks/builtin/memory_md_sync.py`
  - Writes to `.raise/rai/memory/MEMORY.md`
- **CLAUDE.md generator** — function that assembles CLAUDE.md from canonical `.raise/` sources
  - Sources: `identity/core.md`, `methodology.yaml`, `manifest.yaml`, `jira.yaml`
  - Called by `rai init` (both first-time and update modes)
  - Output: `CLAUDE.md` in project root (git-tracked, generated)
- **CLAUDE.local.md** — per-developer override file
  - Added to `.gitignore`
  - Documented as the override mechanism
- **Tests** for hook registration, MEMORY.md generation trigger, CLAUDE.md content assembly

## Out of Scope

- Diff preview before CLAUDE.md overwrite (deferred to S350.5: enhanced init)
- Pattern level awareness in MEMORY.md (deferred to S350.4)
- Interactive conflict resolution (deferred to S350.5)
- Skill projection (already handled by existing `rai init`)
