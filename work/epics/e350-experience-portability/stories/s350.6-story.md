# S350.6: rai skill sync -- Detect Stale Skills Post-Upgrade

## User Story (Connextra)

**As a** developer who has upgraded the rai-cli package,
**I want** a standalone `rai skill sync` command that detects which deployed skills are stale compared to builtins,
**So that** I can check skill freshness without running a full `rai init`.

## Context

S350.5 implemented skill syncing as part of the `rai init` flow. The `scaffold_skills` function in `rai_cli.onboarding.skills` already has the three-hash algorithm (dpkg-style) to classify skills as current, auto-updatable, conflicted, etc.

This story exposes that detection logic as a standalone `rai skill sync` command. The command is read-only by default (report only), with an optional `--apply` flag to actually perform the sync (delegating to `scaffold_skills`).

Key design decisions:
- Reuse `scaffold_skills(dry_run=True)` for detection -- no duplication of sync logic
- `rai skill sync` without flags = dry-run report (safe, read-only)
- `rai skill sync --apply` = actually sync (same as what `rai init` does for skills)
- Exit code 0 if all current, exit code 1 if stale skills detected (useful for CI)

## Acceptance Criteria (Gherkin)

### AC1: Detect stale skills
```gherkin
Given a project with skills deployed from a previous rai-cli version
And the current rai-cli has newer bundled skills
When I run `rai skill sync`
Then it reports which skills are stale (need update)
And which skills are current
And which skills have conflicts (both sides changed)
And the exit code is 1 (stale detected)
```

### AC2: All current -- clean exit
```gherkin
Given a project with all skills matching current builtins
When I run `rai skill sync`
Then it reports all skills as current
And the exit code is 0
```

### AC3: Apply sync
```gherkin
Given a project with stale skills
When I run `rai skill sync --apply`
Then stale skills are updated (auto-update where safe)
And conflicts are reported (not auto-resolved without --force)
And the manifest is updated
```

### AC4: Force apply
```gherkin
Given a project with conflicted skills
When I run `rai skill sync --apply --force`
Then all skills are overwritten with bundled versions
And the manifest reflects the new state
```
