# S350.1: Inventory & Classify Artifacts — Plan

**Size:** S | **Tasks:** 3 | **Dependencies:** Linear (T1 -> T2 -> T3)

## Task Breakdown

### T1: Audit filesystem

**Goal:** Walk every relevant directory and catalog every file/directory that affects Rai behavior.

**Scope:**
- `.raise/` — project-level Rai configuration and manifest
- `.raise/rai/` — identity, memory, framework, relationships, sync, telemetry, personal
- `.claude/` — IDE integration (settings, skills, scripts)
- `src/rai_cli/skills_base/` — builtin skills (ship with package)
- `src/rai_cli/rai_base/` — base Rai runtime files (identity, methodology)
- `CLAUDE.md` — project instructions (root)
- `.gitignore` — distribution boundary definition
- Root config files (`pyproject.toml`, etc.) that affect Rai behavior

**Method:** Use glob/ls to enumerate every path. For each, record: path, purpose, whether git-tracked, whether shipped in package.

**Output:** Raw inventory list (working notes, not committed — feeds T2).

**Verification:** Inventory covers all 14 artifact categories from scope document.

**Depends on:** Nothing

---

### T2: Classify and write document

**Goal:** Take the raw inventory from T1, classify each artifact into base/project/personal/derived, flag mixed concerns, write the classification document.

**Scope:**
- Assign each artifact to exactly one layer using definitions from design document
- Flag artifacts with mixed concerns (e.g., config + credentials, source + projection)
- Write resolution recommendations for flagged items
- Structure as markdown tables grouped by layer

**Output:** `work/epics/e350-experience-portability/stories/s350.1-classification.md`

**Verification:**
- Every artifact from T1 inventory appears in exactly one layer table
- At least one artifact in each of the four layers
- Mixed-concern artifacts have an Issues column entry with resolution recommendation
- Document matches acceptance criteria AC-1 through AC-5

**Depends on:** T1

---

### T3: Verify completeness

**Goal:** Run glob/grep audit to confirm no artifacts were missed. Cross-reference against multiple sources.

**Cross-reference checks:**
1. Glob `.raise/**/*` and verify every path is accounted for in classification
2. Glob `.claude/**/*` and verify coverage
3. Glob `src/rai_cli/rai_base/**/*` and `src/rai_cli/skills_base/**/*` and verify coverage
4. Check `.gitignore` entries that relate to Rai artifacts — confirm those paths are inventoried
5. Check `pyproject.toml` package data includes for base layer artifacts
6. Cross-reference the 14 artifact categories from scope document

**Output:** Completeness verified (or classification document updated with missing items).

**Verification:** All cross-reference checks pass with no uncovered artifacts.

**Depends on:** T2

---

## Execution Order

```
T1: Audit filesystem
 └─► T2: Classify and write document
      └─► T3: Verify completeness
```

Linear chain. Each task depends on the prior task's output.

## Notes

- No code changes, no tests — documentation-only story (AC-6)
- TDD cycle does not apply; verification is by audit (glob/grep) and human review (HITL gate)
- The classification document is the sole deliverable; working notes from T1 are not committed
