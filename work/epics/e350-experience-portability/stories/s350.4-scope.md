# S350.4: Pattern Destillation & 3-Level Model — Scope

## Objective

Implement 3-level pattern architecture (base/project/personal) per ADR-044 D3.

## In Scope

- **Distill ~626 patterns into ~50 base first-principles** — curate, merge, and deduplicate existing patterns into a compact base set
  - Base patterns ship with the package at `src/rai_cli/data/patterns/base.jsonl` (or similar)
  - Each base pattern has clear domain, rationale, and non-overlapping scope
- **Implement base/project/personal separation** — three-tier loading and storage
  - Base: read-only, ships with package
  - Project: `.raise/rai/memory/patterns.jsonl` (git-tracked)
  - Personal: `.raise/rai/memory/personal/patterns.jsonl` (git-ignored)
- **`rai pattern add` defaults to personal** — new patterns go to personal level by default
  - Add `--scope` flag: `personal` (default), `project`, `base` (dev-only)
- **`rai pattern promote` moves personal → project** — explicit promotion command
  - Takes pattern ID, moves from personal to project JSONL
  - Validates pattern exists at personal level before promoting
- **Merge logic** — when loading patterns, merge all three levels with specificity resolution (personal > project > base)
- **Tests** for three-level loading, add, promote, merge conflict resolution

## Out of Scope

- Pattern analytics (frequency, impact metrics)
- Pattern recommendation engine
- Auto-promotion (patterns promoted only by explicit command)
- Pattern demotion (project → personal)
- Organization-level pattern layer (ADR-044 explicitly defers this)

## Dependencies

- S350.1 (done): Artifact inventory — classification of patterns as base/project/personal
- Existing `src/rai_cli/memory/patterns.py` — current pattern loading/storage
- Existing `rai pattern add` CLI command
