# S350.1: Inventory & Classify Artifacts — Scope

## In Scope

- Full inventory of all files, directories, and configuration that constitute the Rai experience
- Classification of each artifact into exactly one of four layers:
  - **Base** — ships with the `rai-cli` package, same for all instances
  - **Project** — git-tracked, shared across team members
  - **Personal** — per-developer, git-ignored
  - **Derived** — generated from other sources (should be regenerable)
- Document the current distribution mechanism for each artifact (git, package, .gitignore, generated)
- Flag artifacts with mixed concerns (e.g., config + credentials in same file)
- Provide resolution recommendations for flagged artifacts

### Artifact Categories to Inventory

1. **Identity** — `core.md`, `perspective.md`
2. **Methodology** — `methodology.yaml`
3. **Skills** — 31 in `skills_base/`, 36 in `.claude/skills/` (5 deployment-only)
4. **Patterns** — `patterns.jsonl` (728 entries)
5. **Memory** — `MEMORY.md`, `index.json`, graph HTMLs
6. **CLAUDE.md** — project instructions (root)
7. **Personal** — sessions, calibration, last-diff, session-output
8. **Integrations** — `jira.yaml`, `confluence.yaml`, MCP configs
9. **Sync state** — `sync/state.json`
10. **Telemetry** — `events.jsonl`, `signals.jsonl`
11. **Relationships** — `humans.jsonl`
12. **IDE config** — `.claude/settings.json`, `.claude/skills/`
13. **Manifest** — `.raise/manifest.yaml`
14. **Skill artifacts** — `.raise/skills/*.yaml` (typed skill definitions)

## Out of Scope

- Implementing any changes to artifact locations or distribution (that is S350.2+)
- Pattern destillation or 3-level pattern model (S350.4)
- CLAUDE.md or MEMORY.md regeneration logic (S350.3)
- Credential separation implementation (S350.7)
- Any code changes — this story produces documentation only

## Done Criteria (Observable Outcomes)

1. **Classification document exists** at `work/epics/e350-experience-portability/stories/s350.1-classification.md`
2. **Every artifact that affects Rai behavior is inventoried** — verified by grep/glob audit against actual file tree
3. **Each artifact has exactly one classification** — base, project, personal, or derived
4. **Mixed-concern artifacts are flagged** with specific resolution recommendations
5. **Classification document reviewed and approved** by human (HITL gate)
6. **No code changes** — this is a documentation-only story
