# S350.7: Jira Credentials Separation — Scope

## Objective

Separate per-developer credentials from per-repo Jira configuration so `jira.yaml` is safely committable and new developers have a clear setup path.

## In Scope

- Audit `jira.yaml` for any credential-adjacent content
- Add explicit comments documenting env var requirements
- Create `.raise/personal/jira.yaml` template for per-dev overrides
- Ensure `.raise/personal/` is in `.gitignore` (done in S350.2)
- Update `rai init` to scaffold personal jira config stub
- Verify `rai backlog` commands work with the separated config

## Out of Scope

- MCP bridge/adapter refactoring
- Multi-instance Jira support
- OAuth authentication

## Done Criteria

- [ ] `jira.yaml` contains only project-level config, no credentials
- [ ] Personal overrides via `.raise/personal/jira.yaml`
- [ ] `rai init` creates personal jira stub
- [ ] All `rai backlog` commands pass smoke test
