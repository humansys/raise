# S350.6: rai skill sync -- Scope

## Objective

Add a standalone `rai skill sync` command that checks skill freshness (stale vs current) without requiring a full `rai init`, reusing the existing three-hash sync engine.

## In Scope

- **`rai skill sync` subcommand** -- added to existing `skill.py` command group
  - Default mode: dry-run report (read-only, no file changes)
  - `--apply` flag: actually perform the sync
  - `--force` flag: overwrite conflicts (requires `--apply`)
  - Exit code 0 = all current, 1 = stale/conflicts detected
- **Tabular output** -- skill name, status (current/stale/conflict/new), action taken
- **Reuse `scaffold_skills`** -- dry_run=True for detection, dry_run=False for apply
  - No duplication of three-hash logic
- **Tests** -- unit tests for the sync command, covering all exit-code paths

## Out of Scope

- Interactive conflict resolution (already in `scaffold_skills`, not wired to this command)
- Skill diffing (showing what changed in each skill)
- Skill rollback
- Changes to `scaffold_skills` core logic

## Dependencies

- S350.5 (done): `scaffold_skills` with three-hash algorithm, `SkillScaffoldResult` model
- Existing `skill.py` command group: `rai skill list|validate|check-name|scaffold`

## Done Criteria
