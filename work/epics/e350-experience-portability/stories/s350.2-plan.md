# S350.2: Sanitize Git State — Plan

**Size:** S | **Tasks:** 4 | **Dependencies:** T1 → T2 → T3 (parallel-safe), T4 after all

## Task Breakdown

### T1: Update .gitignore

**Goal:** Add rules for `.claude/worktrees/`, `.claude/projects/`, and legacy paths to prevent re-addition after removal.

**Scope:**
- Add `.claude/worktrees/` entry (per-developer ephemeral worktree dirs, M10)
- Add `.claude/projects/` entry (machine-specific project memory paths)
- Add `.claude/RAI.md`, `.claude/RAI-naming.md`, `.claude/rai.archive/` entries (prevent re-addition after T2 removal)
- Place entries in the existing "Claude Code local settings" section of `.gitignore`

**Current state:** `.gitignore` already has `.claude/settings.local.json` and `CLAUDE.local.md`. The new entries extend that section.

**Verification:** `git status` shows `.gitignore` as modified. `git check-ignore .claude/worktrees/test` returns match.

**Depends on:** Nothing

---

### T2: Remove legacy files from git tracking

**Goal:** `git rm` legacy files that are superseded and should not ship to new clones.

**Scope:**
- `git rm .claude/RAI.md` — superseded by CLAUDE.md + skills system (M7)
- `git rm .claude/RAI-naming.md` — superseded, no active use (M8)
- `git rm -r .claude/rai.archive/` — 4 historical files, all superseded (M9):
  - `calibration.md`, `identity.md`, `memory.md`, `session-index.md`

**Pre-flight:** Grep for references to these files in source code and docs. If any exist, note them but proceed (references are stale by definition since the files are superseded).

**Verification:** `git status` shows deleted files staged. Files no longer in working tree.

**Depends on:** T1 (gitignore rules in place before removal prevents accidental re-creation)

---

### T3: Ensure rai init creates .raise/rai/personal/ with .gitkeep

**Goal:** After `rai init`, the `.raise/rai/personal/` directory exists with a `.gitkeep` so that the gitignored personal directory structure is ready for use.

**Current state:** `rai init` (`src/rai_cli/cli/commands/init.py`) does NOT create this directory. Multiple subsystems assume it exists (session, telemetry, memory). They create it on demand via `mkdir(parents=True)`, but a `.gitkeep` ensures the directory exists in a fresh clone before any `rai` command runs.

**Scope:**
- Add `.raise/rai/personal/.gitkeep` creation to `rai init` scaffolding logic
- The `.gitkeep` file itself is NOT tracked (the parent directory is gitignored), so this is a runtime concern only — `rai init` creates it for the developer

**Verification:** Run `uv run rai init --detect` in a temp directory and confirm `.raise/rai/personal/.gitkeep` exists.

**Depends on:** Nothing (parallel-safe with T1/T2)
