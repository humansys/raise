## Top 3 Process Improvements

### 1. Pre-implementation boundary checklist
**Pattern:** Code+Missing (34%) — the #1 failure mode is omitting things at implementation time.
**Action:** Before marking a task done, verify: error messages actionable? field mappings complete? edge cases for None/empty handled? Schema documented where parsers consume it?
**Gate:** Could be a lightweight PR checklist, not automated.

### 2. Adapter field normalization contract
**Pattern:** Interface+Code (5 bugs, 14%) — adapters pass raw data to external APIs without format validation.
**Action:** Each adapter declares its field normalization rules. Write-then-read validation for critical fields (assignee, status, parent).
**Gate:** Adapter integration tests that round-trip every field type.

### 3. Config validation at startup
**Pattern:** Configuration+Environment (5 bugs, 14%) — missing configs produce cryptic errors, stale deps produce CVEs.
**Action:** `rai doctor` already exists — extend to validate jira.yaml, mcp configs, and dependency freshness.
**Gate:** Run `rai gate check` at session start.
