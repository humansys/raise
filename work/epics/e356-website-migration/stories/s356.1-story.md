# S356.1: Migrate website from raise-gtm to raise-commons

## User Story

As a **RaiSE developer**,
I want the **marketing website colocated in raise-commons**,
so that **the site can reference source code and stays in sync with releases**.

## Acceptance Criteria

- site/ directory exists in raise-commons with all source files from raise-gtm
- node_modules, dist, and .astro are excluded from the copy and gitignored
- `npm install && npm run build` succeeds inside site/
- No content changes compared to raise-gtm source (pure migration)
