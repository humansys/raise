# S356.4: v2.2 Release Blog Post — Design

## Problem

The website needs a v2.2 release announcement blog post. The existing hello-world test post should be removed.

## Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Create single MDX file for v2.2 release post | Follows existing blog content structure |
| D2 | Delete hello-world.mdx test post | Test content should not ship in production |

## Content: v2.2 Release Highlights

- 37 skills (up from 20)
- New command groups: backlog, mcp, gate, adapter, release, doctor, docs
- Multi-agent support
- Knowledge graph improvements
- MCP server management
- Quality + architecture review skills
- Website now colocated with source

## Components

### 1. `site/src/data/blog/v22-release.mdx` (create)

New blog post MDX file with:
- Frontmatter: title, date, draft: false, description, author
- Content covering v2.2 highlights listed above

### 2. `site/src/data/blog/hello-world.mdx` (delete)

Remove the test post — no longer needed.

## Acceptance Criteria Mapping

| AC | Component | Verification |
|----|-----------|-------------|
| AC1 | v22-release.mdx | File exists in site/src/data/blog/ |
| AC2 | v22-release.mdx | Frontmatter has draft: false |
| AC3 | Build | `npm run build` succeeds, post appears on /blog |

## Risks

- **Low**: MDX frontmatter schema mismatch — mitigated by following existing post structure
