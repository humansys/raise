## What I Learned
1. **Docs drift faster than expected** — CLI commands were `raise` not `rai`, memory subcommand
   was restructured to graph/pattern/signal. Content stories need source-of-truth verification
   against actual runtime output.
2. **QR catches what AR misses** — The quality review found i18n labels and pricing consistency
   issues that acceptance review overlooked. Separate QR pass is valuable.
3. **Astro/Starlight i18n patterns** — Better understanding of content structure, translation
   mirroring, and component prop patterns.

## What I Would Change

2. **Source-of-truth first** — Before updating docs, run `rai --help` and count skills to get
   ground truth. Don't rely on memory or previous docs.
