# S356.1 Design: Migrate website from raise-gtm to raise-commons

## Complexity

**Simple** — 1 component (filesystem copy + gitignore), 2-3 story points, no external integrations, trivial algorithm complexity.

## Problem & Value

- **Problem**: The RaiSE website lives in raise-gtm where it cannot access source code, making accurate documentation impossible.
- **Value**: Colocating the website with the codebase enables source-aware content and single-repo operations.

## Approach

Copy `site/` from `~/Code/raise-gtm/site/` into `raise-commons/site/` using rsync, excluding build artifacts. Add gitignore entries. Verify build.

### Components Affected

| Component | Action | Notes |
|-----------|--------|-------|
| `site/` | Create | New directory, copied from raise-gtm |
| `.gitignore` | Modify | Add site artifact exclusions |

### Migration Command

```bash
rsync -av --exclude='node_modules' --exclude='dist' --exclude='.astro' \
  ~/Code/raise-gtm/site/ ./site/
```

### Verification

```bash
cd site && npm install && npm run build
# Expected: dist/ directory created with static site
```

## Acceptance Criteria

| ID | Criterion | Verifiable |
|----|-----------|------------|
| AC-1 | site/ directory exists in raise-commons with all source files from raise-gtm | Yes |
| AC-2 | node_modules/, dist/, .astro/ are gitignored | Yes |
| AC-3 | `npm run build` succeeds in site/ | Yes |

## Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D-1 | Use rsync with --exclude flags | Handles exclusions natively, preserves structure. Simpler than cp+rm or git subtree for a one-time copy. |

## Boundaries

**MUST NOT:**
- Change any content (that is S356.2)
- Change deployment config (that is S356.3)
