# S356.2: Update website content for v2.2

## User Story
As a potential RaiSE user visiting raiseframework.ai,
I want to see accurate information about v2.2 capabilities,
so that I can evaluate the framework based on current features.

## Acceptance Criteria

### AC1: Landing page reflects v2.2 reality
- Feature cards match current capabilities
- Metrics bar shows current numbers (tests, skills, etc.)
- Code example uses correct import paths

### AC2: Pricing page reflects current offering
- Community tier lists accurate feature count (37 skills, not 20)
- Feature comparison table is current

### AC3: Docs content is accurate
- Getting started uses correct CLI commands
- Concepts pages reflect current architecture
- CLI reference matches v2.2 commands

### AC4: Spanish content matches English updates
- All i18n content updated in parallel
