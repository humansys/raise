# S346.2: Language-Agnostic Quality Review

**Epic:** E346 — Skill Lifecycle Hardening
**Jira:** RAISE-351
**Size:** M
**Branch:** `story/s346.2/language-agnostic-quality-review`

## User Story

As a RaiSE developer working on a non-Python project,
I want the quality review skill to detect my project's language and adapt its checks,
so that I get meaningful semantic bug detection regardless of tech stack.

## Acceptance Criteria

```gherkin
Scenario: QR on a Python project
  Given a project with .raise/manifest.yaml showing Python
  When /rai-quality-review runs
  Then it checks *.py files for type:ignore, cast(), except Exception, __all__

Scenario: QR on a TypeScript project
  Given a project with .raise/manifest.yaml showing TypeScript
  When /rai-quality-review runs
  Then it checks *.ts/*.tsx files for as casts, any types, unhandled promises

Scenario: QR on a C#/.NET project
  Given a project with .raise/manifest.yaml showing C#
  When /rai-quality-review runs
  Then it checks *.cs files for null safety, unchecked casts, disposed resources

Scenario: QR on unknown language
  Given a project without manifest or with unknown language
  When /rai-quality-review runs
  Then it falls back to file extension scanning and applies universal heuristics
```
