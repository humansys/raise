# S346.3: Language-Agnostic Story Review

**Epic:** E346 — Skill Lifecycle Hardening
**Jira:** RAISE-352
**Size:** S
**Branch:** epic branch (S story, no dedicated branch)

## User Story

As a RaiSE developer working on a non-Python project,
I want the story-review skill to use my project's test runner instead of hardcoded `uv run pytest`,
so that the review verification works regardless of tech stack.

## Acceptance Criteria

```gherkin
Scenario: Story review on a Python project
  Given a project with Python as primary language
  When /rai-story-review runs Step 1
  Then it executes `uv run pytest --tb=short` (or project-configured test command)

Scenario: Story review on a TypeScript project
  Given a project with TypeScript as primary language
  When /rai-story-review runs Step 1
  Then it executes `npm test` or `npx vitest` (or project-configured test command)

Scenario: Story review on a C#/.NET project
  Given a project with C# as primary language
  When /rai-story-review runs Step 1
  Then it executes `dotnet test` (or project-configured test command)

Scenario: Story review on unknown project
  Given a project without manifest or with unknown language
  When /rai-story-review runs Step 1
  Then it looks for common test runner patterns and asks developer if unsure
```
