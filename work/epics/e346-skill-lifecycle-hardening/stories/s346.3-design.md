# S346.3: Language-Agnostic Story Review — Design

## Complexity: Simple (1 component, text editing)

## Problem

story-review hardcodes `uv run pytest --tb=short` in Step 1 — fails or is meaningless on non-Python projects.

## Value

Story-review verification gate works for any tech stack, enabling full lifecycle on TS, C#, PHP, Dart projects.

## Approach

Replace `uv run pytest --tb=short` with a language-detection + test runner lookup table. Follow the same pattern established in S346.2 (quality-review Step 0): detect language from `.raise/manifest.yaml`, fallback to file extension scanning.

## Components

| Component | Change |
|-----------|--------|
| `.claude/skills/rai-story-review/SKILL.md` | Modify Step 1 — replace hardcoded pytest |
| `src/rai_cli/skills_base/rai-story-review/SKILL.md` | Modify — sync with deployment |

## Examples

Current:
```bash
uv run pytest --tb=short
```

Target:
```bash
# Python:     uv run pytest --tb=short
# TypeScript: npx vitest run (or npm test)
# C#:         dotnet test --verbosity quiet
# Go:         go test ./...
# PHP:        vendor/bin/phpunit
# Dart:       flutter test
# Unknown:    Ask developer
```

## Acceptance Criteria

- **MUST:** Step 1 detects project language before running tests
- **MUST:** Python test command preserved
- **MUST:** At least 3 other languages documented
- **MUST:** Unknown language asks developer
- **SHOULD:** Reuse QR Step 0 detection pattern
- **MUST NOT:** Break existing Python behavior
