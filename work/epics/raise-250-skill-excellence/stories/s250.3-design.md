# S250.3 Design: Utility Skills Refactor

## Problem

11 utility skills average 346 lines with inconsistent structure. 10/11 missing Quality Checklist section. Total: 3,810 lines → target ≤1,650 (11 × 150). Same bloat patterns as S250.2: inlined templates, context-loading steps, explanatory sections.

## Approach

Mechanical application of ADR-040 contract (3rd batch, PAT-E-442). Two batches:

1. **Batch 1: Discovery skills (4)** — discover-start, discover-scan, discover-validate, discover-document
2. **Batch 2: Meta skills (7)** — research, debug, docs-update, problem-shape, welcome, project-create, project-onboard

### Compression Strategy (same as S250.2)

| Bloat pattern | Action |
|---------------|--------|
| Steps 0/0.1 (context loading) | **Remove** — preamble covers |
| Inlined templates/schemas | **Extract** to `templates/` or compress |
| "Integration with Memory Model" | **Remove** — preamble content |
| Explanatory/notes sections | **Compress** to Quality Checklist items |
| Verbose step descriptions | **Merge** related steps |

### Special Cases

| Skill | Challenge | Strategy |
|-------|-----------|----------|
| project-onboard (697 lines) | Largest skill, 4-dim gate | Aggressive merge: 8→4 steps, extract templates |
| project-create (486 lines) | Second largest | Merge setup steps, extract templates |
| welcome (227 lines, 3/7) | Most non-compliant | Restructure into 7 sections, keep scenario flow |
| discover-validate (374 lines) | Confidence-tier workflow | Keep decision tables, compress explanations |

## Targets

| Metric | Target |
|--------|--------|
| Lines per skill | ≤150 (escape valve: ≤170 for project-onboard if all substance) |
| Sections | 7 canonical (ADR-040) |
| Substance ratio | ≥80% |
| Templates | Extract to `templates/` where >30 lines of inlined content |

## Acceptance Criteria

**MUST:**
- All 11 skills follow ADR-040 7-section canonical structure
- All 11 skills ≤150 lines (170 escape valve for project-onboard)
- Zero context-loading steps (covered by preamble)
- Both locations synced (`.claude/skills/` and `skills_base/`)
- Add Quality Checklist to all 10 skills missing it

**SHOULD:**
- Decision tables replacing prose conditionals
- Affirmative phrasing throughout
- Extract templates for skills with >30 lines of inlined content

**MUST NOT:**
- Lose critical process information (confidence tiers in discover-validate, gate dimensions in project-onboard)
- Remove CLI command references that users depend on
- Break the discover-start → scan → validate → document pipeline
