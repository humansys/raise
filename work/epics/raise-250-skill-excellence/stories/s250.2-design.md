# S250.2 Design: Lifecycle Skills Refactor

## Problem

9 lifecycle skills average 324 lines with 35-75% substance ratio. Common bloat patterns: inlined templates, context-loading steps duplicated from preamble, lifecycle diagrams, memory model sections, and "feature" terminology that should be "story".

## Approach

Apply ADR-040 contract using the proven S250.1 pattern. Two batches with review between:

1. **Batch 1: Epic skills** (4) — epic-start, epic-design, epic-plan, epic-close
2. **Batch 2: Story skills** (5) — story-start, story-design, story-plan, story-review, story-close

### Compression Strategy (common to all 9)

| Bloat pattern | Action |
|---------------|--------|
| Steps 0/0.1/0.2 (context loading) | **Remove** — preamble covers graph queries |
| Inlined templates (100+ lines) | **Extract** to `{skill-dir}/templates/` as separate files |
| "Integration with Memory Model" | **Remove** — preamble content |
| "Relationship to Other Skills" diagram | **Remove** — CLAUDE.md has lifecycle |
| Lifecycle "YOU ARE HERE" diagrams | **Remove** — redundant |
| "Notes" explanatory sections | **Compress** to 1-2 lines in Quality Checklist |
| Summary/Close templates | **Extract** to `{skill-dir}/templates/` |
| Quality Standards tables | **Merge** into Quality Checklist |
| Common Pitfalls sections | **Keep top 3** as Quality Checklist items |
| "feature" terminology | **Replace** with "story" |

### Epic-Design Specific (627 → ≤150)

- 12 steps → 5 steps:
  1. Frame Objective & Scope — merge current steps 1+2
  2. Assess Architecture — merge steps 3+5 (spike + ADRs)
  3. Break Down Stories — merge steps 4+6+8 (candidates + deps + sizing)
  4. Define Done Criteria — merge steps 7+9 (done + risks)
  5. Write Scope Document — merge steps 10+11 (doc + parking lot)
- Extract scope template (~114 lines) to `rai-epic-design/templates/scope.md`

### Epic-Plan Specific (656 → ≤150)

- 11 steps → 5 steps:
  1. Review Epic Scope — current step 1
  2. Sequence & Rationalize — merge steps 2+3+4+8 (critical path + strategy + parallel + rationale)
  3. Define Milestones — merge steps 5+6+9 (milestones + timeline + risks)
  4. Setup Tracking — current step 7
  5. Update Scope Document — current step 10
- Extract plan template (~100 lines) to `rai-epic-plan/templates/plan-section.md`
- Remove external URL references (link rot risk)

## Targets

| Metric | Target |
|--------|--------|
| Lines per skill | ≤150 (escape valve: ≤170 for epic-design/plan if all substance) |
| Sections | 7 canonical (ADR-040) |
| Substance ratio | ≥80% |
| Terminology | "story" not "feature" in all story-level skills |
| Templates | Extracted to `templates/` subdirectory per skill |

## Acceptance Criteria

**MUST:**
- All 9 skills follow ADR-040 7-section canonical structure
- All 9 skills ≤150 lines (170 escape valve for top-2 complex)
- Zero context-loading steps (covered by preamble)
- Inlined templates extracted to separate files
- "feature" → "story" terminology in story-level skills
- Both locations synced (`.claude/skills/` and `skills_base/`)

**SHOULD:**
- Decision tables replacing prose conditionals
- Affirmative phrasing throughout
- Step numbering: clean integers (1, 2, 3 — no 0.1, 1.5)

**MUST NOT:**
- Lose critical process information during compression (risk gates, poka-yokes)
- Remove the `rai pattern reinforce` step from story-review (scoring system depends on it)
- Remove backlog registration from epic-start (PAT-194 fix)
