# Story: S338.9 — Stack-aware MCP recommendations

## User Story

**As a** developer onboarding a project with RaiSE,
**I want** Rai to suggest relevant MCP servers based on my detected tech stack,
**So that** I discover useful MCP integrations without knowing what's available.

## Acceptance Criteria

### Scenario 1: Suggest MCP servers after discovery

```gherkin
Given a project uses Python and TypeScript
When I run /rai-discover-scan or /rai-mcp-status
Then Rai suggests MCP servers relevant to my stack
And shows how to add them via /rai-mcp-add
```

### Scenario 2: No suggestions for unknown stacks

```gherkin
Given a project uses a language with no known MCP servers
When recommendations are checked
Then Rai says "No MCP recommendations for your stack yet"
And suggests checking the MCP ecosystem manually
```

### Scenario 3: Skip already-registered servers

```gherkin
Given context7 is already registered
When Rai checks recommendations
Then context7 is not suggested again
```

## Size

S (data mapping + skill update, no complex logic)

## Dependencies

- S338.8 (`/rai-mcp-add` skill exists for the "add" action)
