# Design: S338.6 — `rai mcp install`

## Complexity: Moderate (3 strategies, subprocess, health check)

## Problem

Registering an MCP server requires knowing the exact command/args format for each package type, then manually writing YAML. This is friction that discourages adoption.

## Value

One command installs, verifies, and registers an MCP server. Lowers the barrier from "read docs + write YAML" to `rai mcp install <pkg> --type npx`.

## Approach

New `install` command in `mcp_app`. Type-specific strategy generates command/args, then reuses scaffold's introspect-and-write logic.

### CLI signature

```
rai mcp install <package> --type uvx|npx|pip --name NAME [--env ENV] [--module MODULE] [--force]
```

- `package`: Package identifier (e.g. `@upstash/context7-mcp`, `mcp-github`)
- `--type`: Required. Package manager type (Literal["uvx", "npx", "pip"])
- `--name`: Required. Server name for config file
- `--env`: Comma-separated env var names
- `--module`: Python module name (required for `--type pip`, ignored otherwise)
- `--force`: Overwrite existing config

### Type strategies

| Type | Pre-install | command | args |
|------|-------------|---------|------|
| npx | (none — npx auto-fetches) | `npx` | `["-y", "<package>"]` |
| uvx | (none — uvx auto-fetches) | `uvx` | `["<package>"]` |
| pip | `pip install <package>` | `python` | `["-m", "<module>"]` |

**Decision (AR Q1-B):** pip requires explicit `--module` flag — no auto-derivation from package name.

### Flow

1. Validate inputs (pip requires --module)
2. For pip: run `pip install <package>` subprocess
3. Build command/args from type strategy
4. Connect via McpBridge, list_tools() (health check)
5. If health fails: warn on stderr, continue (non-fatal)
6. Generate `.raise/mcp/<name>.yaml` with tool comments
7. Report success

### Reuse from S338.7 scaffold

The YAML generation logic (McpServerConfig → yaml.dump with header comments) is identical to scaffold. Extract a shared `_write_mcp_config()` helper to avoid duplication.

### Error handling

- pip install fails → exit 1
- Health check fails → warn, still write config (exit 0)
- Config exists without --force → exit 1

## Examples

```bash
# npx
rai mcp install @upstash/context7-mcp --type npx --name context7

# uvx
rai mcp install mcp-github --type uvx --name github --env GITHUB_TOKEN

# pip (requires --module)
rai mcp install mcp-server-fetch --type pip --name fetch --module mcp_server_fetch
```

## Acceptance Criteria

### MUST
1. Works for uvx, npx, pip types with correct command/args
2. pip requires --module flag (error without it)
3. Health check runs after install, warns on failure (non-fatal)
4. Config YAML generated and parseable by McpServerConfig
5. Overwrite protection (--force)

### MUST NOT
- Auto-detect --type from package name (AR-R3)
- Auto-derive module name from package name
