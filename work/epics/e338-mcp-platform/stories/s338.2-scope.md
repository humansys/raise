# Scope: S338.2 — `rai mcp list` + `health` + `tools`

## In Scope

- `rai mcp list` — show all registered servers (name, description, command)
- `rai mcp health [SERVER]` — ping one or all servers, show status/latency/tool count
- `rai mcp tools SERVER` — list available tools on a server (name, description)
- Error handling: server not found, bridge connection failures
- Human-readable output (Rich tables/panels)

## Out of Scope

- JSON output format for list/health/tools (can add later if needed)
- `rai mcp call` (S338.3, done)
- Telemetry (S338.4)

## Done Criteria

- `rai mcp list` shows registered servers or "no servers" message
- `rai mcp health <server>` shows connectivity status
- `rai mcp tools <server>` shows available tools
- Server-not-found gives clear error + exit 1
- Unit tests cover all commands (happy + error paths)
- All existing tests pass
