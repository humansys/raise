# Design: S338.4 — Telemetry hooks

## Complexity: Simple

3 components, pattern fully established by 10+ existing events in hooks/events.py.

## Problem

No observability on MCP tool calls. Developer can't tell if calls are slow, failing, or which tools are used most.

## Value

Every `rai mcp call` emits a structured event that telemetry hooks can consume. `--verbose` gives immediate human feedback. Foundation for future `rai mcp stats`.

## Approach

1. Add `McpCallEvent` frozen dataclass to `hooks/events.py` (same pattern as all others)
2. Emit from `call` command in `cli/commands/mcp.py` after tool invocation (success or failure)
3. Add `--verbose` flag that prints call details to stderr

Event emitted from CLI command, NOT from bridge (bridge is infrastructure, unaware of hooks — clean layer separation).

## Components

| Component | Change |
|-----------|--------|
| `src/rai_cli/hooks/events.py` | MODIFY — add `McpCallEvent` |
| `src/rai_cli/cli/commands/mcp.py` | MODIFY — emit event + `--verbose` flag |
| `tests/hooks/test_mcp_call_event.py` | CREATE — event structure tests |
| `tests/cli/commands/test_mcp_call.py` | MODIFY — add verbose + emission tests |

## Examples

### McpCallEvent dataclass

```python
@dataclass(frozen=True)
class McpCallEvent(HookEvent):
    """Emitted after an MCP tool call completes (success or failure)."""

    event_name: Literal["mcp:call"] = field(
        default="mcp:call", init=False
    )
    server: str = ""
    tool: str = ""
    success: bool = True
    latency_ms: int = 0
    error: str = ""
```

### CLI with --verbose (stderr)

```
$ rai mcp call context7 resolve-library-id --args '{"query":"next.js","libraryName":"next.js"}' --verbose
mcp:call context7/resolve-library-id — ok (245ms)
{"text": "...", "data": {...}, "is_error": false, "error_message": ""}
```

On failure:
```
$ rai mcp call context7 bad-tool --verbose
mcp:call context7/bad-tool — error (12ms): Tool 'bad-tool' not found
Error: Tool call 'bad-tool' failed: Tool 'bad-tool' not found
```

### Emission in call command

```python
# After tool call (success or failure), before output:
emitter = create_emitter()
emitter.emit(McpCallEvent(
    server=server,
    tool=tool,
    success=True,
    latency_ms=245,
))
```

## Acceptance Criteria

- **MUST**: `McpCallEvent` is a frozen dataclass extending `HookEvent` with event_name `"mcp:call"`
- **MUST**: Event emitted on every `rai mcp call` — both success and failure
- **MUST**: Event fields: server, tool, success, latency_ms, error
- **MUST**: `--verbose` prints call summary to stderr
- **MUST NOT**: Bridge imports hooks (layer boundary)
