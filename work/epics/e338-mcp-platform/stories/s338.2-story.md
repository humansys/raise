# Story: S338.2 — `rai mcp list` + `health` + `tools`

## User Story

**As a** developer using RaiSE,
**I want to** list registered MCP servers, check their health, and discover available tools,
**So that** I know what servers are configured and what tools I can call.

## Size: S

## Acceptance Criteria

```gherkin
Scenario: List registered servers
  Given MCP server configs in .raise/mcp/
  When I run `rai mcp list`
  Then each server name and description is shown

Scenario: List with no servers registered
  When I run `rai mcp list` with empty .raise/mcp/
  Then a message says no servers are registered

Scenario: Health check a specific server
  Given a registered MCP server "context7"
  When I run `rai mcp health context7`
  Then connectivity status, latency, and tool count are shown

Scenario: Health check server not found
  When I run `rai mcp health nonexistent`
  Then exit code is non-zero
  And stderr shows server not found error

Scenario: List tools for a server
  Given a registered MCP server "context7"
  When I run `rai mcp tools context7`
  Then each tool name and description is shown

Scenario: Tools for server not found
  When I run `rai mcp tools nonexistent`
  Then exit code is non-zero
  And stderr shows server not found error
```
