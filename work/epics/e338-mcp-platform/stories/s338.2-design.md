# Design: S338.2 — `rai mcp list` + `health` + `tools`

## Complexity: Simple

3 new commands added to existing `mcp_app` in `cli/commands/mcp.py`. All use existing infrastructure (registry, bridge).

## Problem

Developer has no way to discover what MCP servers are registered or what tools they expose.

## Value

DX completeness — `list`, `health`, `tools` let developers verify config and discover capabilities before calling tools.

## Approach

Add 3 commands to existing `mcp_app`:
- `list` — sync, reads registry only, no bridge needed
- `health` — async, creates bridge, calls `bridge.health()`
- `tools` — async, creates bridge, calls `bridge.list_tools()`

Rich tables for human output. Server lookup reused from `call` command.

## Components

| Component | Change |
|-----------|--------|
| `src/rai_cli/cli/commands/mcp.py` | MODIFY — add `list_servers`, `health`, `tools` commands |
| `tests/cli/commands/test_mcp_list.py` | CREATE — list tests |
| `tests/cli/commands/test_mcp_health.py` | CREATE — health tests |
| `tests/cli/commands/test_mcp_tools.py` | CREATE — tools tests |

## Acceptance Criteria

- **MUST**: `rai mcp list` shows servers or "no servers" message
- **MUST**: `rai mcp health <server>` shows status/latency/tool count
- **MUST**: `rai mcp tools <server>` shows tool names and descriptions
- **MUST**: Server-not-found → stderr error + exit 1
- **MUST NOT**: Import from `rai_cli.adapters`
