# Design: S338.7 — `rai mcp scaffold`

## Complexity: Simple (1 command, reuses bridge + schema)

## Problem

Manually writing `.raise/mcp/<name>.yaml` is error-prone — developers must know the exact YAML schema and can't easily discover what tools a server exposes.

## Value

One command generates a valid, registry-parseable config with tool discovery. Removes friction from MCP server registration.

## Approach

New `scaffold` command in `mcp_app` (same file as list/health/tools/call). Connects to server via McpBridge, introspects tools, generates YAML.

### CLI signature

```
rai mcp scaffold <name> --command CMD [--args ARGS] [--env ENV] [--force]
```

- `name`: Server name (used as filename and `name:` field)
- `--command`: Required. Server command (e.g. `npx`, `uvx`)
- `--args`: Space-separated server args (e.g. `"-y @upstash/context7-mcp"`)
- `--env`: Comma-separated env var names (e.g. `"GITHUB_TOKEN,API_KEY"`)
- `--force`: Overwrite existing config

### Generated YAML format

```yaml
# Generated by: rai mcp scaffold context7
# Discovered tools: resolve-library-id, query-docs
name: context7
description: null
server:
  command: npx
  args:
    - -y
    - '@upstash/context7-mcp'
  env: null
```

### Flow

1. Check if `.raise/mcp/<name>.yaml` exists (abort without `--force`)
2. Create `.raise/mcp/` directory if missing
3. Connect to server via McpBridge(command, args, env)
4. Call `bridge.list_tools()` to discover tools
5. Build `McpServerConfig` model
6. Serialize to YAML with tool names as header comment
7. Write file, report success

### Error handling

- Server unreachable → McpBridgeError → exit 1 with message
- Existing config without --force → exit 1 with message

## Acceptance Criteria

### MUST
1. Generates valid `.raise/mcp/<name>.yaml` parseable by `McpServerConfig`
2. Discovered tool names appear as comments in the YAML
3. Creates `.raise/mcp/` directory if missing
4. Refuses overwrite without `--force`
5. Reports error on unreachable server

### MUST NOT
- Generate declarative adapter YAML
- Auto-detect command from package name
