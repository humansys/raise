# Scope: S338.11 — Semgrep MCP Install (Won't Do)

**Jira:** RAISE-338 (closed)
**Branch:** `story/s338.11/semgrep-mcp-install` (from dev)
**Parent epic:** RAISE-384 (E338: MCP Platform)

## Outcome

MCP integration abandoned due to upstream bug: Semgrep MCP server has a
hardcoded 10s timeout for registry rule downloads that cannot be configured.
The standalone `semgrep-mcp` PyPI package is deprecated (moved to `semgrep mcp`
built-in). Both paths hit the same timeout.

CLI direct (`semgrep scan --config auto`) works perfectly — 249 files, 933 rules,
11 findings in ~30s.

## What Was Done

- Investigated `uvx semgrep-mcp` (deprecated, only exposes deprecation_notice tool)
- Investigated `semgrep mcp` built-in (7 tools, health OK, scan timeout)
- Researched community workarounds (semgrep/semgrep#2620, semgrep/mcp#79)
- Full codebase scan via CLI: found and fixed `shell=True` in publish/check.py
- Created RAISE-395: integrate Semgrep into CI/pre-commit pipeline instead

## Deliverables Kept

- `fix: remove shell=True` in publish/check.py (security fix, independent of MCP)
- Backlog sync: E338 epic created in Jira (RAISE-384), stories mapped
- RAISE-395 created under RAISE-295 (Security Fixes) for CI integration
