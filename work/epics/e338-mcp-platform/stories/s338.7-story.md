# User Story: S338.7 — `rai mcp scaffold`

## Story

**As a** developer registering a new MCP server,
**I want to** run `rai mcp scaffold <server>` against a running server,
**so that** it introspects available tools and generates a valid `.raise/mcp/<name>.yaml` config.

## Acceptance Criteria (Gherkin)

### Scenario 1: Scaffold generates valid YAML from running server

```gherkin
Given a running MCP server at command "npx" with args ["-y", "@upstash/context7-mcp"]
When I run `rai mcp scaffold context7 --command npx --args "-y @upstash/context7-mcp"`
Then a file `.raise/mcp/context7.yaml` is created
And it contains name: "context7", server.command, server.args
And it contains a comment listing discovered tools
```

### Scenario 2: Scaffold reports error for unreachable server

```gherkin
Given a server command that fails to start
When I run `rai mcp scaffold bad-server --command nonexistent`
Then exit code is 1
And error message indicates connection failure
```

### Scenario 3: Scaffold creates .raise/mcp/ directory if missing

```gherkin
Given no .raise/mcp/ directory exists
When I run `rai mcp scaffold myserver --command echo --args "server"`
Then .raise/mcp/ directory is created
And .raise/mcp/myserver.yaml is written
```

### Scenario 4: Scaffold refuses to overwrite existing config

```gherkin
Given .raise/mcp/context7.yaml already exists
When I run `rai mcp scaffold context7 --command npx --args "..."`
Then exit code is 1
And error says config already exists (use --force to overwrite)
```

## Size

S (Small) — CLI command + bridge introspection + YAML generation
