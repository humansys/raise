# Design: S338.5 — Declarative adapter `server.ref`

## Complexity: Simple (2 components, Size S)

## Problem

Declarative adapters duplicate MCP server connection config inline in `.raise/adapters/*.yaml`. When the same server is already registered in `.raise/mcp/*.yaml`, this creates maintenance overhead and drift risk.

## Value

Single source of truth for server connections. Adapter YAML becomes leaner — just `server.ref: context7` instead of repeating command/args/env.

## Approach

**Option B (chosen):** New `ServerRef` model in declarative schema. `ServerConnection` stays unchanged in `rai_cli.mcp.schema` — ref is a declarative adapter concern, not a registry concern.

### Components affected

| Component | Change | File |
|-----------|--------|------|
| `ServerRef` | **create** | `src/rai_cli/adapters/declarative/schema.py` |
| `DeclarativeAdapterConfig.server` | **modify** type from `ServerConnection` to `ServerRef` | same file |
| `_create_bridge()` | **modify** to resolve ref via registry | `src/rai_cli/adapters/declarative/adapter.py` |

### `ServerRef` model

```python
class ServerRef(BaseModel):
    """Server config: reference registry OR inline connection."""
    ref: str | None = Field(default=None, description="Name in .raise/mcp/ registry")
    command: str | None = Field(default=None, description="Server command (inline)")
    args: list[str] = Field(default_factory=list)
    env: list[str] | None = Field(default=None)

    @model_validator(mode="after")
    def _ref_or_command(self) -> ServerRef:
        if self.ref is None and self.command is None:
            raise ValueError("Either 'ref' or 'command' must be provided")
        return self
```

### Resolution in `_create_bridge()`

```python
def _create_bridge(self) -> McpBridge:
    server = self._config.server
    if server.ref is not None:
        registry = discover_mcp_servers()
        if server.ref not in registry:
            raise McpBridgeError(f"Server ref '{server.ref}' not found in MCP registry")
        conn = registry[server.ref].server
    else:
        # Inline — command is guaranteed non-None by validator
        conn = ServerConnection(command=server.command, args=server.args, env=server.env)
    # resolve env
    env: dict[str, str] | None = None
    if conn.env:
        env = {**os.environ, **{k: os.environ.get(k, "") for k in conn.env}}
    return McpBridge(server_command=conn.command, server_args=conn.args, env=env)
```

### YAML examples

**Ref-based (new):**
```yaml
adapter:
  name: jira-context7
  protocol: pm
server:
  ref: context7
methods: ...
```

**Inline (backwards compat, unchanged):**
```yaml
adapter:
  name: github
  protocol: pm
server:
  command: uvx
  args: [mcp-github]
  env: [GITHUB_TOKEN]
methods: ...
```

## Acceptance Criteria

### MUST
1. `ServerRef` validates: ref-only OK, inline-only OK, neither raises `ValidationError`
2. `_create_bridge()` resolves ref from registry → creates bridge with registry server's connection
3. Ref to nonexistent server raises `McpBridgeError`
4. All existing declarative adapter tests pass unchanged (backwards compat)

### MUST NOT
- Modify `ServerConnection` in `rai_cli.mcp.schema`
- Support ref + inline override (deferred — unclear semantics)
