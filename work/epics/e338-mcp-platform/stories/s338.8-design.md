# Design: S338.8 — MCP Skills (add/remove/status)

## Complexity: Moderate

3 skills, conversational flows, CLI orchestration. No Python code — SKILL.md files only.

## Problem

Developers without MCP experience cannot construct `rai mcp install` commands. Skills are the human interface (PAT-E-608). Three skills cover the CRUD lifecycle: add, remove, status.

## Approach

Each skill is a SKILL.md that guides the agent through a conversational flow, collecting information from the human and delegating to `rai mcp` CLI commands underneath. The human never sees or types CLI syntax.

### Components

| Component | Change | Location |
|-----------|--------|----------|
| `/rai-mcp-add` | Create | `src/rai_cli/skills_base/rai-mcp-add/SKILL.md` + `.claude/skills/` |
| `/rai-mcp-remove` | Create | `src/rai_cli/skills_base/rai-mcp-remove/SKILL.md` + `.claude/skills/` |
| `/rai-mcp-status` | Create | `src/rai_cli/skills_base/rai-mcp-status/SKILL.md` + `.claude/skills/` |

### Skill Design

#### `/rai-mcp-add`

Conversational flow:
1. Ask what server the human wants (free text: "Context7", "GitHub", "Snyk")
2. Ask package type: npx / uvx / pip
3. Ask package name (suggest if recognizable from step 1)
4. Ask server name for config (suggest slug from package)
5. Ask about env vars (if needed)
6. If pip: ask for Python module name
7. Run `rai mcp install <package> --type <type> --name <name> [--env ...] [--module ...]`
8. Report result: health status, tools discovered, config path

**Key design decision:** The skill does NOT maintain a "known servers" catalog. It asks the human for package details. Stack-aware suggestions are S338.9's job.

#### `/rai-mcp-remove`

Flow:
1. If no argument, show registered servers (via `rai mcp list`) and ask which to remove
2. Check `.raise/adapters/*.yaml` for any `server.ref` pointing to this server
3. If dependencies found: warn and ask for confirmation
4. Delete `.raise/mcp/<name>.yaml`
5. Confirm removal

**Key design decision:** Remove is a file delete + dependency check. No CLI command needed — skill reads filesystem directly.

#### `/rai-mcp-status`

Flow:
1. Run `rai mcp list` to show registered servers
2. For each server, run `rai mcp health <name>`
3. Present summary table: name, status, tool count, latency

**Key design decision:** Status is read-only, no conversational flow needed. Just run + report.

## Examples

### /rai-mcp-add conversation

```
Human: /rai-mcp-add
Rai: What MCP server would you like to add?
Human: Context7 for documentation lookups
Rai: Got it — Context7. Let me set that up.
     - Package type: npx (Context7 is an npm package)
     - Package: @upstash/context7-mcp
     - Server name: context7
     Does this look right? Any env vars needed?
Human: looks good, no env vars
Rai: Installing... ✓ Created .raise/mcp/context7.yaml
     Health: healthy (2 tools, 1200ms)
     Tools: resolve-library-id, query-docs

     Context7 is ready to use via `rai mcp call context7 <tool>`.
```

### /rai-mcp-remove conversation

```
Human: /rai-mcp-remove context7
Rai: Checking dependencies...
     ⚠ Found 1 adapter referencing context7:
       - .raise/adapters/docs.yaml (server.ref: context7)
     Removing this server will break the docs adapter.
     Proceed anyway? [y/N]
Human: yes
Rai: ✓ Removed .raise/mcp/context7.yaml
     Note: Update .raise/adapters/docs.yaml to use inline config or another server.
```

### /rai-mcp-status output

```
Human: /rai-mcp-status
Rai: MCP Server Status:

     | Server   | Status  | Tools | Latency |
     |----------|---------|-------|---------|
     | context7 | healthy | 2     | 1.2s    |
     | github   | healthy | 15    | 0.8s    |

     All 2 servers healthy.
```

## Acceptance Criteria

### MUST
- `/rai-mcp-add` guides human from intent to working config
- `/rai-mcp-remove` checks adapter dependencies before deleting
- `/rai-mcp-status` shows health of all registered servers
- Skills deployed to both builtin and .claude/skills/
- Follow ADR-040 skill structure (YAML frontmatter, ShuHaRi, steps, verification)

### SHOULD
- `/rai-mcp-add` suggests package names for well-known servers (Context7, GitHub)
- `/rai-mcp-remove` with no argument shows server list first

### MUST NOT
- Skills must NOT duplicate CLI logic — delegate to `rai mcp` commands
- Skills must NOT require the human to know CLI syntax
