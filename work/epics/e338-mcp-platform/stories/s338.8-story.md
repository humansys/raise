# Story: S338.8 — MCP Skills (add/remove/status)

## User Story

**As a** developer with zero MCP experience,
**I want** conversational skills that guide me through MCP server management,
**So that** I can add, remove, and check MCP servers without knowing CLI syntax.

## Acceptance Criteria

### Scenario 1: Add MCP server via skill

```gherkin
Given I invoke /rai-mcp-add
When I say "I want to add Context7"
Then Rai asks for server type (npx/uvx/pip)
And Rai asks for the package name (or suggests one)
And Rai runs `rai mcp install` underneath
And Rai confirms the server is healthy
And Rai shows what tools are now available
```

### Scenario 2: Remove MCP server via skill

```gherkin
Given I invoke /rai-mcp-remove
When I say "remove context7"
Then Rai checks if any declarative adapters reference this server
And Rai warns about dependencies if any
And Rai deletes .raise/mcp/context7.yaml
And Rai confirms removal
```

### Scenario 3: Check MCP status via skill

```gherkin
Given I invoke /rai-mcp-status
Then Rai lists all registered MCP servers
And Rai runs health checks on each
And Rai shows status, tool count, and any issues
```

### Scenario 4: Add with environment variables

```gherkin
Given I invoke /rai-mcp-add
When I say "add GitHub MCP server"
And the server requires GITHUB_TOKEN
Then Rai asks about required environment variables
And Rai validates they are set in the environment
And Rai includes them in the generated config
```

## Size

M (3 skills, conversational flows, dependency checking)

## Dependencies

- S338.6 (`rai mcp install`) — add skill delegates to install
- S338.7 (`rai mcp scaffold`) — add skill can delegate to scaffold
