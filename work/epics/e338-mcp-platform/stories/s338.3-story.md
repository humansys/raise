# Story: S338.3 — `rai mcp call`

## User Story

**As a** developer using RaiSE,
**I want to** invoke any registered MCP server tool via `rai mcp call <server> <tool> [--args JSON]`,
**So that** I can use MCP servers from skills and scripts without injecting tool schemas into agent context.

## Size: S

## Acceptance Criteria

```gherkin
Scenario: Call a registered MCP server tool with arguments
  Given a valid MCP server config at .raise/mcp/context7.yaml
  When I run `rai mcp call context7 resolve-library-id --args '{"query":"nextjs","libraryName":"next.js"}'`
  Then the tool is invoked via McpBridge
  And the result is printed as JSON to stdout

Scenario: Call a tool without arguments
  Given a valid MCP server config at .raise/mcp/echo.yaml
  When I run `rai mcp call echo ping`
  Then the tool is invoked with empty arguments
  And the result is printed as JSON to stdout

Scenario: Server not found in registry
  When I run `rai mcp call nonexistent some-tool`
  Then exit code is non-zero
  And stderr shows "Server 'nonexistent' not found in registry"

Scenario: Tool invocation error
  Given a registered MCP server
  When the bridge call raises McpBridgeError
  Then exit code is non-zero
  And stderr shows the error message

Scenario: JSON output structure
  Given a successful tool call
  Then stdout contains valid JSON with keys: text, data, is_error
```
