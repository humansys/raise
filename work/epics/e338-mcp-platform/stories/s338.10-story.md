# Story: S338.10 — MCP health in session start

## User Story

**As a** developer starting a work session,
**I want** Rai to show MCP server status automatically,
**So that** I know if my tools are healthy without asking.

## Acceptance Criteria

### Scenario 1: Servers registered and healthy

```gherkin
Given MCP servers are registered in .raise/mcp/
When I run /rai-session-start
Then Rai shows a brief MCP status line (e.g. "MCP: 2 servers, all healthy")
```

### Scenario 2: Unhealthy server

```gherkin
Given an MCP server is registered but unreachable
When I run /rai-session-start
Then Rai warns about the unhealthy server
And suggests /rai-mcp-status for details
```

### Scenario 3: No servers registered

```gherkin
Given no MCP servers are registered
When I run /rai-session-start
Then Rai skips MCP status silently (no noise)
```
