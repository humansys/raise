# Retrospective: S338.5 — Declarative adapter `server.ref`

## What Went Well

- **Design decision upfront** — Option B (separate models) resolved cleanly. AR before implementation caught the Q1 inheritance question early.
- **Backwards compat preserved** — All 286 existing adapter tests pass unchanged. Only 1 test assertion updated (error message wording).
- **QR caught real issues** — `type: ignore` replaced with runtime assert (type-safe), lazy import normalized, missing env test added.

## What Could Improve

- `type: ignore` was written knowing it was wrong — should have used `assert` from the start. Pattern: when a validator guarantees a condition, use `assert` to communicate it to both pyright and humans.

## Patterns

- PAT-E-609: When a Pydantic validator guarantees a field is non-None in a specific branch, use `assert field is not None` before use — satisfies pyright without `type: ignore` and documents the invariant.
