# Story: S338.4 — Telemetry hooks

## User Story

**As a** RaiSE framework developer,
**I want** every MCP bridge call to emit a structured `McpCallEvent`,
**So that** telemetry hooks can observe MCP usage (server, tool, latency, success/failure) for monitoring and debugging.

## Size: S

## Acceptance Criteria

```gherkin
Scenario: Successful tool call emits event
  Given a registered MCP server
  When I run `rai mcp call <server> <tool>`
  Then an McpCallEvent is emitted with server, tool, success=True, latency_ms

Scenario: Failed tool call emits event
  Given a registered MCP server
  When the bridge call fails
  Then an McpCallEvent is emitted with success=False and error message

Scenario: Event follows HookEvent pattern
  Given the McpCallEvent class
  Then it is a frozen dataclass extending HookEvent
  And event_name is "mcp:call"

Scenario: --verbose flag shows call details
  When I run `rai mcp call <server> <tool> --verbose`
  Then stderr shows server, tool, latency, and status
```
