# Scope: S338.4 — Telemetry hooks

## In Scope

- `McpCallEvent` frozen dataclass in `hooks/events.py` (follows existing pattern)
- Emit event from `rai mcp call` command after tool invocation
- `--verbose` flag on `rai mcp call` — shows server, tool, latency, status to stderr
- Tests for event structure and emission

## Out of Scope

- Event persistence / aggregation (`rai mcp stats` — deferred per AR-R2)
- Emitting from bridge itself (bridge is infrastructure, unaware of hooks)
- Emitting from health/tools/list commands (not tool invocations)

## Done Criteria

- McpCallEvent emitted on every `rai mcp call` (success and failure)
- `--verbose` shows human-readable call details on stderr
- Event structure tested (frozen, fields, event_name)
- All existing tests pass
