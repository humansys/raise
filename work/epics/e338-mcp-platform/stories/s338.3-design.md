# Design: S338.3 — `rai mcp call`

## Complexity: Simple

3 components, well-established CLI pattern from backlog/adapter commands.

## Problem

No CLI path to invoke MCP server tools. Developers must configure MCP natively in IDE (injecting thousands of tool schema tokens) or write custom scripts.

## Value

`rai mcp call` as CLI command = zero tool schema tokens in agent context. Skills call it via Bash, not MCP protocol.

## Approach

New Typer command `rai mcp call <server> <tool> [--args JSON]` that:
1. Looks up server in registry (`discover_mcp_servers()`)
2. Creates McpBridge with server connection config
3. Calls tool via `bridge.call(tool, args)`
4. Prints result as JSON to stdout
5. Closes bridge

Uses `asyncio.run()` for async-to-sync (same pattern as `_run_sync`).

## Components

| Component | Change |
|-----------|--------|
| `src/rai_cli/cli/commands/mcp.py` | CREATE — `mcp_app` Typer group with `call` command |
| `src/rai_cli/cli/main.py` | MODIFY — register `mcp_app` |
| `tests/cli/commands/test_mcp.py` | CREATE — unit tests |

## Examples

### CLI Usage

```bash
# With arguments
rai mcp call context7 resolve-library-id --args '{"query":"nextjs","libraryName":"next.js"}'

# Without arguments
rai mcp call echo ping

# Server not found
rai mcp call nonexistent tool
# stderr: Error: Server 'nonexistent' not found in registry.
# exit code: 1
```

### Stdout (success)

```json
{"text": "...", "data": {"libraryId": "/vercel/next.js"}, "is_error": false, "error_message": ""}
```

### Stdout (tool error)

```json
{"text": "", "data": {}, "is_error": true, "error_message": "Library not found"}
```

### Command signature

```python
@mcp_app.command()
def call(
    server: Annotated[str, typer.Argument(help="Registered MCP server name")],
    tool: Annotated[str, typer.Argument(help="Tool name to invoke")],
    args: Annotated[str, typer.Option("--args", help="Tool arguments as JSON string")] = "{}",
) -> None:
```

## Acceptance Criteria

Refined from story.md:

- **MUST**: `rai mcp call <server> <tool> --args '{...}'` prints JSON result to stdout
- **MUST**: Server not found → stderr error + exit 1
- **MUST**: Bridge error → stderr error + exit 1
- **MUST**: Invalid --args JSON → stderr error + exit 1
- **SHOULD**: Output is valid JSON parseable by `jq`
- **MUST NOT**: Import from `rai_cli.adapters` (infrastructure independence)
