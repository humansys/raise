# User Story: S338.6 — `rai mcp install`

## Story

**As a** developer adding a new MCP server to my project,
**I want to** run `rai mcp install <package> --type uvx|npx|pip`,
**so that** the server is installed and a `.raise/mcp/<name>.yaml` config is generated automatically.

## Acceptance Criteria (Gherkin)

### Scenario 1: Install via npx

```gherkin
Given the npx package "@upstash/context7-mcp" is available
When I run `rai mcp install @upstash/context7-mcp --type npx --name context7`
Then the server is verified via health check
And .raise/mcp/context7.yaml is generated with command: npx, args: ["-y", "@upstash/context7-mcp"]
```

### Scenario 2: Install via uvx

```gherkin
Given the uvx package "mcp-github" is available
When I run `rai mcp install mcp-github --type uvx --name github`
Then the server is verified via health check
And .raise/mcp/github.yaml is generated with command: uvx, args: ["mcp-github"]
```

### Scenario 3: Install via pip

```gherkin
Given the pip package "mcp-server-fetch" is available
When I run `rai mcp install mcp-server-fetch --type pip --name fetch`
Then the package is pip-installed
And the server is verified via health check
And .raise/mcp/fetch.yaml is generated with command: python, args: ["-m", "mcp_server_fetch"]
```

### Scenario 4: Health check fails after install

```gherkin
Given a package that installs but fails to start
When I run `rai mcp install bad-pkg --type uvx --name bad`
Then config is generated but marked with a warning comment
And stderr reports the health check failure
And exit code is 0 (install succeeded, health failed)
```

### Scenario 5: Unknown --type rejected

```gherkin
Given an unsupported --type value "cargo"
When I run `rai mcp install pkg --type cargo`
Then exit code is 2 (Typer validation error)
```

## Size

M (Medium) — 3 package type strategies, health verification, config generation
