# Design: S338.10 — MCP health in session start

## Complexity: Simple (XS)

Single skill modification — add one substep to `/rai-session-start`.

## Approach

Add MCP health check as a substep in Step 2 of `/rai-session-start`. Runs `rai mcp list` to detect servers, then `rai mcp health` per server. Result appears as one line in the session summary.

### Output format

- All healthy: `**MCP:** 2 servers, all healthy`
- Some unhealthy: `**MCP:** 2 servers, 1 unhealthy — run /rai-mcp-status`
- No servers: line omitted (no noise)

### Components

| Component | Change |
|-----------|--------|
| `/rai-session-start` | Modify — add MCP substep in Step 2 |

## Acceptance Criteria

### MUST
- MCP status line in session summary when servers registered
- Warning for unhealthy servers with `/rai-mcp-status` suggestion
- Silent skip when no servers registered

### MUST NOT
- Block session start on unhealthy servers
- Show full health details (that's `/rai-mcp-status`)
