# S479.2: Git Evidence Extractor

## User Story

**As a** development team lead preparing for ISO 27001 recertification,
**I want** an extractor that collects git history (commits, PRs, tags, branches) as traceable evidence items,
**so that** the audit report includes verifiable change management and configuration management evidence.

## Acceptance Criteria

```gherkin
Feature: Git evidence extraction

  Scenario: Extract commits for a control
    Given a git repository with commits in the audit period
    And the control mapping includes A.8.32 (change management) with git evidence
    When the git extractor runs for A.8.32
    Then it returns EvidenceItem objects with evidence_type "git"
    And each item has commit hash, author, date, and message

  Scenario: Extract version tags
    Given a git repository with version tags (e.g. v0.1.0)
    And the control mapping includes A.8.9 (configuration management)
    When the git extractor runs for A.8.9
    Then it returns EvidenceItems for each tag within the audit period
    And each item has the tag name and associated commit hash

  Scenario: Extract branch naming patterns
    Given a git repository with branches following naming conventions
    And the control mapping includes A.8.9
    When the git extractor runs for A.8.9
    Then it returns EvidenceItems showing branch naming discipline
    And source_ref contains branch names

  Scenario: Detect merge commits as PR evidence
    Given a git repository with merge commits
    And the control mapping includes A.8.32
    When the git extractor runs for A.8.32
    Then merge commits are identified as review evidence
    And each item includes the merge commit message and hash

  Scenario: Filter by date range
    Given a git repository with commits spanning multiple years
    When the git extractor runs with a date range of 2025-01-01 to 2025-12-31
    Then only evidence within that period is returned
    And no evidence outside the period is included

  Scenario: Handle empty results
    Given a git repository with no commits in the audit period
    When the git extractor runs
    Then it returns an empty list
    And no error is raised

  Scenario: Cover all git-sourced controls
    Given the default control mapping
    When the git extractor runs for all controls with git evidence sources
    Then A.8.4, A.8.9, and A.8.32 all have evidence items
```

## Evidence Types Extracted

| Git Artifact | Controls | EvidenceItem Fields |
|-------------|----------|-------------------|
| Commits | A.8.32, A.8.9 | hash, author, date, message |
| Tags | A.8.9 | tag name, commit hash, date |
| Branches | A.8.9 | branch name, naming pattern |
| Merge commits | A.8.32 | hash, message (review proxy) |
