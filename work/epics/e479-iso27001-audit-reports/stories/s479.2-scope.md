# S479.2 Scope: Git Evidence Extractor

## In Scope

- Git evidence extractor module using `subprocess` to call git commands
- Extract commits (hash, author, date, message) as EvidenceItems
- Extract version tags as EvidenceItems
- Extract branch names as EvidenceItems
- Detect merge commits as PR/review evidence
- Date range filter for audit period
- Returns `list[EvidenceItem]` per control
- Reads ControlMapping (from S479.1) to determine which controls need git evidence
- Covers controls: A.8.4 (access to source code), A.8.9 (configuration management), A.8.32 (change management)

## Out of Scope

- Gate evidence extraction (S479.3)
- Session evidence extraction (S479.4)
- Report generation (S479.5+)
- GitHub/GitLab API calls — subprocess git only
- gitpython or other git library dependencies
- Evidence from non-git sources
- Custom git log formats beyond what's needed for evidence

## Done Criteria

- [ ] Extractor produces EvidenceItems from real git history
- [ ] All git-sourced controls (A.8.4, A.8.9, A.8.32) covered
- [ ] Date range filtering works correctly
- [ ] Merge commit detection identifies review evidence
- [ ] Tag extraction captures version releases
- [ ] Branch extraction captures naming patterns
- [ ] Tests verify extraction from real/mock git data
- [ ] Type checks pass (pyright strict)
- [ ] Linting passes
