# S479.1 Scope: Control Mapping Configuration

## In Scope

- YAML configuration file mapping 8 ISO 27001:2022 Annex A.8 controls to RaiSE evidence sources
- Pydantic models for `ControlMapping` and `EvidenceSourceConfig`
- Default mapping file shipped with raise-cli (package data)
- Loader function that reads YAML and returns validated Pydantic model
- Unit tests for loading, validation, and rejection of invalid config

## Out of Scope

- Evidence extractors (S479.2, S479.3, S479.4)
- Report generation (S479.5, S479.6, S479.7)
- Non-A.8 controls (organizational, people, physical)
- Runtime configuration UI or CLI for editing mappings
- Custom user overrides of mapping (future enhancement)

## Done Criteria

- [ ] All 8 A.8 controls mapped to at least one evidence source in YAML
- [ ] Pydantic models validate the mapping structure
- [ ] Config loads successfully from default YAML path
- [ ] Invalid configs are rejected with clear validation errors
- [ ] Tests verify loading, validation, and error cases
- [ ] Type checks pass (pyright strict)
- [ ] Linting passes

## Dependencies

- None (this is the foundation story)

## Enables

- S479.2 (Git extractor)
- S479.3 (Gate extractor)
- S479.4 (Session extractor)
- S479.8 (Coverage dashboard)
