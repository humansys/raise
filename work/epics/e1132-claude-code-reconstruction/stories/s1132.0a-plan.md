## Tasks

### T1: TypeScriptAnalyzer core — parse, detect, extract imports/exports (TDD)

**Description:** Create `TypeScriptAnalyzer` class with `detect()`, `analyze_modules()`, and all internal extraction methods. Follow PythonAnalyzer structure exactly. Reuse `_get_ts_parser` from `scanner.py`.

**Files:**
- **New:** `packages/raise-cli/src/raise_cli/context/analyzers/typescript.py`
- **New:** `packages/raise-cli/tests/context/analyzers/test_typescript.py`

**TDD Cycle:**

1. **RED — detect():** Test `detect()` returns True when `tsconfig.json` exists, True when `package.json` with `.ts` files exists, False when neither exists.
2. **GREEN:** Implement `detect()`.
3. **RED — import extraction:** Test named imports (`import { Foo } from '../bar'`), default imports (`import Config from '../config'`), namespace imports (`import * as utils from '../utils'`), type-only imports (`import type { X } from '../types'`). Test external packages are ignored. Test self-imports are ignored.
4. **GREEN:** Implement `_extract_imports()` using tree-sitter `import_statement` nodes. Resolve relative paths to sibling module names (design section "Import Resolution").
5. **RED — export extraction:** Test named exports (`export class Foo {}`), default exports (`export default class App {}`), re-exports (`export { Foo } from './bar'`), `export type` statements.
6. **GREEN:** Implement `_extract_exports()` using `export_statement`, `export_declaration`, `export_default_declaration` nodes.
7. **RED — barrel file:** Test that when `index.ts` exists, its exports define the module API. Test re-exports contribute both export names AND import dependencies.
8. **GREEN:** Implement barrel file logic in `_extract_exports()`.
9. **RED — component counting:** Test that classes, functions, interfaces, enums, type aliases are counted.
10. **GREEN:** Implement `_count_components()`.
11. **RED — analyze_modules():** Test full module analysis with a fixture tree (multiple modules under `src/`). Verify `ModuleInfo` output matches expected imports, exports, component_count, language="typescript".
12. **GREEN:** Implement `analyze_modules()` — iterate top-level dirs under `src/`, skip dirs without `.ts`/`.tsx` files, aggregate per module.
13. **REFACTOR:** Extract helpers, clean up, ensure `.d.ts` files are skipped.

**AC Coverage:** AC1-AC11 (all MUST), plus SHOULD items (.tsx, .js/.jsx, skip .d.ts)

**Verification:**
```bash
uv run pytest tests/context/analyzers/test_typescript.py --tb=short -v
uv run ruff check src/ tests/ && uv run ruff format --check src/ tests/
uv run pyright
```

**Size:** M (bulk of the work)
**Dependencies:** None — standalone new file + tests

## Execution Order

```
T1 (core + tests)  ──→  T2 (integration)  ──→  T3 (dogfood)
```

All tasks are sequential. T1 is risk-first (tree-sitter parsing is the unknown). T2 is mechanical wiring. T3 is validation.

## Risk Notes

- **Tree-sitter node type names:** The design lists expected node types (`import_statement`, `export_statement`, etc.). If actual tree-sitter-typescript grammar differs, T1 RED phase will reveal this immediately. Mitigation: inspect parsed tree in first failing test.
- **Performance on large codebase:** 1902 files should be fine for tree-sitter (native parser), but T3 will confirm.
- **`.tsx` parsing:** tree-sitter-typescript has separate `language_tsx()` — reuse `_get_ts_parser` dispatch logic from scanner.py.
