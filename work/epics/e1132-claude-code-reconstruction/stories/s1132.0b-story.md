# S1132.0b: Module-Level Dependency Aggregation

**Jira:** RAISE-1171
**Epic:** E1132 (RAISE-1162) — Claude Code Architecture Reconstruction
**Size:** S | **Type:** code

---

## User Story

**As a** RaiSE user analyzing TypeScript codebases,
**I want** file-level import data aggregated into a module-level dependency matrix,
**So that** the knowledge graph shows which modules depend on which, enabling architecture visualization and analysis.

---

## Acceptance Criteria

```gherkin
Feature: Module-level dependency aggregation

  Scenario: Aggregate file imports to module dependencies
    Given module "commands" has files importing from "../core/runner" and "../config/index"
    When analyze_modules() processes the project
    Then the "commands" ModuleInfo.imports contains ["config", "core"]

  Scenario: Root-level files excluded from module resolution
    Given src/ contains both directories (commands/, core/) and root files (commands.ts, Tool.ts)
    When a file imports "../commands" and it resolves to root-level commands.ts
    Then that import is NOT added as a module dependency

  Scenario: Dependency matrix generation
    Given a project with modules ["commands", "core", "config", "utils"]
    When the dependency matrix is generated
    Then it shows directed edges: commands→core, commands→config, etc.

  Scenario: Dogfood against Claude Code
    Given the Claude Code source tree (~1902 TS files, 35 modules)
    When the TypeScriptAnalyzer processes the project
    Then all ModuleInfo.imports reference valid module names (directories, not root files)
```

---

## SbE Examples

### Root-level file exclusion

```
src/
  commands/        <- module (directory)
    execute.ts
  core/            <- module (directory)
    runner.ts
  commands.ts      <- root file (NOT a module)
  Tool.ts          <- root file (NOT a module)
```

File `src/core/runner.ts` with `import { x } from '../commands'`:
- Resolves to `src/commands` — is a directory → valid module dep: `"commands"` ✓
- If it resolved to `src/commands.ts` — is a file → skip ✗

### Dependency matrix output

```
          commands  core  config  utils
commands     -       ✓      ✓      ✓
core         -       -      -      ✓
config       -       ✓      -      -
utils        -       -      -      -
```
