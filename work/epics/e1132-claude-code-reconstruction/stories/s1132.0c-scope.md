# S1132.0c Scope: Signal Scanner

**Story:** S1132.0c | **Jira:** RAISE-1172

---

## In Scope

- New `SignalScanner` class that scans files for configurable regex patterns
- Default signal tags: TODO, HACK, FIXME, XXX, DEPRECATED, @deprecated
- Output: list of `SignalHit` (file, line, tag, message) per file
- Aggregation: count by tag per module directory
- Integration with TypeScriptAnalyzer workflow (same file collection)
- E2E against Claude Code source

## Out of Scope

- Feature flag detection (complex, heuristic — defer)
- Dead code detection (needs call graph — defer)
- Integration into ModuleInfo model (separate concern)
- Integration into builder.py graph (separate story)
- AST-based comment extraction (regex is sufficient for markers)

## Done When

- SignalScanner extracts TODO/HACK/FIXME/XXX/DEPRECATED from .ts files
- Aggregation produces per-module tag counts
- Tests cover: single signal, multiple signals, no signals, custom patterns
- E2E against Claude Code produces plausible results (~438 signals across 90 files)
- pyright strict, ruff, all green
