# Story: Reconnaissance — Architecture Map

## User Story
As a RaiSE framework developer,
I want a complete architectural map of Claude Code's source code,
so that I can make informed decisions about how to extend, integrate with, and build upon it.

## Acceptance Criteria

### Scenario: Module catalog complete
```gherkin
Given the Claude Code source tree (35 top-level modules + root files)
When I complete the reconnaissance scan
Then every module has a catalog entry with: name, size, file count, purpose, key exports, RaiSE relevance rating
```

### Scenario: Dependency map produced
```gherkin
Given the module catalog
When I analyze inter-module imports
Then I produce a dependency map showing which modules import from which
And I identify high-coupling clusters and architectural layers
```

### Scenario: Interest areas prioritized
```gherkin
Given the module catalog and dependency map
When I identify areas relevant to RaiSE
Then each area has a hypothesis question and assigned wave (1-5)
And the wave assignment reflects business value priority
```

## Examples (Specification by Example)

| Module | Catalog Entry | Relevance |
|--------|--------------|-----------|
| skills/ | 20 files, 208K, skill loading/validation/execution | Alta — we build skills |
| buddy/ | 6 files, 88K, companion sprite easter egg | Baja — no business impact |
| coordinator/ | 1 file, 24K, multi-agent orchestration | Alta — informs E3 |
