# S1132.0c: Signal Scanner — TODOs, Feature Flags, Dead Code Markers

**Jira:** RAISE-1172
**Epic:** E1132 (RAISE-1162) — Claude Code Architecture Reconstruction
**Size:** S | **Type:** code

---

## User Story

**As a** RaiSE user performing architecture reconnaissance,
**I want** to scan source files for technical debt signals (TODO, HACK, FIXME, feature flags),
**So that** I can identify areas of uncertainty and technical debt concentration per module.

---

## Acceptance Criteria

```gherkin
Feature: Signal scanning

  Scenario: Detect TODO/HACK/FIXME markers
    Given a TypeScript file containing "// TODO: refactor this"
    When the signal scanner processes the file
    Then it reports a signal with tag "TODO" and the message text

  Scenario: Aggregate signals per module
    Given module "commands" has 3 TODOs and 1 HACK across its files
    When signals are aggregated
    Then the module report shows {"TODO": 3, "HACK": 1}

  Scenario: Dogfood against Claude Code
    Given the Claude Code source (~1902 TS files)
    When the signal scanner processes all modules
    Then it reports signal counts per module with no errors
```
