# S348.1: Documentation audit

## User Story

**As a** documentation maintainer,
**I want** a systematic gap analysis of current vs needed documentation,
**So that** I can prioritize which docs to write, update, or restructure in subsequent stories.

## Acceptance Criteria

- Gap analysis document produced covering all documentation artifacts
- Every Diataxis type (tutorial, how-to, reference, explanation) mapped to current/needed state
- Three audiences addressed: users, developers, AI agents
- Audit covers: docs/ Astro site content, README.md, AGENTS.md, llms.txt, CLI reference, extension point docs, community files
- Priorities assigned for subsequent stories (S348.2-S348.6)
