# S348.2: CLI Reference Completion

## User Story

**As a** RaiSE user, developer, or AI agent,
**I want** complete, accurate CLI reference documentation for all v2.2 commands,
**So that** I can discover and use any `rai` subcommand without reading source code.

## Acceptance Criteria

```gherkin
Feature: CLI Reference Documentation

  Scenario: All current commands documented
    Given the CLI reference page at docs/src/content/docs/docs/cli/index.mdx
    When a user looks up any of the 72 current subcommands
    Then they find its description, flags, and at least one usage example

  Scenario: Deprecated namespaces removed
    Given the CLI reference page
    When a user searches for "rai memory" or "rai publish"
    Then no documentation for those deprecated namespaces appears

  Scenario: Command groups organized consistently
    Given the CLI reference page
    When a user browses the reference
    Then each command group has a heading with its subcommands listed
    And each subcommand shows: synopsis, description, flags/options, example, exit codes

  Scenario: Flags match actual CLI
    Given any documented command
    When compared against `rai <command> --help` output
    Then all flags, their types, and defaults match exactly

  Scenario: Partially documented commands updated
    Given commands like init, session, skill, release, discover
    When viewing their documentation
    Then all current flags and subcommands are present
    And stale/removed flags are not shown
```

## Size

M (Medium) -- 72 subcommands across 20 groups, primarily documentation writing with CLI verification.

## Dependencies

- S348.1 (Documentation audit) -- Done. Gap analysis provides the inventory.

## Notes

- Source of truth: `rai <command> --help` output + actual execution
- Target file: `docs/src/content/docs/docs/cli/index.mdx`
- Format: consistent per-command sections with synopsis, description, flags table, example, exit codes
- 11 entirely new command groups + 4 partially documented groups to update
