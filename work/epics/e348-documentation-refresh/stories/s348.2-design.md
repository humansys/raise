# S348.2 Design: CLI Reference Completion

**Story:** S348.2 | **Epic:** E348 | **Complexity:** moderate
**Skill:** rai-story-design | **Created:** 2026-03-05

## Summary

Replace the stale CLI reference (100% deprecated `rai memory` namespace) with complete documentation for all 72 current v2.2 subcommands across 20 command groups. Multi-file Astro/Starlight pages organized in workflow order, agent-friendly (<5000 chars/page).

## Approach

### Per-command format

Each subcommand gets:

```markdown
### `rai <group> <subcommand>`

<1-2 sentence description>

| Flag | Short | Description |
|------|-------|-------------|
| ... | ... | ... |

\`\`\`bash
# Example usage
rai <group> <subcommand> <args>
\`\`\`

**Exit codes:** 0 success, 1 error [if non-trivial]

**See also:** `rai <related>` [if relevant]
```

### Workflow order (sidebar)

### Source of truth

Every flag, option, and default MUST be verified against `rai <command> --help` output. No writing from memory.

## Decisions

### D1: Multi-file structure
**Rationale:** Agent-friendly (<5000 chars/page), enables llms.txt linking, Starlight supports sidebar autogenerate.
**Alternatives:** Single file (too large for LLM context).

### D2: Lean per-command format
**Rationale:** Description + flags + examples + exit codes covers scripting/CI needs. Common mistakes stay in CLAUDE.md for agents.
**Alternatives:** Minimal (insufficient), full with caveats (wrong venue).

### D3: Remove deprecated entirely
**Rationale:** No installed user base. Clean break, no migration table needed.
**Alternatives:** Migration table (unnecessary), deprecation notices (noise).

### D4: Workflow order
**Rationale:** Mirrors how users discover the CLI progressively.
**Alternatives:** Alphabetical (no narrative), frequency-based (subjective).

## Acceptance Criteria

4. [AC4] Each page <5000 chars (agent-friendly)
