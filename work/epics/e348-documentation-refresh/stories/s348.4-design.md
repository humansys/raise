# S348.4 Design — Developer Extension Guides

## Approach
Diataxis how-to guides: task-oriented, practical, include working examples. Each page is self-contained.

## Architecture
All guides reference actual codebase patterns:
- **Adapters**: Protocol-based (`protocols.py`), entry point registration (`pyproject.toml`), async-first with sync wrappers
- **Skills**: SKILL.md frontmatter + structured sections, scaffold via CLI
- **MCP**: YAML config in `.raise/mcp/`, install via CLI or manual scaffold
- **Hooks**: Shell scripts in `.claude/scripts/`, configured in Claude Code settings

## File Layout
```
docs/src/content/docs/docs/guides/
  extending.mdx         # Overview linking to 4 guides
  create-adapter.mdx    # Adapter creation guide
  create-skill.mdx      # Skill creation guide
  register-mcp.mdx      # MCP registration guide
  create-hook.mdx       # Hook wiring guide
```

## Quality Constraints
- Each page <5000 chars
- Code examples from actual source (not pseudocode)
- Self-contained (no required cross-reading)
- Consistent frontmatter format matching existing guides
