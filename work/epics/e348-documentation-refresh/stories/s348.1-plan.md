## Tasks

### T1: Execute gap analysis

**Goal:** Produce a comprehensive gap matrix covering all documentation artifacts, CLI surface, and audience needs.

**Steps:**
1. Read every existing doc file in `docs/src/content/docs/docs/` (both en and es)
2. Run `rai --help` and every subcommand `--help` to capture the actual CLI surface
3. Compare documented commands vs actual commands — compute coverage percentage
4. Assess each doc file for: accuracy (v2.2), completeness, audience, Diataxis type
5. Check for missing artifacts: llms.txt, AGENTS.md (placeholder vs real), extension docs
6. Assess community files: README.md, CONTRIBUTING.md, CODE_OF_CONDUCT.md, CHANGELOG.md
7. Produce the gap matrix as `work/epics/e348-documentation-refresh/stories/s348.1-gap-analysis.md`

**Verification:** Every CLI command group has a row. All three audiences (user, developer, agent) have explicit coverage. Diataxis types mapped.

**Depends on:** None

---

### T2: Validate and commit

**Goal:** Ensure the gap analysis is complete and commit the artifact.

**Steps:**
1. Review gap analysis for completeness against AC1-AC5 from the design
2. Verify every command group (20 groups) has a row in the matrix
3. Verify priorities (P0/P1/P2) are assigned and align with epic story assignments (S348.2-S348.6)
4. Commit the gap analysis

**Verification:** All acceptance criteria satisfied. Artifact committed to version control.

**Depends on:** T1
