# S337.1: Expression Evaluator + YAML Schema

## User Story

As a **framework developer**,
I want **a mini expression evaluator and Pydantic schema for YAML adapter configs**,
so that **declarative adapters can map protocol methods to MCP tool calls via templates**.

## Acceptance Criteria

```gherkin
Scenario: Evaluate simple variable substitution
  Given a template "{{ name }}"
  And context {"name": "hello"}
  When I evaluate the template
  Then the result is "hello"

Scenario: Evaluate dot-access
  Given a template "{{ issue.summary }}"
  And context {"issue": {"summary": "Fix bug"}}
  When I evaluate the template
  Then the result is "Fix bug"

Scenario: Evaluate filter — str
  Given a template "{{ value | str }}"
  And context {"value": 42}
  When I evaluate the template
  Then the result is "42"

Scenario: Evaluate filter — default
  Given a template "{{ missing | default('fallback') }}"
  And context {}
  When I evaluate the template
  Then the result is "fallback"

Scenario: Evaluate filter — pluck
  Given a template "{{ items | pluck('name') }}"
  And context {"items": [{"name": "a"}, {"name": "b"}]}
  When I evaluate the template
  Then the result is ["a", "b"]

Scenario: Evaluate filter — json
  Given a template "{{ data | json }}"
  And context {"data": {"key": "value"}}
  When I evaluate the template
  Then the result is '{"key": "value"}'

Scenario: Literal passthrough
  Given a template "hello world"
  And context {}
  When I evaluate the template
  Then the result is "hello world"

Scenario: Valid YAML config parses to Pydantic model
  Given a valid YAML adapter config file
  When I parse it with DeclarativeAdapterConfig
  Then all fields are validated and accessible

Scenario: Invalid YAML config raises validation error
  Given a YAML config missing required fields
  When I parse it with DeclarativeAdapterConfig
  Then a ValidationError is raised with clear message
```
