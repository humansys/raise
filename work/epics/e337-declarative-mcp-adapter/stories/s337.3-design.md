# S337.3 Design — YAML Discovery in Resolver

## Complexity: Moderate-Simple

3 components (discovery.py create, _resolve.py modify, tests), S size, no external integrations.

## What & Why

**Problem:** DeclarativeMcpAdapter exists but can't be discovered. Users must write Python to use it — defeating the purpose of YAML-driven adapters.

**Value:** After this story, `rai backlog --adapter github` works if `.raise/adapters/github.yaml` exists. Zero Python required for new MCP integrations.

## Approach

New `discovery.py` module scans `.raise/adapters/*.yaml`, parses each with `DeclarativeAdapterConfig`, filters by protocol, and returns factory closures. `_resolve.py` merges YAML results with entry point results, with entry points winning on name collision.

### Components

| Component | Change | Purpose |
|-----------|--------|---------|
| `src/rai_cli/adapters/declarative/discovery.py` | **create** | `discover_yaml_adapters(protocol)` |
| `src/rai_cli/cli/commands/_resolve.py` | **modify** | Merge YAML into discover lambda, relax type |
| `tests/adapters/declarative/test_discovery.py` | **create** | Unit tests for discovery |
| `tests/cli/commands/test_resolve.py` | **modify** | Integration tests with YAML adapters |

### Key Decisions (from epic design)

- **AR-C1:** Factory closure — `lambda: DeclarativeMcpAdapter(config)` — compatible with `cls()` call in `resolve_entrypoint`
- **AR-C2:** Each closure captures its own config; McpBridge created at instantiation time (shared per adapter lifetime, not per call)
- **Type relaxation:** `resolve_entrypoint.discover` changes from `Callable[[], dict[str, type]]` to `Callable[[], dict[str, Callable[[], Any]]]`
- **Priority:** Entry points dict updates first, YAML dict second → entry points win on collision (dict merge semantics)

## Examples

### discover_yaml_adapters usage

```python
# .raise/adapters/github.yaml exists with adapter.protocol = "pm"
# .raise/adapters/confluence.yaml exists with adapter.protocol = "docs"

from rai_cli.adapters.declarative.discovery import discover_yaml_adapters

pm_adapters = discover_yaml_adapters("pm")
# {"github": <factory closure>}

docs_adapters = discover_yaml_adapters("docs")
# {"confluence": <factory closure>}

# Factory closure returns a DeclarativeMcpAdapter
adapter = pm_adapters["github"]()
assert isinstance(adapter, DeclarativeMcpAdapter)
```

### Merged discovery in _resolve.py

```python
def resolve_adapter(adapter_name: str | None) -> ProjectManagementAdapter:
    return resolve_entrypoint(
        discover=lambda: {**discover_yaml_adapters("pm"), **get_pm_adapters()},
        #                  ^^^^ YAML first, EP second = EP wins on collision
        ...
    )
```

### Error handling

```python
# Malformed YAML → logged warning, skipped (consistent with registry.py pattern)
# Missing env vars → NOT checked at discovery time (checked at instantiation)
# Empty .raise/adapters/ dir → returns {} (no error)
# .raise/adapters/ doesn't exist → returns {} (no error)
```

## Acceptance Criteria

### MUST
1. `discover_yaml_adapters("pm")` returns `dict[str, Callable[[], DeclarativeMcpAdapter]]` from `.raise/adapters/*.yaml`
2. `resolve_adapter(None)` auto-selects when only YAML adapter available
3. `resolve_adapter("github")` selects YAML adapter by name
4. Entry points take priority over YAML on name collision
5. Malformed YAML files are skipped with warning (not crash)

### SHOULD
1. Discovery uses project root detection (not cwd)

### MUST NOT
1. Not touch `registry.py` — discovery.py is separate module
2. Not validate env vars at discovery time
3. Not import discovery module at registry import time (lazy)
