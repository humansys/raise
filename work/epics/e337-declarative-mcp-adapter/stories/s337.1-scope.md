# S337.1 Scope: Expression Evaluator + YAML Schema

## In Scope

- `ExpressionEvaluator` class in `src/rai_cli/adapters/declarative/expressions.py`
  - `evaluate(template, context)` → resolved value
  - `evaluate_args(args_dict, context)` → resolved dict
  - Syntax: `{{ var }}`, `{{ obj.field }}`, literals
  - Filters: `str`, `default('x')`, `pluck('field')`, `json`
- `DeclarativeAdapterConfig` Pydantic models in `src/rai_cli/adapters/declarative/schema.py`
  - `AdapterMeta`: name, protocol, description
  - `ServerConfig`: command, args, env (list of var names — AR-R2)
  - `MethodMapping`: tool, args, response
  - `ResponseMapping`: fields, items_path (no `model` field — AR-R1)
  - `DeclarativeAdapterConfig`: adapter, server, methods
- `__init__.py` public API
- Unit tests for both modules
- YAML test fixtures (valid + invalid configs)

## Out of Scope

- `DeclarativeMcpAdapter` class (S337.2)
- Registry/resolver integration (S337.3)
- CLI commands (S337.5)

- [ ] `ExpressionEvaluator` handles all 6 syntax forms (var, dot-access, 4 filters) + literal passthrough
- [ ] `DeclarativeAdapterConfig` validates valid YAML and rejects invalid with clear errors
- [ ] All tests pass, types check, lint clean
- [ ] ~180 LOC total (expressions ~100, schema ~80)
