# S337.4: Docs Protocol Support

## User Story

**As a** CLI user with a YAML-defined docs adapter
**I want** `DeclarativeMcpAdapter` to also implement `AsyncDocumentationTarget`
**So that** I can use `rai docs --target wiki` with a YAML config for any MCP-based wiki/docs system.

## Acceptance Criteria

```gherkin
Scenario: Docs adapter publishes a page
  Given a YAML config with protocol "docs" and method "publish" mapped
  When adapter.publish(doc_type, content, metadata) is called
  Then the configured MCP tool is invoked with evaluated args
  And a PublishResult is returned with parsed response fields

Scenario: Docs adapter searches pages
  Given a YAML config with method "search" mapped
  When adapter.search(query, limit) is called
  Then a list of PageSummary is returned from the MCP tool response

Scenario: Docs adapter get_page
  Given a YAML config with method "get_page" mapped
  When adapter.get_page(identifier) is called
  Then a PageContent is returned with parsed fields

Scenario: Docs adapter can_publish check
  Given a YAML config with method "can_publish" mapped
  When adapter.can_publish(doc_type, metadata) is called
  Then a boolean is returned

Scenario: Unmapped docs method raises
  Given a YAML config without "get_page" in methods
  When adapter.get_page(identifier) is called
  Then NotImplementedError is raised
```
