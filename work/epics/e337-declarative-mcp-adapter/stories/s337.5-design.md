# Design: S337.5 — Reference Config + Validation CLI

## Complexity: Simple
2 components (CLI command + reference YAML), 0 external integrations.

## Problem
Developers integrating MCP servers have no reference for writing YAML configs and no way to validate them without running the adapter. The existing `valid.yaml` test fixture covers 5 of 11 PM methods and lacks inline documentation.

## Value
Unblocks self-service MCP integration: write YAML, validate offline, deploy. No framework developer needed.

## Approach

### 1. Reference `github.yaml`
Expand existing `valid.yaml` fixture into a full reference config with:
- Inline YAML comments explaining every section and field
- All 11 PM protocol methods mapped (or explicitly `null` for unsupported)
- Realistic GitHub MCP server tool names and expression templates

Lives at: `src/rai_cli/adapters/declarative/reference/github.yaml`
(Shipped with package — importable via `importlib.resources` for validation tests and future `rai adapter init` scaffolding.)

### 2. `rai adapters validate` command
Add to existing `adapters_app` in `adapters.py`:

```python
@adapters_app.command("validate")
def validate_command(
    file: Annotated[Path, typer.Argument(help="Path to YAML adapter config")],
) -> None:
```

**Behavior:**
- Read YAML file → parse with `yaml.safe_load` → validate with `DeclarativeAdapterConfig.model_validate()`
- Success: print adapter name, protocol, description, mapped methods count
- Failure: print Pydantic validation errors with field paths, exit code 1
- Edge cases: file not found → error + exit 1, empty/invalid YAML → error + exit 1

**No `--format` option.** This is a dev-time validation tool, not a CI pipeline output. Human-readable only (KISS).

## Examples

### Success
```
$ rai adapters validate .raise/adapters/github.yaml
✓ Valid adapter config
  Name:        github
  Protocol:    pm
  Description: GitHub Issues via mcp-github
  Methods:     8 mapped, 3 unsupported
  Server:      uvx mcp-github
```

### Schema error
```
$ rai adapters validate broken.yaml
✗ Invalid adapter config: broken.yaml
  adapter.name
    Field required
  server.command
    Field required
```

### File not found
```
$ rai adapters validate nonexistent.yaml
Error: File not found: nonexistent.yaml
```

### Empty / non-YAML
```
$ rai adapters validate empty.yaml
✗ Invalid adapter config: empty.yaml
  Cannot parse YAML content
```

## Acceptance Criteria

### MUST
1. Reference github.yaml validates successfully with `DeclarativeAdapterConfig`
2. All 11 PM methods present (mapped or `null`)
3. `rai adapters validate <file>` exits 0 on valid, 1 on invalid
4. Validation errors show field paths from Pydantic
5. Tests cover: valid, invalid schema, missing file, empty file, non-YAML

### SHOULD
1. Reference YAML has inline comments for every section

### MUST NOT
1. No runtime MCP server calls — validation is schema-only
2. No changes to existing schema models or adapter code
