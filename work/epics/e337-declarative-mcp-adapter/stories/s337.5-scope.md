# Scope: S337.5 — Reference Config + Validation CLI

## In Scope

- Reference `github.yaml` config with inline comments for all PM methods
- `rai adapter validate <file>` CLI command (Typer)
- Validation uses existing `DeclarativeAdapterConfig.model_validate()`
- Clear error messages for invalid configs (field-level Pydantic errors)
- Success output shows adapter name, protocol, mapped methods count

## Out of Scope

- Runtime testing against actual MCP server (validation is schema-only)
- Auto-generation of YAML configs from MCP tool discovery

- Docs protocol methods in reference config (github is PM-only)

## Done Criteria

- `rai adapter validate github.yaml` succeeds with clear output
- `rai adapter validate broken.yaml` fails with specific error messages
- Reference github.yaml has inline comments for every section
- Tests cover valid, invalid, and edge cases (missing file, empty file)
- All gates pass (pytest, pyright, ruff)
