# Story: S337.5 — Reference Config + Validation CLI

## User Story

**As a** developer integrating an MCP server,
**I want** a reference YAML config and a validation command,
**So that** I can author correct adapter configs without reading framework source code.

## Acceptance Criteria

### Scenario: Validate a correct YAML config
```gherkin
Given a valid github.yaml config in .raise/adapters/
When I run `rai adapter validate github.yaml`
Then the output shows "Valid" with adapter name and protocol
```

### Scenario: Validate catches schema errors
```gherkin
Given a YAML config with missing required fields
When I run `rai adapter validate broken.yaml`
Then the output shows specific validation errors
And the exit code is non-zero
```

### Scenario: Reference config serves as documentation
```gherkin
Given the reference github.yaml
When I read its contents
Then every section has inline comments explaining the config structure
And it maps all 11 PM protocol methods
```
