### Task 1: Add 4 docs methods to DeclarativeMcpAdapter (S)

**Description:** Add `can_publish`, `publish`, `get_page`, `search` (docs variant) to `DeclarativeMcpAdapter`. `health()` already exists and is protocol-agnostic — shared. The docs `search` returns `list[PageSummary]` (not `list[IssueSummary]`), so it needs a separate method name internally. Use `docs_search` in config methods to avoid collision with PM `search`.

Wait — re-reading the protocol: both PM and docs have `search`, but the adapter is one class serving one protocol at a time (per YAML config). The protocol field determines which `search` signature applies. So the same `search` key in methods dict works — the return type changes based on protocol. We need a way to dispatch correctly.

Simplest approach: add dedicated docs methods (`can_publish`, `publish`, `get_page`) and make `search` protocol-aware based on `adapter.protocol`. Or even simpler: just add the 4 new methods and let the consumer (SyncDocsAdapter) call the right ones. The adapter class exposes both PM and docs methods — structural typing picks the right protocol.
