# S337.2: DeclarativeMcpAdapter (PM Protocol)

## User Story

As a **framework developer**,
I want **a generic adapter class that implements all 11 PM protocol methods by dispatching to MCP tools via YAML config**,
so that **any MCP server can satisfy `AsyncProjectManagementAdapter` without writing Python code**.

## Acceptance Criteria

```gherkin
Scenario: Adapter dispatches create_issue to configured MCP tool
  Given a YAML config mapping create_issue to "github_create_issue"
  And a mocked McpBridge returning {"number": 42, "html_url": "https://..."}
  When I call adapter.create_issue("PROJ", IssueSpec(summary="Bug"))
  Then the bridge is called with tool="github_create_issue" and evaluated args
  And the result is an IssueRef with key="42" and url="https://..."

Scenario: Adapter dispatches search with list response
  Given a YAML config mapping search with items_path="data.items"
  And a mocked McpBridge returning {"data": {"items": [{"number": 1, "title": "Bug", "state": "open"}]}}
  When I call adapter.search("is:issue", limit=10)
  Then the result is a list with one IssueSummary(key="1", summary="Bug", status="open")

Scenario: Null method raises NotImplementedError
  Given a YAML config with link_to_parent: null
  When I call adapter.link_to_parent("A-1", "A-2")
  Then NotImplementedError is raised

Scenario: Undeclared method raises NotImplementedError
  Given a YAML config with no get_issue method
  When I call adapter.get_issue("A-1")
  Then NotImplementedError is raised

Scenario: batch_transition auto-loops over transition_issue
  Given a YAML config with transition_issue mapped but no batch_transition
  And a mocked McpBridge
  When I call adapter.batch_transition(["A-1", "A-2"], "done")
  Then transition_issue is called twice (once per key)
  And the result is a BatchResult with 2 succeeded

Scenario: health delegates to McpBridge.health
  Given an adapter with a mocked bridge
  When I call adapter.health()
  Then the result is an AdapterHealth with the adapter name

Scenario: aclose delegates to bridge
  Given an adapter with a mocked bridge
  When I call adapter.aclose()
  Then bridge.aclose() is called
```
