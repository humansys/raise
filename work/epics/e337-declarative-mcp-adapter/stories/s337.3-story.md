# S337.3: YAML Discovery in Resolver

## User Story

**As a** CLI user
**I want** YAML-defined adapters to be automatically discovered
**So that** I can use `rai backlog --adapter github` with a YAML config without writing Python code.

## Acceptance Criteria

```gherkin
Scenario: YAML adapter discovered alongside entry points
  Given a YAML config file exists at .raise/adapters/github.yaml
  And a Python entry point adapter "jira" is registered
  When the resolver discovers available PM adapters
  Then both "github" (YAML) and "jira" (entry point) are available
  And entry points take priority over YAML on name collision

Scenario: YAML adapter auto-selected when only one available
  Given a YAML config file exists at .raise/adapters/github.yaml
  And no Python entry point adapters are registered
  When resolve_adapter(None) is called
  Then the YAML adapter is instantiated via factory closure

Scenario: Factory closure produces working adapter
  Given a valid YAML config for protocol "pm"
  When discover_yaml_adapters("pm") is called
  Then each value in the result dict is a no-arg callable
  And calling it returns a DeclarativeMcpAdapter instance
```
