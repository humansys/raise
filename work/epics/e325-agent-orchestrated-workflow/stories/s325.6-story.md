# S325.6 — Story Orchestrator Skill

## User Story

**As a** developer using RaiSE,
**I want** a `/rai-story-run` meta-skill that chains the full story lifecycle (start → design → plan → implement → review → close),
**So that** I can say "implement story X" and the agent executes the full cycle, pausing only at pre-agreed delegation gates.

## Acceptance Criteria

```gherkin
Scenario: Full story cycle with delegation profile
  Given a developer with experience_level "ha" and no delegation overrides
  When I invoke /rai-story-run with story_id "S999.1"
  Then the orchestrator chains: start → design → plan → implement → review → close
  And delegation level "notify" is applied at each skill transition
  And the developer is shown a summary after each skill completes

Scenario: Git-derived phase detection for resumption
  Given story S999.1 has design_md and plan_md artifacts but no code commits
  When I invoke /rai-story-run with story_id "S999.1"
  Then the orchestrator detects phase as "implement" from artifact presence
  And resumes from /rai-story-implement (skipping start, design, plan)

Scenario: Delegation override respects per-skill config
  Given developer.yaml has delegation.overrides.rai-story-design = "review"
  When the orchestrator reaches the design phase
  Then it pauses for explicit human approval before continuing to plan

Scenario: Manual skill invocation still works
  Given /rai-story-run exists as a meta-skill
  When I invoke /rai-story-start directly
  Then it works exactly as before (backward compatible)

Scenario: Graceful degradation without backlog
  Given rai backlog CLI is not configured (no jira.yaml)
  When /rai-story-run completes
  Then the story cycle completes normally
  And backlog sync errors are logged but do not block the workflow
```

## Specification by Example

**Input:** `/rai-story-run S325.6`

**Phase detection logic:**
| Artifact | Exists? | Inferred Phase |
|----------|---------|----------------|
| story branch | no | start (begin from scratch) |
| story_md + scope_md | yes | design |
| design_md | yes | plan |
| plan_md | yes | implement |
| retrospective_md | yes | close |

**Delegation at each transition:**
```
[start] → check delegation("rai-story-start") → NOTIFY → show summary, continue
[design] → check delegation("rai-story-design") → NOTIFY → show summary, continue
[plan] → check delegation("rai-story-plan") → NOTIFY → show summary, continue
[implement] → check delegation("rai-story-implement") → NOTIFY → show summary, continue
[review] → check delegation("rai-story-review") → NOTIFY → show summary, continue
[close] → check delegation("rai-story-close") → NOTIFY → show summary, continue
```
