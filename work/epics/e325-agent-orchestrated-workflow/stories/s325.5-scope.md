# Scope: S325.5 — Discovery/Onboard Compaction

## In Scope

- **Compact discover-start**: Replace manual find commands with `rai discover scan` output for language/directory detection. Auto-populate context file.
- **Compact discover-scan**: Auto-describe high-confidence components. Batch-present medium-confidence by module with suggested descriptions.
- **Compact discover-validate**: Auto-validate high-confidence without asking. Present medium-confidence module batches with LLM-suggested descriptions for quick approve/edit.
- **Compact project-onboard**: Pre-populate governance docs from discovery output. Ask only for gaps not found by discovery.
- **Update I/O wiring**: Ensure skill outputs match next skill's inputs (auto-wire via I/O contracts from S325.1).
- **Backward compatibility**: All skills still work when invoked manually.

## Out of Scope

- Code changes to `rai discover scan` CLI command
- Code changes to `rai_cli/discovery/scanner.py` or any Python module
- New CLI commands or flags
- project-create changes (greenfield is inherently conversational)
- discover-document changes (already semi-automated, low ROI for compaction)

## Done Criteria

- [ ] Discovery pipeline completes with ≤50% of current manual steps (M2 criteria)
- [ ] Skills auto-wire: output of one skill feeds input of next without manual copy
- [ ] High-confidence components (≥70) never require human review
- [ ] Manual skill invocation still works (backward compatible)
- [ ] All changes are SKILL.md modifications only (no code changes)

## Approach

SKILL.md-only changes across 4 discovery/onboard skills. Compact manual steps into automated steps, add auto-wiring instructions that reference I/O contracts, and reduce HITL to decision gates only.
