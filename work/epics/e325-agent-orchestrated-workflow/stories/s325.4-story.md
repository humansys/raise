# User Story: S325.4 — Backlog-Aware Skills

**Epic:** E325 Agent-Orchestrated Workflow
**Jira:** RAISE-333
**Size:** M

## User Story

**As a** developer using RaiSE lifecycle skills,
**I want** Jira to reflect story/epic state automatically when I run lifecycle skills,
**So that** I don't manually transition tickets or add comments — the process keeps Jira in sync.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Story start creates/transitions Jira issue
  Given a story scope is committed via /rai-story-start
  When the skill completes
  Then a Jira issue exists for the story (or is transitioned to In Progress)

Scenario: Story close transitions Jira to Done
  Given a story is merged via /rai-story-close
  When the skill completes
  Then the Jira issue is transitioned to Done

Scenario: Story review adds retrospective comment
  Given a retrospective is written via /rai-story-review
  When the skill completes
  Then a summary comment is added to the Jira issue

Scenario: Epic start creates/transitions Jira epic
  Given an epic scope is committed via /rai-epic-start
  When the skill completes
  Then a Jira epic exists (or is transitioned to In Progress)

Scenario: Epic close transitions Jira to Done
  Given an epic is merged via /rai-epic-close
  When the skill completes
  Then the Jira epic is transitioned to Done

Scenario: Backlog CLI unavailable — graceful degradation
  Given rai backlog commands are not available
  When a lifecycle skill runs
  Then the skill completes normally with a warning (no failure)
```
