# User Story: S325.5 — Discovery/Onboard Compaction

## User Story

**As a** developer onboarding a project with RaiSE,
**I want** the discovery and onboarding skills to auto-wire between phases with fewer manual steps,
**so that** I can go from `rai init --detect` to a working graph in ≤50% of the current manual steps.

## Context

The current discovery pipeline (discover-start → scan → validate → document) and onboarding skills (project-create, project-onboard) require many manual steps that could be compacted:

- Language/directory detection in discover-start uses manual find commands
- Medium-confidence components require per-module human synthesis in discover-scan
- discover-validate requires O(modules) approval decisions for medium-confidence components
- Module documentation in discover-document requires human prose per module
- project-onboard re-asks for info that discovery already found

S325.1 added I/O contracts to all 12 lifecycle skills. This story uses those contracts to auto-wire the discovery pipeline steps and reduce HITL intervention to decision gates only.

## Acceptance Criteria

```gherkin
Scenario: Discovery pipeline auto-wires between phases
  Given a brownfield project with `rai init --detect` completed
  When I run `/rai-discover-start`
  Then language detection uses `rai discover scan` output (not manual find commands)
  And the context file is auto-populated with detected languages and directories

Scenario: Reduced manual steps in discover-scan
  Given discovery context exists
  When I run `/rai-discover-scan`
  Then high-confidence components (≥70) are auto-described without human review
  And medium-confidence components are batch-presented by module (not individually)

Scenario: Streamlined validation
  Given a components draft exists
  When I run `/rai-discover-validate`
  Then high-confidence auto-validates without asking
  And medium-confidence presents module batches with LLM-suggested descriptions

Scenario: Onboard compaction
  Given discovery pipeline is complete
  When I run `/rai-project-onboard`
  Then discovered modules, languages, and conventions pre-populate governance docs
  And the human is only asked for information not found by discovery

Scenario: Manual invocation still works
  Given any discovery or onboard skill
  When I invoke it directly (not via pipeline)
  Then it works as before (backward compatible)
```

## Size

M (medium) — touches 4-6 SKILL.md files, no code changes.

## Dependencies

- S325.1 (done) — I/O contracts provide typed metadata for auto-wiring
