# Scope: S325.4 — Backlog-Aware Skills

**Epic:** E325 Agent-Orchestrated Workflow
**Jira:** RAISE-333
**Size:** M

## In Scope

- `BacklogHook` — a LifecycleHook (ADR-039) that listens to `emit-work` signals and calls `rai backlog` commands deterministically
- Event-to-backlog mapping:
  - `story start` → `rai backlog create` (if needed) + `transition in-progress`
  - `story complete` → `rai backlog transition done`
  - `epic start` → `rai backlog create` (if needed) + `transition in-progress`
  - `epic complete` → `rai backlog transition done`
- Bridge: connect `rai signal emit-work` to the hook system so BacklogHook receives events
- Read `.raise/jira.yaml` for project key and transition mapping
- Graceful degradation: hook failure is logged and skipped (hooks never crash the CLI, per ADR-039)

## Out of Scope

- Jira comments on review (can be added later as hook extension)
- Issue creation with full metadata (parent link, labels) — bare create + transition only
- SKILL.md modifications (hooks handle it mechanically, not behaviorally)
- New CLI commands

## Done When

- `BacklogHook` registered as builtin hook
- `rai signal emit-work story X --event start` triggers Jira create+transition
- `rai signal emit-work story X --event complete` triggers Jira transition to done
- Same for epic work type
- Hook failure does not crash CLI (error isolation)
- Tests cover: event→backlog mapping, graceful degradation, jira.yaml loading
- All gates pass (pytest, pyright, ruff)
