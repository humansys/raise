# Scope: S325.4 — Backlog-Aware Skills

**Epic:** E325 Agent-Orchestrated Workflow
**Jira:** RAISE-333
**Size:** M

## In Scope

- Add `rai backlog` instructions to 5 lifecycle skills at natural integration points:
  - `/rai-story-start` → `rai backlog create` (if no Jira key) + `transition in-progress` (after scope commit)
  - `/rai-story-review` → `rai backlog comment {key} "retrospective summary"` (after retrospective)
  - `/rai-story-close` → `rai backlog transition {key} done` (after merge)
  - `/rai-epic-start` → `rai backlog create` (if no Jira key) + `transition in-progress` (after scope commit)
  - `/rai-epic-close` → `rai backlog transition {key} done` (after merge)
- Issue creation: story-start and epic-start create Jira issues if none exists (M2 criteria)
- Graceful degradation: skills instruct agent to warn but continue if CLI unavailable
- Use `--format agent` where applicable (S325.3 output)

## Out of Scope

- Jira integration in design/plan/implement skills (no natural Jira touchpoint)
- CLI code changes — this story only modifies SKILL.md instruction files
- Runtime enforcement of backlog calls (agent follows instructions, not code)

## Done When

- 5 lifecycle skills have `rai backlog` instructions at correct integration points
- Instructions include graceful degradation guidance
- Builtin and deployment copies are identical
- All existing tests still pass (no code changes, only SKILL.md modifications)
