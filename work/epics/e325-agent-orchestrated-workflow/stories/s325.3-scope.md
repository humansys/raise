# Scope: S325.3 — CLI Output as ACI

**Story:** S325.3
**Epic:** E325 Agent-Orchestrated Workflow
**Size:** S
**Branch:** epic/e325/agent-orchestrated-workflow (S-sized, stays on epic)
**Jira:** RAISE-332

---

## In Scope

- Add `--format` option (`human`|`agent`) to `rai backlog` subcommands: search, create
- Add `--format` option (`human`|`agent`) to `rai graph` subcommands: query, context, list
- Agent format: compact, structured, minimal tokens — one entity per line
- Default format remains `human` (current behavior, backward compatible)

## Out of Scope

- JSON/YAML machine-readable serialization (agent format is structured text, not data interchange)
- Changes to `rai backlog` subcommands that don't return query results (transition, update, link, comment, batch-transition)
- Changes to `rai graph` subcommands that are operations (build, validate, extract, viz)
- Output piping or streaming changes
- Agent-specific error formatting (use same errors for both formats)

## Done Criteria

- [ ] `--format agent` works on: backlog search, backlog create, graph query, graph context, graph list
- [ ] Default output (`--format human` or no flag) is identical to current behavior
- [ ] Agent output is ≤50% token count of human output for same data
- [ ] Tests cover both formats for each modified subcommand
- [ ] Existing tests pass (no regressions)
