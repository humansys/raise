# Retrospective: S325.S1 — Incremental Journal for Compaction Resilience (Spike)

**Story:** S325.S1 (spike) | **Size:** S | **Actual:** S
**Sessions:** SES-305, SES-306 | **Date:** 2026-03-01
**Velocity:** ~1.0x (heavy debugging offset implementation speed)

---

## Summary

Built an append-only JSONL journal system (`rai session journal`) for incremental memory persistence across Claude Code context compactions. The implementation (T1-T4) was straightforward, but four layers of bugs in the hook integration consumed most of the effort, ultimately leading to a pivot from hook-based to instruction-based post-compaction restoration.

## What Went Well

- Journal model + writer were clean TDD (10 tests, first try)
- CLI integration via Typer subcommand was mechanical
- Bug investigation was systematic — each layer peeled back revealed the next
- Pivot decision was fast once evidence was clear (3 known Claude Code bugs)
- Final solution (CLAUDE.md instruction) is simpler and more portable than hooks

## What To Improve

- Started with hook-based approach without first verifying Claude Code hook capabilities for SessionStart:compact — should have spiked the hook alone before building the journal
- Four debugging sessions across compactions to reach root cause — a focused 15-min hook test at the start would have revealed the limitation immediately
- Hook config caching (changes don't take effect until Claude Code restart) was a surprise that cost an extra debug cycle

## Bugs Discovered (4 layers)

| Layer | Bug | Fix |
|:-----:|-----|-----|
| 1 | `.envrc` stdout pollution via BASH_ENV chain — broke hook JSON parsing | Interactive guard: `[[ $- == *i* ]] && echo` |
| 2 | SessionStart hooks require JSON `hookSpecificOutput` format, not plain text | Rewrote scripts to output JSON (moot after pivot) |
| 3 | Ghost hook entry for deleted `session-init.sh` script | Removed from settings.local.json |
| 4 | Hook config cached in Claude Code session — removal doesn't take effect | Restart Claude Code to clear |

## Key Decision: Pivot from Hooks to Instructions

**Evidence:** Claude Code bugs #12671, #15174, #16538 confirm SessionStart hooks cannot reliably inject context post-compaction. Hook stdout is discarded, additionalContext ignored, compact matcher broken.

**Decision:** Replace hook-based restoration with CLAUDE.md instruction that tells the agent to read the journal when it detects a continuation summary. Simpler, portable, no runtime dependency.

**Trade-off:** Agent must follow instruction (behavioral, not mechanical). Validated: works when CLAUDE.md instruction is followed. Failed once when agent didn't follow it (user had to remind).
