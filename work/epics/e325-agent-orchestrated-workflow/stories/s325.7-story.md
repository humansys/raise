# S325.7 — Epic Orchestrator Skill

## User Story

**As a** developer using RaiSE,
**I want** a `/rai-epic-run` meta-skill that chains the full epic lifecycle (start → design → plan → [story iterations] → close),
**So that** I can say "run epic X" and the agent orchestrates the entire epic including iterating through stories via `/rai-story-run`.

## Acceptance Criteria

```gherkin
Scenario: Full epic cycle from scratch
  Given no epic branch or artifacts exist for E999
  When I invoke /rai-epic-run E999
  Then the orchestrator chains: start → design → plan → [stories via /rai-story-run] → close
  And delegation gates are applied at post-design and post-plan
  And each story is executed via /rai-story-run

Scenario: Phase detection for resumption
  Given epic E999 has scope.md with implementation plan and 2/5 stories done
  When I invoke /rai-epic-run E999
  Then the orchestrator detects phase as "stories" and resumes from next pending story
  And already-completed stories are skipped

Scenario: Story iteration with ordering
  Given epic plan lists stories S999.1 through S999.5 with dependencies
  When the orchestrator reaches the stories phase
  Then it executes stories in plan order
  And respects dependency ordering (blocked stories wait)

Scenario: Delegation at natural gates
  Given developer delegation level is REVIEW
  When the orchestrator completes epic design
  Then it pauses for human approval of the story breakdown before planning

Scenario: Backward compatibility
  Given /rai-epic-run exists
  When I invoke /rai-epic-start directly
  Then it works exactly as before
```

## Specification by Example

**Input:** `/rai-epic-run E325`

**Epic phase detection:**
| Artifact | Exists? | Inferred Phase |
|----------|---------|----------------|
| epic branch | no | start |
| scope.md (no plan) | yes | design |
| scope.md (with plan) | yes | plan or stories |
| stories in progress/done | yes | stories (iterate remaining) |
| all stories done | yes | close |
