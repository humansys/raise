# User Story: S325.2 — Delegation Profile Model

**Epic:** E325 Agent-Orchestrated Workflow
**Jira:** RAISE-331
**Size:** S

## User Story

**As a** developer using RaiSE with an orchestrator agent,
**I want** my delegation preferences (review/notify/auto) stored in my developer profile,
**So that** the orchestrator respects my HITL preferences without asking each time.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Profile loads delegation defaults from ShuHaRi level
  Given a developer profile with experience_level "ha"
  When the delegation section is loaded
  Then default_level is "notify"
  And per-skill overrides are empty

Scenario: Profile loads with explicit delegation overrides
  Given a developer profile with delegation.overrides for "rai-story-design"
  When delegation is resolved for "rai-story-design"
  Then the override value is returned instead of the ShuHaRi default

Scenario: Profile without delegation section uses defaults
  Given a developer profile with no delegation section
  When delegation is resolved
  Then ShuHaRi-derived defaults are used
  And no error occurs (backward compatible)
```

## SbE Examples

```yaml
# Ha-level developer, no overrides
experience_level: ha
delegation:
  default_level: notify

# Ri-level developer with one override
experience_level: ri
delegation:
  default_level: auto
  overrides:
    rai-story-design: review
```
