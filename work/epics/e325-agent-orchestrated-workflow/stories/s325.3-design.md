# Design: S325.3 — CLI Output as ACI

**Story:** S325.3
**Epic:** E325 Agent-Orchestrated Workflow
**Size:** S
**Complexity:** Moderate
**Jira:** RAISE-332

---

## Problem

`rai backlog` and `rai graph` commands produce output optimized for humans (rich tables, markdown headers, padding, decorative markup) that wastes context tokens when consumed by an orchestrator agent. The agent needs to parse information, not read it.

## Value

The orchestrator agent (S325.6) can consume CLI results with maximum semantic density — every token carries information, not decoration. This reduces context cost and improves parsing accuracy in the skill→CLI→skill chain.

## Approach

Add `--format agent` to 5 query subcommands, following the pipe-delimited pattern. Design principle: **semantic density, not arbitrary compression** — each token is an identifier, state, or semantic content. Nothing else.

### Components Affected

| File | Change | Detail |
|------|--------|--------|
| `src/rai_cli/cli/commands/backlog.py` | modify | Add `--format` option to `search` and `create` |
| `src/rai_cli/cli/commands/graph.py` | modify | Add `agent` format value to `query`, `context`, `list` (already have `--format`) |

### Design Decisions

| Decision | Rationale |
|----------|-----------|
| No headers in agent output | Schema lives in I/O contracts (S325.1), headers are redundant tokens |
| Content complete, no truncation | Agent decides what it needs, artificial truncation loses semantics |
| `graph list` returns type summary, not full dump | 1633 concepts would overflow context; agent queries specific types via `graph query` |
| `graph context` constraints as IDs grouped by severity | IDs have semantic naming patterns; avoids second round-trip vs count-only |
| `agent` as new format value, don't rename `compact` | Backward compat; "agent" communicates intent better |
| Pipe delimiter | Unambiguous, rare in content, easy to split |
| Errors as `error|message` | Consistent pattern, parseable |

### Cross-cutting Conventions

| Aspect | Decision |
|--------|----------|
| Delimiter | `\|` (pipe) |
| No results | Empty stdout or `empty` |
| Errors | `error\|message` |
| Truncation | None (complete content) |
| Headers | None (schema in I/O contract) |
| Encoding | UTF-8, no rich markup |

## Acceptance Criteria

### MUST

1. `--format agent` produces pipe-delimited output without markup in: `backlog search`, `backlog create`, `graph query`, `graph context`, `graph list`
2. `--format human` (or no flag) produces output identical to current behavior — zero regression
3. Complete content without truncation in agent formats (backlog search summary, graph query content)
4. `graph list --format agent` produces type summary (count), not full listing
5. Errors in agent format follow `error|message` pattern

### SHOULD

1. `graph query --format compact` continues working (backward compat)
2. Tests cover both formats for each modified subcommand

### MUST NOT

1. Do not change output of mutation commands (transition, update, link, comment, batch-transition)
2. Do not add headers/metadata to agent format (schema lives in I/O contract)
3. Do not use rich markup (`[bold]`, `**`, `#`) in agent output
