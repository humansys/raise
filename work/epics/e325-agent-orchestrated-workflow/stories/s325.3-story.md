# Story: CLI Output as ACI

## User Story
As an orchestrator agent,
I want `rai backlog` and `rai graph` commands to offer `--format agent` output,
so that I can parse structured, compact responses without wasting context tokens on human-oriented formatting.

## Acceptance Criteria

### Scenario: Backlog search returns agent-optimized output
```gherkin
Given the rai backlog search command
When I pass --format agent
Then output is compact structured text (key, summary, status per line)
And no decorative formatting (boxes, colors, padding) is included
```

### Scenario: Graph query returns agent-optimized output
```gherkin
Given the rai graph query command
When I pass --format agent
Then output is compact structured text (id, type, description per line)
And token count is ≤50% of default human output for same data
```

### Scenario: Default output unchanged
```gherkin
Given any rai backlog or rai graph command
When I do NOT pass --format
Then output is identical to current behavior (human-friendly)
```

### Scenario: All relevant subcommands support --format
```gherkin
Given the subcommands: backlog search, backlog create, graph query, graph context, graph list
When I pass --format agent to each
Then each produces agent-optimized output without error
```

## Examples (Specification by Example)

| Command | Human Output (current) | Agent Output (new) |
|---------|----------------------|-------------------|
| `rai backlog search "status=Done" --limit 3` | Rich table with borders | `RAISE-301 \| Done \| Title...\nRAISE-302 \| Done \| Title...` |
| `rai graph query "authentication"` | Formatted cards with headers | `mod-auth \| module \| Handles JWT auth...` |
| `rai graph context mod-memory` | Multi-section formatted output | Compact YAML-like key:value pairs |

## Notes
- ACI = Agent-Computer Interface — optimize for parseability, not readability
- `--format` option: `human` (default, current behavior), `agent` (new compact format)
- Backlog commands to cover: search, create (return created key)
- Graph commands to cover: query, context, list
- Do NOT change default behavior — backward compatible
