# S325.7 — Epic Orchestrator Skill: Scope

## In Scope

- Create `/rai-epic-run` meta-skill (SKILL.md) that chains: start → design → plan → [story iterations] → close
- Git-derived phase detection — infer epic phase from artifacts (branch, scope.md, plan section, story status)
- Story iteration — read story list from scope.md plan, execute each via `/rai-story-run` in order
- Delegation profile integration — gates at post-design and post-plan
- Graceful degradation when backlog CLI unavailable

## Out of Scope

- Python code changes (SKILL.md-only, consistent with S325.5 and S325.6)
- New CLI commands
- Parallel story execution (sequential only, single developer)
- Cross-epic dependency management
- Automatic story creation from design (stories defined in design phase, not auto-generated)

## Done Criteria

1. `/rai-epic-run {epic_id}` chains the full epic lifecycle in a single invocation
2. Phase detection from git artifacts resumes at correct epic phase
3. Story iteration reads plan from scope.md and executes via `/rai-story-run`
4. Delegation profile controls pause at post-design and post-plan
5. Individual epic skills unchanged (backward compatible)
6. SKILL.md validates with `rai skill validate`
7. Deployed to both `src/rai_cli/skills_base/` and `.claude/skills/`
