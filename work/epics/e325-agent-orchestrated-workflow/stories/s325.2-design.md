# Design: S325.2 — Delegation Profile Model

**Epic:** E325 Agent-Orchestrated Workflow
**Jira:** RAISE-331
**Size:** S
**Date:** 2026-03-01
**Complexity:** Simple (1 file, 1 enum, 1 model, 1 function)

---

## What & Why

**Problem:** The orchestrator (S325.6) needs to know when to pause for HITL and when to continue autonomously. Today there is no place to express that preference.

**Value:** Each developer declares their delegation level once; the orchestrator respects it every cycle without asking.

## Approach

Extend `DeveloperProfile` in `src/rai_cli/onboarding/profile.py`:

- `DelegationLevel` enum: `review` | `notify` | `auto`
- `DelegationConfig` model: `default_level` + `overrides: dict[str, DelegationLevel]`
- Field `delegation: DelegationConfig | None` on `DeveloperProfile` (None = backward compat)
- Function `resolve_delegation(profile, skill_name)` — override > explicit default > ShuHaRi derivation

**Architecture decision (Q3, SES-305):** Minimal first — `delegation.default_level` in DeveloperProfile, derive per-skill from ShuHaRi. Per-skill overrides only if needed.

## ShuHaRi Derivation

| Level | Default Delegation | Rationale |
|-------|-------------------|-----------|
| shu | review | Beginner reviews everything |
| ha | notify | Intermediate gets notified, can intervene |
| ri | auto | Expert trusts the process, only gates stop |

## Examples

```python
# Ha developer, no delegation section (backward compat)
profile = DeveloperProfile(name="Dev", experience_level="ha")
resolve_delegation(profile, "rai-story-design")  # → DelegationLevel.NOTIFY

# Ri developer with override
profile = DeveloperProfile(
    name="Dev",
    experience_level="ri",
    delegation=DelegationConfig(
        default_level=DelegationLevel.AUTO,
        overrides={"rai-story-design": DelegationLevel.REVIEW},
    ),
)
resolve_delegation(profile, "rai-story-design")  # → DelegationLevel.REVIEW
resolve_delegation(profile, "rai-story-plan")     # → DelegationLevel.AUTO
```

## Acceptance Criteria

**MUST:**
1. `DelegationLevel` enum with 3 values (review/notify/auto)
2. `DelegationConfig` Pydantic model validates correctly
3. `resolve_delegation()` respects: override > default_level > ShuHaRi
4. Profiles without `delegation` field load without error
5. All gates pass (pytest, pyright, ruff)

**MUST NOT:**
- Break existing profile loading/saving
- Add CLI commands (out of scope)
