# S325.6 — Story Orchestrator Skill: Scope

## In Scope

- Create `/rai-story-run` meta-skill (SKILL.md) that chains: start → design → plan → implement → review → close
- Git-derived phase detection — infer current phase from artifact presence, resume from there
- Delegation profile integration — resolve DelegationLevel per skill, control pause behavior
- Graceful degradation when backlog CLI unavailable
- Backward compatibility — individual skills still work independently

## Out of Scope

- Python code changes (this is a SKILL.md-only story, like S325.5)
- New CLI commands or flags
- State file persistence (Q5 decision: git-derived state only)
- Epic orchestrator (S325.7)
- Delegation auto-tuning
- Multi-agent orchestration

## Done Criteria

1. `/rai-story-run {story_id}` chains the full story lifecycle in a single invocation
2. Orchestrator detects current phase from git artifacts and resumes correctly
3. Delegation profile (from developer.yaml) controls pause/notify/auto at each skill transition
4. Individual skill invocation unchanged (backward compatible)
5. Backlog sync failures logged but don't block workflow
6. SKILL.md validates with `rai skill validate`
7. Deployed to both `src/rai_cli/skills_base/` and `.claude/skills/`

## Dependencies

| Dependency | Status | What It Provides |
|-----------|--------|-----------------|
| S325.1 — I/O Contracts | Done | raise.inputs/outputs metadata on all lifecycle skills |
| S325.2 — Delegation Profile | Done | DelegationLevel enum, DelegationConfig, resolve_delegation() |
| S325.4 — Backlog-Aware Skills | Done | rai signal emit-work + BacklogHook for Jira sync |

## Size

M — single SKILL.md but complex orchestration logic, phase detection, delegation integration.
