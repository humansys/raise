# S301.4: `rai docs` CLI Group

## User Story

**As** an AI agent working on a RaiSE project,
**I want** to publish, retrieve, and search documentation via `rai docs` commands,
**So that** I don't need to know Confluence page IDs, API details, or MCP tool names.

## Acceptance Criteria

```gherkin
Scenario: Publish a governance document
  Given a docs target adapter is registered
  When I run `rai docs publish governance/roadmap.md`
  Then the document is published via the adapter
  And compact output shows the result URL

Scenario: Get a page by identifier
  Given a docs target adapter is registered
  When I run `rai docs get <page-id>`
  Then the page content is returned in markdown format

Scenario: Search documentation
  Given a docs target adapter is registered
  When I run `rai docs search "architecture decisions"`
  Then matching pages are listed with title, space, and URL

Scenario: No adapter installed
  Given no docs target is registered
  When I run any `rai docs` command
  Then an error message explains how to install one

Scenario: Multiple adapters installed
  Given two docs target adapters are registered
  When I run `rai docs get <id>` without --target flag
  Then an error lists available adapters and requests --target flag
```
