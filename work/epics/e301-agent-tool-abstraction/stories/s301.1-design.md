# S301.1: Protocols + Models — Design

> **Size:** M (upgraded from S after architecture review)
> **Revised:** 2026-02-27 (post architecture review)

## Problem

The existing adapter protocols cover only basic CRUD (4 PM methods, 2 Docs methods).
Real daily operations — searching issues, commenting, batch transitions, reading pages —
have no typed contract, blocking all downstream stories (S301.2–S301.7).

## Value

Unblocks both epic streams. Every signature decided here propagates to 6 stories.
Getting it right now avoids compounding rework.

## Approach

Extend the two existing protocols with async as primary interface, add a generic sync
wrapper for CLI consumption, and add new Pydantic models to the existing models file.
The sync `ProjectManagementAdapter` and `DocumentationTarget` protocols remain as the
sync facade — their signatures are updated but they stay sync. New async protocols are
added alongside.

### Components Affected

| Component | Change | Details |
|-----------|--------|---------|
| `src/rai_cli/adapters/protocols.py` | Modify | Update sync PM (4→11 methods), sync Docs (2→5 methods). Add async PM (11 methods), async Docs (5 methods). |
| `src/rai_cli/adapters/models.py` | Modify | Add 9 new models (8 + FailureDetail) |
| `src/rai_cli/adapters/sync.py` | Create | Generic `SyncPMAdapter` and `SyncDocsAdapter` wrappers |
| `src/rai_cli/adapters/__init__.py` | Modify | Re-export new models and protocols in `__all__` |
| `tests/adapters/test_protocols.py` | Modify | Update stubs, add async protocol tests, add new model tests |

## Design Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| 1 | Extension strategy | Add methods inline | Single protocol per concern, no mixin complexity |
| 2 | Existing signatures | Break now (`ref: IssueRef` → `key: str`) | No concrete implementations exist. Cleaner caller DX |
| 3 | IssueDetail inheritance | Extends IssueRef | Liskov substitution — works anywhere IssueRef expected |
| 4 | BatchResult.failed | Structured `FailureDetail` model | Programmatic access without string parsing |
| 5 | update_issue fields | Keep `dict[str, Any]` | Platform-agnostic, adapter validates |
| 6 | Sync/async strategy | Async primary + sync wrapper | One adapter impl, two consumption modes. httpx.AsyncClient natural fit |
| 7 | transition_issue param | `status: str` (not `transition`) | Caller says WHERE, adapter resolves HOW. Platform-agnostic |
| 8 | get_page identifier | `identifier: str`, adapter resolves | Adapter interprets as ID, title, or URL. Protocol stays simple |
| 9 | search query | Passthrough `query: str` | Platform-specific query language (JQL, CQL). YAGNI on mini-DSL |
| 10 | health() symmetry | Both PM and Docs have `health()` | Both talk to remote APIs, both need connectivity checks |
| 11 | Timestamp format | `str` with ISO 8601 convention | Adapters MUST normalize. Empty string = not available |
| 12 | `__init__.py` exports | Update in S301.1 | New models/protocols must be accessible from public API |
| 13 | Adapter instantiation | Factory function in CLI command | Registry discovers types, CLI owns config + wiring (S301.2+) |

## Protocol Contracts (Final)

### Sync Protocols (CLI consumption)

These are the sync facades. The CLI and tests consume these directly.

#### ProjectManagementAdapter (11 methods)

```python
@runtime_checkable
class ProjectManagementAdapter(Protocol):
    """Sync PM adapter contract. CLI commands consume this.

    For concrete adapters, implement AsyncProjectManagementAdapter and wrap
    with SyncPMAdapter for CLI consumption.
    """

    # CRUD (4, signatures updated from original)
    def create_issue(self, project_key: str, issue: IssueSpec) -> IssueRef: ...
    def get_issue(self, key: str) -> IssueDetail: ...
    def update_issue(self, key: str, fields: dict[str, Any]) -> IssueRef: ...
    def transition_issue(self, key: str, status: str) -> IssueRef: ...

    # Batch (1)
    def batch_transition(self, keys: list[str], status: str) -> BatchResult: ...

    # Relationships (2)
    def link_to_parent(self, child_key: str, parent_key: str) -> None: ...
    def link_issues(self, source: str, target: str, link_type: str) -> None: ...

    # Comments (2)
    def add_comment(self, key: str, body: str) -> CommentRef: ...
    def get_comments(self, key: str, limit: int = 10) -> list[Comment]: ...

    # Query (1) — query is adapter-specific (JQL for Jira, etc.)
    def search(self, query: str, limit: int = 50) -> list[IssueSummary]: ...

    # Health (1)

```

### Sync Wrappers (`adapters/sync.py`)

Generic wrappers that bridge async adapters to sync protocol.

```python
import asyncio

class SyncPMAdapter:
    """Wraps AsyncProjectManagementAdapter for sync CLI consumption.

    Satisfies ProjectManagementAdapter protocol.
    """

    def __init__(self, async_adapter: AsyncProjectManagementAdapter) -> None:
        self._adapter = async_adapter

    def create_issue(self, project_key: str, issue: IssueSpec) -> IssueRef:
        return asyncio.run(self._adapter.create_issue(project_key, issue))

    def get_issue(self, key: str) -> IssueDetail:
        return asyncio.run(self._adapter.get_issue(key))

    # ... same pattern for all 11 methods


class SyncDocsAdapter:
    """Wraps AsyncDocumentationTarget for sync CLI consumption.

    Satisfies DocumentationTarget protocol.
    """

    def __init__(self, async_target: AsyncDocumentationTarget) -> None:
        self._target = async_target

    def can_publish(self, doc_type: str, metadata: dict[str, Any]) -> bool:
        return asyncio.run(self._target.can_publish(doc_type, metadata))

    # ... same pattern for all 5 methods
```

## Models (Final)

### Timestamp Convention

All timestamp fields use ISO 8601 format (e.g. `"2026-02-27T10:30:00Z"`).
Adapters MUST normalize to this format. Empty string means not available.

### New models (9)

```python
class IssueDetail(IssueRef):
    """Full issue details — extends IssueRef (inherits key, url, metadata).

    Timestamps use ISO 8601 format. Empty string = not available.
    """
    summary: str
    description: str = ""
    status: str
    issue_type: str
    parent_key: str | None = None
    labels: list[str] = Field(default_factory=list)
    assignee: str | None = None
    priority: str | None = None
    created: str = ""   # ISO 8601 or empty
    updated: str = ""   # ISO 8601 or empty

class IssueSummary(BaseModel):
    """Compact issue for search results and listings."""
    key: str
    summary: str
    status: str
    issue_type: str
    parent_key: str | None = None

class Comment(BaseModel):
    """Issue comment. Timestamps use ISO 8601 format."""
    id: str
    body: str
    author: str
    created: str  # ISO 8601 (required)

class CommentRef(BaseModel):
    """Reference to a created comment."""
    id: str
    url: str = ""

class FailureDetail(BaseModel):
    """A single failure in a batch operation."""
    key: str
    error: str

class BatchResult(BaseModel):
    """Result of a batch operation."""
    succeeded: list[IssueRef] = Field(default_factory=list)
    failed: list[FailureDetail] = Field(default_factory=list)

class PageContent(BaseModel):
    """Full page content from documentation target."""
    id: str
    title: str
    content: str  # markdown
    url: str = ""
    space_key: str = ""
    version: int = 1

class PageSummary(BaseModel):
    """Compact page for search results. Timestamps use ISO 8601 format."""
    id: str
    title: str
    url: str = ""
    space_key: str = ""
    updated: str = ""  # ISO 8601 or empty

class AdapterHealth(BaseModel):
    """Health check result for an adapter."""
    name: str
    healthy: bool
    message: str = ""
    latency_ms: int | None = None
```

## Examples

### Model instantiation

```python
# IssueDetail extends IssueRef — Liskov holds
detail = IssueDetail(
    key="RAISE-301", summary="Agent Tool Abstraction",
    status="In Progress", issue_type="Epic",
    labels=["epic", "e301"], assignee="emilio",
)
ref: IssueRef = detail  # ✓ valid substitution
assert detail.url == ""  # inherited default

# BatchResult with structured failures
result = BatchResult(
    succeeded=[IssueRef(key="RAISE-1"), IssueRef(key="RAISE-2")],
    failed=[FailureDetail(key="RAISE-3", error="not found")],
)
assert len(result.succeeded) == 2
assert result.failed[0].key == "RAISE-3"

# AdapterHealth
health = AdapterHealth(name="jira", healthy=True, latency_ms=42)
assert health.message == ""  # default
```

## Acceptance Criteria

### MUST

1. `ProjectManagementAdapter` (sync) has exactly 11 methods with signatures matching design
2. `AsyncProjectManagementAdapter` has exactly 11 async methods mirroring sync signatures
3. `DocumentationTarget` (sync) has exactly 5 methods with signatures matching design
4. `AsyncDocumentationTarget` has exactly 5 async methods mirroring sync signatures
5. `SyncPMAdapter` wraps any `AsyncProjectManagementAdapter` and satisfies `ProjectManagementAdapter`
6. `SyncDocsAdapter` wraps any `AsyncDocumentationTarget` and satisfies `DocumentationTarget`
7. All 9 new Pydantic models instantiate, validate, and serialize correctly
8. `IssueDetail` substitutes for `IssueRef` (Liskov)
9. `@runtime_checkable` preserved on all 4 protocols — isinstance() works with mock classes
10. Existing models (`IssueRef`, `IssueSpec`, `PublishResult`, `ArtifactLocator`) unchanged
11. `__init__.py` re-exports all new models and protocols in `__all__`
12. Tests + types (pyright) + lint (ruff) pass

### SHOULD

1. Docstrings on each new model class with timestamp convention
2. Docstrings on protocol classes explaining sync/async relationship
3. Docstrings on protocol method groups (# CRUD, # Batch, etc.)

### MUST NOT

1. Must not add concrete adapter implementations — protocols and wrappers only
2. Must not change `GovernanceSchemaProvider` or `GovernanceParser` protocols
3. Must not change existing model classes (`IssueRef`, `IssueSpec`, `PublishResult`, `ArtifactLocator`, `CoreArtifactType`)
