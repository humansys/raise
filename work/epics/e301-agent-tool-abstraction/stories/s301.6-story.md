# S301.6: Skill Auto-Sync Hooks

## User Story

**As** a developer using RaiSE lifecycle skills,
**I want** Jira issues to transition automatically when I run `/rai-story-start` or `/rai-story-close`,
**so that** Jira stays in sync without manual `rai backlog transition` calls.

## Acceptance Criteria

```gherkin
Scenario: Story start triggers In Progress transition
  Given a JiraSyncHook is registered via entry points
  And lifecycle_mapping in .raise/jira.yaml maps story_start → 31
  When a WorkStartEvent with work_type="story" and issue_key="RAISE-301" is emitted
  Then the hook calls adapter.transition_issue("RAISE-301", "in-progress")
  And returns HookResult(status="ok")

Scenario: Story close triggers Done transition
  Given a JiraSyncHook is registered
  And lifecycle_mapping maps story_close → 41
  When a WorkCloseEvent with work_type="story" and issue_key="RAISE-301" is emitted
  Then the hook calls adapter.transition_issue("RAISE-301", "done")
  And returns HookResult(status="ok")

Scenario: Epic lifecycle events also trigger transitions
  Given lifecycle_mapping maps epic_start → 31 and epic_close → 41
  When work events with work_type="epic" are emitted
  Then the corresponding Jira transitions fire

Scenario: Graceful no-op when unconfigured
  Given no lifecycle_mapping exists in jira.yaml
  When a work event is emitted
  Then the hook returns HookResult(status="ok") with no Jira call
  And a debug log message explains "no lifecycle_mapping configured"

Scenario: Graceful degradation on adapter failure
  Given the Jira adapter is unavailable (no mcp-atlassian installed)
  When a work event is emitted
  Then the hook returns HookResult(status="error", message="...")
  And the CLI does NOT crash (error isolation)
```
