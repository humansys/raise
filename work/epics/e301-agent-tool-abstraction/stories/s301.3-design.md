# S301.3 Design: McpBridge + McpJiraAdapter

> **Complexity:** Moderate (4 components, custom logic, 1 external integration)
> **Risk gate:** Low — protocols and sync wrapper are proven infrastructure
> **UX gate:** N/A — no new human interaction patterns
> **Integration gate:** AC includes manual smoke test against real Jira

## Problem & Value

**Problem:** `rai backlog` commands exist but have no PM adapter registered —
`resolve_adapter(None)` exits with "No PM adapter installed." There's no bridge
to execute MCP tools from Python, and no Jira-specific adapter to map protocol
methods to `mcp-atlassian` tool calls.

**Value:** After this story, `rai backlog transition RAISE-301 done` works
end-to-end against real Jira. The generic McpBridge enables any future MCP
server integration (Confluence, GitHub, Grafana) with zero bridge code changes.

## Open Questions Resolution

### OQ1: `mcp` SDK as dependency or optional?

**Decision: Optional `[mcp]` group.** Base install stays light. Registry's
try/except in `_discover()` already handles missing imports gracefully (AR-4).
McpJiraAdapter import fails → entry point skipped → `resolve_adapter()` shows
"No PM adapter installed. Install: `pip install rai-cli[mcp]`".

### OQ2: How to handle `mcp-atlassian` env vars?

**Decision: Env vars only (MVP).** McpBridge passes `os.environ` to the
subprocess by default. The adapter reads `JIRA_URL`, `JIRA_USERNAME`,
`JIRA_API_TOKEN` from env for validation/error messages. No `.env` file
loading — users set env vars or use their shell's `.env` sourcing.

Rationale: `.env` loading adds a dependency (python-dotenv) or custom code
for a convenience feature. Users already source `.env` in their shells.
YAGNI for MVP.

### OQ3: Batch operations via bridge?

**Decision: Sequential calls in single async context.** `batch_transition()`
is one async method → `_run_sync()` creates one event loop → one MCP session
for N calls. No concurrency within session (Phase 2). This is exactly what
the epic design described.

### OQ4: Test strategy for McpBridge?

**Decision: Mock at `ClientSession` level for unit tests.** We mock
`stdio_client` as async context manager returning mock read/write streams,
and mock `ClientSession.call_tool()` / `list_tools()` responses. No FastMCP
in-process server needed for this story — integration tests use real
`mcp-atlassian` in manual smoke test.

### Key design choices

**McpBridge owns session lifecycle, not the adapter.**
The bridge manages `stdio_client` + `ClientSession` as nested async context
managers. The adapter just calls `self._bridge.call(tool, args)`.

**Bridge is a concrete class, not a protocol.**
Only one bridge implementation exists (MCP Python SDK stdio). No need for
abstraction. If HTTP transport is needed later, extend the class or add a
factory method.

**Adapter reads config once in `__init__`, validates eagerly.**
`McpJiraAdapter.__init__()` reads `jira.yaml` and validates `status_mapping`
exists. Fails fast with clear error if config is missing. Does NOT connect
to MCP server in `__init__` — connection is lazy on first `call()`.

**Auto-wrap in resolver (AR-1).**
`resolve_adapter()` detects `AsyncProjectManagementAdapter` instances and
wraps with `SyncPMAdapter`. This is ~6 lines in `_adapter_resolve.py`.

### McpBridge internals — session management

```python
class McpBridge:
    def __init__(self, server_command: str, server_args: list[str] | None = None,
                 env: dict[str, str] | None = None) -> None:
        self._server_command = server_command
        self._server_args = server_args or []
        self._env = env  # None = inherit os.environ
        self._session: ClientSession | None = None
        self._cm_stack: AsyncExitStack | None = None

    async def _ensure_session(self) -> ClientSession:
        """Lazy connect. Reconnects if session is closed/dead."""
        if self._session is not None:
            try:
                # Verify session is alive — list_tools is cheap
                await self._session.list_tools()
                return self._session
            except Exception:
                await self._cleanup()

        # Fresh connection
        params = StdioServerParameters(
            command=self._server_command,
            args=self._server_args,
            env=self._env,
        )
        stack = AsyncExitStack()
        try:
            read, write = await stack.enter_async_context(stdio_client(params))
            session = await stack.enter_async_context(ClientSession(read, write))
            await session.initialize()
        except FileNotFoundError:
            await stack.aclose()
            raise McpBridgeError(
                f"MCP server '{self._server_command}' not found. "
                f"Install the server and ensure it's on PATH."
            )
        except Exception as exc:
            await stack.aclose()
            raise McpBridgeError(f"Failed to connect to MCP server '{self._server_command}': {exc}")

        self._session = session
        self._cm_stack = stack
        return session

    async def call(self, tool_name: str, arguments: dict[str, Any]) -> McpToolResult:
        with logfire.span("mcp.tool_call", mcp_server=self._server_command,
                          mcp_tool=tool_name) as span:
            start = time.monotonic()
            session = await self._ensure_session()
            try:
                result = await session.call_tool(tool_name, arguments)
                elapsed_ms = int((time.monotonic() - start) * 1000)
                span.set_attribute("duration_ms", elapsed_ms)
                span.set_attribute("success", not result.isError)
                return self._parse_result(result)
            except Exception as exc:
                elapsed_ms = int((time.monotonic() - start) * 1000)
                span.set_attribute("duration_ms", elapsed_ms)
                span.set_attribute("success", False)
                raise McpBridgeError(f"Tool call '{tool_name}' failed: {exc}") from exc

    async def _cleanup(self) -> None:
        if self._cm_stack:
            await self._cm_stack.aclose()
        self._session = None
        self._cm_stack = None
```

### McpBridge._parse_result (AR-S3-1)

`mcp-atlassian` returns JSON as `TextContent.text`, not `structuredContent`.
`CallToolResult.content` can be empty, multi-item, or non-TextContent.
The parser must handle all cases defensively.

```python
from mcp import types as mcp_types

def _parse_result(self, result: CallToolResult) -> McpToolResult:
    """Parse CallToolResult into McpToolResult.

    Handles: empty content, multiple text items, non-text content (ignored),
    isError flag, and JSON auto-parsing.
    """
    if result.isError:
        error_text = ""
        if result.content:
            texts = [c.text for c in result.content if isinstance(c, mcp_types.TextContent)]
            error_text = "\n".join(texts) if texts else "Unknown error"
        return McpToolResult(is_error=True, error_message=error_text or "Unknown error")

    # Collect all text content, ignore non-text (images, embedded resources)
    texts = [c.text for c in result.content if isinstance(c, mcp_types.TextContent)]
    text = "\n".join(texts) if texts else ""

    # Try parse as JSON for structured access
    data: dict[str, Any] = {}
    if text:
        try:
            parsed = json.loads(text)
            if isinstance(parsed, dict):
                data = parsed
        except json.JSONDecodeError:
            pass  # text stays as-is, data stays empty

    return McpToolResult(text=text, data=data)
```

### Data models

```python
class McpToolResult(BaseModel):
    """Parsed result from an MCP tool call."""
    text: str = ""
    data: dict[str, Any] = Field(default_factory=dict)
    is_error: bool = False
    error_message: str = ""

class McpToolInfo(BaseModel):
    """Tool metadata from server discovery."""
    name: str
    description: str = ""

class McpBridgeError(Exception):
    """Raised when MCP bridge operations fail."""
    pass
```

## Acceptance Criteria

### MUST

1. `resolve_adapter(None)` returns a working `ProjectManagementAdapter` (sync) when `rai-cli[mcp]` is installed and `mcp-atlassian` is on PATH
2. `McpBridge.call()` connects to MCP server via stdio, calls tools, and returns parsed `McpToolResult`
3. All 11 `AsyncProjectManagementAdapter` methods implemented in `McpJiraAdapter`, mapping to correct `mcp-atlassian` tools
4. Status name → transition ID resolution via `jira.yaml` `status_mapping` with clear error for unknown status
5. logfire span `mcp.tool_call` emitted on every bridge call (no-op without full SDK)

### SHOULD

1. `McpBridge.health()` returns `AdapterHealth` with tool count and latency
2. Clear error message when `mcp-atlassian` is not installed (install guidance)
3. `_ensure_session()` reconnects on dead/closed session

### MUST NOT

1. Import `mcp` at module top level in adapter files — use lazy import or let entry point loading handle it
2. Add `mcp` to base dependencies — must stay in optional `[mcp]` group
3. Modify existing protocol, model, or sync wrapper files

## Architecture Review Findings

| ID | Finding | Severity | Resolution |
|----|---------|----------|------------|
| AR-S3-1 | `_parse_result` frágil — content can be empty, multi-item, or non-TextContent | High | **Resolved in design.** Added explicit `_parse_result` method that handles all `CallToolResult` shapes defensively. |
| AR-S3-2 | `link_to_parent` → `jira_link_to_epic` only works for epics, not subtasks | Medium | **Resolved in design.** Changed mapping to `jira_update_issue` with `{"parent": key}` — works for all parent-child relationships. |
| AR-S3-3 | `StdioServerParameters.env=None` — verify inherits parent env | Medium | **Verify during implementation.** If `None` doesn't inherit, pass `env=dict(os.environ)` explicitly. Add test. |
| AR-S3-4 | `_ensure_session` probe with `list_tools()` adds roundtrip in async context | Low | **Accept for MVP.** In CLI sync the probe never executes (session always new). Optimize if measured in async context. |
| AR-S3-5 | `get_comments` doesn't propagate `limit` to `comment_limit` | Low | **Resolve during implementation.** Pass protocol `limit` as `comment_limit` param to `jira_get_issue`. |
| AR-S3-6 | Zombie processes if `AsyncExitStack.aclose()` doesn't terminate subprocess | Low | **Trust SDK.** `stdio_client` CM terminates subprocess. SHOULD-level test for no orphan processes. |
