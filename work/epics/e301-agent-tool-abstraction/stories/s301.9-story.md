# S301.9: FilesystemPMAdapter

## User Story

**As a** developer using RaiSE without Jira (open-core),
**I want** `rai backlog` commands to read/write `governance/backlog.md`,
**So that** I get PM functionality out of the box without configuring a paid adapter.

## Acceptance Criteria

```gherkin
Scenario: Search returns epics from backlog.md
  Given no PM adapter is configured (no Jira env vars)
  And governance/backlog.md exists with epics
  When I run `rai backlog search "status = pending"`
  Then I get IssueSummary list parsed from the backlog table

Scenario: Get issue returns detail from backlog.md
  Given governance/backlog.md contains epic E301
  When I run `rai backlog get E301`
  Then I get IssueDetail with summary, status, and metadata

Scenario: Health reports filesystem adapter
  Given FilesystemPMAdapter is the resolved adapter
  When I call health()
  Then status is "healthy" and adapter_name is "filesystem"

Scenario: Graceful fallback when no backlog exists
  Given governance/backlog.md does not exist
  When I run `rai backlog search "any"`
  Then I get an empty list (not an error)
```
