# S301.7: E2E Dogfood — Plan

- **Story:** S301.7
- **Size:** S (3 tasks)
- **Date:** 2026-03-01
- **Type:** Validation (not code — E2E commands are the tests)

## Tasks

### T1: Jira Lifecycle E2E [M]

**Goal:** Execute full Jira lifecycle via `rai backlog` CLI against live Jira Cloud.

**Steps:**
1. `rai backlog create` — create dogfood task in RAISE project
2. `rai backlog search` — verify issue exists
3. `rai backlog transition <key> in-progress` — move to In Progress
4. `rai backlog comment <key> "..."` — add comment
5. `rai backlog transition <key> done` — close issue
6. `rai backlog search` — verify final state

**Verification:** All 6 commands succeed without errors. Issue visible in Jira with correct state.

**Bug protocol:** If a command fails, diagnose, fix, add test, commit, then continue.

**Depends on:** nothing

---

### T2: Confluence Docs E2E [S]

**Goal:** Execute publish/search/get via `rai docs` CLI against live Confluence.

**Steps:**
1. `rai docs publish roadmap --title "E301 Dogfood Test"` — publish page
2. `rai docs search "E301 Dogfood"` — find published page
3. `rai docs get <page-id>` — retrieve content

**Verification:** All 3 commands succeed. Page visible in Confluence.

**Cleanup:** Delete test page from Confluence after validation.

**Bug protocol:** Same as T1.

**Depends on:** nothing (parallel with T1)

---

### T3: Token Measurement + Friction Log [S]

**Goal:** Document token reduction and capture friction points.

**Steps:**
1. Measure CLI output size (chars/tokens) from T1 and T2 commands
2. Compare against raw MCP equivalent (reference: ~8K tokens per Jira search)
3. Calculate reduction ratio
4. Write friction log: gaps, rough edges, missing commands
5. Verify lifecycle hook registration (entry point check)
6. Clean up: transition dogfood issue to Done if not already, delete Confluence test page

**Verification:** Token measurement documented. Friction log captured. No orphaned test artifacts.

**Deliverable:** Append results to s301.7-retrospective.md (during `/rai-story-review`)

**Depends on:** T1, T2

## Risks

| Risk | Impact | Mitigation |
|------|--------|------------|

| Jira rate limiting | Delayed | Space commands, don't retry aggressively |
| Confluence space permissions | Blocked | Verify space key + write perms before publish |
| Bug discovered blocks further testing | Delayed | Fix inline, commit, continue |
