# S301.6: Skill Auto-Sync Hooks — Design

> **Complexity:** Moderate (3-4 components, custom mapping logic)
> **Date:** 2026-03-01 (SES-306)

## Problem & Value

**Problem:** Jira status drifts from reality because developers forget to run `rai backlog transition` after starting/closing work. The PM board becomes unreliable.

**Value:** Zero-friction Jira sync — `/rai-story-start` moves Jira to In Progress, `/rai-story-close` moves to Done, automatically. The board is always accurate without manual effort.

## Design Decisions

| # | Decision | Rationale |
|---|----------|-----------|
| D1 | New hook events (`WorkStartEvent`, `WorkCloseEvent`) in hook system | Clean separation: skills emit intent, hooks react. Not coupled to telemetry JSONL. |
| D2 | `resolve_adapter()` called in `handle()` each time | Stateless, lazy. If adapter unavailable → catch → error result. Aligns with CLI per-call pattern (PAT-E-561). |
| D3 | `issue_key` field on event (optional) | Explicit, no search/lookup needed. Hook skips if empty — graceful no-op. |
| D4 | Hook infrastructure only, no skill wiring | Keeps S301.6 small and testable. Wiring into skills → S301.7 (dogfood). |

## Approach

### New Events (events.py)

Two frozen dataclasses following existing pattern:

```python
@dataclass(frozen=True)
class WorkStartEvent(HookEvent):
    """Emitted when a work item (story/epic) starts."""
    event_name: Literal["work:start"] = field(
        default="work:start", init=False
    )
    work_type: str = ""     # "story" | "epic"
    work_id: str = ""       # "S301.6"
    issue_key: str = ""     # "RAISE-301" (optional, hook skips if empty)

@dataclass(frozen=True)
class WorkCloseEvent(HookEvent):
    """Emitted when a work item (story/epic) closes."""
    event_name: Literal["work:close"] = field(
        default="work:close", init=False
    )
    work_type: str = ""
    work_id: str = ""
    issue_key: str = ""
```

### JiraSyncHook (hooks/builtin/jira_sync.py)

```python
class JiraSyncHook:
    """Auto-sync lifecycle events to Jira transitions.

    Reads lifecycle_mapping from .raise/jira.yaml:
        lifecycle_mapping:
            story_start: 31   # → In Progress
            story_close: 41   # → Done
            epic_start: 31
            epic_close: 41

    Maps work:start + work_type=story → story_start → transition ID 31.
    """

    events: ClassVar[list[str]] = ["work:start", "work:close"]
    priority: ClassVar[int] = -10  # after telemetry (priority 0)

    def handle(self, event: HookEvent) -> HookResult:
        # 1. Type-narrow to work events
        if not isinstance(event, (WorkStartEvent, WorkCloseEvent)):
            return HookResult(status="ok")

        # 2. Skip if no issue key
        if not event.issue_key:
            logger.debug("JiraSyncHook: no issue_key, skipping")
            return HookResult(status="ok")

        # 3. Load lifecycle_mapping
        mapping = _load_lifecycle_mapping()
        if not mapping:
            logger.debug("JiraSyncHook: no lifecycle_mapping configured")
            return HookResult(status="ok")

        # 4. Resolve mapping key: "{work_type}_{action}"
        action = "start" if isinstance(event, WorkStartEvent) else "close"
        mapping_key = f"{event.work_type}_{action}"  # e.g. "story_start"
        transition_id = mapping.get(mapping_key)
        if transition_id is None:
            logger.debug("JiraSyncHook: no mapping for %s", mapping_key)
            return HookResult(status="ok")

        # 5. Resolve status name from transition ID (reverse lookup)
        status_name = _resolve_status_name(transition_id)

        # 6. Call adapter
        try:
            adapter = resolve_adapter(None)
            adapter.transition_issue(event.issue_key, status_name)
            logger.info(
                "JiraSyncHook: %s → %s (%s)",
                event.issue_key, status_name, mapping_key,
            )
            return HookResult(status="ok")
        except Exception as exc:
            msg = f"JiraSyncHook: {type(exc).__name__}: {exc}"
            logger.warning(msg)
            return HookResult(status="error", message=msg)
```

### Config Reader

```python
def _load_lifecycle_mapping() -> dict[str, int] | None:
    """Read lifecycle_mapping from .raise/jira.yaml."""
    path = Path(".raise/jira.yaml")
    if not path.exists():
        return None
    data = yaml.safe_load(path.read_text())
    return data.get("workflow", {}).get("lifecycle_mapping")

def _resolve_status_name(transition_id: int) -> str:
    """Reverse-lookup: transition ID → status name from status_mapping."""
    path = Path(".raise/jira.yaml")
    data = yaml.safe_load(path.read_text())
    status_mapping = data.get("workflow", {}).get("status_mapping", {})
    for name, tid in status_mapping.items():
        if tid == transition_id:
            return name
    return str(transition_id)  # fallback: pass ID as-is
```

## Examples

### Happy path: story start → Jira In Progress

```python
from rai_cli.hooks.events import WorkStartEvent
from rai_cli.hooks.emitter import create_emitter

emitter = create_emitter()  # discovers JiraSyncHook via entry points
event = WorkStartEvent(
    work_type="story",
    work_id="S301.6",
    issue_key="RAISE-301",
)
result = emitter.emit(event)
# → JiraSyncHook reads lifecycle_mapping: story_start → 31
# → resolve_adapter() → McpJiraAdapter
# → adapter.transition_issue("RAISE-301", "in-progress")
# → result.aborted == False, result.handler_errors == ()
```

### No-op: no issue key

```python
event = WorkStartEvent(work_type="story", work_id="S301.6", issue_key="")
result = emitter.emit(event)
# → JiraSyncHook skips (debug log: "no issue_key")
# → result.handler_errors == ()
```

### No-op: no lifecycle_mapping configured

```python
# .raise/jira.yaml has no lifecycle_mapping section
event = WorkStartEvent(work_type="story", work_id="S301.6", issue_key="RAISE-301")
result = emitter.emit(event)
# → JiraSyncHook skips (debug log: "no lifecycle_mapping configured")
# → result.handler_errors == ()
```

### Error: adapter unavailable

```python
# mcp-atlassian not installed
event = WorkStartEvent(work_type="story", work_id="S301.6", issue_key="RAISE-301")
result = emitter.emit(event)
# → JiraSyncHook catches McpBridgeError
# → HookResult(status="error", message="MCP server 'mcp-atlassian' not found...")
# → result.handler_errors has 0 entries (hook returned error, didn't raise)
# → CLI does NOT crash
```

## Acceptance Criteria

### MUST

1. `WorkStartEvent` and `WorkCloseEvent` are frozen dataclasses with `work_type`, `work_id`, `issue_key` fields
2. `JiraSyncHook` implements `LifecycleHook` Protocol (isinstance check passes)
3. Hook reads `lifecycle_mapping` from `.raise/jira.yaml` and calls `adapter.transition_issue()`
4. Graceful no-op when: no issue_key, no lifecycle_mapping, no mapping for event type
5. Graceful error when adapter fails (HookResult error, no crash, no raise)
6. Entry point `jira-sync` registered in pyproject.toml

### SHOULD

1. Priority -10 (runs after telemetry at priority 0)
2. Debug logging for all skip/no-op paths
3. Info logging for successful transitions

### MUST NOT

1. NOT wire events into existing skills (scope of S301.7)
2. NOT crash the CLI on any failure path
3. NOT make Jira calls when unconfigured
