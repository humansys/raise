# Story: rai backlog CLI group

## User Story
As a developer using rai,
I want a `rai backlog` CLI group with commands for create, transition, update, link, comment, search, and batch-transition,
so that I can manage project backlog items from the terminal with compact, token-efficient output instead of raw MCP responses.

## Acceptance Criteria

### Scenario: Transition a backlog item
```gherkin
Given a configured ProjectManagementAdapter (mock or real)
When I run `rai backlog transition RAISE-123 done`
Then the adapter's transition_issue method is called with correct arguments
And output is compact (<200 tokens) showing the result
```

### Scenario: Search backlog items
```gherkin
Given a configured ProjectManagementAdapter
When I run `rai backlog search "project=RAISE AND status=Open"`
Then the adapter's search_issues method is called with the JQL query
And output lists issues in compact format (key, summary, status per line)
```

### Scenario: Create a backlog item
```gherkin
Given a configured ProjectManagementAdapter
When I run `rai backlog create --type Story --parent RAISE-301 "New story title"`
Then the adapter's create_issue method is called with correct fields
And output shows the created issue key
```

### Scenario: Mock adapter substitutes real adapter
```gherkin
Given no Jira credentials configured
When any `rai backlog` command is invoked with --adapter mock
Then a StubPMAdapter handles the call without network access
And output reflects stub data
```

### Scenario: Invalid command arguments
```gherkin
Given a `rai backlog` command
When required arguments are missing or invalid
Then Typer shows a clear usage error with exit code != 0
And no adapter call is made
```

## Examples (Specification by Example)

| Command | Expected Output |
|---------|-----------------|
| `rai backlog transition RAISE-123 done` | `RAISE-123: transitioned → Done` |
| `rai backlog search "status=Open"` | `RAISE-100 Open  Fix auth bug\nRAISE-101 Open  Add tests` |
| `rai backlog create --type Task "Write docs"` | `Created: RAISE-999` |
| `rai backlog update RAISE-123 --priority high` | `RAISE-123: updated` |
| `rai backlog link RAISE-123 blocks RAISE-124` | `RAISE-123 → blocks → RAISE-124: linked` |
| `rai backlog comment RAISE-123 "Looks good"` | `RAISE-123: comment added` |
| `rai backlog batch-transition RAISE-1,RAISE-2 done` | `2/2 transitioned → Done` |

## Notes
- Factory pattern: adapter instantiation lives in the CLI command, not imported globally
- Compact output format is a key goal — avoid dumping raw API responses
- StubPMAdapter from S301.1 test stubs can be reused for CLI tests
- Commands delegate 100% to adapter — no business logic in CLI layer
- See epic design: `work/epics/e301-agent-tool-abstraction/design.md`
- See S301.1 adapters: `src/rai_cli/adapters/`
