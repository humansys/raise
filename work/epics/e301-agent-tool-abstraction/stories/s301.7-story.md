# S301.7: E2E Dogfood

## User Story

**As** a developer using RaiSE,
**I want** to complete a full story lifecycle using only `rai backlog` and `rai docs` CLI commands (no direct MCP calls),
**so that** the agent tool abstraction is validated end-to-end against real Jira and Confluence instances.

## Acceptance Criteria

```gherkin
Scenario: Create and track a story in Jira via CLI
  Given the McpJiraAdapter is configured in .raise/jira.yaml
  When I run `rai backlog create RAISE --summary "Dogfood test" --type Task`
  Then a Jira issue is created and the key is returned
  When I run `rai backlog transition <key> in-progress`
  Then the issue status changes to In Progress
  When I run `rai backlog comment <key> "Work complete"`
  Then the comment appears on the issue
  When I run `rai backlog transition <key> done`
  Then the issue status changes to Done

Scenario: Publish and retrieve a doc via CLI
  Given the McpConfluenceAdapter is configured
  When I run `rai docs publish` with a markdown file
  Then the page is created/updated in Confluence
  When I run `rai docs get` with the page reference
  Then the content is retrieved successfully

Scenario: Lifecycle hooks fire on story events
  Given JiraSyncHook is registered via entry points
  And lifecycle_mapping is configured in jira.yaml
  When a story lifecycle event fires
  Then the corresponding Jira transition executes automatically

Scenario: Token efficiency validated
  Given a complete story lifecycle executed via CLI
  When comparing token usage vs equivalent raw MCP calls
  Then CLI usage is at least 10x more token-efficient
```
