# S301.4: `rai docs` CLI Group — Design

## Complexity: Moderate

3 components (CLI group, resolver, config schema), mirrors S301.2 pattern, governance-scoped semantics.

## What & Why

**Problem:** Agents need to publish/retrieve/search governance documentation (ADRs, roadmap, constitution) on remote targets (Confluence), but currently must use raw MCP tools with platform-specific IDs and 8K+ tokens per operation.

**Value:** `rai docs` provides governed documentation operations at ~200 tokens/op. Unlike a generic Confluence MCP bridge, this is scoped to **governance artifacts that live in the knowledge graph** — the graph is the source of truth, remote targets are synchronized publication destinations. This increases reliability because the CLI knows the artifact types, their locations, and their publishing rules.

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Publish input | Artifact type from CoreArtifactType/schema | `rai docs publish roadmap` — resolves path from GovernanceSchema. Knows where artifacts live. |
| D2 | Config location | `.raise/docs.yaml` | Maps artifact types to space+parent. Parallel to jira.yaml for backlog. |
| D3 | `get` source | Remote (adapter) | Validates what's actually published. Local files are read directly. |
| D4 | Resolver flag | `--target` | Matches `DocumentationTarget` protocol name. Semantically clearer than --adapter. |
| D5 | Search scope | Remote target | Searches Confluence (or whatever target). `rai graph query` exists for local. |
| D6 | Config timing | Include in S301.4 | Define config schema now so CLI can reference it. |
| D7 | Config schema | Type-to-space mapping | Each artifact type maps to space_key + optional parent_page_id. |

## Approach

### Components

| Component | Change | Description |
|-----------|--------|-------------|
| `src/rai_cli/cli/commands/docs.py` | **Create** | CLI group: publish, get, search |
| `src/rai_cli/cli/commands/_docs_resolve.py` | **Create** | Resolve DocumentationTarget from entry points (mirrors `_adapter_resolve.py`) |
| `src/rai_cli/config/docs_config.py` | **Create** | Pydantic model for `.raise/docs.yaml` + loader |
| `src/rai_cli/cli/main.py` | **Modify** | Register `docs_app` |
| `.raise/docs.yaml` | **Create** | Project-level config (this project's mapping) |

### Config Schema

```yaml
# .raise/docs.yaml
target: confluence  # default target name (entry point)

mappings:
  roadmap:
    space_key: RAISE
    parent_page_id: "123456"  # optional
  adr:
    space_key: RAISE
    parent_page_id: "789012"
  constitution:
    space_key: RAISE
```

```python
class DocMapping(BaseModel):
    space_key: str
    parent_page_id: str | None = None

class DocsConfig(BaseModel):
    target: str = "confluence"
    mappings: dict[str, DocMapping] = Field(default_factory=dict)
```

### Artifact Resolution Flow

```
rai docs publish roadmap
    │
    ▼
Load .raise/docs.yaml → DocsConfig
    │
    ▼
Resolve artifact path: GovernanceSchemaProvider.locate("roadmap")
  ↳ If no schema registered: fallback to convention (governance/roadmap.md)
    │
    ▼
Read file content
    │
    ▼
Build metadata: {space_key, parent_page_id, title} from DocsConfig.mappings["roadmap"]
    │
    ▼
resolve_docs_target() → DocumentationTarget (from entry points)
    │
    ▼
target.publish(doc_type="roadmap", content=..., metadata=...)
    │
    ▼
Compact output: "Published: roadmap → https://..."
```

## Examples

### `rai docs publish`

```bash
# Publish roadmap to Confluence
$ rai docs publish roadmap
Published: roadmap → https://raise.atlassian.net/wiki/spaces/RAISE/pages/123

# Publish ADR
$ rai docs publish adr --title "ADR-041: Config Schema"
Published: adr → https://raise.atlassian.net/wiki/spaces/RAISE/pages/456

# Unknown artifact type
$ rai docs publish foobar
Error: Unknown artifact type 'foobar'. Available: roadmap, adr, constitution, ...

# No mapping configured
$ rai docs publish roadmap
Error: No mapping for 'roadmap' in .raise/docs.yaml

# No target installed
$ rai docs publish roadmap
Error: No docs target installed.
Install one or register via entry points in group 'rai.docs.targets'.
```

### DocsConfig

```python
# config/docs_config.py
from pydantic import BaseModel, Field
from pathlib import Path
import yaml

class DocMapping(BaseModel):
    space_key: str
    parent_page_id: str | None = None

class DocsConfig(BaseModel):
    target: str = "confluence"
    mappings: dict[str, DocMapping] = Field(default_factory=dict)

def load_docs_config(project_root: Path) -> DocsConfig:
    path = project_root / ".raise" / "docs.yaml"
    if not path.exists():
        return DocsConfig()
    data = yaml.safe_load(path.read_text())
    return DocsConfig.model_validate(data or {})
```

## Acceptance Criteria

### MUST

1. `rai docs publish <artifact-type>` publishes via DocumentationTarget, outputs URL
2. `rai docs get <identifier>` retrieves page content as markdown from remote target
3. `rai docs search <query>` lists matching pages (id, space, title)
4. `resolve_docs_target()` auto-detects from `rai.docs.targets` entry points
5. `.raise/docs.yaml` Pydantic config model with type-to-space mapping
6. All commands fail gracefully when no target installed or no mapping configured

### SHOULD

7. `rai docs get <artifact-type>` resolves identifier from config mapping (not just page ID)
8. `--target` flag for explicit target selection when multiple registered

### MUST NOT

- Generic Confluence operations (this is governance-scoped, not a general Confluence CLI)
- File content parsing or transformation (adapter handles format conversion)
- Creating the McpConfluenceAdapter (S301.5)
