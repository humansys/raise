# S301.4: `rai docs` CLI Group — Design

> **v1:** 7 decisions, config layer, GovernanceSchemaProvider dependency.
> **v2:** KISS/DRY/YAGNI review — removed config, phantom deps, raw path input.
> **v3 (current):** Restored artifact type as input (governance domain intelligence).
> CLI owns domain concept (type → path convention), adapter owns platform config.

## Complexity: Simple-Moderate

2 components (CLI group, generic resolver refactor). CLI has minimal domain logic
(artifact type → path convention), same separation of concerns as `rai backlog`.

## What & Why

**Problem:** Agents need to publish/retrieve/search governance documentation
(ADRs, roadmap, constitution) on remote targets (Confluence), but currently must
use raw MCP tools with platform-specific IDs and 8K+ tokens per operation.

**Value:** `rai docs` provides governed documentation operations at ~200 tokens/op.
Unlike a generic Confluence MCP bridge, this is scoped to governance artifacts —
the CLI knows artifact types and their local paths (domain), the adapter knows
where to publish them remotely (platform). Same pattern as `rai backlog`: CLI
speaks domain concepts (`done`, `roadmap`), adapter translates to platform
specifics (transition ID 41, Confluence space RAISE).

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Publish input | Artifact type | `rai docs publish roadmap` — CLI resolves to `governance/roadmap.md` by convention. Domain intelligence, not raw paths. |
| D2 | Path resolution | Convention | `governance/{type}.md` — no GovernanceSchemaProvider (doesn't exist), no config file. Simple convention. |
| D3 | Config ownership | Adapter (S301.5) | jira.yaml → McpJiraAdapter. docs config → McpConfluenceAdapter. CLI doesn't know about spaces/pages. |
| D4 | `get` source | Remote (adapter) | `target.get_page(id)` — validates what's actually published. |
| D5 | Resolver flag | `--target` | Matches `DocumentationTarget` protocol name. |
| D6 | Search scope | Remote target | `target.search(query)`. `rai graph query` exists for local. |
| D7 | Resolver impl | Generic refactor | Extract `_resolve.py` from `_adapter_resolve.py` — parametrized, both CLI groups consume it. DRY. |

## Approach

### Components

| Component | Change | Description |
|-----------|--------|-------------|
| `src/rai_cli/cli/commands/_resolve.py` | **Create** (refactor) | Generic entry-point resolver. Replaces `_adapter_resolve.py`. |
| `src/rai_cli/cli/commands/_adapter_resolve.py` | **Delete** | Replaced by `_resolve.py`. |
| `src/rai_cli/cli/commands/docs.py` | **Create** | CLI group: publish, get, search. Thin + domain convention. |
| `src/rai_cli/cli/commands/backlog.py` | **Modify** | Import from `_resolve.py` instead of `_adapter_resolve.py`. |
| `src/rai_cli/cli/main.py` | **Modify** | Register `docs_app`. |

### Separation of Concerns

```
                     Domain (CLI)              Platform (Adapter)
                     ──────────────            ──────────────────
rai backlog          "done" → status name      transition ID 41, JQL
rai docs publish     "roadmap" → governance/   space RAISE, parent page 123
                     roadmap.md (convention)
```

```python
    Auto-wraps async implementations with sync_wrapper if provided.
    """
    ...

# Convenience wrappers (one-liners)
def resolve_adapter(name: str | None) -> ProjectManagementAdapter:
    return resolve_entrypoint(get_pm_adapters, SyncPMAdapter, "get_issue",
                              "PM adapter", "--adapter", name)

def resolve_docs_target(name: str | None) -> DocumentationTarget:
    return resolve_entrypoint(get_doc_targets, SyncDocsAdapter, "get_page",

```

### Publish Flow

```
rai docs publish roadmap --title "Project Roadmap"
    │
    ▼
CLI: resolve path by convention → governance/roadmap.md
    │  (domain logic: artifact_type → path)
    ▼
CLI: read file content
    ▼
resolve_docs_target() → DocumentationTarget (from entry points)
    ▼
target.publish(doc_type="roadmap", content=..., metadata={"title": ...})
    │  ↑ adapter owns space/parent mapping (its own config, like jira.yaml)
    ▼
"Published: roadmap → https://..."
```

## Examples

### `rai docs publish`

```bash
# Publish roadmap
$ rai docs publish roadmap
Published: roadmap → https://raise.atlassian.net/wiki/spaces/RAISE/pages/123

# Publish ADR with explicit title
$ rai docs publish adr --title "ADR-041: Config Schema"
Published: adr → https://raise.atlassian.net/wiki/spaces/RAISE/pages/456

# Unknown artifact type
$ rai docs publish foobar
Error: File not found: governance/foobar.md

# No target installed
$ rai docs publish roadmap
Error: No docs target installed.
Install one or register via entry points in group 'rai.docs.targets'.
```

### `rai docs get`

```bash
$ rai docs get 123456
```


## Acceptance Criteria

### MUST

1. `rai docs publish <artifact-type>` resolves path by convention, reads file, publishes via DocumentationTarget
2. `rai docs get <page-id>` retrieves page content as markdown from remote target
3. `rai docs search <query>` lists matching pages (id, space, title)
4. Generic `resolve_entrypoint()` replaces `_adapter_resolve.py` — both backlog and docs use it
5. All commands fail gracefully when no target installed or file not found

### SHOULD

6. `--target` flag for explicit target selection when multiple registered
7. `--title` flag on publish for explicit page title (default: artifact type as title)

### MUST NOT

- Own config for space/parent mapping (adapter's responsibility, like jira.yaml)
- Depend on GovernanceSchemaProvider (doesn't exist)
- Generic Confluence operations (governance-scoped)
- Create the McpConfluenceAdapter (S301.5)
