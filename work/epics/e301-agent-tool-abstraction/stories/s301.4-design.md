# S301.4: `rai docs` CLI Group — Design

> **Revised:** KISS/DRY/YAGNI review (SES-302). Removed config layer, phantom
> GovernanceSchemaProvider dependency, and artifact-type resolution. CLI is thin
> — same pattern as `rai backlog`.

## Complexity: Simple-Moderate

2 components (CLI group, generic resolver refactor), mirrors S301.2 pattern exactly.

## What & Why

**Problem:** Agents need to publish/retrieve/search governance documentation
(ADRs, roadmap, constitution) on remote targets (Confluence), but currently must
use raw MCP tools with platform-specific IDs and 8K+ tokens per operation.

**Value:** `rai docs` provides governed documentation operations at ~200 tokens/op.
Unlike a generic Confluence MCP bridge, this is scoped to governance artifacts that
live in the knowledge graph. The adapter (S301.5) owns platform-specific config
(space mapping, parent pages) — the CLI is a thin pass-through, just like `rai backlog`.

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Publish input | File path | `rai docs publish governance/roadmap.md` — explicit, no phantom dependencies. Adapter infers doc_type from path if needed. |
| D2 | Config ownership | Adapter (S301.5) | jira.yaml is consumed by McpJiraAdapter, not by backlog.py. Same pattern: docs config belongs to McpConfluenceAdapter. |
| D3 | `get` source | Remote (adapter) | `target.get_page(id)` — validates what's actually published. |
| D4 | Resolver flag | `--target` | Matches `DocumentationTarget` protocol name. |
| D5 | Search scope | Remote target | `target.search(query)`. `rai graph query` exists for local. |
| D6 | Resolver impl | Generic refactor | Extract `_resolve.py` from `_adapter_resolve.py` — parametrized, both CLI groups consume it. DRY. |

## Approach

### Components

| Component | Change | Description |
|-----------|--------|-------------|
| `src/rai_cli/cli/commands/_resolve.py` | **Create** (refactor) | Generic entry-point resolver. Replaces `_adapter_resolve.py`. |
| `src/rai_cli/cli/commands/docs.py` | **Create** | CLI group: publish, get, search. Thin — delegates to DocumentationTarget. |
| `src/rai_cli/cli/commands/backlog.py` | **Modify** | Import from `_resolve.py` instead of `_adapter_resolve.py`. |
| `src/rai_cli/cli/main.py` | **Modify** | Register `docs_app`. |

**Removed vs v1 design:** `docs_config.py`, `.raise/docs.yaml`, `GovernanceSchemaProvider` dependency.

### Generic Resolver

```python
# _resolve.py — replaces _adapter_resolve.py
def resolve_entrypoint(
    discover: Callable[[], dict[str, type]],
    sync_wrapper: type | None,
    async_check_method: str,
    group_label: str,
    flag_name: str,
    selected: str | None,
) -> Any:
    """Resolve and instantiate an adapter/target from entry points.

    Auto-detect: 1 registered → auto, 0 → error, 2+ → error + flag hint.
    Auto-wraps async implementations with sync_wrapper if provided.
    """
    ...

# Convenience wrappers (one-liners)
def resolve_adapter(name: str | None) -> ProjectManagementAdapter:
    return resolve_entrypoint(get_pm_adapters, SyncPMAdapter, "get_issue",
                              "PM adapter", "--adapter", name)

def resolve_docs_target(name: str | None) -> DocumentationTarget:
    return resolve_entrypoint(get_doc_targets, SyncDocsAdapter, "get_page",
                              "docs target", "--target", name)
```

### Publish Flow (thin CLI)

```
rai docs publish governance/roadmap.md --title "Project Roadmap"
    │
    ▼
Read file content from path
    │
    ▼
resolve_docs_target() → DocumentationTarget (from entry points)
    │
    ▼
target.publish(doc_type="governance", content=..., metadata={"path": ..., "title": ...})
    │  ↑ adapter owns space/parent mapping (its own config, like jira.yaml)
    ▼
Compact output: "Published: → https://..."
```

## Examples

### `rai docs publish`

```bash
# Publish a governance doc
$ rai docs publish governance/roadmap.md
Published: governance/roadmap.md → https://raise.atlassian.net/wiki/spaces/RAISE/pages/123

# Publish with explicit title
$ rai docs publish governance/adrs/adr-041.md --title "ADR-041: Config Schema"
Published: governance/adrs/adr-041.md → https://raise.atlassian.net/wiki/spaces/RAISE/pages/456

# File not found
$ rai docs publish governance/nonexistent.md
Error: File not found: governance/nonexistent.md

# No target installed
$ rai docs publish governance/roadmap.md
Error: No docs target installed.
Install one or register via entry points in group 'rai.docs.targets'.
```

### `rai docs get`

```bash
# Get page by ID
$ rai docs get 123456
# Roadmap (v3)
Space: RAISE | Version: 3 | https://raise.atlassian.net/wiki/...

## Q1 2026
- Feature A ...
```

### `rai docs search`

```bash
$ rai docs search "architecture decisions"
123456  RAISE  ADR-033: Open-core adapter architecture
789012  RAISE  ADR-034: Governance extensibility
345678  RAISE  ADR-036: KnowledgeGraphBackend

$ rai docs search "nonexistent"
No results.
```

## Acceptance Criteria

### MUST

1. `rai docs publish <path>` reads file and publishes via DocumentationTarget, outputs URL
2. `rai docs get <page-id>` retrieves page content as markdown from remote target
3. `rai docs search <query>` lists matching pages (id, space, title)
4. Generic `resolve_entrypoint()` replaces `_adapter_resolve.py` — both backlog and docs use it
5. All commands fail gracefully when no target installed or file not found

### SHOULD

6. `--target` flag for explicit target selection when multiple registered
7. `--title` flag on publish for explicit page title (default: filename stem)

### MUST NOT

- Own config for space/parent mapping (adapter's responsibility, like jira.yaml)
- Depend on GovernanceSchemaProvider (doesn't exist)
- Generic Confluence operations (governance-scoped)
- Create the McpConfluenceAdapter (S301.5)
