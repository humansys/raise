# S301.8: Complete Backlog CLI

## User Story

**As** an AI agent using `rai backlog`,
**I want** `get` and `get-comments` commands,
**so that** I can retrieve issue details and comments without falling back to raw MCP calls.

## Acceptance Criteria

```gherkin
Scenario: Get a single issue by key
  Given the McpJiraAdapter is configured
  When I run `rai backlog get RAISE-328`
  Then the issue details are displayed in compact format
  And the output includes key, status, summary, assignee, and description

Scenario: Get comments for an issue
  Given RAISE-328 has at least one comment
  When I run `rai backlog get-comments RAISE-328`
  Then the comments are displayed with author and timestamp
  And the output is compact (not raw MCP JSON)

Scenario: Get issue that does not exist
  When I run `rai backlog get RAISE-99999`
  Then an error message is displayed
  And the exit code is non-zero
```
