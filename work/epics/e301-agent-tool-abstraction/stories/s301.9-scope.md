# S301.9: FilesystemPMAdapter — Scope

## In Scope

- `FilesystemPMAdapter` class implementing `ProjectManagementAdapter` (sync protocol)
- Read-only operations: `search`, `get_issue`, `health`, `get_comments`
- Parse `governance/backlog.md` epic table into `IssueSummary` / `IssueDetail` models
- Reuse existing `BacklogParser` logic from `governance/parsers/backlog.py`
- Register as entry point `filesystem = rai_cli.adapters.filesystem:FilesystemPMAdapter` in `rai.adapters.pm`
- Auto-resolve: when no other PM adapter available, filesystem is the fallback

## Out of Scope

- Write operations (create, update, transition, link, comment) — return NotImplementedError or no-op
- Batch operations
- Story-level parsing (only epics from backlog table)
- Watching for file changes
- Write-back to backlog.md (future story)

## Done When

- FilesystemPMAdapter passes isinstance check against ProjectManagementAdapter
- `rai backlog search` works without Jira configured (reads backlog.md)
- `rai backlog get E301` returns IssueDetail from backlog.md
- Entry point registered and discoverable via registry
- Tests pass (TDD, unit tests for adapter + integration for CLI fallback)
- Type checks + lint pass
