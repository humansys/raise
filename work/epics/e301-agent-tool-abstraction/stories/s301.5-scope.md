# S301.5: McpConfluenceAdapter — Scope

> **Status:** In Progress
> **Size:** S
> **Branch:** `story/s301.5/mcp-confluence-adapter`
> **Dependencies:** S301.3 (McpBridge) ✓, S301.4 (rai docs CLI) ✓

## In Scope

- `McpConfluenceAdapter` class implementing `AsyncDocumentationTarget` protocol
- 5 methods: `can_publish`, `publish`, `get_page`, `search`, `health`
- MCP tool mapping via existing `McpBridge` (reuse from S301.3)
- Response parsers for Confluence MCP tool output → adapter models
- `aclose()` lifecycle method (RAISE-324 pattern)
- Entry point registration in pyproject.toml (`rai.docs.targets = confluence`)
- Confluence config from environment variables (CONFLUENCE_URL, CONFLUENCE_USERNAME, CONFLUENCE_API_TOKEN)
- Space key config from `.raise/jira.yaml` or environment

## Out of Scope

- New CLI commands (S301.4 already provides publish/get/search)
- Confluence-specific config file (reuse jira.yaml or env vars)
- Attachment handling
- Page hierarchy/parent management (beyond what publish needs)
- Permission checks
