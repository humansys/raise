# S301.4: `rai docs` CLI Group — Scope

> **Revised:** KISS/DRY/YAGNI review (SES-302).

## In Scope

- `rai docs publish <path>` — read local file, publish via DocumentationTarget
- `rai docs get <page-id>` — retrieve page content as markdown from remote
- `rai docs search <query>` — search pages on remote target, compact output
- Generic `_resolve.py` — refactor `_adapter_resolve.py` to parametrized resolver, both backlog and docs use it
- Registration in main CLI (`main.py`)
- Tests for all commands + generic resolver

## Out of Scope

- McpConfluenceAdapter (S301.5)
- `.raise/docs.yaml` config (adapter owns its config, like jira.yaml → S301.5)
- GovernanceSchemaProvider integration (doesn't exist yet)
- Artifact type resolution (publish takes a path, not a type)
- Batch publish
- `rai docs delete`

## Done Criteria

- [ ] `rai docs --help` shows publish, get, search commands
- [ ] `resolve_entrypoint()` generic resolver works for both PM and docs
- [ ] `_adapter_resolve.py` removed, backlog.py uses `_resolve.py`
- [ ] All commands delegate to DocumentationTarget protocol
- [ ] Compact output (~200 tokens per operation)
- [ ] Tests pass (TDD, red-green-refactor)
- [ ] Type checks pass (pyright)
- [ ] Lint passes (ruff)
