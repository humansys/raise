# S301.3 Scope: McpBridge + McpJiraAdapter

## In Scope

- **McpBridge** (`src/rai_cli/adapters/mcp_bridge.py`):
  - Generic async class wrapping MCP Python SDK `ClientSession`
  - `call(tool_name, arguments) → McpToolResult`
  - `list_tools() → list[McpToolInfo]`
  - `health() → AdapterHealth`
  - Lazy session with auto-reconnect (D3)
  - logfire spans on every call (D4) — no-op when logfire not installed
  - Clear error messages when MCP server not found

- **McpJiraAdapter** (`src/rai_cli/adapters/mcp_jira.py`):
  - Implements `AsyncProjectManagementAdapter` (11 methods)
  - Maps each method to corresponding `mcp-atlassian` Jira tool
  - Reads `.raise/jira.yaml` for status → transition ID mapping
  - Reads env vars for Jira auth (JIRA_URL, JIRA_USERNAME, JIRA_API_TOKEN)
  - Registered as entry point `rai.adapters.pm = "jira"`

- **McpToolResult / McpToolInfo** models in mcp_bridge.py

- **pyproject.toml changes**:
  - Add `logfire-api>=4.0` to dependencies
  - Add `mcp>=1.26,<2` to optional `[mcp]` group
  - Add `logfire>=4.0` to optional `[observability]` group
  - Add entry point `rai.adapters.pm` → McpJiraAdapter

- **Tests** (TDD, mock at ClientSession level):
  - McpBridge: call, health, error handling, reconnect
  - McpJiraAdapter: all 11 method mappings, config loading, transition ID resolution
