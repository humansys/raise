# S301.8: Complete Backlog CLI — Design

## Complexity: Simple

3 components (CLI, adapter already implemented, models exist). No new protocol methods. No integration gate.

## What & Why

**Problem:** `rai backlog` has 7 write commands but no read commands. An agent that creates an issue can't verify its state without raw MCP calls (F3 from S301.7 dogfood).

**Value:** Closes the read/write gap. Agents get predictable, token-efficient issue retrieval.

## Approach

Add two CLI commands that delegate to already-implemented adapter methods:

| Component | Change | Notes |
|-----------|--------|-------|
| `src/rai_cli/cli/commands/backlog.py` | modify | Add `get` and `get-comments` commands |
| `src/rai_cli/adapters/mcp_jira.py` | none | `get_issue` and `get_comments` already implemented |
| `src/rai_cli/adapters/models.py` | none | `IssueDetail` and `Comment` already exist |
| `tests/test_cli/test_backlog.py` | modify | Add tests for new commands |

**D1: Compact multi-line output for `get`** — header line (key/status/type) + fields only if non-empty + truncated description. ~50-80 tokens. `search` gives single-line; `get` gives more without exploding to raw MCP JSON.

**D2: Separate `get-comments` command** — agents control context budget. `get` = ~80 tokens (predictable). `get-comments` = variable (depends on conversation length). Combining via flag makes `get` output unpredictable. Protocol has separate methods → CLI mirrors that.

**Missing entity strategy (PAT-E-523):** Non-existent issue key → adapter raises error → CLI prints `Error: Issue not found: RAISE-99999` + exit code 1.

## Examples

### `rai backlog get`

```
$ rai backlog get RAISE-328
RAISE-328  Done  Task
S301.7 Dogfood Test — E2E validation
Assignee: emilio@humansys.ai
Created:  2026-03-01T10:30:00Z

Automated E2E validation of agent tool abstraction.
Will be deleted after test.
```

Fields shown only when non-empty: Assignee, Labels, Parent, Priority.
Description truncated at 500 chars (configurable via `--max-desc` in future, not now).

Error case:
```
$ rai backlog get RAISE-99999
Error: RAISE-99999 not found
```

### `rai backlog get-comments`

```
$ rai backlog get-comments RAISE-328
[2026-03-01 10:35] emilio@humansys.ai:
  E2E dogfood validation — testing full CLI abstraction layer (S301.7).
  This issue will be transitioned to Done and left as evidence.
```

With `--limit`:
```
$ rai backlog get-comments RAISE-328 --limit 5
```

No comments:
```
$ rai backlog get-comments RAISE-328
No comments.
```

## Acceptance Criteria

### MUST
1. `rai backlog get KEY` returns compact multi-line issue details
2. `rai backlog get-comments KEY` returns comment list with author/timestamp
3. Non-existent issue returns error + exit code 1
4. Output is token-efficient (<100 tokens for typical issue)
5. Tests pass (TDD)

### SHOULD
1. `get-comments --limit N` controls max comments returned

### MUST NOT
1. No new protocol methods or models
2. No raw MCP JSON in output
