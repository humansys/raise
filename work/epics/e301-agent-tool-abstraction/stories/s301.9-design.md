# S301.9: FilesystemPMAdapter ŌĆö Design

## Problem

When no Jira adapter is configured, `rai backlog` commands exit with error
"No PM adapter installed." Open-core users without Jira get zero PM
functionality, even though `governance/backlog.md` already contains
structured epic data that the existing `BacklogParser` can parse.

## Value

Every RaiSE user gets working `rai backlog search/get` out of the box.
The filesystem adapter is the zero-config fallback ŌĆö no env vars, no
external services, no cost.

## Approach

Create `FilesystemPMAdapter` implementing `ProjectManagementAdapter` (sync
protocol directly ŌĆö no async needed for file I/O). Reuse `extract_epics()`
and `extract_project()` from `governance/parsers/backlog.py` to parse
`governance/backlog.md` into adapter models.

Register as entry point `filesystem` in `rai.adapters.pm`. When both
`jira` and `filesystem` are registered, the resolver requires `--adapter`
flag (existing behavior for 2+ adapters).

Read-only operations: `search`, `get_issue`, `get_comments`, `health`.
Write operations (`create_issue`, `update_issue`, `transition_issue`,
`link_to_parent`, `link_issues`, `add_comment`, `batch_transition`)
raise `NotImplementedError` with a clear message pointing to Jira adapter.

### Components

| Component | Change | Notes |
|-----------|--------|-------|
| `src/rai_cli/adapters/filesystem.py` | Create | FilesystemPMAdapter class |
| `pyproject.toml` | Modify | Add entry point `filesystem` |
| `tests/adapters/test_filesystem.py` | Create | Unit tests |

## Examples

### search ŌĆö filter by status

```python
adapter = FilesystemPMAdapter(project_root=Path("."))
results = adapter.search("status = pending", limit=10)
# ŌåÆ [IssueSummary(key="E301", summary="Agent Tool Abstraction", status="in_progress", issue_type="Epic")]
```

Query syntax: simple `field = value` matching on epic metadata fields
(`status`, `name`, `priority`). Not JQL ŌĆö just key=value filters.
Bare text (no `=`) matches against name.

### get_issue ŌĆö epic detail

```python
detail = adapter.get_issue("E301")
# ŌåÆ IssueDetail(
#     key="E301",
#     summary="Agent Tool Abstraction",
#     status="in_progress",
#     issue_type="Epic",
#     description="",  # no description in backlog table
#     labels=[],
#     metadata={"scope_doc": "scope.md", "priority": "P0"}
# )
```

### health

```python
h = adapter.health()
# ŌåÆ AdapterHealth(name="filesystem", healthy=True, message="governance/backlog.md (12 epics)")
# If no backlog.md:
# ŌåÆ AdapterHealth(name="filesystem", healthy=False, message="governance/backlog.md not found")
```

### get_comments ŌĆö always empty

```python
comments = adapter.get_comments("E301")
# ŌåÆ []  (filesystem has no comments)
```

### write operation ŌĆö NotImplementedError

```python
adapter.create_issue("PROJ", IssueSpec(summary="x"))
# raises NotImplementedError("FilesystemPMAdapter is read-only. Install jira adapter for write operations.")
```

### CLI fallback

```bash
# No Jira configured, filesystem is only adapter ŌåÆ auto-selected
$ rai backlog search "status = pending"
E301  Agent Tool Abstraction  in_progress  Epic

# Both installed ŌåÆ must specify
$ rai backlog search --adapter filesystem "status = pending"
```

## Acceptance Criteria

### MUST
- FilesystemPMAdapter passes `isinstance(adapter, ProjectManagementAdapter)`
- `search()` returns `IssueSummary` list from `governance/backlog.md` epic table
- `get_issue()` returns `IssueDetail` for a valid epic ID
- `health()` returns healthy=True when backlog.md exists, False when missing
- Entry point registered and discoverable via `get_pm_adapters()`
- Write operations raise `NotImplementedError` with install guidance

### SHOULD
- Bare text search matches against epic name (case-insensitive)

### MUST NOT
- Must not import or depend on MCP, Jira, or any external service
- Must not modify `governance/backlog.md` (read-only)
