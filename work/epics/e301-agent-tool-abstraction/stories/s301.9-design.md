# S301.9: FilesystemPMAdapter — Design

## Problem

When no Jira adapter is configured, `rai backlog` commands exit with error
"No PM adapter installed." Open-core users without Jira get zero PM
functionality, even though `governance/backlog.md` already contains
structured epic data that the existing `BacklogParser` can parse.

## Value

Every RaiSE user gets working `rai backlog` out of the box — read AND write.
The filesystem adapter is the zero-config fallback — no env vars, no
external services, no cost. Users can manage their backlog entirely through
CLI without needing Jira.

## Approach

Create `FilesystemPMAdapter` implementing `ProjectManagementAdapter` (sync
protocol directly — no async needed for file I/O). Reuse `extract_epics()`
from `governance/parsers/backlog.py` for reads. Write operations manipulate
the markdown table directly (append row, update row fields).

Register as entry point `filesystem` in `rai.adapters.pm`. When both
`jira` and `filesystem` are registered, the resolver requires `--adapter`
flag (existing behavior for 2+ adapters).

### Operation Support

| Operation | Supported | How |
|-----------|-----------|-----|
| `search` | Yes | Parse epic table, filter by field=value |
| `get_issue` | Yes | Find row by epic ID |
| `get_comments` | No-op | Returns `[]` (no comment storage in md) |
| `health` | Yes | Check backlog.md exists + count epics |
| `create_issue` | Yes | Append row to epic table |
| `update_issue` | Yes | Update fields in existing row |
| `transition_issue` | Yes | Update status column |
| `batch_transition` | Yes | Loop of transition_issue |
| `link_to_parent` | No-op | No link structure in md table |
| `link_issues` | No-op | No link structure in md table |
| `add_comment` | No-op | No comment storage in md |

No-op methods return empty results (not errors) — silent success for
operations that don't map to the filesystem model. This keeps the adapter
composable without forcing callers to check capabilities.

### Table format (write target)

```markdown
| ID | Epic | Status | Scope Doc | Priority |
|----|------|--------|-----------|----------|
| E1 | **Core Foundation** | ✅ Complete | `scope.md` | — |
```

Writes must preserve: bold on name, emoji+text status format, backtick on
scope doc, `—` for empty fields.

### Components

| Component | Change | Notes |
|-----------|--------|-------|
| `src/rai_cli/adapters/filesystem.py` | Create | FilesystemPMAdapter class |
| `pyproject.toml` | Modify | Add entry point `filesystem` |
| `tests/adapters/test_filesystem.py` | Create | Unit tests |

## Examples

### search — filter by status

```python
adapter = FilesystemPMAdapter(project_root=Path("."))
results = adapter.search("status = pending", limit=10)
# → [IssueSummary(key="E301", summary="Agent Tool Abstraction", status="in_progress", issue_type="Epic")]
```

Query syntax: simple `field = value` matching on epic metadata fields
(`status`, `name`, `priority`). Not JQL — just key=value filters.
Bare text (no `=`) matches against name (case-insensitive).

### get_issue — epic detail

```python
detail = adapter.get_issue("E301")
# → IssueDetail(
#     key="E301",
#     summary="Agent Tool Abstraction",
#     status="in_progress",
#     issue_type="Epic",
#     description="",
#     labels=[],
#     metadata={"scope_doc": "scope.md", "priority": "P0"}
# )
```

### create_issue — append epic row

```python
from rai_cli.adapters.models import IssueSpec
ref = adapter.create_issue("RAISE", IssueSpec(
    summary="New Feature",
    issue_type="Epic",
    metadata={"priority": "P1", "scope_doc": "scope.md"},
))
# → IssueRef(key="E50", url="")
# backlog.md now has:
# | E50 | **New Feature** | 📋 Backlog | `scope.md` | P1 |
```

ID auto-assigned: finds max E{N} in table, increments.

### transition_issue — update status

```python
ref = adapter.transition_issue("E301", "complete")
# → IssueRef(key="E301", url="")
# Row updated: status column → "✅ Complete"
```

Status mapping: `complete` → `✅ Complete`, `in_progress` → `🚀 In Progress`,
`pending`/`backlog` → `📋 Backlog`, `draft` → `📋 DRAFT`,
`deferred` → `→ Deferred`.

### update_issue — update fields

```python
ref = adapter.update_issue("E301", {"summary": "New Name", "priority": "P0"})
# → IssueRef(key="E301", url="")
# Row updated: name → **New Name**, priority → P0
```

### health

```python
h = adapter.health()
# → AdapterHealth(name="filesystem", healthy=True, message="governance/backlog.md (12 epics)")
```

### get_comments / add_comment — no-op

```python
adapter.get_comments("E301")       # → []
adapter.add_comment("E301", "hi")  # → CommentRef(id="", url="")
```

### CLI usage

```bash
# No Jira configured → filesystem auto-selected
$ rai backlog search "status = pending"
E301  Agent Tool Abstraction  in_progress  Epic

$ rai backlog create RAISE --summary "New Feature" --type Epic
Created: E50

$ rai backlog transition E301 complete
Transitioned: E301 → complete
```

## Acceptance Criteria

### MUST
- FilesystemPMAdapter passes `isinstance(adapter, ProjectManagementAdapter)`
- `search()` returns `IssueSummary` list from `governance/backlog.md` epic table
- `get_issue()` returns `IssueDetail` for a valid epic ID
- `create_issue()` appends a new row to the epic table with auto-ID
- `transition_issue()` updates the status column with emoji format
- `update_issue()` modifies name/priority/scope_doc fields in-place
- `health()` returns healthy=True when backlog.md exists, False when missing
- Entry point registered and discoverable via `get_pm_adapters()`

### SHOULD
- `batch_transition()` transitions multiple epics in one call
- Bare text search matches against epic name (case-insensitive)

### MUST NOT
- Must not import or depend on MCP, Jira, or any external service
- Must not corrupt existing table formatting on writes
