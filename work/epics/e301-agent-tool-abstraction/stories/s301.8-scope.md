# S301.8: Complete Backlog CLI — Scope

## In Scope

- `rai backlog get KEY` — retrieve single issue details (maps to `get_issue` protocol method)
- `rai backlog get-comments KEY` — retrieve issue comments (maps to `get_comments` protocol method)
- McpJiraAdapter implementation for `get_issue` and `get_comments` (if not already implemented)
- Compact output format consistent with existing commands
- Tests for both commands (TDD)

## Out of Scope

- New protocol methods beyond what exists (versions, sprints, dev-info, projects)
- Protocol extensions — those belong in a future epic
- Extending the docs CLI (already complete: publish, get, search)

## Done Criteria

- [ ] `rai backlog get KEY` returns compact issue details
- [ ] `rai backlog get-comments KEY` returns compact comment list
- [ ] Error handling for non-existent issues
- [ ] Tests pass (TDD, red-green-refactor)
- [ ] Quality checks pass (ruff, pyright)
