## Execution Order

```
T1 (models) → T2 (protocols) → T3 (sync wrappers) → T4 (exports)
```

Linear chain — each task depends on the previous. No parallelism possible because:
- Protocols import models (T2 needs T1)
- Wrappers reference async protocols (T3 needs T2)
- Exports re-export everything (T4 needs T1+T2+T3)

Risk-first: models are lowest risk (pure Pydantic), protocols are medium (signature design already decided), wrappers are highest novelty (asyncio.run pattern). But dependency order forces bottom-up anyway, which is natural.

## Risks

| Risk | L/I | Mitigation |
|------|:---:|------------|
| `asyncio.run()` in sync wrapper fails if event loop already running | M/M | Test in isolation. CLI (Typer) does not run an event loop. If nested loop needed, `nest_asyncio` as fallback. |
| `@runtime_checkable` with async methods may not work as expected | L/H | Test explicitly in T2. Python 3.12+ supports async Protocol methods. |
| Existing test imports break when protocol signatures change | L/L | Only `StubPM` in test_protocols.py affected — update in T2. |
