# S301.5: McpConfluenceAdapter

## User Story

**As** Rai (AI agent),
**I want** a Confluence adapter that maps DocumentationTarget methods to mcp-atlassian tools via McpBridge,
**so that** `rai docs publish/get/search` works against Confluence Cloud without raw MCP calls.

## Acceptance Criteria

```gherkin
Scenario: Publish governance artifact to Confluence
  Given a configured McpConfluenceAdapter with Confluence credentials
  When I call publish("roadmap", content, {"title": "Roadmap"})
  Then a page is created/updated in Confluence
  And a PublishResult with success=True and URL is returned

Scenario: Get page by ID
  Given a page exists in Confluence with ID "123456"
  When I call get_page("123456")
  Then a PageContent is returned with title, content (markdown), space_key, version

Scenario: Search pages
  Given pages exist in Confluence
  When I call search("roadmap", limit=5)
  Then a list of PageSummary is returned with id, title, space_key, url

Scenario: Can publish check
  Given a configured adapter
  When I call can_publish("roadmap", {})
  Then True is returned (adapter accepts all doc types)

Scenario: Health check
  Given Confluence is reachable
  When I call health()
  Then AdapterHealth with healthy=True and latency is returned

Scenario: Entry point registration
  Given rai-cli is installed
  When entry points for "rai.docs.targets" are discovered
  Then "confluence" maps to McpConfluenceAdapter
```

## SbE Examples

| Method | MCP Tool | Key Params |
|--------|----------|------------|
| publish | confluence_create_page or confluence_update_page | space_key, title, content |
| get_page | confluence_get_page | page_id |
| search | confluence_search | query, limit |
| health | confluence_search (minimal) | limit=1 |
