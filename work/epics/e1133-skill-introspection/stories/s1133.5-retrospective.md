## Patterns

### PAT: First-run discovery premium
**Context:** dogfood, introspection, infrastructure
**Pattern:** First execution of a new system reveals infrastructure bugs (graph build failure) and data gaps (empty subtypes) that are invisible in design. Budget 50%+ extra for first-run stories. The debug time is investment, not waste — it fixes the foundation for all subsequent runs.
**Learned from:** S1133.5

## Heutagogical Checkpoint

**What did I learn?** The introspection system works end-to-end but revealed that the graph infrastructure had latent bugs. The LEARN record format is simple enough for manual production — good design signal.

**What would I do differently?** Run `rai graph build` as a gate before starting introspection-aware stories. The debug session was valuable but could have been avoided with a pre-check.

**What's next?** S1133.6 dogfood with a real epic cycle. The system is functional — now we need volume to validate the metrics and refine the query templates.
