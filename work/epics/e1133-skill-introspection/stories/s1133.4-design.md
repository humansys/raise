# S1133.4 Design: Story Lifecycle Integration

## Problem & Value

**Problem:** Story lifecycle skills produce designs, plans, implementations, and evaluations without systematic evidence grounding or measurement. story-implement has a manual graph query; story-review captures patterns — but neither is measured for effectiveness.

**Value:** Wrapping existing logic with PRIME/JIT/LEARN enables measurement without disruption. Learning records will tell us whether current queries and capture are optimal. Measurement first, optimization later.

## Complexity: Moderate

4 skill files. Same pattern as S1133.3 but with richer chain reads (story chain has downstream enrichment).

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Existing logic | Wrap, don't replace | Can't optimize without measurement data. LEARN records will tell us what's working. |
| D2 | JIT placement | Every evidence-grounding decision point | Grounded generation principle from S1133.3 |
| D3 | Chain enrichment | story-plan enriches story-design, story-implement enriches story-design | Per aspect chain mapping |

## Approach

### story-design (5 steps)

**Metadata:**
```yaml
raise.aspects: introspection
raise.introspection:
  phase: story.design
  context_source: scope doc
  affected_modules: []
  max_tier1_queries: 3
  max_jit_queries: 5
  tier1_queries:
    - "patterns for {affected_modules} design decisions"
    - "prior designs for similar scope in {phase}"
    - "risks and lessons from related epics"
```

**Markers:** PRIME before Step 1 (no chain read — first in story chain). JIT at Steps 1, 2, 3, 5. LEARN after Step 5.

### story-plan (4 steps)

**Metadata:**
```yaml
raise.aspects: introspection
raise.introspection:
  phase: story.plan
  context_source: design doc
  affected_modules: []
  max_tier1_queries: 3
  max_jit_queries: 2
  tier1_queries:
    - "decomposition patterns for {complexity} stories"
    - "TDD patterns for {affected_modules}"
    - "estimation calibration for {size} stories"
```

**Markers:** PRIME before Step 1 (chain read: story-design's record, enrich with `downstream: {plan_derivable, tasks_clear}`). JIT at Steps 1, 3. LEARN after Step 4.

### story-implement (5 steps, loop)

**Metadata:**
```yaml
raise.aspects: introspection
raise.introspection:
  phase: story.implement
  context_source: plan doc
  affected_modules: []
  max_tier1_queries: 3
  max_jit_queries: 3
  tier1_queries:
    - "implementation patterns for {affected_modules}"
    - "testing patterns for {test_type} in {language}"
    - "integration patterns for {upstream_dependencies}"
```

**Markers:** PRIME before Step 1 (chain read: story-plan's record). Existing graph query in Step 1 stays — PRIME adds the framework around it. JIT at Step 1. LEARN after Step 5 (enriches story-design's record with `downstream: {design_gaps_found}`).

### story-review (5 steps)

**Metadata:**
```yaml
raise.aspects: introspection
raise.introspection:
  phase: story.review
  context_source: all story artifacts
  affected_modules: []
  max_tier1_queries: 2
  max_jit_queries: 3
  tier1_queries:
    - "evaluation patterns for {affected_modules}"
    - "process patterns from recent stories"
```

**Markers:** PRIME before Step 1 (chain read: ALL previous story records). Existing pattern capture in Steps 3-4 stays — LEARN adds the measurement framework. JIT at Steps 2, 3. LEARN after Step 5.

## Acceptance Criteria

### MUST
1. All 4 story skills declare introspection metadata
2. All 4 have PRIME before Step 1 and LEARN after last step
3. JIT at evidence-grounding points
4. story-plan reads and enriches story-design's record
5. story-implement enriches story-design's record with design_gaps_found
6. story-review reads all previous records
7. Existing graph queries and pattern capture untouched

### MUST NOT
1. Not modify existing skill logic — only add markers and metadata
2. Not optimize queries without measurement data
