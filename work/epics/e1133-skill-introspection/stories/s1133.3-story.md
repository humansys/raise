# S1133.3: Epic Lifecycle Integration

**Epic:** E1133 — Skill Introspection
**Tracker:** RAISE-1141
**Type:** docs
**Size:** M

## User Story

As an epic lifecycle skill (epic-design, epic-plan, epic-close), I need to participate in the introspection aspect via metadata declaration and PRIME/JIT/LEARN markers, so that my executions are informed by accumulated knowledge and produce learning records for the chain.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Epic skills declare introspection metadata
  Given epic-design, epic-plan, and epic-close skill files
  When a developer reads the frontmatter
  Then each declares raise.aspects: introspection with phase, context_source, affected_modules, query limits, and tier1_queries

Scenario: PRIME marker before first step
  Given an epic skill with introspection metadata
  When the skill begins execution
  Then PRIME fires before Step 1, executing deterministic graph queries

Scenario: JIT marker at decision bifurcations
  Given an epic skill at a step with multiple valid approaches
  When consequences extend beyond the current step
  Then the JIT heuristic triggers a graph query

Scenario: LEARN marker after last step
  Given an epic skill that has completed execution
  When the LEARN protocol fires
  Then a flat learning record is produced at .raise/rai/learnings/{skill}/{work_id}/record.yaml
```
