# S1133.2: Graph Schema Enhancements

**Epic:** E1133 — Skill Introspection
**Tracker:** RAISE-1140
**Type:** code
**Size:** S

## User Story

As the introspection aspect, I need the knowledge graph to support `approach` and `risk` pattern types, story outcome metadata, and standardized context tags, so that PRIME queries return relevant patterns and LEARN records store structured data.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Pattern type taxonomy supports approach and risk
  Given the knowledge graph schema
  When a pattern is added with type "approach" or "risk"
  Then the pattern is accepted and queryable by type

Scenario: Learning records directory exists
  Given the introspection aspect produces learning records
  When LEARN writes a record
  Then it is stored in `.raise/rai/learnings/{skill}/{work_id}/record.yaml`

Scenario: Context tags follow standardized taxonomy
  Given the graph query templates use context tags
  When a query includes context tags like "design decisions" or "architecture"
  Then results are filtered by the standardized tag set

Scenario: Story outcome metadata is available
  Given a story has completed with a retrospective
  When the graph is queried for story outcomes
  Then outcome metadata (patterns captured, gaps identified) is accessible
```
