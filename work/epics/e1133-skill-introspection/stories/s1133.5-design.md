# S1133.5 Design: Story-Review Refactor

## Problem & Value

**Problem:** story-review was the only skill evaluating patterns and capturing learnings. With LEARN distributed across all skills, story-review duplicates evaluation that already happens per-skill.

**Value:** Add learning record aggregation to story-review so it produces cross-skill metrics. Simplify by leveraging the records each skill already produces.

## Complexity: S

One step added, one step simplified. No code changes — skill file modification only.

## PRIME Observations (dogfood)

- 2 PRIME queries returned ~10 patterns, none specific to learning aggregation
- `--subtypes approach,risk` returned nothing (new subtypes not yet populated)
- 1 JIT query returned 0 results → gap: "No patterns for learning record aggregation"
- **Graph build was blocked** by duplicate epic IDs (RAISE-1199) — fixed with data cleanup

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Aggregation placement | New Step 2.5 between Reflect and Persist | Aggregation informs pattern persistence decisions |
| D2 | Metrics | 3 simple metrics: acceptance rate, gap rate, pattern utility | Derivable from learning records, no new infrastructure |
| D3 | Existing steps | Keep all — don't remove pattern capture | Measurement first. LEARN records will tell us if pattern capture here is redundant |

## Approach

### Change 1: Add "Aggregate Learning Records" step

Insert between current Step 2 (Reflect) and Step 3 (Persist Patterns):

```markdown
### Step 3: Aggregate Learning Records

Read all learning records produced during this story:
- `.raise/rai/learnings/rai-story-design/{work_id}/record.yaml`
- `.raise/rai/learnings/rai-story-plan/{work_id}/record.yaml`
- `.raise/rai/learnings/rai-story-implement/{work_id}/record.yaml`

If any record is missing (silent node or execution gap), note it and continue.

Produce aggregate summary:

| Metric | How to calculate |
|--------|-----------------|
| **Acceptance rate** | Patterns voted +1 / total patterns primed |
| **Gap rate** | Total gaps across all records / total JIT queries |
| **Pattern utility** | Patterns voted +1 / (patterns voted +1 + patterns voted -1) |

Include the aggregate summary in the retrospective (Step 5).
```

Current Steps 3-5 become Steps 4-6.

### Change 2: Update retrospective template

Add "Learning Chain Summary" section to retrospective with:
- Records found/missing
- Aggregate metrics
- Notable gaps (merged from all records)
- Downstream enrichments received

### Change 3: Renumber steps

| Old | New | Name |
|-----|-----|------|
| Step 1 | Step 1 | Verify Tests Pass |
| Step 2 | Step 2 | Gather Data & Reflect |
| (new) | Step 3 | Aggregate Learning Records |
| Step 3 | Step 4 | Persist Patterns & Reinforce |
| Step 4 | Step 5 | Document Retrospective |
| Step 5 | Step 6 | Emit Calibration Telemetry |

PRIME stays before Step 1. JIT markers shift to new step numbers. LEARN moves after Step 6.

## Acceptance Criteria

### MUST
1. New step reads all story learning records
2. 3 metrics calculated: acceptance rate, gap rate, pattern utility
3. Aggregate summary included in retrospective
4. Existing pattern capture steps unchanged
5. LEARN marker after final step

### MUST NOT
1. Not remove existing functionality
2. Not require code changes (skill file only)
