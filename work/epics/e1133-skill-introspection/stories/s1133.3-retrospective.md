# S1133.3 Retrospective: Epic Lifecycle Integration

## What happened

Integrated the introspection aspect into 3 epic lifecycle skills (epic-design, epic-plan, epic-close). Each skill now declares introspection metadata and has PRIME/JIT/LEARN markers at appropriate positions.

**Key design conversation:** The JIT heuristic was reframed during this story. The original framing ("bifurcation resolution") was too narrow — it only triggered at choice points. The corrected framing ("grounded decision") triggers at any point where the agent produces output that should be governed by accumulated evidence. This is the epistemological foundation of RaiSE: decisions are governed by experience, not hallucinated.

## Patterns

### PAT: Grounded generation over functional hallucination
**Context:** AI skill design, decision quality
**Pattern:** JIT queries should trigger whenever the agent produces decisions that need epistemological grounding — not just at choice bifurcations. The heuristic is "am I about to generate something that should be governed by evidence?" not "do I have multiple options?"
**Learned from:** S1133.3 design conversation
**Why it matters:** Without this, AI agents produce plausible-sounding recommendations that aren't anchored to project experience. RaiSE distinguishes itself by governing decisions with accumulated knowledge.

## Heutagogical Checkpoint

**What did I learn?** The distinction between bifurcation and grounded decision is fundamental to RaiSE's epistemic model. It changes JIT from a "help me choose" mechanism to an "I must be informed" discipline.

**What would I do differently?** Start with the grounded decision framing from S1133.1 rather than discovering it in S1133.3.

**What's next?** S1133.4 will apply the same integration to 4 story lifecycle skills, using this corrected JIT framing from the start.
