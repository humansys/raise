# S1133.4: Story Lifecycle Integration

**Epic:** E1133 — Skill Introspection
**Tracker:** RAISE-1142
**Type:** docs
**Size:** M

## User Story

As a story lifecycle skill (story-design, story-plan, story-implement, story-review), I need to participate in the introspection aspect via metadata declaration and PRIME/JIT/LEARN markers, so that each story execution is governed by accumulated evidence and produces learning records that flow through the chain.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Story skills declare introspection metadata
  Given story-design, story-plan, story-implement, and story-review skill files
  When a developer reads the frontmatter
  Then each declares raise.aspects: introspection with phase-specific metadata

Scenario: Learning chain flows through story lifecycle
  Given story-design has produced a learning record
  When story-plan executes PRIME
  Then it reads story-design's record and enriches it with downstream observations

Scenario: story-implement enriches story-design's record
  Given story-implement has completed
  When LEARN fires
  Then it enriches story-design's record with design_gaps_found

Scenario: story-review aggregates all learning records
  Given all previous story skills have produced records
  When story-review executes PRIME
  Then it reads all records and produces an aggregate learning summary
```
