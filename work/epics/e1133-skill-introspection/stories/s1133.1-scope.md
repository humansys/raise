# S1133.1: Introspection Aspect Definition — Scope

## In Scope

1. **`aspects/introspection.md`** — the shared aspect file defining:
   - PRIME protocol: guided Tier 1 query templates per lifecycle phase
   - JIT protocol: Tier 2 heuristic with concrete trigger examples
   - LEARN protocol: produces learning record YAML
   - Downstream enrichment protocol: how skills read/write previous learning records

2. **Learning record schema** — YAML structure with 3 sections:
   - `process_fidelity`: tier1_primed, queries, jit_queries, artifacts_produced
   - `pattern_metrics`: primed patterns with votes+why, gaps
   - `downstream_impact`: slots filled by subsequent skills
   - ~~`output_quality`~~: removed — acceptance implicit in chain (Poppendieck: eliminate waste)

3. **Phase metadata format** — how skills declare participation:
   - `aspect: introspection`
   - `phase: {lifecycle_phase}`
   - `context_source: {primary input}`
   - `affected_modules: [...]`

4. **Evaluation semantics** — recall failure vs use failure distinction (Governed Memory)

## Out of Scope

- Modifying any lifecycle skill files (S1133.3, S1133.4)
- Graph schema changes (S1133.2)
- Story-review refactor (S1133.5)
- `.raise/rai/learnings/` directory structure (S1133.2)
- Automated skill evolution / rai-skill-improve
