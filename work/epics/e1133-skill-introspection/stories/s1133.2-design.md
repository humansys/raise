# S1133.2 Design: Graph Schema Enhancements

## Problem & Value

**Problem:** The introspection aspect (S1133.1) defines PRIME queries that filter by `approach` and `risk` types, but the graph only supports 4 pattern subtypes (`process`, `codebase`, `architecture`, `technical`). There's no `--subtypes` filter in the query engine, no learning record infrastructure, and context tags are free-form.

**Value:** This story builds the infrastructure that S1133.3/S1133.4 need. Without it, PRIME queries can't filter by approach/risk, and LEARN has no model or storage for records.

## Complexity: Small

Additive changes to existing code. No architectural decisions beyond what was resolved interactively.

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | Pattern types | Extend `PatternSubType` enum + add `--subtypes` flag to query | Node type ≠ subtype. Keep semantics clean. |
| D2 | Learning infrastructure | Pydantic model + read/write functions + directory | Single source of truth for schema. Infra for S1133.3/S1133.4. |
| D3 | Context tags | Documented taxonomy, not enforced | Simple First. Tags are organic and useful. Enforce with evidence from dogfood. |
| D4 | Story outcome metadata | Learning record IS the outcome | Don't duplicate concepts. Record already captures votes, gaps, artifacts, downstream. |

## Approach

### Component 1: PatternSubType Extension

**File:** `packages/raise-cli/src/raise_cli/memory/models.py`

Add two values to `PatternSubType` enum:
```python
class PatternSubType(StrEnum):
    CODEBASE = "codebase"
    PROCESS = "process"
    ARCHITECTURE = "architecture"
    TECHNICAL = "technical"
    APPROACH = "approach"      # NEW: design approaches, patterns for solving
    RISK = "risk"              # NEW: known risks, failure modes, pitfalls
```

**File:** `packages/raise-cli/src/raise_cli/cli/commands/pattern.py`

Update help text for `--type` to include new values. Validation via enum is automatic.

### Component 2: Query Engine `--subtypes` Filter

**File:** `packages/raise-core/src/raise_core/graph/query.py`

Add `subtypes` field to `Query` model:
```python
class Query(BaseModel):
    # ... existing fields ...
    subtypes: list[str] | None = Field(
        default=None,
        description="Filter by pattern subtypes (approach, risk, etc.)",
    )
```

Add filtering in `_keyword_search()` and `_concept_lookup()`:
```python
# After type filter (line ~311):
if query.subtypes and concept.metadata.get("sub_type") not in query.subtypes:
    continue
```

**File:** `packages/raise-cli/src/raise_cli/cli/commands/graph.py`

Add `--subtypes` option to `query` command, pass through to `Query`.

### Component 3: Learning Record Model & I/O

**File:** `packages/raise-cli/src/raise_cli/memory/learning.py` (NEW)

```python

    """Write learning record to .raise/rai/learnings/{skill}/{work_id}/record.yaml"""

def read_record(skill: str, work_id: str, base_dir: Path) -> LearningRecord | None:
    """Read learning record. Returns None if not found (silent node or not yet written)."""

def enrich_downstream(skill: str, work_id: str, fields: dict[str, Any], base_dir: Path) -> bool:
    """Best-effort downstream enrichment. Returns False on any failure, never raises."""
```

**Directory:** `.raise/rai/learnings/README.md` — convention documentation.

### Component 4: Context Tag Taxonomy

**File:** `docs/guides/context-tag-taxonomy.md` (NEW)

Documented taxonomy organized by domain:
- **Architecture:** architecture, module-design, cross-cutting, layers, boundaries
- **Testing:** testing, tdd, pytest, integration, mocking
- **Process:** process, workflow, lifecycle, estimation, retrospective
- **Security:** security, sast, cve, dependency, authentication
- **Code:** typing, patterns, refactoring, naming, idioms
- **Infrastructure:** ci, deployment, configuration, tooling

Not enforced. Guidance for consistency.

### Component 5: Aspect Query Template Update

**File:** `aspects/introspection.md`

Update all query examples from:
```bash
rai graph query "..." --types approach,risk --limit 5
```
to:
```bash
rai graph query "..." --types pattern --subtypes approach,risk --limit 5
```

## Acceptance Criteria

### MUST
1. `PatternSubType` includes `approach` and `risk`
2. `rai pattern add --type approach` works
3. `rai graph query --subtypes approach,risk` filters correctly
4. `LearningRecord` Pydantic model matches aspect schema exactly
5. `write_record()` creates YAML at correct path
6. `read_record()` returns `None` for missing records (not error)
7. `enrich_downstream()` is best-effort (never raises)
8. `.raise/rai/learnings/` directory exists with README
9. Context tag taxonomy documented
10. `aspects/introspection.md` query templates updated to use `--subtypes`

### SHOULD
1. Tests cover all new code (TDD)
2. Existing tests still pass

### MUST NOT
1. Not break existing pattern type validation
2. Not enforce context tags at write time
3. Not create a separate story outcome schema
