# S1133.3 Design: Epic Lifecycle Integration

## Problem & Value

**Problem:** Epic lifecycle skills (design, plan, close) generate decisions, plans, and evaluations without consulting accumulated project knowledge. Output is plausible but ungrounded — functional hallucination.

**Value:** With introspection, every epic skill execution is governed by evidence: PRIME loads relevant patterns before generating, JIT grounds decisions in project experience during execution, LEARN captures what was useful for the next skill in the chain.

## Complexity: Moderate

3 skill files to modify. Each needs metadata, PRIME marker, JIT markers at evidence-grounding points, and LEARN marker. The JIT placement requires understanding which steps produce decisions vs mechanical output.

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| D1 | JIT placement | At every step producing decisions that should be evidence-grounded | Grounded generation, not just bifurcation resolution. RaiSE principle: epistemological rigor. |
| D2 | JIT heuristic update | Updated in aspect from "bifurcation" to "grounded decision" | Captures the real intent — decisions governed by experience, not hallucinated |
| D3 | Chain reads | epic-plan reads epic-design record, epic-close reads all | Natural information flow through the lifecycle |

## Approach

### Changes per skill

#### rai-epic-design (5 steps)

**Metadata:**
```yaml
metadata:
  raise.aspects: introspection
  raise.introspection:
    phase: epic.design
    context_source: problem brief or strategic objective
    affected_modules: [determined at runtime from scope]
    max_tier1_queries: 3
    max_jit_queries: 5
    tier1_queries:
      - "patterns for {affected_modules} architecture decisions"
      - "risks and failure modes in {domain} epics"
      - "prior epic designs with similar scope ({story_count} stories)"
```

**Markers:**
- PRIME before Step 1
- JIT at Step 1 (framing objective — consult prior similar designs)
- JIT at Step 2 (architecture decisions — consult patterns and risks)
- JIT at Step 3 (story decomposition — consult sizing patterns)
- JIT at Step 4 (risk assessment — consult known risks from similar epics)
- LEARN after Step 5

**`max_jit_queries: 5`** because epic-design has 4 JIT points. Budget allows ~1 query per point plus 1 spare.

#### rai-epic-plan (5 steps)

**Metadata:**
```yaml
metadata:
  raise.aspects: introspection
  raise.introspection:
    phase: epic.plan
    context_source: scope doc from epic-design
    affected_modules: [from scope]
    max_tier1_queries: 3
    max_jit_queries: 3
    tier1_queries:
      - "sequencing patterns for {strategy} ordering"
      - "estimation patterns for {size} epics"
      - "milestone patterns for multi-story epics"
```

**Markers:**
- PRIME before Step 1 (includes chain read: epic-design's learning record)
- JIT at Step 2 (sequencing strategy — consult ordering patterns)
- JIT at Step 3 (milestones — consult calibration and checkpoint patterns)
- LEARN after Step 5

**Chain read:** PRIME reads epic-design's learning record. Enriches it with `downstream: {scope_clear: bool, stories_sequenceable: bool}`.

#### rai-epic-close (5 steps)

**Metadata:**
```yaml
metadata:
  raise.aspects: introspection
  raise.introspection:
    phase: epic.close
    context_source: all epic artifacts
    affected_modules: [from scope]
    max_tier1_queries: 2
    max_jit_queries: 3
    tier1_queries:
      - "retrospective patterns for {domain} epics"
      - "process improvement patterns from similar epics"
```

**Markers:**
- PRIME before Step 1 (includes chain read: all previous learning records)
- JIT at Step 1 (descoping decision — consult completion patterns)
- JIT at Step 2 (retrospective — consult process improvement patterns)
- LEARN after Step 5

**Chain read:** PRIME reads ALL learning records from the epic chain. Produces aggregate learning summary.

### Aspect update (already done)

Updated JIT heuristic from "bifurcation" to "grounded decision":
- Old: "Multiple valid approaches exist AND consequences extend beyond this step"
- New: "You are about to produce a decision that should be grounded in project experience"

## Acceptance Criteria

### MUST
1. All 3 epic skills declare introspection metadata in frontmatter
2. All 3 skills have PRIME marker before Step 1
3. All 3 skills have LEARN marker after last step
4. JIT markers at every step that produces evidence-grounded decisions
5. Query templates use correct syntax (`--types pattern --subtypes approach,risk`)
6. epic-plan reads epic-design's learning record in PRIME
7. epic-close reads all previous learning records in PRIME
8. JIT heuristic updated to "grounded decision" framing

### MUST NOT
1. Not reimplement protocol logic in skills (markers + metadata only)
2. Not add JIT markers to mechanical steps (tag, push, tracking setup)
3. Not modify skill logic — only add metadata and markers
