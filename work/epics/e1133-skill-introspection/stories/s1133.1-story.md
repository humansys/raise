# Story: Introspection Aspect Definition

## User Story
As a lifecycle skill author,
I want a shared introspection aspect that defines PRIME/JIT/LEARN protocol,
so that all skills can participate in the learning chain without each implementing the protocol independently.

## Acceptance Criteria

### Scenario: Skill references introspection aspect
```gherkin
Given a lifecycle skill (e.g., rai-story-design)
When the skill declares `aspect: introspection` with phase metadata
Then the PRIME/JIT/LEARN protocol applies to that skill execution
```

### Scenario: LEARN produces learning record
```gherkin
Given a skill execution completes
When the LEARN phase runs
Then a learning record YAML is produced at `.raise/rai/learnings/{skill}/{work_id}/record.yaml`
And it contains process_fidelity, output_quality, pattern_metrics, and downstream_impact sections
```

### Scenario: Tier 1 guided prime
```gherkin
Given a skill with phase metadata declares Tier 1 queries
When the skill starts execution
Then suggested queries are presented to the LLM for refinement
And 3-5 relevant patterns are retrieved from the graph
```

### Scenario: Downstream enrichment protocol
```gherkin
Given story-design produced a learning record
When story-plan executes
Then story-plan reads story-design's learning record
And story-plan can enrich story-design's downstream_impact fields
```

## Examples (Specification by Example)

| Phase | Tier 1 Query Suggestion | LLM Refined Query | Results |
|-------|------------------------|-------------------|---------|
| story.design | "patterns for {module} design decisions" | "patterns for memory module design with graph integration" | 3 patterns |
| epic.plan | "sizing patterns for {complexity} epics" | "sizing patterns for M-sized epics with cross-cutting concerns" | 2 patterns |

## Notes
- ADR-014 documents the architectural decisions behind this design
- The aspect is defined once, referenced by all 10 lifecycle skills (S1133.3 and S1133.4)
- Bitter lesson: Tier 1 is guidance, not rigid template — LLM can reformulate queries
- Learning record schema must be rich enough for future MetaHarness/MCE-style self-improvement
- Poppendieck: measurement IS operation — LEARN is natural output, not extra work
