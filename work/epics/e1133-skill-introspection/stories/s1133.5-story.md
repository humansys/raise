# S1133.5: Story-Review Refactor

**Epic:** E1133 — Skill Introspection
**Tracker:** RAISE-1143
**Type:** docs
**Size:** S

## User Story

As the story-review skill, I need to aggregate learning records from all story skills, produce learning metrics, and simplify my evaluation responsibilities now that each skill measures its own effectiveness via LEARN.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: story-review aggregates learning records
  Given previous skills produced learning records
  When story-review executes
  Then it reads all records and produces an aggregate learning summary

Scenario: Learning metrics are produced
  Given aggregate learning data exists
  When story-review completes its retrospective
  Then it includes acceptance rate, gap rate, and pattern utility metrics

Scenario: Evaluation responsibility simplified
  Given each skill now measures via LEARN
  When story-review evaluates the story
  Then it focuses on retrospective, calibration, and meta-pattern synthesis
  And does not duplicate per-skill evaluation
```
