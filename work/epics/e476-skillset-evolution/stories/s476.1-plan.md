## Tasks

### T1: RED -- Regression tests for manifest-driven gate commands

**Description:** Write tests that verify each gate reads its command from
`load_manifest()` when a manifest is present, and falls back to defaults when
absent. Tests will fail because gates currently hardcode commands.

**Files to create/modify:**
- `tests/test_gates/test_builtin_gates.py` -- add new test classes

**TDD Cycle:**

- **RED:** Add 8 tests (2 per gate: manifest-present reads custom command,
  manifest-absent falls back to default). Each test patches `load_manifest` to
  return a `ProjectManifest` with a custom command string and asserts
  `subprocess.run` is called with the parsed command. These tests will fail
  because current gates never call `load_manifest`.
- **GREEN:** Deferred to T2.
- **REFACTOR:** Deferred to T2.

**Test details per gate:**

| Gate | Manifest field | Custom command | Expected argv |
|------|---------------|----------------|---------------|
| TestGate | `project.test_command` | `"npm test"` | `["npm", "test"]` |
| LintGate | `project.lint_command` | `"eslint src/"` | `["eslint", "src/"]` |
| TypeGate | `project.type_check_command` | `"tsc --noEmit"` | `["tsc", "--noEmit"]` |
| CoverageGate | `project.test_command` | `"npm test"` | `["npm", "test", "--cov", "--cov-report=term-missing", "-q"]` |

**Fallback tests:** Each gate also gets a test confirming that when
`load_manifest()` returns `None`, the original hardcoded command is used
(these tests should pass immediately since that is current behavior, but they
document the contract).

**Verification:**
```bash

# Expected: 8 new tests -- 4 manifest-present FAIL, 4 manifest-absent PASS
```

**Commit:** `test(s476.1): RED -- manifest-driven gate command tests`

### T2: GREEN + REFACTOR -- Gates read manifest with fallback

**Description:** Add `_get_command(context)` method to each gate class. The
method calls `load_manifest(context.working_dir)`, reads the relevant field,
uses `shlex.split()` to parse it into argv, and falls back to the current
hardcoded default. Update `evaluate()` to use `_get_command()`.

**Files to modify:**
- `src/raise_cli/gates/builtin/tests.py` -- add `_get_command`, update `evaluate`
- `src/raise_cli/gates/builtin/lint.py` -- same pattern, reads `lint_command`
- `src/raise_cli/gates/builtin/types.py` -- same pattern, reads `type_check_command`
- `src/raise_cli/gates/builtin/coverage.py` -- same pattern, reads `test_command` + appends coverage flags

**TDD Cycle:**

- **RED:** Tests from T1 are already failing.
- **GREEN:** Implement `_get_command()` in each gate:
  ```python
  def _get_command(self, context: GateContext) -> list[str]:
      manifest = load_manifest(context.working_dir)
      if manifest and manifest.project.<field>:
          return shlex.split(manifest.project.<field>)
      return [<current default>]
  ```
  Update `evaluate()` to call `self._get_command(context)` instead of inline list.
- **REFACTOR:** Add `import shlex` and `from raise_cli.onboarding.manifest import load_manifest` to each gate module. No shared base class (D4 decision: 3 lines of duplication is acceptable).

**Verification:**
```bash
uv run pytest tests/test_gates/test_builtin_gates.py -x --tb=short
# Expected: all tests pass (old + new)
uv run pyright src/raise_cli/gates/builtin/
uv run ruff check src/raise_cli/gates/builtin/
```

**Commit:** `feat(s476.1): gates read manifest commands with fallback`
