# S476.1 Retrospective: Builtin Gates Read Manifest

## What Was Delivered

Made the 4 builtin quality gates (TestGate, LintGate, TypeGate, CoverageGate) read
their commands from `.raise/manifest.yaml` instead of hardcoding Python-specific tools.
Each gate got a `_get_command(context)` method that calls `load_manifest()`, parses the
relevant field with `shlex.split()`, and falls back to the original hardcoded default
when the manifest is absent or the field is unset. Fully backwards compatible.

## What Went Well

- **Clean pattern application.** The same 3-line `_get_command()` pattern applied
  identically to all 4 gates. Design decision D4 (no shared base class, accept minor
  duplication) kept each gate self-contained and easy to understand.
- **TDD discipline.** RED phase wrote 8 tests first (2 per gate: manifest-present and
  manifest-absent). 4 failed as expected, 4 passed immediately (fallback = current
  behavior). GREEN phase made all 8 pass with minimal code changes.
- **Small blast radius.** Only 4 production files changed, each by ~10 lines. No new
  files, no schema changes, no new dependencies beyond `shlex` (stdlib).
- **Full suite green.** All 3693 tests pass after the change; pyright and ruff clean.

## What Could Improve

- **Test helper placement.** The `_manifest_with()` helper was added at the bottom of
  the test file. If more gates or manifest-driven tests appear later, this helper should
  move to a shared test fixture module.
- **CoverageGate coupling.** CoverageGate appends `--cov` flags to the test command,
  which assumes pytest-cov compatibility. For truly language-agnostic coverage, a
  separate `coverage_command` manifest field may be needed in a future story.

## Patterns Learned

- **Manifest-first with fallback** is now the standard pattern for all gates, matching
  the 5 skills that already use it. This makes RaiSE gates usable on non-Python projects
  without code changes.
- **`shlex.split()` for command parsing** is the right tool for converting a single
  manifest string (`"npm test"`) into an argv list (`["npm", "test"]`). Handles quoting
  and escaping correctly.
