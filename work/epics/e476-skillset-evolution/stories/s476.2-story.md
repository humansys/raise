# S476.2: Skills use manifest-first pattern

## User Story
As a developer using RaiSE on a non-Python project,
I want core skills to read tool commands from manifest
so that I don't need to edit skill files to use my stack's tools.

## Acceptance Criteria

Given a project with `.raise/manifest.yaml` containing `project.test_command`, `project.lint_command`, `project.type_check_command`
When I run `/rai-story-implement`, `/rai-story-plan`, or `/rai-bugfix`
Then the skill references the manifest commands for verification steps
And falls back to language detection + defaults when manifest is absent

## Scope
- Modify 3 skill SKILL.md files: rai-story-implement, rai-story-plan, rai-bugfix
- Use the same pattern already proven in rai-story-review and rai-story-close
- Changes are markdown only — no Python code changes
