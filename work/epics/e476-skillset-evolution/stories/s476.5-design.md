# S476.5: Design — Skillset Documentation

## Decision: Location

**Choice:** Astro docs site at `docs/src/content/docs/docs/guides/skillsets.mdx`

**Rationale:**
- `docs/` is an existing Starlight site with CLI reference pages already
- A guide page is discoverable alongside `cli/skill.mdx` and `cli/gate.mdx`
- Keeps project root clean (no loose markdown guides)
- Alternative rejected: `work/epics/` artifacts are internal, not user-facing

## Document Structure

Single `.mdx` page with three sections:

1. **What is a skillset** — overlay concept, directory layout under `.raise/skills/<name>/`
2. **Creating a custom skillset** — step-by-step: create dir, add SKILL.md overlays with frontmatter (`name`, `overlay`, `replaces`, `description`), reference manifest commands
3. **Gate configuration** — how `manifest.yaml` fields (`test_command`, `lint_command`, `type_check_command`) drive gate checks; how overlays reference these

## Reference Examples

- `raise-dev` (Python): pytest + ruff + pyright
- `raise-dev-ts` (TypeScript): vitest + eslint + tsc

Both live at `.raise/skills/` and will be shown inline as code blocks.

## Constraints

- No code changes — documentation only
- Single file deliverable
- XS scope: aim for ~80-120 lines of content
