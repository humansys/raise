# S476.3 Design: Create raise-dev Skillset

## Decisions

### 1. Overlay Format

Each overlay is a **full SKILL.md** placed at `.raise/skills/raise-dev/{skill-name}/SKILL.md`.
The `rai skill set diff` command compares these against builtins by matching directory names.
There is no partial-merge mechanism -- the overlay file replaces the entire builtin skill when active.

However, the overlay should contain **only the sections that differ** from the base skill,
plus enough frontmatter for identification. The resolution engine (S476.4) will handle merging.
For now: each overlay is a minimal markdown file with just the verification replacement content.

### 2. Content Per Overlay

All three overlays replace the verification/command-resolution sections with explicit Python commands:

```markdown
### Verification Commands (Python)

- **Test:** `uv run pytest --tb=short -x`
- **Lint:** `uv run ruff check`
- **Type check:** `uv run pyright --pythonversion 3.11`
```

Per-skill specifics:

| Skill | Section Replaced | Notes |
|-------|-----------------|-------|
| rai-story-implement | Step 3 (Verify Task) | Replace manifest lookup + language table with direct commands |
| rai-bugfix | Step 4 (Fix) verification | Replace priority chain with direct commands |
| rai-story-plan | Step 2 task template | Verification commands in task decomposition guidance |

### 3. Creation Method

Use `rai skill set create raise-dev --empty` to scaffold the directory, then manually create the 3 overlay subdirectories with SKILL.md files.

## Structure

```
.raise/skills/raise-dev/
  rai-story-implement/SKILL.md
  rai-bugfix/SKILL.md
  rai-story-plan/SKILL.md
```

## Risks

- **R1:** No merge/resolution engine yet (S476.4) -- overlays exist but are not automatically applied. Acceptable: they serve as documentation and future input.
- **R2:** Overlay format may evolve when S476.4 implements resolution. Keep overlays minimal to reduce rework.
