# S476.4 Design: Create raise-dev-ts Skillset

## Decisions

### 1. Mirror raise-dev Structure Exactly

Same 3 overlays, same directory layout, same frontmatter format. Only the commands differ.

```
.raise/skills/raise-dev-ts/
  rai-story-implement/SKILL.md
  rai-bugfix/SKILL.md
  rai-story-plan/SKILL.md
```

### 2. TypeScript Verification Commands

All three overlays replace verification sections with:

```bash
# Tests
npx vitest run

# Lint
npx eslint src/

# Type check
npx tsc --noEmit
```

### 3. Content Per Overlay

| Skill | Section Replaced | Notes |
|-------|-----------------|-------|
| rai-story-implement | Step 3 (Verify Task) | Same as raise-dev but TypeScript commands |
| rai-bugfix | Step 4 (Fix) verification | Same as raise-dev but TypeScript commands |
| rai-story-plan | Step 2 task template | Same as raise-dev but TypeScript commands |

### 4. Creation Method

Manual creation mirroring raise-dev. Each overlay SKILL.md has frontmatter with `overlay: raise-dev-ts` and minimal body with just the replacement commands.

## Risks

- **R1:** Same as S476.3 -- no merge engine yet. Overlays exist as documentation and future input.
