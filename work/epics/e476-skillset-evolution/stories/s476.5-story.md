# S476.5: Skillset documentation

## User Story
As a developer adopting RaiSE for a new project,
I want documentation on how to create custom skillsets and configure gates
so that I can adapt the framework to my stack without reading source code.

## Acceptance Criteria

Given a developer reads the skillset documentation
When they follow the guide to create a custom skillset
Then they understand overlay structure, gate configuration, and manifest integration

## Scope
- How to create custom skillsets (overlay directories under .raise/skills/)
- Gate configuration guide (manifest.yaml gate commands)
- Reference existing raise-dev and raise-dev-ts as examples
- XS story: documentation only, no code changes
