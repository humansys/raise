# S476.1 Design: Builtin Gates Read Manifest

## What & Why

The 4 builtin gates (TestGate, LintGate, TypeGate, CoverageGate) hardcode Python-specific
commands (`pytest`, `ruff check .`, `pyright`). The manifest already has fields for
`project.test_command`, `project.lint_command`, and `project.type_check_command`, and
`load_manifest()` already returns `ProjectManifest | None`. The gates just never read it.

This story closes that gap: each gate reads from manifest first, falls back to current defaults.

## Approach

Each gate gets a `_get_command(context)` method:

1. Call `load_manifest(context.working_dir)`
2. If manifest exists and the relevant field is set, use `shlex.split(field_value)`
3. Otherwise, return the current hardcoded default

The `evaluate()` method uses `_get_command()` output instead of its inline list.

## Concrete Examples

### Before (TestGate, hardcoded)

```python
def evaluate(self, context: GateContext) -> GateResult:
    result = subprocess.run(
        ["pytest", "-x", "--tb=short"],  # <-- hardcoded
        capture_output=True, text=True,
        cwd=str(context.working_dir),
    )
```

### After (TestGate, manifest-first with fallback)

```python
def _get_command(self, context: GateContext) -> list[str]:
    manifest = load_manifest(context.working_dir)
    if manifest and manifest.project.test_command:
        return shlex.split(manifest.project.test_command)
    return ["pytest", "-x", "--tb=short"]

def evaluate(self, context: GateContext) -> GateResult:
    cmd = self._get_command(context)
    result = subprocess.run(
        cmd,
        capture_output=True, text=True,
        cwd=str(context.working_dir),
    )
```

### When manifest is missing

`load_manifest()` returns `None`, so `_get_command()` returns the hardcoded default.
Behavior is identical to today. No exception, no log noise beyond existing debug line.

### CoverageGate specifics

CoverageGate reads `test_command` (not a separate coverage field) and appends coverage flags:

```python
def _get_command(self, context: GateContext) -> list[str]:
    manifest = load_manifest(context.working_dir)
    if manifest and manifest.project.test_command:
        base = shlex.split(manifest.project.test_command)
        return base + ["--cov", "--cov-report=term-missing", "-q"]
    return ["pytest", "--cov", "--cov-report=term-missing", "-q"]
```

## Components Affected

| File | Change |
|------|--------|
| `src/raise_cli/gates/builtin/tests.py` | Add `_get_command()`, use in `evaluate()` |
| `src/raise_cli/gates/builtin/lint.py` | Same pattern, reads `lint_command` |
| `src/raise_cli/gates/builtin/types.py` | Same pattern, reads `type_check_command` |
| `src/raise_cli/gates/builtin/coverage.py` | Same pattern, reads `test_command` + coverage flags |

No new files. No schema changes. No new dependencies (shlex is stdlib).

## Acceptance Criteria

| ID | Criterion | Testable |
|----|-----------|----------|
| AC1 | TestGate reads `project.test_command` from manifest; falls back to `pytest -x --tb=short` | Yes |
| AC2 | LintGate reads `project.lint_command` from manifest; falls back to `ruff check .` | Yes |
| AC3 | TypeGate reads `project.type_check_command` from manifest; falls back to `pyright` | Yes |
| AC4 | CoverageGate reads `project.test_command` + appends coverage flags; falls back to current default | Yes |
| AC5 | No manifest on disk = identical behavior to today (backwards compatible) | Yes |

## Key Decisions

- **D1:** Extract `_get_command()` helper per gate (not inline, not shared base class)
- **D2:** Use `shlex.split()` to parse manifest strings into argv lists
- **D3:** CoverageGate appends coverage flags to the test command from manifest
- **D4:** No shared utility module -- 3 lines of duplication per gate is acceptable for clarity
