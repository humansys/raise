# S476.1: Scope

## In Scope

- Modify TestGate to read `project.test_command` from `.raise/manifest.yaml`
- Modify LintGate to read `project.lint_command` from `.raise/manifest.yaml`
- Modify TypeGate to read `project.type_check_command` from `.raise/manifest.yaml`
- Modify CoverageGate to read `project.test_command` from `.raise/manifest.yaml` (coverage wraps test command)
- Fallback to current hardcoded defaults when manifest is absent or field is missing
- Unit tests for each gate with manifest present and absent

## Out of Scope

- Changing the manifest schema (fields already exist)
- Modifying skill markdown files (that is S476.2)
- Adding new gate types
- CI/CD integration
- Auto-detection of project language

## Done Criteria

- All four gates read from manifest when available
- All four gates fall back to hardcoded defaults when manifest is absent
- Existing test suite passes (no regressions)
- New tests cover manifest-present and manifest-absent scenarios for each gate
- Type checks pass (pyright strict)
- Linting passes (ruff)
