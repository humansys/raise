## Story Scope: S16.1 — Graph Diff Engine

**Epic:** E16 (Incremental Coherence)
**Size:** M
**Milestone:** M1 (Walking Skeleton)

**In Scope:**
- Pydantic models for graph diff: GraphDiff, NodeChange, EdgeChange, ChangeType
- Comparison algorithm: diff two UnifiedGraph snapshots (old vs new)
- Structured change set output: added/removed/modified nodes and edges
- Impact categorization (what changed and severity)
- CLI entry point: `raise memory build --diff` (or similar)

**Out of Scope:**
- Doc updating (S16.2)
- AI regeneration (S16.3)
- Lifecycle integration (S16.4)
- Graph persistence/caching (use in-memory for now)

**Done Criteria:**
- [ ] GraphDiff Pydantic model with NodeChange/EdgeChange types
- [ ] Diff algorithm compares two UnifiedGraph instances correctly
- [ ] Handles: added nodes, removed nodes, modified nodes, edge changes
- [ ] Conservative matching: ignores metadata-only noise
- [ ] Unit tests with >90% coverage
- [ ] CLI command produces diff output
- [ ] All quality gates pass (pyright, ruff, tests)
- [ ] Retrospective complete
