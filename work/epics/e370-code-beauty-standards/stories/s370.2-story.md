# Story S370.2: Expand Tooling Gates

## User Story

**As a** developer working on raise-commons,
**I want** expanded static analysis rules, architectural layer contracts, and a complexity cap enforced by tooling,
**So that** code quality standards are automatically verified and violations are caught before code review.

## Context

S370.1 drafted the code quality standard with 5 dimensions. This story makes Dimension 1 (Correctness & Safety) and parts of Dimension 2 (Architecture & Design) machine-enforceable by expanding our tooling configuration.

## Scope

### Ruff Expansion (+11 Rule Sets)

Enable incrementally, fix violations, commit per rule set:

| Rule Set | Category | Why |
|----------|----------|-----|
| S | flake8-bandit (security) | Catch security anti-patterns |
| C90 | McCabe complexity | Enforce max-complexity = 10 |
| PL | Pylint subset | Broad code quality checks |
| PERF | Perflint | Performance anti-patterns |
| FURB | refurb | Modern Python idioms |
| RET | flake8-return | Clean return patterns |
| ARG | flake8-unused-arguments | Dead parameter detection |
| ERA | eradicate | Commented-out code detection |
| PT | flake8-pytest-style | Test consistency |
| TC | flake8-type-checking | TYPE_CHECKING block optimization |
| A | flake8-builtins | Builtin shadowing prevention |

### import-linter

- Install import-linter as dev dependency
- Configure layer contracts per ADR-001:
  - Presentation (cli) -> Application (skills, session, onboarding, doctor, governance, artifacts, context, config) -> Domain (adapters, providers) -> Core (models, schemas)
- All contracts must pass

### McCabe Complexity Cap

- Set max-complexity = 10 in Ruff C90 config
- Fix any functions exceeding the cap

## Strategy

Enable one rule set at a time, fix all violations, commit. Not a big bang.

## Acceptance Criteria

1. All 11 new Ruff rule sets enabled in pyproject.toml
2. Zero violations across all enabled rules
3. import-linter configured with ADR-001 layer contracts
4. All import-linter contracts pass
5. McCabe max-complexity = 10, all functions comply
6. All existing tests pass
7. Type checks pass (pyright)
