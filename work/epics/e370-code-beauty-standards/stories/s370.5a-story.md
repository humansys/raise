# Story: Quick Wins

## User Story
As a project maintainer,
I want the quick-win audit findings fixed,
so that the codebase passes basic API surface and observability checks.

## Acceptance Criteria

### Scenario: Flat API surface
```gherkin
Given schemas/, backlog/, tier/, engines/ lack __init__.py re-exports
When I add re-exports and __all__ to each
Then consumers can import from the package directly
```

### Scenario: Silent exceptions get logging
```gherkin
Given config/settings.py:71 and agents/copilot_plugin.py:113 catch Exception silently
When I add logger.debug() to each
Then failures are observable at debug level
```

### Scenario: Graph formatters moved
```gherkin
Given cli/commands/graph.py contains 5 _format_* functions
When I move them to output/formatters/graph.py
Then cli/commands/graph.py imports from output/formatters/
And the output/ pattern is consistent
```

### Scenario: Global removed
```gherkin
Given cli/main.py has _current_output_format module-level global
When I remove it and route through ctx.obj
Then no module-level mutable state remains in main.py
```
