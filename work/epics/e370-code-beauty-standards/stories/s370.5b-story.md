# S370.5b: Extract Helpers

## User Story (Connextra)

As a maintainer of raise-commons,
I want long functions decomposed into focused helper functions,
so that each unit is testable in isolation and readable at a glance.

## Acceptance Criteria (Gherkin)

```gherkin
Feature: Extract Helpers

  Scenario: CLI command functions decomposed (R5)
    Given the top 8 CLI command functions by size
    When helpers are extracted for discrete logic blocks
    Then no CLI command function exceeds 100 lines
    And all existing tests pass

  Scenario: Governance parser helpers extracted (R6)
    Given governance functions exceeding 100 lines in extractor.py, epic.py, backlog.py
    When parsing logic is extracted into focused helpers
    Then no governance function exceeds 100 lines
    And all existing tests pass

  Scenario: Discovery scanner split by language (R7)
    Given discovery/scanner.py contains all language-specific logic
    When language handlers are moved to sub-modules
    Then scanner.py delegates to language-specific modules
    And each sub-module is independently testable

  Scenario: GovernanceParser Protocol added (R8)
    Given governance parsers share a common interface
    When a GovernanceParser Protocol is defined
    Then each parser conforms to the Protocol
    And legacy except Exception clauses are justified with comments

  Scenario: ThreadPoolExecutor lifecycle fixed (R9)
    Given hooks/emitter.py creates ThreadPoolExecutor without proper lifecycle
    When lifecycle management is added
    Then the executor is properly shut down on cleanup
    And no resource leak warnings in tests

  Scenario: MCP Jira format handlers extracted (R10)
    Given adapters/mcp_jira.py has duplicated flat/nested branching
    When format handlers are extracted and DRY'd
    Then branching logic is consolidated
    And no function exceeds 100 lines

  Scenario: Global output format removed (R4, deferred from S370.5a)
    Given cli/main.py uses _current_output_format global
    When the error handler architecture is refactored
    Then the global is removed
    And output format is passed explicitly
```
