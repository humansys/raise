# Story S370.3: Code Audit Skill

## User Story

**As** Rai,
**I need** a systematic skill to audit code modules against our code-standards-draft.md,
**So that** I can evaluate the entire codebase consistently and produce actionable findings.

## Context

S370.1 defined the code quality standard with 5 dimensions (D1-D5). S370.2 expanded tooling gates to cover TOOL-classified criteria automatically (Ruff, Pyright, import-linter). What remains uncovered are the HUMAN and BOTH criteria that require judgment:

- **Naming quality** — Are names intention-revealing? Do they follow domain vocabulary?
- **Idiom quality** — Does the code use Pythonic patterns? Are there missed simplifications?
- **Test specification** — Do tests specify behavior or just exercise code?
- **API surface design** — Are public interfaces minimal, consistent, well-typed?
- **Architectural intent** — Does module structure communicate purpose? Are responsibilities clear?

This skill bridges the gap between what tools enforce and what a senior developer would evaluate in a code review.

## Acceptance Criteria

1. Skill file exists at `.raise/skills/code-audit.yaml`
2. Skill validates via `rai skill validate`
3. Follows existing skill template structure (YAML frontmatter, input/output contracts)
4. Covers all 5 dimensions with focus on HUMAN and BOTH criteria
5. Input contract accepts module path or list of modules
6. Output contract produces a scorecard with findings categorized by severity
7. Skill references code-standards-draft.md as the evaluation baseline
