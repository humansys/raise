### T2 (M): Write evaluation steps

**Goal:** Define the 5 evaluation steps that form the core of the audit skill.

**Steps:**
1. **Step 1 — Identify Scope:** Accept module path(s) or "all". Read `.raise/governance/code-standards-draft.md` as reference. List target files.
2. **Step 2 — Group by ADR-001 Layer:** Classify modules into Presentation (cli/), Application (handlers/), Domain (engines/, skills/), Core (schemas/, config/). Include mapping table.
3. **Step 3 — Evaluate per Module Group:** For each layer group, evaluate against HUMAN and BOTH criteria from all 5 dimensions. Include the specific criteria IDs to evaluate per dimension. For BOTH criteria, specify "evaluate only the human-judgment portion." Include Appendix A (AI tells) as additional checks.
4. **Step 4 — Categorize Findings:** Assign severity: critical (correctness/safety risk), must-fix (violates standard, clear fix), recommended (improves quality, judgment call), observation (pattern noted, no action needed). Each finding: file:line, criterion ID, description, severity, fix suggestion.
5. **Step 5 — Generate Scorecard:** Per-module scorecard with pass/partial/fail per dimension. Summary report aggregating across all modules. Highlight systemic patterns (same issue across multiple modules).

**Verification:**
- All 5 dimensions have evaluation guidance with specific criteria IDs
- HUMAN criteria from each dimension are listed
- BOTH criteria specify what the human-judgment portion is
- TOOL-only criteria are explicitly marked as "skip"
- Severity classification scheme documented

**Commit:** `feat(s370.3): T2 evaluation steps with criteria mapping`
