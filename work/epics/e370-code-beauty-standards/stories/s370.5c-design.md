# S370.5c Design: God Class Decomposition

## Problem

`context/builder.py` is 1,569 lines with 30 methods in a single class. It handles 7 distinct responsibilities: orchestration, loading, identity parsing, architecture parsing, structural extraction, utilities, and relationship inference. This violates SRP and makes the module hard to navigate.

## Value

Each responsibility becomes a focused module readable in isolation. The orchestrator reads like a table of contents. Contributors can find and modify loader logic without scrolling through 1,500+ lines.

## Approach

Extract method clusters into focused modules under `context/`. GraphBuilder becomes a thin orchestrator that delegates to loader functions.

**Key decision:** Extract as **module-level functions**, not new classes. The methods don't share state beyond `project_root` and `ide_config` â€” passing these as parameters is simpler than creating loader classes with the same constructor.

### Decomposition Plan

| --- | --- | --- | --- |

| `context/loaders/architecture.py` | `load_architecture`, `_parse_architecture_doc`, `_parse_module_doc`, `_parse_architecture_context`, `_parse_architecture_design`, `_parse_architecture_domain_model` | ~217 | Architecture doc parsing |
| `context/loaders/identity.py` | `load_identity`, `_extract_identity_values`, `_extract_identity_boundaries` | ~180 | Identity (values, boundaries) |
| `context/loaders/components.py` | `load_components` | ~37 | Component discovery docs |
| `context/loaders/artifacts.py` | `load_artifacts` | ~46 | Typed YAML artifacts |
| `context/extractors/structure.py` | `_extract_bounded_contexts`, `_extract_layers`, `_extract_constraints` | ~247 | Structural nodes/edges |
| `context/extractors/relationships.py` | `infer_relationships`, `_infer_learned_from`, `_infer_part_of`, `_infer_skill_edges`, `_infer_depends_on`, `_infer_release_part_of`, `_infer_keyword_relationships`, `_extract_keywords` | ~316 | Edge inference |
| `context/builder.py` | `__init__`, `build`, `load_governance`, `load_skills`, `_get_governance_extractor`, `_concept_to_node` | ~143 | Orchestration only |

### What stays in builder.py

- `__init__` â€” constructor
- `build()` â€” orchestration (calls loaders, builds graph)
- `load_governance()` â€” thin wrapper around GovernanceExtractor
- `load_skills()` â€” thin wrapper around extract_all_skills
- `_get_governance_extractor()` â€” factory
- `_concept_to_node()` â€” shared utility used by governance/skills

```python

    def build(self) -> Graph:
```
