# S370.6: Distill Governance

## User Story (Connextra)

As a contributor reading raise-commons governance, I want the code quality standard promoted from draft to permanent and the audit/review skills clearly delineated so that governance is codified and skills don't overlap.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Code standard is permanent
  Given the draft standard validated by S370.4 audit
  When I read .raise/governance/code-standards.md
  Then it has no "DRAFT" status
  And criteria reflect learnings from the audit

Scenario: Skills are reconciled
  Given /rai-code-audit and /rai-quality-review
  When I read their descriptions
  Then their scope is clearly differentiated
  And the code-audit skill references the permanent standard

Scenario: Patterns captured
  Given audit learnings from S370.1-S370.5
  When I count E370-sourced patterns
  Then there are at least 5

Scenario: Epic done criteria met
  Given S370.6 complete
  When I check E370 done criteria
  Then all applicable criteria are satisfied
```
