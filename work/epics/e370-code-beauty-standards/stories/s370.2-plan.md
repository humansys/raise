## T1 (M): Early Signal -- Dimension Violation Landscape

**Goal:** Run Ruff with all 11 new rule sets in check-only mode to measure the violation surface before fixing anything.

**Steps:**
1. Temporarily add all 11 rule sets to `[tool.ruff.lint] select` in pyproject.toml
2. Add per-file-ignores for tests: `S101`, `ARG001`
3. Add `[tool.ruff.lint.mccabe]` with `max-complexity = 10`
4. Run `uv run ruff check src/ tests/` -- capture output
5. Record total violations per rule set (S, C90, PL, PERF, FURB, RET, ARG, ERA, PT, TC, A)
6. Revert pyproject.toml to current state (this was measurement only)
7. Commit the violation counts in the commit message as a signal record

**Verification:** Violation counts captured in commit message. No code changes.

**Artifacts:** Commit message with violation inventory.

## T2 (L): Ruff Expansion -- Enable and Fix Incrementally

**Goal:** Enable all 11 rule sets with zero violations. One rule set at a time, risk-first order.

**Sub-tasks (one commit each):**

| # | Rule Set | Category | Action |
|---|----------|----------|--------|
| T2.1 | S | flake8-bandit | Enable, fix security findings, per-file-ignore S101 for tests |
| T2.2 | C90 | McCabe complexity | Enable with max-complexity=10, decompose complex functions |
| T2.3 | PL | Pylint subset | Enable, fix convention/error/refactor violations; per-rule ignores for noisy PLR09xx if needed |
| T2.4 | PERF | Perflint | Enable, fix unnecessary allocations |
| T2.5 | FURB | refurb | Enable, modernize idioms |
| T2.6 | RET | flake8-return | Enable, fix mixed return styles |
| T2.7 | ARG | flake8-unused-arguments | Enable, fix dead parameters; per-file-ignore ARG001 for tests |
| T2.8 | ERA | eradicate | Enable, remove commented-out code |
| T2.9 | PT | flake8-pytest-style | Enable, fix test style inconsistencies |
| T2.10 | TC | flake8-type-checking | Enable, move imports behind TYPE_CHECKING |
| T2.11 | A | flake8-builtins | Enable, fix builtin shadowing |

**Decision rules per sub-task:**
- Prefer fix over noqa (design decision D1)
- noqa only for: false positives, intentional patterns (PAT-E-589), or fixes requiring larger refactor (defer to S370.5)
- Every noqa must have a justification comment
- Run `uv run ruff check src/ tests/` after each enable -- must pass with zero violations before next rule set

**Verification:** After T2.11, `uv run ruff check src/ tests/` passes with all 20 rule sets (9 existing + 11 new) and zero violations.

## T3 (M): import-linter Setup -- Architecture Enforcement

**Goal:** Add import-linter and configure ADR-001 layer contracts.

**Steps:**
1. Add import-linter as dev dependency: `uv add --dev import-linter`
2. Configure `[tool.importlinter]` in pyproject.toml:
   - `root_packages = ["raise_cli"]`
   - Layer contract matching ADR-001's 4-layer architecture:
     - L1 (Presentation): `raise_cli.cli | raise_cli.output | raise_cli.viz`
     - L2 (Orchestration): `raise_cli.session | raise_cli.context | raise_cli.onboarding | raise_cli.handlers | raise_cli.doctor | raise_cli.governance | raise_cli.artifacts | raise_cli.skills | raise_cli.backlog | raise_cli.hooks | raise_cli.gates | raise_cli.publish | raise_cli.agents`
     - L3 (Engine): `raise_cli.engines | raise_cli.discovery | raise_cli.graph | raise_cli.tier | raise_cli.memory`
     - L4 (Foundation): `raise_cli.core | raise_cli.config | raise_cli.schemas | raise_cli.adapters | raise_cli.rai_base`
3. Run `uv run lint-imports`
4. Fix any layer violations by adjusting imports (not architecture, per scope.md)
5. Commit

**Verification:** `uv run lint-imports` passes with zero violations.
