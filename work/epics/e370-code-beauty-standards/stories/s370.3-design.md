# S370.3 Design: Code Audit Skill

## Problem

We have a code standard (37 criteria across 5 dimensions) and tooling gates (Ruff, Pyright, import-linter) covering the 11 TOOL-only criteria. But 15 criteria are HUMAN-only and 11 are BOTH — these require systematic judgment that no tool can enforce. Without a structured skill, audit quality depends on whoever remembers to check what.

## Approach

Create `src/raise_cli/skills_base/rai-code-audit/SKILL.md` that:

1. Takes input: module path(s) or "all" for full codebase
2. Loads `.raise/governance/code-standards-draft.md` as the evaluation reference
3. Groups modules by ADR-001 architectural layer (Presentation, Application, Domain, Core)
4. For each module group: reads all files, evaluates against HUMAN and BOTH criteria from each dimension
5. Skips TOOL-only criteria (already enforced by S370.2 tooling)
6. Produces findings with severity (critical / must-fix / recommended / observation)
7. Outputs a scorecard per module showing pass/partial/fail per dimension
8. Aggregates into a summary report

## Key Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Evaluate by ADR-001 layer, not alphabetically | Groups related modules for cross-module pattern detection |
| D2 | HUMAN + BOTH criteria only | TOOL-only criteria already green from S370.2 gates |
| D3 | Findings cite specific file:line | Consistent with quality-review; actionable |
| D4 | pass/partial/fail per dimension | No false precision from numeric scores |

## Differences from Existing Skills

| Aspect | /rai-quality-review | /rai-code-audit |
|--------|--------------------|--------------------|
| Scope | Story (changed files) | Codebase (modules) |
| Trigger | After story-implement | On-demand or scheduled |
| Evaluation basis | Semantic correctness heuristics | Governance standard criteria |
| Input | Story ID + git diff | Module paths or "all" |
| Output | Findings + verdict | Scorecard + findings |

## Acceptance Criteria

### MUST
- AC1: Skill file at `src/raise_cli/skills_base/rai-code-audit/SKILL.md`
- AC2: Covers all 5 dimensions (D1-D5) with focus on HUMAN/BOTH criteria
- AC3: Produces per-module scorecard with severity-categorized findings
- AC4: Groups evaluation by ADR-001 architectural layer
- AC5: `uv run rai skill validate rai-code-audit` passes

### SHOULD
- SC1: References code-standards-draft.md criteria IDs (D1.1, D2.6, etc.)
- SC2: Includes guidance on what to skip (TOOL-only criteria)
