# S370.5f: Migrate Loaders to YAML

## User Story (Connextra)

As a contributor reading raise-commons, I want identity and architecture loaders to read structured YAML files so that data loading is deterministic and free of fragile regex parsing.

## Acceptance Criteria (Gherkin)

```gherkin
Scenario: Identity loader reads from core.yaml
  Given a core.yaml with values, boundaries, and principles
  When load_identity() is called
  Then it returns the same graph nodes as before
  And no regex or markdown parsing is used

Scenario: Architecture loader reads pure YAML
  Given architecture docs as pure YAML files
  When load_architecture() is called
  Then it returns the same graph nodes as before
  And no markdown frontmatter extraction is used

Scenario: rai init generates YAML artifacts
  Given a fresh project initialization
  When rai init runs
  Then core.yaml is generated instead of core.md for identity
  And architecture docs are generated as pure YAML

Scenario: All existing tests pass
  Given the migrated loaders
  When I run the full test suite
  Then all tests pass with no modifications

Scenario: Type checks and lint pass
  Given the migrated code
  When I run pyright and ruff
  Then 0 errors reported
```
