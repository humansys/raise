# S474.5 Scope: Document CI=Skills Guardrail

## In Scope

- Document the CI=Skills parity principle as a governance guardrail
- Explain the manifest-driven architecture: how gates read commands from `.raise/manifest.yaml`
- Explain how to add, modify, or remove gate commands (developer workflow)
- Reference the pre-commit hook from S474.4 as the local enforcement mechanism
- Explain the relationship: manifest.yaml -> gates -> pre-commit hook -> CI pipeline

## Out of Scope

- Code changes (this is a docs-only story)
- Changes to gate framework or manifest structure
- CI pipeline modifications
- New gates or hooks

## Technical Approach

1. Create a guardrail document (e.g., `docs/guardrails/ci-skills-parity.md` or within `.raise/`) that:
   - States the principle: "CI and local gates MUST check the same things"
   - Explains the single source of truth: `.raise/manifest.yaml`
   - Shows the flow: manifest -> `rai gate check` -> pre-commit hook -> CI jobs
   - Provides a checklist for adding a new gate command

2. Ensure the document is referenced from the epic scope and discoverable via project navigation
