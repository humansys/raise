## T1: Write the Guardrail Document

**Goal:** Create `.raise/governance/guardrails/ci-skills-parity.md` codifying the CI=Skills parity principle.

**Steps:**

1. Create directory `.raise/governance/guardrails/`
2. Write the guardrail document with these sections:
   - **Principle statement** -- CI pipeline and local gates execute the same commands, driven by `.raise/manifest.yaml`
   - **Architecture flow** -- Diagram showing manifest as single source of truth feeding `rai gate check`, pre-commit hook, and CI pipeline
   - **Current commands** -- Table mapping manifest keys to actual commands and where they run (pre-commit, CI, `rai gate check`)
   - **How to add/modify a gate command** -- Developer checklist (update manifest, verify local, verify CI, verify hook)
   - **Known asymmetries** -- Intentional exceptions: pre-commit skips tests, CI adds security scanning, pre-commit has housekeeping hooks
   - **Cross-references** -- Links to `gate-code.md`, pre-commit config, CI pipeline, manifest
3. Verify document renders correctly (section headers, table, code blocks)

**Input files to read:**
- `.raise/manifest.yaml` -- current gate commands
- `.raise/gates/gate-code.md` -- existing gate documentation
- `.pre-commit-config.yaml` (if exists) or `.git/hooks/pre-commit` -- pre-commit setup

**Files:**
