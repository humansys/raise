# Design: S474.2 — Configurable Gates + FormatGate

**Story:** S474.2 | **Epic:** E474 Gate Reliability
**Complexity:** Moderate (4 components, established pattern)

## Problem

Gates hardcode Python commands (pytest, ruff, pyright). A TypeScript project using RaiSE would get false failures. Also, no FormatGate exists — `ruff format --check` is not covered.

## Value

Gates become language-agnostic by reading commands from `.raise/manifest.yaml`. Any project that runs `rai init` gets gates that match its stack. FormatGate closes the CI parity gap.

## Approach

1. Add `format_command` to `ProjectInfo` model and language detection table
2. Create a shared helper that reads a command from manifest and runs it
3. Refactor all 4 existing gates to use the helper instead of hardcoded commands
4. Create FormatGate using the same helper
5. Update gate-code.md to reference manifest-driven gates

## Example: Gate Helper

```python
# src/raise_cli/gates/builtin/_runner.py
def run_manifest_command(
    gate_id: str,
    manifest_key: str,
    description: str,
    context: GateContext,
) -> GateResult:
    """Read command from manifest and execute it."""
    manifest = load_manifest(context.working_dir)
    if manifest is None:
        return GateResult(passed=False, gate_id=gate_id,
                          message="No .raise/manifest.yaml found")

    command = getattr(manifest.project, manifest_key, None)
    if command is None:
        return GateResult(passed=True, gate_id=gate_id,
                          message=f"Skipped — {manifest_key} not configured")

    result = subprocess.run(
        command.split(), capture_output=True, text=True,
        cwd=str(context.working_dir),
    )
    passed = result.returncode == 0
    return GateResult(
        passed=passed, gate_id=gate_id,
        message=description if passed else f"{description} failed",
        details=(result.stdout,) if not passed else (),
    )
```

## Example: Refactored LintGate

```python
class LintGate:
    gate_id: ClassVar[str] = "gate-lint"
    description: ClassVar[str] = "Linting passes"
    workflow_point: ClassVar[str] = "before:release:publish"

    def evaluate(self, context: GateContext) -> GateResult:
        return run_manifest_command(
            self.gate_id, "lint_command", self.description, context,
        )
```

## Decision

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|-------------|
| D1 | Shared helper reads manifest | DRY — 5 gates share same pattern | Each gate reads manifest independently |
| D2 | Skip (pass) if command not configured | Non-blocking for projects without that tool | Fail if not configured |
| D3 | `command.split()` for execution | Simple, covers 99% of cases | `shlex.split()` (overkill for manifest commands) |

## Acceptance Criteria

- **AC1:** FormatGate reads `format_command` from manifest
- **AC2:** All 4 existing gates read from manifest instead of hardcoding
- **AC3:** `rai gate check --all` shows 5 gates, all pass
- **AC4:** Gates skip gracefully if their command is not configured
- **AC5:** `format_command` added to detection for all 14 languages
- **AC6:** gate-code.md updated to reference manifest-driven gates
- **AC7:** Tests cover: configured command passes, configured command fails, command not configured (skip)
