# Plan: S474.3 — Code Audit Against S370.1 Standard

**Story:** S474.3 | **Epic:** E474 Gate Reliability
**Size:** M (5-8 SP) | **Tasks:** 5

## Execution Order

Risk-first: T1 dimensions the problem, T2 enables rules (highest risk — unknown violation count), T3 fixes violations, T4 documents results, T5 validates everything passes clean.

---

## T1: Gap Analysis — Baseline Current Violations

**Goal:** Run all existing tools as-is, capture current violation counts per tool, establish baseline.

**Steps:**
1. Run `uv run ruff check src/ tests/` — capture output and count
2. Run `uv run pyright` — capture error count
3. Run `uv run lint-imports` — capture contract pass/fail
4. Compare current `pyproject.toml` Ruff select against the S370.1 standard table
5. Record baseline counts for each tool

**Files to read:**
- `pyproject.toml` (ruff + pyright config)
- `.raise/governance/code-standards-draft.md` (the standard)
- `.importlinter` or `pyproject.toml` (import-linter config)

**Verification:** Baseline counts documented (can be notes, not committed artifact)

**Size:** S (1 SP)
**Dependencies:** None

---

## T2: Config Expansion — Enable ANN, D Rules and Lower McCabe

**Goal:** Modify Ruff configuration to add missing rule families and tighten McCabe threshold. Run in check mode only (no fixes yet) to count new violations.

**Steps:**
1. Add `"ANN"` to `[tool.ruff.lint.select]`
2. Add `"D"` to `[tool.ruff.lint.select]`
3. Add per-file ignores for tests:
   - `ANN001`, `ANN201` — test function annotations optional
   - `D100`, `D103`, `D104` — test module/function/package docstrings optional
4. Lower `[tool.ruff.lint.mccabe] max-complexity` from 15 to 10
5. Add necessary `[tool.ruff.lint.pydocstyle]` convention (e.g., `convention = "google"`)
6. Run `uv run ruff check src/ tests/` — count violations per rule code
7. Triage: which violations are auto-fixable (`ruff check --fix --diff`) vs manual

**Files to modify:**
- `pyproject.toml`

**Verification:** `uv run ruff check src/ tests/ --statistics` shows violation counts by rule

**Size:** M (2 SP)
**Dependencies:** T1 (need baseline to compare)

---

## T3: Fix Violations — Apply Fixes Iteratively

**Goal:** Reach zero violations for all expanded rules across `src/` and `tests/`.

**Steps:**
1. Run `uv run ruff check src/ tests/ --fix` for auto-fixable violations
2. Manually fix remaining ANN violations (add type annotations)
3. Manually fix remaining D violations (add/fix docstrings)
4. For McCabe C901 violations (functions with complexity 11-15):
   - Refactor if straightforward
   - Add `# noqa: C901` with justification comment if refactoring is disproportionate
5. Run `uv run pyright` — fix any new errors caused by annotation changes
6. Run `uv run lint-imports` — fix or document any contract violations
7. Iterate until all three tools exit 0

**Files to modify:**
- Multiple files in `src/` and `tests/` (violations determine which)

**Verification:**
- `uv run ruff check src/ tests/` exits 0
- `uv run pyright` exits 0
- `uv run lint-imports` exits 0

**Size:** L (3 SP)
**Dependencies:** T2 (config must be expanded first)

**Risk:** This is the largest task. If violations exceed ~50 manual fixes, consider splitting by rule family (ANN first, then D). Commit after each rule family is clean.

---

## T4: Document Audit Results

**Steps:**

2. For each criterion, classify:
   - **PASS** — tool exits 0, cite command and zero output
   - **DEFERRED** — HUMAN-only or disproportionate to fix, with justification
3. Write `work/epics/e474-gate-reliability/stories/s474.3-audit-results.md`
4. Include decision D3 rationale (pyright `reportUnknownMemberType=false` kept)

**Files to create:**
- `work/epics/e474-gate-reliability/stories/s474.3-audit-results.md`

**Verification:** Document exists, all 5 dimensions listed, all deferred items have justification

**Size:** S (1 SP)
**Dependencies:** T3 (need clean tool runs for evidence)
