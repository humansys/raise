### Decision D3: pyright reportUnknownMemberType=false

Kept as-is. Enabling it produces hundreds of false positives from third-party libraries without complete type stubs (atlassian-python-api, tree-sitter, etc.). The strict mode already enforces all other type safety rules. This is a pragmatic trade-off documented in pyproject.toml.
