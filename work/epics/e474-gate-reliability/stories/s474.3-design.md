# Design: S474.3 — Code Audit Against S370.1 Standard

**Story:** S474.3 | **Epic:** E474 Gate Reliability
**Complexity:** Moderate (configuration + fixes across codebase, no new components)

## Problem

The codebase has not been systematically audited against the S370.1 code standard. Two rule families from the standard are not yet enabled in Ruff (`ANN`, `D`), McCabe complexity threshold is looser than the standard recommends (15 vs 10), and import-linter is not run in CI. Without this audit, we cannot claim the codebase meets its own quality criteria.

## Value

Validates that tooling enforcement works end-to-end. Ensures code quality meets the published standard before the framework is used by others. Surfaces any gaps between the standard and reality.

## Approach

### Phase 1: Gap Analysis (configuration audit)

Compare current tooling configuration against what the standard requires:

| Standard Ref | Required | Current Status | Gap |
|---|---|---|---|
| D1.1 (E722) | `E722` | Enabled via `E` | None |
| D1.2 (B904) | `B904` | Enabled via `B` | None |
| D1.3 (SIM115) | `SIM115` | Enabled via `SIM` | None |
| D1.4 (B006) | `B006` | Enabled via `B` | None |
| D1.5 (RET) | `RET` | Enabled | None |
| D1.6 (S) | `S` | Enabled | None |
| D2.1 (C4,SIM) | `C4`, `SIM` | Enabled | None |
| D2.4 (C901) | `C901` max=10 | Enabled, max=15 | Threshold too loose |
| D2.8 (UP) | `UP` | Enabled | None |
| D3.1 (ANN) | `ANN` | **Not enabled** | Must add |
| D5.1 (D) | `D` | **Not enabled** | Must add |
| D5.2 (I,F401) | `I`, `F401` | Enabled via `I`, `F` | None |
| D5.3 (PT) | `PT` | Enabled | None |
| D4.1/D4.8 | import-linter | Configured, 2 contracts | Not in CI |
| D3.1 (pyright) | strict mode | Enabled | `reportUnknownMemberType=false` |

### Phase 2: Enable Missing Rules

1. **Add `ANN` to Ruff select** — annotation enforcement. Likely needs per-file ignores for tests (`ANN001`, `ANN201`) and some inline ignores for complex overrides.
2. **Add `D` to Ruff select** — docstring enforcement. Needs per-file ignores for tests. May need specific D-rule ignores (e.g., `D100` module docstrings in tests, `D104` package docstrings).
3. **Lower McCabe to 10** — the standard says 10. Run first to see how many functions exceed it. Functions at 11-15 either get refactored or get an inline ignore with justification.

### Phase 3: Fix Violations

Run each tool and fix violations iteratively:

1. `uv run ruff check src/ tests/` — fix all new violations from ANN, D rules
2. `uv run pyright` — ensure strict still passes after annotation changes
3. `uv run lint-imports` — verify existing contracts pass
4. Iterate until zero violations

### Phase 4: Document Audit Results

Create an audit results document listing each of the 5 dimensions with:
- TOOL-enforced criteria: pass/fail with evidence (tool command + zero output)
- HUMAN-only criteria: noted as "ongoing review" (out of scope per s474.3-scope.md)
- Deferred items: any criteria where fixing would require disproportionate effort, with justification

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|-------------|
| D1 | Add ANN + D rules incrementally with per-file ignores | Pragmatic — getting to zero violations requires some ignores for tests and edge cases | Enable everything and fix the entire codebase (too risky for M story) |
| D2 | Lower McCabe to 10 | Matches standard. Functions 11-15 get reviewed; refactor or justify inline | Keep at 15 (contradicts standard) |
| D3 | Do not change pyright `reportUnknownMemberType=false` | Third-party libs without stubs would flood with false positives. Already documented in pyproject.toml | Enable it (too many false positives) |
| D4 | Do not add import-linter to CI in this story | CI changes are S474.5 scope. Verify contracts pass locally | Add to CI now (scope creep) |

## Example: Audit Results Format

```markdown
## D1: Correctness & Safety

| Criterion | Enforcement | Status | Notes |
|---|---|---|---|
| D1.1 Specific exceptions | TOOL (E722) | PASS | ruff check --select E722: 0 violations |
| D1.2 Exception chaining | BOTH (B904 + review) | PASS | B904: 0 violations |
| D1.7 Thread safety | HUMAN | DEFERRED | Ongoing review concern |
```

## Acceptance Criteria

- **AC1:** Ruff config includes `ANN` and `D` rule families
- **AC2:** McCabe max-complexity set to 10
- **AC3:** `uv run ruff check src/ tests/` exits 0 with expanded rules
- **AC4:** `uv run pyright` exits 0 in strict mode
- **AC5:** `uv run lint-imports` exits 0
- **AC6:** Audit results document exists listing all 5 dimensions with pass/fail/deferred status
- **AC7:** All deferred items have written justification
