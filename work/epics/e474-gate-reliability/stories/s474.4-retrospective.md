## What Was Done

1. Created `src/raise_cli/gates/hook.py` -- manifest-driven pre-commit hook module
   - Loads `.raise/manifest.yaml`, runs lint/format/type-check commands
   - Deliberately excludes `test_command` (too slow for commit-time)

2. Added CLI commands in `src/raise_cli/cli/commands/gate.py`:
   - `rai gate install-hook` -- writes bash shim to `.git/hooks/pre-commit`
   - `rai gate uninstall-hook` -- removes only rai-installed hooks (marker-based safety)
   - `--force` flag for overwriting non-rai hooks

3. 13 new tests (6 hook module + 7 CLI commands), all passing
4. AR verdict: PASS (clean, proportionate)
5. QR verdict: PASS WITH RECOMMENDATIONS -- fixed R1 (shlex.split for command parsing)

## What Went Well

- **Thin shim pattern worked perfectly.** Bash shim delegates to Python module, keeping all logic testable. Zero bash logic to debug.
- **Marker-based ownership** is a clean solution for safe uninstall. No external state, just a grep-able comment.
- **shlex.split fix** caught during QR was the right call -- str.split breaks on quoted arguments in commands.
- **Under estimate.** Story came in at ~60 min vs 90 min estimated. Clean design + reuse of existing manifest infrastructure.

## What Could Be Better

- The initial implementation used `command.split()` instead of `shlex.split()`. This was a real bug that would have surfaced with more complex manifest commands. QR caught it, but TDD should have included a test with quoted arguments from the start.

## Heutagogical Checkpoint

**What did I learn that changes how I work?**

1. **shlex.split is mandatory for shell command parsing.** Any time we pass a string command to `subprocess.run()`, use `shlex.split()`, not `str.split()`. This is now a committed pattern (PAT-E-019).

2. **Marker comments as ownership protocol.** Writing a marker comment into generated files is a lightweight alternative to maintaining a registry of "files we own." The marker serves as both documentation and machine-readable ownership signal.

3. **Thin shim > embedded logic.** When bridging Python and shell (git hooks, CI scripts), write the thinnest possible shell wrapper and keep all logic in Python where it's testable. The shim is 4 lines and trivially correct.

**Specific examples:**
- `shlex.split("ruff check --config \"pyproject.toml\"")` correctly produces `['ruff', 'check', '--config', 'pyproject.toml']` vs `str.split()` which would break.
- The `_HOOK_MARKER` constant in `gate.py` is used by both `install-hook` (to detect rai-owned hooks for safe re-install) and `uninstall-hook` (to refuse removing non-rai hooks).
- `run_hook()` is a pure function taking `working_dir: Path` and returning `int` -- fully testable with mocked `load_manifest` and `subprocess.run`.
