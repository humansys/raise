# S1040.2: Scope

## In Scope

- Migrate `session/close.py` — `save_session_state()` uses `adapter.write()`
- Migrate `session/journal.py` — `append_journal_entry()` uses `adapter.append()`
- Migrate `memory/writer.py` — `append_pattern()`, `append_session()` use `adapter.append()`
- Migrate developer profile writes to use `adapter.write()` with `root=~/.rai`
- Timestamp comparison before session state overwrite (closes RAISE-697)
- Consistent UTF-8 encoding on all writes via adapter
- Atomic writes via adapter prevent partial/corrupt files
- Full test coverage with TDD for all migrated paths

## Out of Scope

- Changing CLI function signatures (public API unchanged)
- Moving Pydantic validation (stays in CLI per DD-8, migrates to Domain Gates in 3.0)
- Changing file layout or directory structure
- New classes or abstractions (FilesystemAdapter instantiated where needed)
- Read-path migration (reads stay as-is for now)
- Steward/Domain layer (removed per DD-8)
- Concurrency/locking (single-process assumption)
