## T1: Add frontmatter validation to FilesystemDocsTarget

**Goal:** Add `_validate_frontmatter()` private method and call it from `publish()` before `write_text()`. Define `FrontmatterValidationError` in same module.

**Red:**
- Test: valid frontmatter with all required fields — file written, `PublishResult.success is True`
- Test: unparseable YAML frontmatter — `FrontmatterValidationError` raised, file not written
- Test: frontmatter with non-mapping YAML (e.g. a list) — `FrontmatterValidationError` raised
- Test: missing required base fields (title, status) — `FrontmatterValidationError` raised with `missing_fields`
- Test: story-level artifact missing `story_id` — error raised listing `story_id`
- Test: epic-level artifact with `epic_id` but no `story_id` — passes
- Test: content without `---` delimiters — file written as-is, no validation
- Test: existing tests still pass unchanged (publish writes file, creates dirs, etc.)

**Green:**
1. Define `FrontmatterValidationError` in `filesystem_docs.py`:
   - `message: str`, `missing_fields: list[str]`
2. Add `_validate_frontmatter(self, content: str, doc_type: str) -> None`:
   - Detect frontmatter: `content.startswith("---\n")`
   - Find closing `\n---\n` (or `\n---` at EOF)
   - No frontmatter → return (pass through)
   - Parse with `yaml.safe_load()` — catch `yaml.YAMLError`, wrap in `FrontmatterValidationError`
   - Check `isinstance(data, dict)` — reject non-mapping
   - Build required fields set: base `{"title", "status"}`, add `epic_id`/`story_id` based on frontmatter keys and `doc_type`
   - Compute `missing = required - data.keys()` — raise if non-empty
3. Add `import yaml` to `filesystem_docs.py`
4. Call `self._validate_frontmatter(content, doc_type)` in `publish()` after path check, before `write_text()`

**Refactor:** None expected — method is small and self-contained.

**Verify:**
- All new tests pass
- All existing `TestFilesystemDocsTarget` tests pass unchanged
- `uv run pytest packages/raise-cli/tests/adapters/test_filesystem_docs.py`
- `uv run pyright packages/raise-cli/src/raise_cli/adapters/filesystem_docs.py`
