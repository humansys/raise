# Design: S1040.1 — FilesystemAdapter Atomic I/O Primitives

**Story:** S1040.1 | **Epic:** E1040 Local Persistence Adapter
**Complexity:** Simple (1 component, no external integrations)

## Problem

Stewards need atomic I/O primitives that prevent data corruption. Today, CLI code uses raw `path.write_text()` which can leave partial or corrupt files on crash. This is the class of bugs exemplified by RAISE-697.

## Value

Foundation for the Steward + Adapter architecture (DD-1). All three stewards (session, memory, developer) depend on this adapter for safe file operations.

## Approach

Single `FilesystemAdapter` class with 4 methods:

1. `write(path, content)` — write to temp file in same directory, then atomic `os.rename`
2. `append(path, line)` — read existing content + new line, write atomically (temp + rename)
3. `read(path) -> str` — read file content, raise `FileNotFoundError` if missing
4. `list(pattern) -> list[Path]` — glob relative to root

Constructor: `FilesystemAdapter(root: Path)` — stores resolved absolute path. All method paths are resolved relative to root.

Design decisions from epic:
- **DD-4:** Single concrete class, no protocol until second backend
- **DD-3:** Zero domain knowledge — stewards handle validation, adapter handles bytes

## Examples

```python
from pathlib import Path
from raise_cli.adapters.filesystem_adapter import FilesystemAdapter

# Construction
adapter = FilesystemAdapter(root=Path("/project/.raise"))

# Atomic write — creates parent dirs, writes via temp + rename
adapter.write(Path("rai/personal/session-state.yaml"), "current_work:\n  epic: E1040\n")
# Result: /project/.raise/rai/personal/session-state.yaml exists with content

# Atomic append — for JSONL files
adapter.append(Path("rai/memory/patterns.jsonl"), '{"id": "PAT-599", "content": "atomic writes"}')
# Result: line appended with trailing newline, or new file created

# Read
content = adapter.read(Path("rai/personal/session-state.yaml"))
# Returns: "current_work:\n  epic: E1040\n"

# Read non-existent — raises FileNotFoundError
adapter.read(Path("does/not/exist.yaml"))

# List — returns relative paths matching glob
paths = adapter.list("rai/personal/sessions/*/state.yaml")
# Returns: [Path("rai/personal/sessions/SES-032/state.yaml"), ...]
```

Error cases:
```python
# Write to read-only directory — PermissionError propagates
adapter.write(Path("readonly/file.yaml"), "content")

# Append to non-existent parent — creates directories first
adapter.append(Path("new/dir/file.jsonl"), '{"data": true}')
```

## Acceptance Criteria

| ID | Description | Type |
|----|-------------|------|
| AC1 | `write()` produces identical file content to direct `write_text()` | MUST |
| AC2 | `write()` is atomic — crash mid-write leaves original file unchanged | MUST |
| AC3 | `append()` adds exactly one line (with trailing newline) to file | MUST |
| AC4 | `append()` to non-existent file creates it with parent dirs | MUST |
| AC5 | `read()` returns content, raises `FileNotFoundError` for missing | MUST |
| AC6 | `list()` returns paths matching glob pattern relative to root | MUST |
| AC7 | All methods resolve paths relative to constructor root | MUST |
| AC8 | Stale temp files from crashed writes don't accumulate indefinitely | SHOULD |
| AC9 | No domain knowledge in the adapter (no YAML parsing, no Pydantic) | MUST NOT |
| AC10 | No concurrency handling (single-process only for now) | MUST NOT |

## Technical Notes

- Use `tempfile.NamedTemporaryFile(dir=target_dir, delete=False)` for temp file in same directory
- `os.rename()` is atomic on POSIX when source and target are on same filesystem (guaranteed by same-dir temp)
- Append reads existing content, appends line with `\n`, writes atomically (not `open('a')` — that's not atomic for the full file)
- Constructor stores `root` as `Path.resolve()` absolute path
- Parent directories created with `mkdir(parents=True, exist_ok=True)` before temp file creation
