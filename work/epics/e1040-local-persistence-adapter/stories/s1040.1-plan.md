## Task 1: FilesystemAdapter — write + read + list (core)

**Goal:** Create `FilesystemAdapter` class with constructor and three core methods.

**Files:**
- `src/raise_cli/adapters/filesystem_adapter.py` (new)
- `tests/test_filesystem_adapter.py` (new)

**Implementation:**
1. Constructor `__init__(self, root: Path)` — stores `root.resolve()` as absolute path
2. `write(path: Path, content: str) -> None` — create parent dirs, write to `NamedTemporaryFile` in target dir, `os.rename` to target
3. `read(path: Path) -> str` — resolve relative to root, read content, raise `FileNotFoundError` if missing
4. `list(pattern: str) -> list[Path]` — glob relative to root, return relative paths

**TDD (RED first):**
- `test_write_creates_file` — AC1
- `test_write_creates_parent_dirs` — AC1, AC7
- `test_write_overwrites_existing` — AC2
- `test_write_content_matches` — AC1
- `test_read_returns_content` — AC5
- `test_read_missing_raises_file_not_found` — AC6
- `test_list_returns_matching_paths` — AC7
- `test_list_no_matches_returns_empty` — AC7
- `test_paths_are_relative_to_root` — AC8

**Verification:**

## Task 2: FilesystemAdapter — append (JSONL support)

**Goal:** Add `append()` method for atomic single-line append (JSONL use case).

**Files:**
- `src/raise_cli/adapters/filesystem_adapter.py` (modify)
- `tests/test_filesystem_adapter.py` (modify)

**Implementation:**
1. `append(path: Path, line: str) -> None` — read existing content (or empty string if file missing), append line with trailing `\n`, write atomically via temp + rename
2. Create parent dirs if needed
3. No `open('a')` — full read + append + atomic write for consistency (per design doc DD-3)
