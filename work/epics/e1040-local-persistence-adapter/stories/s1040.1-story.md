# S1040.1: FilesystemAdapter — Atomic I/O Primitives

## User Story (Connextra)

**As a** steward (session, memory, or developer),
**I want** atomic I/O primitives for write, append, read, and list,
**so that** partial writes never corrupt existing process-critical files.

## Acceptance Criteria (Gherkin)

### AC1: Atomic write

```gherkin
Scenario: Write creates file atomically via temp + rename
  Given a FilesystemAdapter with root "/tmp/test-root"
  When I call write("data/config.yaml", "key: value\n")
  Then the file at root / "data/config.yaml" contains "key: value\n"
  And no partial content was ever visible at that path
```

### AC2: Atomic write overwrites existing file

```gherkin
Scenario: Write replaces existing file without corruption window
  Given a file "data/config.yaml" with content "old"
  When I call write("data/config.yaml", "new")
  Then the file contains "new"
  And at no point was the file empty or partially written
```

### AC3: Atomic append for JSONL

```gherkin
Scenario: Append adds a line atomically
  Given a file "log/events.jsonl" with content '{"a":1}\n'
  When I call append("log/events.jsonl", '{"b":2}')
  Then the file contains '{"a":1}\n{"b":2}\n'
```

### AC4: Append creates file if not exists

```gherkin
Scenario: Append to non-existent file creates it
  Given no file at "log/new.jsonl"
  When I call append("log/new.jsonl", '{"first":true}')
  Then the file contains '{"first":true}\n'
```

### AC5: Read returns file content

```gherkin
Scenario: Read returns string content
  Given a file "data/config.yaml" with content "key: value\n"
  When I call read("data/config.yaml")
  Then the result is "key: value\n"
```

### AC6: Read raises on missing file

```gherkin
Scenario: Read raises FileNotFoundError for missing file
  Given no file at "data/missing.yaml"
  When I call read("data/missing.yaml")
  Then a FileNotFoundError is raised
```

### AC7: List returns matching paths

```gherkin
Scenario: List returns paths matching glob pattern
  Given files "data/a.yaml" and "data/b.yaml" and "data/c.json"
  When I call list("data/*.yaml")
  Then the result contains Path("data/a.yaml") and Path("data/b.yaml")
  And the result does not contain Path("data/c.json")
```

### AC8: All paths are relative to root

```gherkin
Scenario: Paths are scoped to adapter root
  Given a FilesystemAdapter with root "/project/.raise"
  When I call write("sessions/current.yaml", "data")
  Then the file is written to "/project/.raise/sessions/current.yaml"
```
