# S1040.1: Scope

## In Scope

- `FilesystemAdapter` class with constructor `__init__(self, root: Path)`
- `write(path, content)` — atomic overwrite via temp file + `os.rename`
- `append(path, line)` — atomic append for JSONL (single line + newline)
- `read(path) -> str` — read file content, raise `FileNotFoundError` if missing
- `list(glob) -> list[Path]` — return paths matching glob, relative to root
- All paths relative to root (no escaping root directory)
- Parent directory auto-creation for write and append
- Full test coverage with TDD

## Out of Scope

- Protocol/ABC for adapter (DD-4: no protocol until second backend)
- Domain validation (steward responsibility, not adapter)
- Concurrency/locking (single-process assumption for v1)
- Binary file support
- Compression or encoding options
- Any CLI integration (that's S1040.3)

## Done Criteria

- [ ] `FilesystemAdapter` class exists with typed constructor
- [ ] `write()` is atomic (temp + rename, never partial content visible)
- [ ] `append()` is atomic for single-line JSONL entries
- [ ] `read()` returns content or raises `FileNotFoundError`
- [ ] `list()` returns relative paths matching glob pattern
- [ ] All paths are relative to root, no path traversal possible
- [ ] Full test coverage (all 8 acceptance criteria)
- [ ] Type annotations on all code, pyright strict passes
- [ ] Linting passes

## Technical Notes

- Use `tempfile.NamedTemporaryFile` in same directory as target for atomic rename
- `os.rename` is atomic on POSIX when source and target are on same filesystem
- Append uses `open(path, 'a')` with explicit newline termination
- Constructor stores `root` as resolved absolute `Path`

## Dependencies

- None (foundation story)
