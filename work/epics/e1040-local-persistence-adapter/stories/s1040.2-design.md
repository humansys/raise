# S1040.2 Design: Migrate CLI Writes to FilesystemAdapter

## Problem & Value

CLI write operations (session state, journal, patterns, sessions, developer profile) use raw `Path.write_text()` and `open("a")` — no atomicity, no crash safety, no stale-overwrite protection. FilesystemAdapter (S1040.1) provides atomic write/append primitives. Migrating closes the gap between having the adapter and actually using it, and enables timestamp protection for RAISE-697.

## Complexity: Moderate

- 4 files to modify, 1 schema to extend
- Mechanical migration pattern (same for each site)
- One non-trivial addition: timestamp protection on session state

## Approach

Instantiate `FilesystemAdapter(root=...)` at each write call site. Replace raw `write_text`/`open("a")` with `adapter.write()`/`adapter.append()`. Function signatures stay the same — internal refactoring only.

For timestamp protection (RAISE-697): add a `last_modified` field to `SessionState`, compare on-disk timestamp vs incoming before overwriting.

No new classes. No Steward layer. No read-path changes.

## Write Site Audit

### Site 1: `save_session_state` — session/state.py, line 187

**Current:**
```python
def save_session_state(project_path: Path, state: SessionState, session_id: str | None = None) -> None:
    state_path = get_session_state_path(project_path, session_id)
    state_path.parent.mkdir(parents=True, exist_ok=True)
    data = state.model_dump(mode="json")
    content = yaml.dump(data, default_flow_style=False, allow_unicode=True, sort_keys=False)
    state_path.write_text(content, encoding="utf-8")  # <-- raw write
```

**After:**
```python
def save_session_state(project_path: Path, state: SessionState, session_id: str | None = None) -> None:
    state_path = get_session_state_path(project_path, session_id)
    rel_path = state_path.relative_to(project_path)

    # Timestamp protection: reject stale overwrites (RAISE-697)
    if state_path.exists():
        existing = load_session_state(project_path, session_id)
        if existing and existing.last_modified and state.last_modified:
            if state.last_modified < existing.last_modified:
                raise StaleWriteError(state_path, existing.last_modified, state.last_modified)

    # Stamp outgoing data
    state = state.model_copy(update={"last_modified": datetime.now(UTC).isoformat()})
    data = state.model_dump(mode="json")
    content = yaml.dump(data, default_flow_style=False, allow_unicode=True, sort_keys=False)

    adapter = FilesystemAdapter(root=project_path)
    adapter.write(rel_path, content)
```

## Schema Change: `SessionState.last_modified`

Add `last_modified: str | None = None` to `SessionState` in `schemas/session_state.py`:

```python
class SessionState(BaseModel):
    # ... existing fields ...
    last_modified: str | None = Field(
        default=None,
        description="ISO 8601 timestamp of last write. Used for stale-overwrite protection.",
    )
```

## Error Type: `StaleWriteError`

New exception in `adapters/` or `session/state.py`:

```python
class StaleWriteError(Exception):
    """Raised when a session state write would overwrite newer data."""
    def __init__(self, path: Path, on_disk: str, incoming: str) -> None:
        super().__init__(f"Stale write rejected: {path} (on-disk={on_disk}, incoming={incoming})")
        self.path = path
        self.on_disk_timestamp = on_disk
        self.incoming_timestamp = incoming
```

## Acceptance Criteria (refined)

### MUST

1. `save_session_state` writes via `FilesystemAdapter.write()` — atomic temp+rename
2. `append_journal_entry` appends via `FilesystemAdapter.append()` — atomic rewrite
3. `_append_jsonl` (patterns, sessions, calibrations) uses `FilesystemAdapter.append()`
4. `reinforce_pattern` uses `FilesystemAdapter.write()` instead of manual tmp+replace
5. `save_developer_profile` uses `FilesystemAdapter(root=~/.rai).write()`
6. Stale session state overwrites raise `StaleWriteError` (RAISE-697)
7. All writes are UTF-8 (enforced by adapter)

### SHOULD

1. `StaleWriteError` includes both timestamps for debugging
2. Crash during write preserves original file (inherited from adapter)

### MUST NOT

1. Change any public function signature
2. Modify read paths
3. Introduce new abstraction layers or classes (beyond StaleWriteError)

## Decisions

| ID | Choice | Rationale | Alternatives |
|----|--------|-----------|-------------|
| D1 | Instantiate adapter at call site | No DI needed — these are leaf functions, simple `FilesystemAdapter(root=X)` is clear | Inject adapter via parameter (changes signatures), singleton (global state) |
| D2 | `last_modified` as ISO string field on SessionState | Simplest — no schema migration, None means "legacy file, skip check" | Separate metadata file, filesystem mtime (unreliable across git ops) |
| D3 | `_append_jsonl` is the single migration point for all JSONL appends | One change migrates append_pattern, append_calibration, append_session | Migrate each caller individually (more code, more risk of missing one) |
| D4 | `StaleWriteError` defined in session/state.py | Colocated with the only function that raises it | adapters/ package (too generic), shared exceptions module (over-engineering) |
