# Design: S347.6 — `rai backlog sync` mirror generation

## Problem & Value

**Problem:** `governance/backlog.md` is manually maintained and drifts from Jira.
Developers must hand-edit the markdown table whenever they create or transition
epics in Jira, leading to stale data and duplicated effort.

**Value:** A single command regenerates the local backlog mirror from the
source-of-truth adapter. The file stays current, is available offline, and
feeds into session context bundles without manual editing.

## Approach

Add a `sync` subcommand to `rai backlog` that:

1. Resolves the active adapter via `resolve_adapter()` (same as all other commands)
2. Detects if it is the `FilesystemPMAdapter` — if so, short-circuit with an
   informative message (filesystem IS the source of truth, nothing to sync)
3. Queries the adapter via `search()` to fetch all epics
4. Formats results into the markdown table format matching current `backlog.md`
5. Writes the file atomically (write to temp, then rename) to prevent partial writes
6. On any adapter error, exits with clear message and does NOT touch the file

### Components Affected

| Component | Change | Notes |
|-----------|--------|-------|
| `src/rai_cli/cli/commands/backlog.py` | modify | Add `sync` subcommand |

| `tests/test_backlog_sync.py` | create | Unit tests for sync logic |

### Data Mutation Strategy (PAT-E-523)

The sync command is a **full regeneration** (not incremental merge). It replaces
the entire generated section of `governance/backlog.md`. If the adapter returns
an empty result set, the file is still written (with empty table + timestamp) —
this is the correct representation of "no epics in the remote adapter."

If the adapter raises an error (timeout, auth failure), the file is left
untouched. No partial writes, no silent fallback.

### Value Preservation (PAT-E-572)

The sync module provides domain value beyond a generic pass-through:
- Adapter type detection (filesystem no-op vs remote sync)
- Markdown formatting matching existing backlog.md conventions
- Atomic write guarantee
- Timestamp and source metadata for staleness detection

## Examples

### CLI Usage

```bash
# Default: sync from Jira (manifest default adapter)
$ rai backlog sync
Synced governance/backlog.md from jira (12 epics, 2026-03-03T14:30:00Z)

# With project filter
$ rai backlog sync -p RAISE
Synced governance/backlog.md from jira (8 epics, 2026-03-03T14:30:00Z)

# Explicit adapter
$ rai backlog sync -a jira
Synced governance/backlog.md from jira (12 epics, 2026-03-03T14:30:00Z)

# Filesystem adapter — no-op
$ rai backlog sync -a filesystem
Filesystem adapter is source of truth — nothing to sync.

# Adapter unavailable
$ rai backlog sync
Error: adapter 'jira' unavailable: MCP connection timeout
```

### Filesystem Detection Pattern

```python
from rai_cli.adapters.filesystem import FilesystemPMAdapter

# In the sync command handler:
pm = resolve_adapter(adapter)
if isinstance(pm, FilesystemPMAdapter):
    console.print("Filesystem adapter is source of truth — nothing to sync.")
    raise typer.Exit(0)
```

### Search Query for Epics

```python
# For Jira adapter — JQL to fetch epics
query = 'issuetype = Epic ORDER BY key ASC'
if project_filter:
    query = f'project = "{project_filter}" AND issuetype = Epic ORDER BY key ASC'

results = adapter.search(query, limit=200)
```

Note: The query format is adapter-specific (JQL for Jira). The `-p` filter
is injected into the JQL query, not applied post-hoc.

### Atomic Write Pattern

```python
import tempfile
from pathlib import Path

def _atomic_write(path: Path, content: str) -> None:
    """Write content to path atomically via temp file + rename."""
    tmp = path.with_suffix(".tmp")
    tmp.write_text(content, encoding="utf-8")
    tmp.rename(path)
```

## Acceptance Criteria

### MUST

1. `rai backlog sync` regenerates `governance/backlog.md` with data from the active adapter
2. Generated file includes timestamp header and source adapter name
3. `FilesystemPMAdapter` detection produces informative no-op message (exit 0)
4. Adapter failure (exception from `search()`) produces clear error and leaves file untouched
5. `-p`/`--project` flag filters results to a single project

### SHOULD

1. Table format is clean and readable (aligned columns)
2. `-a`/`--adapter` flag overrides default adapter (already handled by `resolve_adapter`)

### MUST NOT

1. Must NOT modify `governance/backlog.md` on adapter error (partial writes)
2. Must NOT silently fall back to another adapter
3. Must NOT add sync to the `ProjectManagementAdapter` protocol (this is CLI-only logic)
