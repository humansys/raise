# Scope: S347.6 — `rai backlog sync` mirror generation

## In Scope

- New `sync` subcommand in `rai backlog` CLI
- Query active adapter for epics (and optionally stories) via protocol operations
- Generate `governance/backlog.md` from query results
- Include generation timestamp and source adapter in output
- Filesystem adapter detection: no-op with informative message
- Fail-fast on adapter errors: no partial writes to backlog.md
- Optional `-p`/`--project` filter flag

## Out of Scope

- Automatic sync on every backlog write operation
- Bidirectional sync (backlog.md changes back to Jira)
- Sync of attachments, sprint data, or custom fields
- Webhook-triggered sync

## Done Criteria

1. `rai backlog sync` regenerates `governance/backlog.md` from active Jira adapter
2. Filesystem adapter returns informative no-op message
3. Adapter failure produces clear error, file untouched
4. Generated file includes timestamp and adapter source
5. Tests cover: Jira sync, filesystem no-op, adapter failure
6. Type checks and linting pass
