# Plan: S347.4 — BacklogHook Adapter-Aware

## Overview

Make `BacklogHook` adapter-agnostic: use manifest-aware resolution (`resolve_adapter(None)`), adapter-aware search strategy (label-first for Jira, direct key for FileAdapter), and rename internal helpers.

Size S — 4 tasks, sequential dependency chain.

## Tasks

### T1: Rename `_resolve_jira_key` to `_resolve_issue_key` and update `resolve_adapter(None)` (D1 + D3)

**Description:** Rename the function, update its call site, change `resolve_adapter()` to pass `None` instead of `"jira"`, and update module/class docstrings to adapter-agnostic language.

**Files:**
- `src/rai_cli/hooks/builtin/backlog.py` — rename function, change call arg, update docstrings

**TDD Cycle:**
- RED: Update existing tests to assert `resolve_adapter` is called with no hardcoded `"jira"` — verify old `_resolve_jira_key` name no longer importable.
- GREEN: Rename function, change `_resolve("jira")` → `_resolve(None)`, update docstrings.
- REFACTOR: Verify all existing tests still pass (no behavior change yet in search logic).

**Verification:**
```bash

uv run ruff check src/rai_cli/hooks/builtin/backlog.py
uv run pyright src/rai_cli/hooks/builtin/backlog.py
```

**AC refs:** Scenario 1 (manifest default), Scenario 2 (auto-detect)
**Size:** XS
**Dependencies:** None

### T2: Adapter-aware search — Jira label-first with summary fallback (D2)

**Description:** Replace the single summary-based JQL query with a label-first strategy: search `labels = "rai:{work_id}"` first, fall back to `summary ~ "{work_id}"` with a warning if no label match.

**Files:**
- `src/rai_cli/hooks/builtin/backlog.py` — rewrite `_resolve_issue_key` body
- `tests/test_hooks/test_backlog_hook.py` — new tests for label match, summary fallback, no match

**TDD Cycle:**
- RED: Write `test_jira_label_search_finds_issue` — assert adapter.search called with label JQL first. Fails because current code uses summary JQL.
- RED: Write `test_jira_label_miss_falls_back_to_summary` — assert two search calls (label, then summary) and warning logged.
- RED: Write `test_jira_both_searches_miss_returns_none` — assert None returned after both fail.
- GREEN: Implement label-first + summary-fallback in `_resolve_issue_key` with isinstance check for non-FileAdapter path.
- REFACTOR: Update existing tests whose `adapter.search` assertions now break due to changed JQL query.

**Verification:**
```bash

uv run ruff check src/rai_cli/hooks/builtin/backlog.py
uv run pyright src/rai_cli/hooks/builtin/backlog.py
```

**AC refs:** Scenario 3 (label match), Scenario 4 (summary fallback)
**Size:** S
**Dependencies:** T1
