### T2: _fetch_live_status() with timeout — success, timeout, unavailable tests

**Size:** S
**Files:** `src/rai_cli/session/bundle.py`, `tests/session/test_bundle.py`
**AC ref:** Scenarios "Live status merged", "Backlog adapter unavailable", "Timeout budget respected"
**Dependencies:** T1

**Description:**
Implement `_fetch_live_status()` with full logic:
1. Extract epic/story keys from `state.current_work`
2. Resolve adapter via `resolve_adapter(None)` (lazy import from
   `rai_cli.cli.commands._resolve`)
3. Call `adapter.get_issue(key)` for each non-empty key inside
   `ThreadPoolExecutor` with 5s timeout
4. Build `LiveBacklogStatus` from `IssueDetail.status` and
   `IssueDetail.summary`
5. On any exception (ImportError, SystemExit from resolve_adapter,
   TimeoutError, Exception), return degraded `LiveBacklogStatus` with
   `warning` describing the failure

Write three tests (all mock `resolve_adapter` at module level):
- `test_fetch_live_status_success` — mock adapter returns IssueDetail for
  epic and story; assert status/summary fields populated.
- `test_fetch_live_status_timeout` — mock adapter.get_issue to sleep > 5s;
  assert warning contains "timeout".
- `test_fetch_live_status_unavailable` — mock resolve_adapter to raise
  SystemExit; assert warning contains "unavailable".

**TDD cycle:**
- RED: Write all three tests. They fail (no implementation yet beyond stub).
- GREEN: Implement `_fetch_live_status()` with ThreadPoolExecutor, timeout,
  exception handling.
- REFACTOR: Extract helper if needed. Ensure no bare `except` — use
  `except Exception`.

**Verification:**
```bash

uv run pyright src/rai_cli/session/bundle.py
uv run ruff check src/rai_cli/session/bundle.py
```

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| `resolve_adapter` calls `sys.exit(1)` on failure | Kills process | Catch `SystemExit` in `_fetch_live_status()` |
| ThreadPoolExecutor leaks thread on timeout | Resource leak | Use context manager (`with`); thread completes eventually |
| Existing tests break if `_format_work_section` signature changes | CI red | Default parameter `live=None` maintains backward compat |
