# Scope: S347.4 — BacklogHook Adapter-Aware

## In Scope

- Remove hardcoded `"jira"` from `resolve_adapter()` call in BacklogHook — use `None` to respect manifest default
- Replace fuzzy JQL summary search with exact label match (`labels = "rai:{work_id}"`)
- Add fallback to summary search when label not found, with deprecation warning
- Update tests to cover both resolution paths (label-first, summary-fallback)

## Out of Scope

- Adding labels to existing Jira issues (manual or migration script)
- Changing the adapter resolution logic itself (done in S347.1)
- Modifying other hooks or the hook dispatch system
- FileAdapter key resolution (FileAdapter uses direct key lookup already)

## Done Criteria

1. `resolve_adapter()` in BacklogHook calls `_resolve("jira")` replaced with `_resolve(None)` — adapter comes from manifest default or auto-detect
2. `_resolve_jira_key()` searches by label first (`labels = "rai:{work_id}"`), falls back to summary search
3. Fallback path logs a warning recommending label adoption
4. All existing BacklogHook tests pass (no regression)
5. New tests cover: label match, summary fallback, no match, adapter resolution without hardcoded name
6. Types pass (`pyright`), lint passes (`ruff`)
