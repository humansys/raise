# Story: Session-start live backlog query

## User Story
As a RaiSE developer starting a session,
I want session-start to query the backlog adapter for the current epic/story status,
so that I see real state instead of a stale snapshot from session-state.yaml.

## Acceptance Criteria

### Scenario: Live status merged into context bundle
```gherkin
Given a session-state.yaml references epic E347 and story S347.5
  And the backlog adapter is available
When assemble_context_bundle() runs
Then the bundle includes live status from `rai backlog get`
  And the live status replaces stale session-state values
```

### Scenario: Backlog adapter unavailable (graceful degradation)
```gherkin
Given a session-state.yaml references current work
  And the backlog adapter is unreachable (MCP timeout, no adapter configured)
When assemble_context_bundle() runs
Then the bundle falls back to session-state.yaml values
  And a warning line is added: "backlog unavailable, showing cached state"
```

### Scenario: No current work in session state
```gherkin
Given session-state.yaml has no current epic or story
When assemble_context_bundle() runs
Then no backlog query is attempted
  And the bundle renders normally without live status
```

### Scenario: Timeout budget respected
```gherkin
Given the backlog adapter responds slowly (>5 seconds)
When the live query is attempted
Then it times out at 5 seconds
  And falls back to cached state with a warning
```

## Examples (Specification by Example)

| State | Adapter Available | Expected Behavior |
|-------|-------------------|-------------------|
| epic=E347, story=S347.5 | Yes | Bundle shows live status from adapter |
| epic=E347, story=S347.5 | No (timeout) | Bundle shows cached state + warning |
| epic=E347, story=None | Yes | Query epic only, show live epic status |
| No current work | N/A | No query attempted |

## Notes
- Design doc specifies: `assemble_context_bundle()` queries `rai backlog get {key}` when current_work.epic or current_work.story is set
- Failure mode: add warning "backlog unavailable, showing cached state" — never block session-start
- Timeout: 5 seconds (aggressive, per risk table in scope.md)
- The adapter default from S347.1 (manifest.yaml) means no `-a` flag needed
- Surface: `src/rai_cli/session/bundle.py` (primary), may need subprocess or adapter call
