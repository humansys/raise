# Design: S347.1 â€” Adapter Default in Manifest

## Problem
Every `rai backlog` command requires `-a jira` when both adapters are registered. No project-level default.

## Value
Eliminates per-command friction. Foundation for S347.3-6 which depend on transparent adapter resolution.

## Approach

Add `backlog` section to `ProjectManifest` with `adapter_default`. Modify `resolve_adapter()` to read it when no explicit flag is passed.

### Components Affected

| Component | Change | Type |
|-----------|--------|------|
| `src/rai_cli/onboarding/manifest.py` | Add `BacklogConfig` model + `backlog` field to `ProjectManifest` | modify |
| `src/rai_cli/cli/commands/_resolve.py` | Read manifest default in `resolve_adapter()` | modify |
| `.raise/manifest.yaml` | Add `backlog.adapter_default: jira` | modify |
| `tests/cli/commands/test_resolve.py` | Tests for 4 resolution scenarios | create |

### Resolution Logic (updated)

```python
def resolve_adapter(adapter_name: str | None) -> ProjectManagementAdapter:
    # 1. Explicit flag provided â†’ use it
    effective = adapter_name

    # 2. No flag â†’ check manifest default
    if effective is None:
        manifest = load_manifest(Path.cwd())
        if manifest and manifest.backlog and manifest.backlog.adapter_default:
            effective = manifest.backlog.adapter_default

    # 3. Pass to resolve_entrypoint (handles auto-detect / error)
    return resolve_entrypoint(..., selected=effective)
```

Key: `resolve_entrypoint()` stays generic. Manifest awareness lives only in `resolve_adapter()`.

### Manifest Extension

```yaml
# .raise/manifest.yaml
backlog:
  adapter_default: jira
```

### Pydantic Model

```python
class BacklogConfig(BaseModel):
    adapter_default: str | None = None

class ProjectManifest(BaseModel):
    # ... existing fields ...
    backlog: BacklogConfig | None = None
```

## Examples

```bash
# Before (required -a every time):
rai backlog search "project = RAISE" -a jira

# After (manifest default):
rai backlog search "project = RAISE"

# Override still works:
rai backlog search "project = RAISE" -a filesystem
```

## Acceptance Criteria

### MUST
1. `rai backlog` resolves adapter from manifest when no `-a` flag
2. Explicit `-a` flag overrides manifest default
3. Auto-detect (single adapter) still works without manifest config
4. Clear error when ambiguous (multiple adapters, no default, no flag)
5. Tests cover all 4 resolution paths

### MUST NOT
- Break existing `-a` flag behavior
- Change `resolve_entrypoint()` signature (it's generic)
- Add mandatory fields to manifest (backward compat)
