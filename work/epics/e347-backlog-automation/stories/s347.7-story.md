# Story: Integration Tests + Dogfood

## User Story
As a RaiSE developer,
I want a shared integration test suite that runs both adapters against the same protocol assertions,
so that I can trust adapter parity and catch regressions before release.

## Acceptance Criteria

### Scenario: Both adapters pass the same test suite
```gherkin
Given a parameterized test suite covering all ProjectManagementAdapter protocol methods
When run against McpJiraAdapter (mocked MCP) and FilesystemPMAdapter
Then both adapters produce equivalent results for identical inputs
```

### Scenario: Protocol method coverage is complete
```gherkin
Given the ProjectManagementAdapter protocol defines N methods
When the integration test suite is counted
Then every protocol method has at least one test case
```

### Scenario: Dogfood on real epic lifecycle
```gherkin
Given E347 has been developed using rai backlog CLI throughout
When the epic lifecycle is reviewed
Then all transitions (start, in_progress, done) were made via rai backlog
And session-start showed live state at least once
```

### Scenario: Adapter default resolution is tested
```gherkin
Given manifest.yaml has backlog.adapter_default set
When rai backlog commands run without -a flag
Then the configured default adapter is used
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| `create("Title", project="TEST")` | Run on both adapters | Both return valid issue key |
| `search("project=TEST")` | Run on both adapters | Both return list of issues |
| `transition(key, "in_progress")` | Run on both adapters | Both update status |
| `get(key)` | Run on both adapters | Both return issue with same structure |
| `comment(key, "text")` | Run on both adapters | Both attach comment |
| `link(src, dst, "blocks")` | Run on both adapters | Both create link |
