# Scope: S347.1 — Adapter Default in Manifest

## In Scope

1. Add `backlog.adapter_default` field to `.raise/manifest.yaml`
2. Update `resolve_adapter()` in `_resolve.py` to read manifest default
3. Resolution priority chain: flag > manifest > auto-detect > error

## Out of Scope

- workflows.yaml (deferred — ADR-043 ships defaults in code, config is future)
- State validation on `rai backlog transition` (S347.3+ concern)
- New CLI commands or subcommands
- Changes to adapter protocol

## Done Criteria

1. `rai backlog` commands work without `-a` when `backlog.adapter_default` is set in manifest
2. Explicit `-a` flag still overrides the default
3. Auto-detect (single adapter) still works when no default configured
4. Clear error with guidance when ambiguous (multiple adapters, no default, no flag)
5. Existing tests pass, new tests cover all 4 resolution scenarios
