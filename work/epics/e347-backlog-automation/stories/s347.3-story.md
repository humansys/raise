# Story: Skills via `rai backlog` CLI

## User Story
As a developer running lifecycle skills (epic-start, epic-close, story-start, story-close),
I want the skills to use `rai backlog` CLI for all backlog operations,
so that backlog state is always consistent regardless of which adapter is active and no manual edits to `governance/backlog.md` are needed.

## Acceptance Criteria

### Scenario: epic-start registers epic via CLI
```gherkin
Given a developer runs /rai-epic-start
When the skill reaches the "register epic in backlog" step
Then the skill instructs to run `rai backlog create` (not edit backlog.md directly)
And the command uses the project key and epic summary
```

### Scenario: epic-close marks epic complete via CLI
```gherkin
Given a developer runs /rai-epic-close for a completed epic
When the skill reaches the "mark epic complete" step
Then the skill instructs to run `rai backlog transition {key} done`
And no direct edit to governance/backlog.md is mentioned
```

### Scenario: story-start transitions story to in_progress
```gherkin
Given a developer runs /rai-story-start for a story with a Jira ticket
When the skill completes the scope commit
Then the skill instructs to run `rai backlog transition {key} in_progress`
Or `rai backlog search` to find the ticket and transition it
```

### Scenario: story-close transitions story to done
```gherkin
Given a developer runs /rai-story-close for a completed story
When the skill reaches the "update backlog" step
Then the skill instructs to run `rai backlog transition {key} done`
```

### Scenario: skills work without -a flag
```gherkin
Given manifest.yaml has backlog.adapter_default configured
When any lifecycle skill runs a `rai backlog` command
Then no `-a` flag is needed (adapter resolved from manifest)
```

### Scenario: backlog.md references replaced with CLI references
```gherkin
Given the 4 lifecycle skill SKILL.md files
When inspected after this story
Then zero references to "edit governance/backlog.md" remain
And all backlog operations reference `rai backlog` CLI commands
```

## Examples (Specification by Example)

| Skill | Old Instruction | New Instruction |
|-------|-----------------|-----------------|
| epic-start | "Register epic in `governance/backlog.md`" | `rai backlog create "Epic title" -p PROJECT -t Epic` |
| epic-close | "Mark epic complete in `governance/backlog.md`" | `rai backlog transition {key} done` |
| story-start | (no backlog step) | `rai backlog transition {key} in_progress` (if ticket exists) |
| story-close | (no backlog step) | `rai backlog transition {key} done` |

## Notes
- The 4 target skills are in `src/rai_cli/skills_base/`: rai-epic-start, rai-epic-close, rai-story-start, rai-story-close.
- Each skill must be edited in builtin first, then replicated to `.claude/skills/`.
- S347.1 (adapter default) is a prerequisite -- the `-a` flag is no longer mandatory.
- The `rai backlog` CLI already supports all needed operations (create, transition, search, comment).
- Design decision: skills reference CLI commands in their prose, not adapter internals.
