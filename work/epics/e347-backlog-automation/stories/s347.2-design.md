# Design: S347.2 — FileAdapter Parity (YAML Store)

## Problem

FilesystemPMAdapter only handles epics from `governance/backlog.md` via markdown table parsing. Stories, links, and comments are no-ops. Extending this with more regex on markdown tables (scope docs, progress tables, comment sections) creates a fragile parser tangle that is hard to test and harder to evolve.

## Decision

**D1: YAML file store at `.raise/backlog/items/{KEY}.yaml`, one file per issue.**

- Each issue (epic, story, task) is a single YAML file validated by a Pydantic model on load/dump.
- Pattern consistent with `.raise/mcp/*.yaml` (already established in project).
- Human readability is via CLI (`rai backlog get`, `rai backlog search`), NOT via file format — YAML is machine-first storage.
- No markdown parsing for PM operations. The backlog.md and scope docs remain human documentation; the YAML store is the FileAdapter's source of truth.
- Comments and links are embedded in the issue YAML file (no separate files).
- Search loads all YAML files and filters in memory (small scale, no DB needed).

**Migration from existing markdown files is NOT in scope for S347.2.** Existing `governance/backlog.md` and scope docs stay untouched. The YAML store is a new parallel backend.

## Schema

### `BacklogItem` (Pydantic model)

```python
class BacklogLink(BaseModel):
    target: str          # target issue key
    link_type: str       # "depends_on", "blocks", "relates_to"

class BacklogComment(BaseModel):
    id: str              # "{KEY}-{N}" sequential
    body: str
    author: str          # default "rai"
    created: str         # ISO 8601

class BacklogItem(BaseModel):
    key: str             # "E347", "S347.1", "RAISE-301"
    summary: str
    issue_type: str      # "Epic", "Story", "Task"
    status: str          # "pending", "in_progress", "complete"
    parent: str | None = None  # parent issue key
    description: str = ""
    labels: list[str] = []
    priority: str | None = None
    assignee: str | None = None
    comments: list[BacklogComment] = []
    links: list[BacklogLink] = []
    created: str = ""    # ISO 8601
    updated: str = ""    # ISO 8601
```

### YAML file example (`.raise/backlog/items/S347.1.yaml`)

```yaml
key: S347.1
summary: Adapter default in manifest
issue_type: Story
status: complete
parent: E347
description: Add backlog.adapter_default to manifest.yaml
labels: [v2.2]
comments:
  - id: S347.1-1
    body: Implementation started
    author: rai
    created: "2026-03-03T10:00:00Z"
  - id: S347.1-2
    body: Tests passing, merged
    author: rai
    created: "2026-03-03T11:30:00Z"
links:
  - target: E347
    link_type: depends_on
created: "2026-03-03T09:00:00Z"
updated: "2026-03-03T11:30:00Z"
```

## Operations Mapping

| Protocol Method | Implementation |
|----------------|----------------|
| `get_issue(key)` | Load `.raise/backlog/items/{key}.yaml` -> `IssueDetail`. KeyError if file missing. |
| `search(query)` | Load all YAML files, filter by substring match on key/summary or `field=value` syntax. |
| `create_issue(project, spec)` | Generate next sequential key, write new YAML file. |
| `update_issue(key, fields)` | Load YAML, merge fields, write back. KeyError if missing. |
| `transition_issue(key, status)` | Load YAML, update status + updated timestamp, write back. |
| `add_comment(key, body)` | Load YAML, append to comments list, write back. KeyError if missing. |
| `get_comments(key, limit)` | Load YAML, return comments list (sliced by limit). Empty list if no file. |
| `link_issues(source, target, type)` | Load source YAML, append to links list, write back. |
| `link_to_parent(child, parent)` | Load child YAML, set parent field, write back. |
| `batch_transition(keys, status)` | Loop transition_issue, collect successes/failures. |
| `health()` | Check `.raise/backlog/items/` dir exists, count YAML files. |

### Key Generation

- Epics: scan existing `E{N}` files, return `E{N+1}`.
- Stories: `parent` required in `IssueSpec.metadata`. Scan `S{epic_num}.{M}` files, return `S{epic_num}.{M+1}`.
- Tasks: similar pattern with `T{N}` or delegate to caller.

### Missing-Entity Strategy

| Operation | Missing Entity | Behavior |
|-----------|---------------|----------|
| `get_issue("S999.1")` | No YAML file | `KeyError("S999.1")` |
| `create_issue(type=Story, no parent)` | Missing parent_key | `KeyError("Story creation requires parent_key")` |
| `get_comments("S999.1")` | No YAML file | Return `[]` (not an error) |
| `add_comment("S999.1", ...)` | No YAML file | `KeyError("S999.1")` |
| `search("anything")` | No items dir | Return `[]` |

## Acceptance Criteria

### MUST

1. All `ProjectManagementAdapter` protocol methods are implemented (no more no-ops)
2. `get_issue` / `search` / `create_issue` / `transition_issue` work for epics AND stories
3. `add_comment` / `get_comments` persist and retrieve comments from YAML
4. `link_issues` persists links in the source issue's YAML file
5. Missing issue raises `KeyError`; missing comments returns `[]`
6. Pydantic validation on load (malformed YAML -> clear error)

### SHOULD

1. `health()` reports item count and store path
2. `search("")` returns all items
3. Timestamps auto-set on create/update

### MUST NOT

- Parse markdown tables for PM operations
- Modify `governance/backlog.md` or scope docs
- Change the `ProjectManagementAdapter` protocol
- Include migration from markdown (separate story)

## Test Strategy

| Class | Covers |
|-------|--------|
| `TestYamlStoreGetIssue` | Load existing, missing file, malformed YAML |
| `TestYamlStoreSearch` | Substring match, field=value, empty query, no items |
| `TestYamlStoreCreate` | Epic creation, story creation, key sequencing, missing parent |
| `TestYamlStoreTransition` | Status update, timestamp update, missing issue |
| `TestYamlStoreComments` | Add comment, get comments, sequential IDs, missing issue |
| `TestYamlStoreLinks` | Add link, append to existing, link_to_parent |
| `TestYamlStoreHealth` | Healthy with items, empty store, missing directory |
