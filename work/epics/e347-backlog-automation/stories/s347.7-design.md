# Design: S347.7 Integration Tests + Dogfood

## Step 1: Complexity Assessment

| Criterion | Value | Rating |
|-----------|-------|--------|
| Components | 4 (FileAdapter, BacklogHook, sync, bundle) | Moderate |
| Story points | M (5) | Moderate |
| External integrations | 0 (all mocked/temp) | Simple |
| Algorithm complexity | Test orchestration, no novel logic | Simple |

**Result:** Moderate — core sections only.

**Integration gate (PAT-E-539):** Story name includes "integration" and "dogfood". However, scope explicitly excludes live Jira (not CI-friendly). The "real infrastructure" AC is satisfied by **FilesystemPMAdapter running against actual temp directories** (real I/O, real YAML parsing, real file writes) — not mocked. This is a genuine round-trip through real code, not mock-to-mock.

## Step 2: What & Why

**Problem:** Unit tests for S347.1-6 components each validate their own contract in isolation, using mocks for cross-component boundaries. No test verifies that the components actually work together — e.g., that BacklogHook calling `resolve_adapter()` → `FilesystemPMAdapter.search()` → `create_issue()` → `transition_issue()` produces the expected YAML files on disk with correct data.

**Value:** A parameterized integration suite proves protocol parity across adapters and catches contract mismatches (wrong field names, missing model conversions, argument order bugs) that unit tests with mocks cannot detect. Dogfood review validates the epic's own lifecycle was managed with the tools it built.

## Step 3: Approach

**Solution:** Create a single integration test module (`tests/integration/test_backlog_integration.py`) containing:

1. **Protocol parity suite** — parameterized tests running identical operations against `FilesystemPMAdapter` (real temp dirs). McpJiraAdapter parity is covered by mocked MCP bridge (consistent with scope: no live Jira).
2. **BacklogHook → FileAdapter flow** — real BacklogHook with real FilesystemPMAdapter (no mocks), verifying emit-work events produce correct YAML files on disk.
3. **Session-start live query** — `_fetch_live_status()` with real FilesystemPMAdapter (real session state, real temp files).
4. **Sync guard** — verify sync_backlog rejects FilesystemPMAdapter (already unit-tested, but included for completeness in cross-component context).
5. **Dogfood review** — documented checklist in the test module docstring (not executable tests, but evidence).

**Components affected:**

| Component | Change Type |
|-----------|-------------|
| `tests/integration/test_backlog_integration.py` | Create |
| `tests/integration/__init__.py` | Create (if missing) |
| `tests/integration/conftest.py` | Create (shared fixtures) |

**Value preservation gate (PAT-E-572):** These tests provide integration verification that no unit test can — they exercise real I/O, real Pydantic validation, and real cross-module imports with actual data flowing between components.

## Step 4: Examples

### 4.1 Protocol parity — full lifecycle through FileAdapter (real I/O)

```python
def test_full_lifecycle_round_trip(tmp_path: Path) -> None:
    """Create → transition → comment → link → search → get → update → batch."""
    adapter = FilesystemPMAdapter(project_root=tmp_path)
    (tmp_path / ".raise" / "backlog" / "items").mkdir(parents=True)

    # Create epic
    epic = adapter.create_issue("TEST", IssueSpec(summary="Epic One", issue_type="Epic"))
    assert epic.key == "E1"

    # Create story under epic
    story = adapter.create_issue(
        "TEST",
        IssueSpec(summary="Story One", issue_type="Story", metadata={"parent_key": "E1"}),
    )
    assert story.key == "S1.1"

    # Transition
    adapter.transition_issue("S1.1", "in_progress")
    detail = adapter.get_issue("S1.1")
    assert detail.status == "in_progress"

    # Comment
    cref = adapter.add_comment("S1.1", "Work started")
    assert cref.id == "S1.1-1"
    comments = adapter.get_comments("S1.1")
    assert len(comments) == 1

    # Link
    adapter.link_issues("S1.1", "E1", "depends_on")

    # Search
    results = adapter.search("")
    assert len(results) == 2

    # Update
    adapter.update_issue("E1", {"summary": "Renamed Epic", "priority": "P0"})
    epic_detail = adapter.get_issue("E1")
    assert epic_detail.summary == "Renamed Epic"

    # Batch transition
    result = adapter.batch_transition(["E1", "S1.1"], "done")
    assert len(result.succeeded) == 2
```

def test_hook_creates_and_transitions_via_file_adapter(tmp_path: Path) -> None:

def test_hook_with_unavailable_adapter_returns_error(tmp_path: Path) -> None:
    """BacklogHook degrades gracefully when adapter is unavailable."""
    _setup_jira_yaml(tmp_path)

## Step 5: Acceptance Criteria

### MUST

1. **Protocol method coverage** — integration tests exercise all 11 `ProjectManagementAdapter` protocol methods (`create_issue`, `get_issue`, `update_issue`, `transition_issue`, `batch_transition`, `link_to_parent`, `link_issues`, `add_comment`, `get_comments`, `search`, `health`) through real `FilesystemPMAdapter` with real file I/O.

2. **BacklogHook → FileAdapter round-trip** — at least 2 tests (start + complete) where `BacklogHook.handle()` calls a real `FilesystemPMAdapter` and produces verifiable YAML files on disk. No adapter mocks — only `resolve_adapter` is patched to inject the real FileAdapter.

3. **Session-start live query** — at least 1 test where `_fetch_live_status()` reads real YAML items via `FilesystemPMAdapter` and returns correct `LiveBacklogStatus` fields.

4. **Sync guard cross-component** — at least 1 test confirming `sync_backlog` rejects `FilesystemPMAdapter` in the integration test context (validates the guard works with a real adapter instance, not a mock).

5. **All existing tests pass** — no regressions. `uv run pytest` green.

### SHOULD

1. **Dogfood review checklist** — documented evidence (in test module docstring or separate section) that E347 lifecycle used `rai backlog` CLI.
2. **Adapter default resolution** — test that `manifest.yaml` with `backlog.adapter_default: filesystem` causes `resolve_adapter(None)` to select `FilesystemPMAdapter`.

### MUST NOT

1. **No live Jira calls** — all tests run without network, credentials, or external services.
2. **No new production code** — this is a test-only story. No changes to `src/`.
3. **No test pollution** — each test uses its own `tmp_path`, no shared mutable state.
