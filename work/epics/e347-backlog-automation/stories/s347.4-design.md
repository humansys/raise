# Design: S347.4 — BacklogHook Adapter-Aware

## Problem

`BacklogHook` hardcodes two Jira-specific assumptions:
1. `resolve_adapter("jira")` — ignores manifest default
2. `summary ~ "{work_id}"` — fragile fuzzy JQL search

## Decisions

### D1: Use `resolve_adapter(None)` for manifest-aware resolution

Replace `_resolve("jira")` with `_resolve(None)`. This respects:
- `manifest.yaml → backlog.adapter_default` (if set)
- Auto-detect (if single adapter registered)
- Explicit error (if 0 or 2+ adapters without default)

### D2: Adapter-aware search strategy

The hook checks the resolved adapter type and builds the appropriate query:

```python
def _resolve_issue_key(adapter, work_id, project_key):
    if isinstance(adapter, FilesystemPMAdapter):
        # FileAdapter: direct key lookup
        results = adapter.search(work_id, limit=1)
    else:
        # Jira/MCP: label-first, summary-fallback
        results = adapter.search(
            f'project = "{project_key}" AND labels = "rai:{work_id}"',
            limit=1,
        )
        if not results:
            logger.warning("No label match for %s — falling back to summary search", work_id)
            results = adapter.search(
                f'project = "{project_key}" AND summary ~ "{work_id}"',
                limit=1,
            )
    return results[0].key if results else None
```

### D3: Rename for adapter-agnostic semantics

- `_resolve_jira_key()` → `_resolve_issue_key()`
- `resolve_adapter()` signature unchanged (already returns `Any`)
- Module docstring: "syncs backlog state" (not "syncs Jira state")
- `_JiraConfig` stays — it reads jira.yaml, name is accurate

## Acceptance Criteria

- MUST: `resolve_adapter(None)` replaces `resolve_adapter("jira")`
- MUST: Label-first search for Jira adapter (`labels = "rai:{work_id}"`)
- MUST: Summary fallback with warning when label not found
- MUST: FileAdapter uses direct key search (no JQL)
- MUST: All existing tests pass (no regression)
- MUST: New tests for label match, summary fallback, no match, FileAdapter path
- SHOULD: Module docstring updated to adapter-agnostic language

## Missing-Entity Strategy (PAT-E-523)

| Scenario | Strategy |
|----------|----------|
| No adapter resolved | Return `HookResult(status="error")` |
| Label search finds nothing | Fall back to summary search |
| Summary search finds nothing | Return `None` (caller handles) |
| FileAdapter key not found | Return `None` (caller handles) |
