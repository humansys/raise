# Story: FileAdapter Parity (Stories, Links, Comments)

## User Story
As a developer using `rai backlog` with the filesystem adapter,
I want the FileAdapter to support stories, links, and comments (not just epics),
so that FileAdapter achieves full protocol parity and can serve as a standalone backlog backend.

## Acceptance Criteria

### Scenario: Search returns stories from scope docs
```gherkin
Given an epic scope doc exists at work/epics/e347-backlog-automation/scope.md
And the scope doc contains a Stories table with S347.1 and S347.2
When I search with query "S347"
Then both S347.1 and S347.2 are returned as IssueSummary with issue_type="Story"
```

### Scenario: Get issue returns story detail
```gherkin
Given S347.1 is listed in the epic scope doc stories table
When I call get_issue("S347.1")
Then an IssueDetail is returned with summary, status, and issue_type="Story"
```

### Scenario: Get issue returns epic detail (existing behavior preserved)
```gherkin
Given E347 exists in governance/backlog.md
When I call get_issue("E347")
Then an IssueDetail is returned with issue_type="Epic"
```

### Scenario: Create story appends to scope doc
```gherkin
Given an epic scope doc exists
When I create_issue with issue_type="Story" and parent_key="E347"
Then a new row is appended to the Stories table in the scope doc
And the story gets a sequential ID (S347.N+1)
```

### Scenario: Transition story updates status in scope doc
```gherkin
Given S347.1 exists in the scope doc
When I call transition_issue("S347.1", "in_progress")
Then the status column for S347.1 is updated in the scope doc
```

### Scenario: Add comment persists to story file
```gherkin
Given S347.1 has a story file at stories/s347.1-story.md
When I call add_comment("S347.1", "Implementation started")
Then a Comments section is appended/updated in the story file
And the comment includes a timestamp
```

### Scenario: Get comments returns from story file
```gherkin
Given S347.1 story file has a Comments section with 2 entries
When I call get_comments("S347.1")
Then 2 Comment objects are returned with body and created_at
```

### Scenario: Link issues records in scope doc
```gherkin
Given S347.2 and S347.1 exist in scope doc
When I call link_issues("S347.2", "S347.1", "depends_on")
Then the dependency is recorded (e.g., in the Dependencies column)
```

### Scenario: Key not found raises KeyError
```gherkin
Given no issue with key "S999.1" exists
When I call get_issue("S999.1")
Then a KeyError is raised
```

## Examples (Specification by Example)

| Key Pattern | Type Detected | Source File |
|-------------|---------------|-------------|
| `E347` | Epic | governance/backlog.md |
| `S347.1` | Story | work/epics/e347-*/scope.md |
| `S347.1` (detail) | Story | + stories/s347.1-story.md |

| Operation | Epic | Story |
|-----------|------|-------|
| search | backlog.md table | scope.md Stories table |
| get_issue | backlog.md row | scope.md row + story file |
| create_issue | append to backlog.md | append to scope.md Stories table |
| transition | update backlog.md status | update scope.md status |
| add_comment | no-op (no file) | append to story file Comments section |
| get_comments | empty list | parse story file Comments section |
| link_issues | no-op | update Dependencies column in scope.md |

## Notes
- Protocol (`ProjectManagementAdapter`) is already complete -- no changes needed there.
- Key pattern determines routing: `E{N}` -> epic operations, `S{N}.{M}` -> story operations.
- Stories live in epic scope docs (`work/epics/e{N}-{name}/scope.md`) -- Option A from design.md.
- Comments stored in story markdown files under a `## Comments` section.
- Links stored as Dependencies column in scope.md Stories table.
- Existing epic operations must remain unchanged (backward compatible).
- See design.md SS FileAdapter Stories Model for context.
