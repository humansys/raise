# Design: Skills via `rai backlog` CLI

## Problem

The lifecycle skills (`rai-epic-start`, `rai-epic-close`, `rai-story-start`, `rai-story-close`) contain hardcoded references to `governance/backlog.md` for backlog operations. This couples skills to one specific backend (the markdown file) and bypasses the adapter layer that S347.1 and S347.2 established.

## Value

After this change, skills are adapter-agnostic. The same skill prose works whether the project uses Jira, the filesystem YAML store, or any future adapter. The `rai backlog` CLI resolves the adapter automatically via `manifest.yaml` (S347.1).

## Approach

Replace all direct `governance/backlog.md` references in the 4 lifecycle skills with equivalent `rai backlog` CLI commands. This is a prose-only change to SKILL.md files -- no Python code changes.

### Components Affected

| File | Location | Change Type |
|------|----------|-------------|
| `rai-epic-start/SKILL.md` | builtin + deployment | modify |
| `rai-epic-close/SKILL.md` | builtin + deployment | modify |
| `rai-story-start/SKILL.md` | builtin + deployment | modify |
| `rai-story-close/SKILL.md` | builtin + deployment | modify |

### Change Map

#### rai-epic-start

**Step 3 (Define Scope & Commit):** Replace the line:
> Register epic in `governance/backlog.md` -- add or update the row with status `In Progress`.

With a CLI instruction block:
```
Register epic in the backlog:
rai backlog create "{Epic title}" -p {PROJECT} -t Epic
```

Plus an `<if-blocked>` for missing adapter config instead of missing backlog file.

**Output table:** Replace `| Backlog entry | governance/backlog.md |` with `| Backlog entry | via rai backlog create |`.

**Quality Checklist:** Replace `Epic registered in governance/backlog.md` with `Epic registered via rai backlog create`.

#### rai-epic-close

**Step 5 (Update Backlog & Context):** Replace:
> 1. Mark epic complete in `governance/backlog.md` (status -> "Complete")

With:
```
1. Mark epic complete: `rai backlog transition {epic_key} done`
```

**Output table:** Replace `| Backlog update | governance/backlog.md |` with `| Backlog update | via rai backlog transition |`.

**References:** Replace `Backlog: governance/backlog.md` with `Backlog: via rai backlog CLI`.

#### rai-story-start

**New step after Step 3 (scope commit), before Step 4 (present next steps):** Add a backlog transition step:
```
### Step 3b: Update Backlog Status

If the story has a backlog ticket (Jira key or local key):
rai backlog transition {story_key} in_progress

| Condition | Action |
|-----------|--------|
| Story has ticket | Transition to in_progress |
| No ticket found | Skip (not all stories are tracked externally) |
```

This is a conditional step -- not all stories have backlog tickets.

#### rai-story-close

**New Step 6b (after Update Context & Emit) or fold into Step 6:** Add a backlog transition:
```
3. Mark story complete: `rai backlog transition {story_key} done`
```

If the story has a backlog ticket. Same conditional pattern as story-start.

## Examples

### epic-start: registering an epic

```bash
# Old: manually edit governance/backlog.md table row
# New:
rai backlog create "Backlog Automation" -p RAISE -t Epic
# Output: Created RAISE-347 (or E1 for filesystem adapter)
```

### epic-close: marking complete

```bash
# Old: manually change status emoji in governance/backlog.md
# New:
rai backlog transition RAISE-347 done
# Output: RAISE-347 transitioned to done
```

### story-start: transitioning to in_progress

```bash
# Old: no backlog step existed
# New:
rai backlog transition RAISE-380 in_progress
# Output: RAISE-380 transitioned to in_progress

# If no ticket:
# Skip -- not all stories are tracked externally
```

### story-close: marking done

```bash
# Old: no backlog step existed
# New:
rai backlog transition RAISE-380 done
# Output: RAISE-380 transitioned to done
```

### Error case: no adapter configured

```bash
rai backlog transition RAISE-380 done
# Error: No backlog adapter configured. Set backlog.adapter_default in .raise/manifest.yaml
# or pass -a <adapter_name>
```

## Acceptance Criteria

### MUST

1. Zero occurrences of `governance/backlog.md` in the 4 lifecycle skills (grep verification)
2. `rai-epic-start` references `rai backlog create` for epic registration
3. `rai-epic-close` references `rai backlog transition {key} done` for completion
4. `rai-story-start` includes conditional step to transition ticket to `in_progress`
5. `rai-story-close` includes conditional step to transition ticket to `done`

### SHOULD

1. All 4 builtin skills match their `.claude/skills/` deployments (verified with `diff`)
2. `rai skill validate` passes for all 4 skills
3. No `-a` flag in CLI commands (adapter resolved from manifest per S347.1)

### MUST NOT

1. Must NOT change `rai-project-create` or `rai-project-onboard` (those still bootstrap `governance/backlog.md` as initial structure)
2. Must NOT add Python code -- this story is prose-only skill edits
3. Must NOT make backlog transition mandatory in story-start/story-close (conditional on ticket existence)
