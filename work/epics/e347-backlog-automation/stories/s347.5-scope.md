# Scope: S347.5 — Session-start live backlog query

## In Scope

- Add live backlog query to `assemble_context_bundle()` in `bundle.py`
- Query `rai backlog get {key}` (via subprocess or adapter API) for current epic/story
- Merge live status into the work section of the context bundle
- Graceful degradation: fall back to session-state.yaml on failure with explicit warning
- Timeout budget: 5 seconds max for backlog query
- Unit tests: mock adapter responses (success, timeout, unavailable)

## Out of Scope

- Querying multiple issues (e.g., all stories in an epic) — only current work items
- Caching live results back to session-state.yaml — this is read-only enrichment
- Changing session-state.yaml schema
- UI/formatting changes beyond adding live status to existing work section
- Retry logic — fail once, fall back

## Done Criteria

1. `assemble_context_bundle()` includes live backlog status when adapter is available
2. Session-start does not break when adapter is unavailable (graceful degradation with warning)
3. Timeout at 5 seconds prevents slow adapters from blocking session-start
4. Tests cover: success path, timeout path, no-current-work path, adapter-unavailable path
5. Type-checked (`pyright`), linted, all tests pass
