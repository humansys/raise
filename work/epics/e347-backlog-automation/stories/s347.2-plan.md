# Plan: S347.2 — FileAdapter Parity (YAML Store)

## Overview

Rewrite `FilesystemPMAdapter` to use a YAML file store at `.raise/backlog/items/{KEY}.yaml` instead of parsing markdown tables. Each issue (epic, story, task) becomes a single YAML file validated by Pydantic models on load/dump. All protocol methods become fully functional — no more no-ops.

**Design doc:** `s347.2-design.md` (YAML store approach confirmed)

**Components:**
- `src/rai_cli/adapters/models.py` — add `BacklogItem`, `BacklogComment`, `BacklogLink`
- `src/rai_cli/adapters/filesystem.py` — rewrite: YAML load/dump replaces markdown parsing
- `tests/adapters/test_filesystem.py` — rewrite: YAML fixtures replace markdown fixtures

## Tasks

## Execution Order

```
T1 (models)
 └── T2 (YAML I/O + get_issue + health)
      ├── T3 (search)
      ├── T4 (create + keygen)
      ├── T5 (transition + update + batch)
      └── T6 (comments + links)
           T7 (protocol + integration) — after all
```

**Rationale:**
1. **T1 first** — pure data models, zero risk, enables everything.
2. **T2 second** — YAML I/O is the core foundation and biggest technical risk. If load/dump doesn't work, nothing works. Risk-first.
3. **T3, T4, T5, T6 parallel** after T2 — all are independent load-modify-save operations. Ordered T3 → T4 → T5 → T6 for read-before-write progression.
4. **T7 last** — integration verification after all pieces are in place.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Pydantic YAML round-trip loses types (e.g., None vs empty string) | Medium | Medium | Test round-trip explicitly in T1. Use `model_dump(exclude_none=True)` for clean YAML. |
| Key generation race condition (two creates in parallel) | Low | Low | FileAdapter is single-process. Document limitation in docstring. |
| Existing code depends on `extract_epics` import path | Low | Medium | Keep import in place until T7 cleanup. Search codebase for other callers before removing. |
| Old tests import `SAMPLE_BACKLOG` fixture from test file | Low | Low | Replace entire test file progressively. Old test classes removed as new ones replace them. |

## Fixture Design

Shared fixture `yaml_store` creates:

```
tmp_path/
  .raise/
    backlog/
      items/
        E1.yaml      # key: E1, summary: Core Foundation, status: complete
        E2.yaml      # key: E2, summary: API Layer, status: in_progress
        S1.1.yaml    # key: S1.1, summary: First Story, status: pending, parent: E1
```

Each YAML file is a valid `BacklogItem` serialization. Individual test classes may add/modify files as needed. The fixture replaces the old `SAMPLE_BACKLOG` markdown constant.
