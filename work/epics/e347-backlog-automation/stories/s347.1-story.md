# Story: Adapter Default in Manifest

## User Story
As a developer using `rai backlog`,
I want the CLI to resolve the active adapter automatically from project config,
so that I never need to pass `-a jira` on every command.

## Acceptance Criteria

### Scenario: Manifest default configured
```gherkin
Given manifest.yaml has backlog.adapter_default: jira
When I run `rai backlog search "project = RAISE"` (no -a flag)
Then the Jira adapter is used
```

### Scenario: Explicit flag overrides default
```gherkin
Given manifest.yaml has backlog.adapter_default: jira
When I run `rai backlog search "query" -a filesystem`
Then the filesystem adapter is used (flag wins)
```

### Scenario: No default, single adapter registered
```gherkin
Given manifest.yaml has no backlog.adapter_default
And exactly 1 adapter is registered
When I run `rai backlog search "query"`
Then the single adapter is auto-detected
```

### Scenario: No default, multiple adapters, no flag
```gherkin
Given manifest.yaml has no backlog.adapter_default
And 2+ adapters are registered
When I run `rai backlog search "query"` (no -a flag)
Then an error is raised with guidance to set default or use -a
```

## Examples (Specification by Example)

| Config | Flag | Registered | Result |
|--------|------|-----------|--------|
| `adapter_default: jira` | (none) | jira, filesystem | jira used |
| `adapter_default: jira` | `-a filesystem` | jira, filesystem | filesystem used |
| (none) | (none) | jira | jira auto-detected |
| (none) | (none) | jira, filesystem | error with guidance |

## Notes
- Resolution priority: explicit flag > manifest default > auto-detect (1 adapter) > error
- This is the foundation story — S347.3-6 all depend on this working
- See design.md § Adapter Resolution (Target) for full resolution logic
