# Story: `rai backlog sync` — mirror generation

## User Story
As a RaiSE developer using Jira as my backlog source of truth,
I want a `rai backlog sync` command that regenerates `governance/backlog.md` from the active adapter,
so that the local markdown mirror stays current without manual editing.

## Acceptance Criteria

### Scenario: Sync regenerates backlog.md from Jira
```gherkin
Given the active adapter is Jira (via manifest default or -a flag)
  And the project has epics and stories in Jira
When I run `rai backlog sync`
Then governance/backlog.md is regenerated with current epic/story data
  And the file includes a generation timestamp and source indicator
```

### Scenario: Sync with FileAdapter (no-op or identity)
```gherkin
Given the active adapter is filesystem
  And governance/backlog.md is the source of truth
When I run `rai backlog sync`
Then the command completes successfully
  And a message indicates "filesystem adapter is source of truth — nothing to sync"
```

### Scenario: Adapter unavailable
```gherkin
Given the backlog adapter is unreachable (MCP timeout, misconfigured)
When I run `rai backlog sync`
Then the command exits with a clear error message
  And governance/backlog.md is NOT modified (fail-fast, no partial writes)
```

### Scenario: Sync with project filter
```gherkin
Given the adapter has multiple projects
When I run `rai backlog sync -p RAISE`
Then only issues from project RAISE are included in the generated file
```

## Examples (Specification by Example)

| Adapter | State | Command | Expected Output |
|---------|-------|---------|-----------------|
| Jira (default) | 3 epics, 12 stories | `rai backlog sync` | backlog.md regenerated with 3 epics table |
| Jira | Adapter timeout | `rai backlog sync` | Error: "adapter unavailable", file untouched |
| Filesystem | backlog.md exists | `rai backlog sync` | "filesystem is source of truth — nothing to sync" |
| Jira | No epics | `rai backlog sync` | backlog.md with empty table + timestamp |

## Notes
- Design doc specifies: backlog.md when Jira configured = read-only mirror, regenerated via `rai backlog sync`
- Fail-fast contract: never silently fall back, never write partial data
- Surface: `src/rai_cli/cli/commands/backlog.py` (new `sync` subcommand)
- Must use adapter protocol — query via `search` + `get` operations
- Generation timestamp at top of file helps detect staleness
- Depends on S347.1 (adapter default) for `-a`-free operation
