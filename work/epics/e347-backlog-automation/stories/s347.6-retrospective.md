## What to Improve

- **isinstance coupling flagged in AR.** The filesystem adapter detection uses `isinstance` check against `FilesystemAdapter`, which creates a concrete coupling. A capability-based check (e.g., `hasattr(adapter, 'search')` or a protocol attribute) would be more extensible. Deferred to future refactoring.
- **CWD hardcoding.** The default output path is `Path("governance/backlog.md")` relative to CWD. This works for CLI invocation but is fragile for programmatic use. Should accept an explicit base path parameter.
- **JQL coupling.** The sync function builds a JQL query string (`issuetype = Epic`), coupling it to Jira's query language. If another adapter type is added (e.g., GitHub Issues), this will need abstraction. Acceptable for now since Jira is the only remote adapter.

## Heutagogical Checkpoint

### 1. What did you learn?

- **Atomic file writes in Python** are best done with `os.replace()` (not `os.rename()`) because `os.replace` is atomic on POSIX and handles cross-device scenarios on Windows. The `.tmp` suffix convention in the same directory avoids cross-filesystem rename failures.
- **Markdown table escaping** is a frequently overlooked edge case — any user-generated content (Jira summaries, labels) can contain pipe characters that break table formatting.
- **Adapter capability detection** is a design decision with trade-offs: isinstance is explicit and discoverable but creates coupling; protocol-based checks (duck typing) are more flexible but less obvious. For a 2-adapter system, isinstance is acceptable.

### 2. What would you change about the process?

- The QR found 3 issues that should have been caught earlier — the tmp suffix naming, os.replace vs os.rename, and pipe escaping. These are all "defensive programming" concerns that could be part of a pre-commit checklist for any file-writing code. Would benefit from a "filesystem operations" pattern checklist in the skill template.

### 3. Are there improvements for the framework?

- **Pattern idea:** A "filesystem write checklist" pattern that triggers when creating code that writes files — check atomic write, check temp file naming, check content escaping for the target format. Not implemented yet, captured as idea.
- The review skill's AR/QR split continues to work well — AR catches architectural concerns (isinstance coupling, CWD hardcoding), QR catches implementation defects (tmp suffix, os.replace, escaping).

### 4. What are you more capable of now?

- Building CLI commands that bridge adapter protocols to local file artifacts. The sync command is a reusable pattern: query adapter -> format output -> atomic write. This same shape applies to future commands like `rai backlog export`, `rai docs sync`, etc.
- Better instinct for markdown generation edge cases — pipe escaping, timestamp formatting, header conventions.
