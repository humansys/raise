## What to Improve

- **Estimation was accurate** — S sizing for 4 tasks was correct. No surprises.

## Heutagogical Checkpoint

### 1. What did I learn?

Label-based search (`labels = "rai:S347.4"`) is strictly better than summary search (`summary ~ "S347.4"`) for deterministic matching. Summary search is fuzzy by nature — `S347.4` matches `S347.40`, `S347.41`, etc. Labels are exact. The fallback-with-warning pattern gives migration path without breaking existing workflows.

### 2. What would I change about the process?

Start with typed function signatures in the RED phase. The `adapter: Any` that slipped through T1-T3 and needed a QR fix commit could have been caught by writing `adapter: ProjectManagementAdapter` in the test mocks from the beginning.

### 3. Are there improvements for the framework?

The isinstance dispatch pattern (D2) works for 2-3 adapter types but has a clear scaling limit. If we add a third adapter with different search semantics, we should refactor to a `SearchStrategy` protocol dispatched by adapter type. No framework change needed now — just awareness.

### 4. What am I more capable of now?

Better at adapter-agnostic design. The pattern of `resolve_adapter(None)` + isinstance dispatch + label-first search is reusable for any hook that needs to query the backlog adapter without assuming Jira.
