# Scope: Skills via `rai backlog` CLI

## In Scope

- Rewrite `rai-epic-start` SKILL.md: replace "register in governance/backlog.md" with `rai backlog create` command
- Rewrite `rai-epic-close` SKILL.md: replace "mark complete in governance/backlog.md" with `rai backlog transition {key} done`
- Add backlog transition step to `rai-story-start` SKILL.md: `rai backlog transition {key} in_progress` (when ticket found)
- Add backlog transition step to `rai-story-close` SKILL.md: `rai backlog transition {key} done`
- Edit builtin skills first (`src/rai_cli/skills_base/`), then replicate to `.claude/skills/`
- Verify all 4 skills have zero references to direct backlog.md editing

## Out of Scope

- Changing the `rai backlog` CLI implementation (already done in S347.1/S347.2)
- Adding new skill steps unrelated to backlog operations
- Modifying skill metadata or structure beyond the backlog-related steps
- Automating the `rai backlog` calls (skills are prompts for the agent, not scripts)
- BacklogHook changes (that is S347.4)
- Session-start changes (that is S347.5)

## Done Criteria

1. `rai-epic-start` SKILL.md references `rai backlog create` instead of direct backlog.md edit
2. `rai-epic-close` SKILL.md references `rai backlog transition` instead of direct backlog.md edit
3. `rai-story-start` SKILL.md includes a step to transition story ticket to in_progress via CLI
4. `rai-story-close` SKILL.md includes a step to transition story ticket to done via CLI
5. Zero occurrences of "edit governance/backlog.md" or "register in backlog.md" in the 4 skills
6. All 4 builtin skills match their `.claude/skills/` deployments (verified with diff)
7. `rai skill validate` passes for all 4 modified skills
8. Existing skill structure and flow preserved (only backlog steps changed)
