# Story: Clean up stale epic-branch references across live files

## User Story
As a developer using RaiSE,
I want all live files to reflect the current branch model (dev → story, no epic branches),
so that documentation and skills don't contradict the actual workflow.

## Acceptance Criteria

### Scenario: Live files reference correct branch model
```gherkin
Given S355.1 removed epic branches from the branch model
When I grep for "epic/" branch patterns in live files
Then no live file references epic branches as part of the workflow
```

### Scenario: Historical artifacts are untouched
```gherkin
Given retrospectives, past stories, and ADRs reference the old model
When the cleanup runs
Then historical artifacts are not modified
```

## Examples (Specification by Example)

| File | Old Reference | New Reference |
|------|--------------|---------------|
| methodology.yaml | "branch from epic" | "branch from dev" |
| MEMORY.md | worktree/epic branch protocol | updated or removed |
| README.md | 3-level branch diagram | 2-level branch diagram |
| scope templates | epic branch references | dev-direct references |
