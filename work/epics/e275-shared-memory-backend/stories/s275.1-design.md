# S275.1 Design: Extract rai-core Package

**Complexity:** Moderate (broad blast radius, mechanical work, new uv workspace setup)
**Architecture review:** 2026-02-25 — PASS (R1 incorporated, no shims)

## Problem

The graph domain (models, engine, query, backends) lives inside `rai_cli`, making it impossible for `rai-server` (PRO) to use shared models without depending on the full CLI package. This blocks the entire E275 epic.

## Value

Unblocks all subsequent stories (S275.2–S275.7). Establishes the package boundary between COMMUNITY (rai-core + rai-cli) and PRO (rai-server). Done when: `from rai_core.graph.models import GraphNode` works and all 2790 existing tests pass.

## Approach

Pure extraction — no new logic. Move files, rename classes (drop "Unified"), update ALL imports directly. **No re-export shims** — there are no external consumers, all code is ours, and 2790 tests catch any breakage. Clean cut.

**uv workspace** is the mechanism: root `pyproject.toml` declares workspace members, each member has its own `pyproject.toml`. `uv sync` resolves local packages as editable installs.

### Migration Strategy: Direct Update (no shims)

Decision: Since all code is in this repo and there are no external consumers on old import paths, we update every import directly to the new `rai_core` paths. No backward compat layer, no re-exports, no dead shim files. Tests are our safety net.

**Consequence:** Old files (`rai_cli/context/models.py`, `rai_cli/context/graph.py`, `rai_cli/graph/filesystem_backend.py`) are **deleted** after extraction, not converted to shims. `rai_cli/context/query.py` is deleted — `from_file()` is replaced by inline 2-liner at the 3 call sites. `rai_cli/graph/` directory is removed entirely.

Files that **remain but are modified** (they still have their own code):
- `rai_cli/adapters/protocols.py` — keeps 4 protocols, imports `KnowledgeGraphBackend` from itself no longer (removed)
- `rai_cli/adapters/models.py` — keeps 5 models, `BackendHealth` removed
- `rai_cli/context/__init__.py` — updated to import from `rai_core` paths
- `rai_cli/context/diff.py` — updated imports
- `rai_cli/context/builder.py` — updated imports

### Dependency Chain (move order matters)

```
rai_core/graph/models.py           ← no deps (move first)
rai_core/graph/backends/models.py  ← no deps (BackendHealth)
rai_core/graph/backends/protocol.py ← imports graph/models + backends/models
rai_core/graph/engine.py           ← imports graph/models
rai_core/graph/backends/filesystem.py ← imports engine + backends/models
rai_core/graph/query.py            ← imports graph/models + engine (move last)
```

### `from_file()` Elimination

3 call sites use `UnifiedQueryEngine.from_file(path)`. Replace each with inline equivalent:

```python
# Before:
engine = UnifiedQueryEngine.from_file(path)

# After:
from rai_core.graph.backends.filesystem import get_active_backend
from rai_core.graph.query import QueryEngine
graph = get_active_backend(path).load()
engine = QueryEngine(graph)
```

Call sites:
- `src/rai_cli/session/bundle.py:134`
- `src/rai_cli/cli/commands/graph.py:135`
- `src/rai_cli/cli/commands/graph.py:347`

### Entry Point Update

```toml
# Before:
[project.entry-points."rai.graph.backends"]
local = "rai_cli.graph.filesystem_backend:FilesystemGraphBackend"

# After:
[project.entry-points."rai.graph.backends"]
local = "rai_core.graph.backends.filesystem:FilesystemGraphBackend"
```

## Acceptance Criteria

**MUST:**
1. `uv sync` resolves rai-core as workspace package without errors
2. `from rai_core.graph.models import GraphNode` works
3. `from rai_core.graph.engine import Graph` works (new name)
4. Full test suite passes (2790 tests, zero regressions)
5. `pyright` passes with zero new errors
6. `ruff check` passes
7. No circular imports between `rai_core` and `rai_cli`
8. Old files deleted (`rai_cli/context/models.py`, `graph.py`, `query.py`, `rai_cli/graph/`)

**SHOULD:**
1. `rai_core/workflow/__init__.py` and `rai_core/governance/__init__.py` exist with docstrings
2. `ConceptNode`/`ConceptEdge` aliases kept in `rai_core/graph/models.py` for readability

**MUST NOT:**
1. Move `ProjectManagementAdapter`, `GovernanceSchemaProvider`, `GovernanceParser`, `DocumentationTarget` to core
2. Move `CoreArtifactType`, `ArtifactLocator`, `IssueSpec`, `IssueRef`, `PublishResult` to core
3. Move `builder.py` to core
4. Leave re-export shims or dead files
5. Add any new functionality — pure extraction + rename
6. Break any existing CLI command behavior
