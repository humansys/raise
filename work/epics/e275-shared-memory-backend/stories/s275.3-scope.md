# S275.3 Scope: FastAPI Server Bootstrap

## In Scope
- FastAPI app factory (`create_app()`) with lifespan
- Configuration via pydantic-settings (`ServerConfig`)
- API key auth middleware (validate hash, check is_active, inject OrgContext)
- Health endpoint (`GET /health` — public, checks DB)
- Dockerfile.server (multi-stage, uv-based)
- Docker Compose update (add server service)
- Tests for all components

## Out of Scope
- Graph CRUD endpoints (S275.4)
- Query/subgraph endpoints (S275.5)
- Rate limiting / throttling
- CORS configuration (not needed until frontend)
- OpenAPI customization beyond auto-generated
- CI/CD pipeline changes
- Production deployment configuration

## Done When
- `GET /health` returns 200 with DB connectivity check
- Requests without valid API key get 401
- Requests with valid API key get OrgContext injected
- `docker compose up` starts server + PG, health reachable
- All tests pass (new + existing 2797)
- pyright + ruff clean
