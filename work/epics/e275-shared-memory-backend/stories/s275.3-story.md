# S275.3: FastAPI Server Bootstrap

**JIRA:** [RAISE-278](https://humansys.atlassian.net/browse/RAISE-278)
**Size:** M
**Epic:** E275 — Shared Memory Backend

## User Story

As a RaiSE platform developer,
I want a FastAPI application with API key authentication and health endpoint running in Docker,
so that the rai-server is ready to receive graph CRUD endpoints in subsequent stories.

## Acceptance Criteria

```gherkin
Scenario: FastAPI app starts and serves health endpoint
  Given the rai-server package
  When I start the application
  Then GET /health returns 200 with {"status": "ok", "database": "connected"}

Scenario: Configuration via environment variables
  Given the rai-server app
  When DATABASE_URL, API_KEY_HASH_ALGORITHM, and LOG_LEVEL are set
  Then the app uses those values
  And missing required vars cause a clear startup error

Scenario: API key authentication middleware
  Given an active API key in the database
  When I send a request with header "Authorization: Bearer rsk_<key>"
  Then the request is authenticated
  And OrgContext is available with the org_id from the key

Scenario: Unauthenticated request is rejected
  Given no Authorization header or an invalid key
  When I send a request to a protected endpoint
  Then I receive 401 Unauthorized with a clear error message

Scenario: Health endpoint is public
  Given no Authorization header
  When I send GET /health
  Then I receive 200 (no auth required)

Scenario: Dockerfile builds and runs
  Given the Dockerfile.server at repo root
  When I run `docker compose up`
  Then the server container starts and connects to PostgreSQL
  And the health endpoint is reachable

Scenario: All quality gates pass
  When I run pytest, pyright, and ruff
  Then all pass with zero regressions
```

## Notes
- Builds on S275.2: models, session factory, Docker Compose with PG 16
- API key uses `rsk_` prefix convention from S275.2 ApiKey model
- Auth middleware validates key_hash, checks is_active, resolves org_id
- Health endpoint checks DB connectivity (async session ping)
- Config via pydantic-settings (BaseSettings with env vars)
