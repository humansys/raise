# S275.6 — ApiGraphBackend + DualWrite

## User Story

**As a** developer using raise-cli with a shared server,
**I want** my graph builds to persist both locally and to the remote server,
**So that** my team can share knowledge graphs across repos without changing my workflow.

## Acceptance Criteria

```gherkin
Feature: ApiGraphBackend + DualWrite

  Scenario: ApiGraphBackend persists graph to server
    Given RAI_SERVER_URL is set to a running rai-server
    And RAI_API_KEY is set to a valid API key
    When I call persist(graph) on ApiGraphBackend
    Then the graph is sent to POST /api/v1/graph/sync
    And nodes and edges are persisted on the server

  Scenario: ApiGraphBackend loads graph from server
    Given a graph has been synced to the server
    When I call load() on ApiGraphBackend
    Then the graph is fetched from GET /api/v1/graph/query
    And a Graph instance is returned with the server data

  Scenario: ApiGraphBackend health check
    When I call health() on ApiGraphBackend
    Then it returns healthy when server responds to GET /health
    And it returns unavailable when server is unreachable

  Scenario: DualWriteBackend writes to both backends
    Given RAI_SERVER_URL is configured
    When I call persist(graph) on DualWriteBackend
    Then the graph is persisted locally (FilesystemGraphBackend)
    And the graph is sent to the remote server (ApiGraphBackend)
    And local always succeeds even if remote fails

  Scenario: DualWriteBackend remote failure is best-effort
    Given RAI_SERVER_URL is configured but server is down
    When I call persist(graph) on DualWriteBackend
    Then local persist succeeds
    And remote failure is logged as warning (not raised)

  Scenario: DualWriteBackend loads from local
    When I call load() on DualWriteBackend
    Then it loads from the local FilesystemGraphBackend
    And never loads from the remote (local is source of truth)

  Scenario: get_active_backend selects backend by env vars
    Given RAI_SERVER_URL is not set
    When get_active_backend is called
    Then FilesystemGraphBackend is returned

  Scenario: get_active_backend with server configured
    Given RAI_SERVER_URL is set
    And RAI_API_KEY is set
    When get_active_backend is called
    Then DualWriteBackend is returned (local + remote)
```
