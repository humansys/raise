## Execution Order

```
T1 (config + app + health)
  ↓
T2 (auth middleware)
  ↓
T3 (Dockerfile + integration)
```

Sequential. Each builds on the previous. Risk-first: T1 establishes the running server — if the app factory pattern doesn't work, we find out immediately.

## Risks

| Risk | L/I | Mitigation |
|------|:---:|------------|
| uv workspace + Dockerfile complexity | M/M | Follow uv Docker docs. Test build before compose. |
| AsyncSession in auth dep (request scope) | L/M | Use `session_factory()` from app.state, not engine directly. |
| pydantic-settings env prefix conflicts | L/L | Use `RAI_` prefix, test explicitly. |
