# S275.5 — Skills & Templates sync + serve — Design

## Problem

Rovo AI agents need access to RaiSE skills (methodology prompts) and governance templates to execute workflows. Currently these live only in the CLI filesystem. Fernando needs API endpoints to build Forge tools that serve this content to Rovo.

## Value

Unblocks Fernando's Forge app POC: Rovo agent retrieves a skill prompt + template via API, follows the steps, and creates governance documents in Confluence.

## Approach

Replicate the three-layer pattern from S275.4 (graph sync) for two new domains: skills and templates. Both are markdown blobs with metadata, stored per-org, synced from CLI (source of truth).

**Key decision:** Skills and templates share the same shape (name, description, content). One DB table with a `kind` discriminator (`skill` | `template`) avoids schema duplication.

### Missing-Entity Strategy (PAT-E-523)

- `GET /skills/{name}` where name doesn't exist → **404** `{"detail": "Skill not found"}`
- `GET /templates/{name}` where name doesn't exist → **404** `{"detail": "Template not found"}`
- Sync replaces all items for (org_id, kind) — same as graph sync

## Examples

### DB Model

```python
class ContentItemRow(Base):
    __tablename__ = "content_items"
    __table_args__ = (
        UniqueConstraint("org_id", "kind", "name", name="uq_content_org_kind_name"),
    )

    id: Mapped[uuid.UUID] = mapped_column(Uuid, primary_key=True, server_default=func.gen_random_uuid())
    org_id: Mapped[uuid.UUID] = mapped_column(ForeignKey("organizations.id"), index=True)
    kind: Mapped[str] = mapped_column(String(20), index=True)  # "skill" | "template"
    name: Mapped[str] = mapped_column(String(255))
    description: Mapped[str] = mapped_column(Text, server_default="")
    content: Mapped[str] = mapped_column(Text)
    properties: Mapped[dict] = mapped_column(JSONB, server_default="{}")
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now(), onupdate=func.now())
```

### Schemas

```python
class ContentItemInput(BaseModel):

    items: list[ContentItemSummary]

    name: str
    description: str
    content: str
    properties: dict[str, Any]
```

**Sync skills:**

```
    }

→ 200 {"status": "ok", "kind": "skill", "items_synced": 2}
```

**List skills:**
```
GET /api/v1/skills
Authorization: Bearer rsk_abc123

→ 200 {"items": [{"name": "lean-business-case", "description": "Create a lean..."}, ...], "total": 2, "kind": "skill"}
```

**Get skill:**
```
GET /api/v1/skills/lean-business-case
Authorization: Bearer rsk_abc123

→ 200 {"name": "lean-business-case", "description": "Create a lean...", "content": "# Lean Business Case\n...", "properties": {}}
```

**Skill not found:**
```
GET /api/v1/skills/nonexistent
Authorization: Bearer rsk_abc123

→ 404 {"detail": "Skill not found"}
```

**Templates — identical pattern:**
```
POST /api/v1/templates/sync   → sync templates
GET  /api/v1/templates        → list templates
GET  /api/v1/templates/{name} → get template content
```

### Service Layer

```python
async def sync_content(session_factory, org_id, kind, request):
    async with session_factory() as session, session.begin():
        count = await upsert_content_items(session, org_id, kind, request.items)
        await prune_content_items(session, org_id, kind, keep_names=[i.name for i in request.items])
    return ContentSyncResponse(kind=kind, items_synced=count)

async def list_content(session_factory, org_id, kind):
    async with session_factory() as session:
        items = await get_content_items(session, org_id, kind)
    return ContentListResponse(items=items, total=len(items), kind=kind)

async def get_content(session_factory, org_id, kind, name):
    async with session_factory() as session:
        item = await get_content_item(session, org_id, kind, name)
    if item is None:
        label = "Skill" if kind == "skill" else "Template"
        raise HTTPException(status_code=404, detail=f"{label} not found")
    return ContentItemDetail(**item)
```

## Acceptance Criteria

### MUST
1. `POST /api/v1/skills/sync` stores skills, returns `items_synced` count
2. `GET /api/v1/skills` lists skills (name + description)
3. `GET /api/v1/skills/{name}` returns full content; 404 if missing
4. Same three endpoints for templates (`/api/v1/templates/*`)
5. Auth required on all endpoints (OrgContext)
6. Sync is replace strategy (upsert + prune orphans)
7. All tests pass at 4 layers (schemas, db, services, API)

### SHOULD
1. `properties` JSONB field for extensibility (metadata, tags, version)

### MUST NOT
1. No skill execution (server-side inference) — out of scope
2. No template rendering / variable substitution
3. No versioning / history
