## What Went Well

- Clean 3-task decomposition: model → integration → validation
- Optional `raise_dir` param preserved backward compat — zero changes to existing tests
- QR caught a real semantic bug (`_path` vs `path`) that would have silently broken marker content in production

## What to Improve

- Should have checked `FilesystemGraphBackend.path` attribute name during implementation, not relied on memory. The `getattr` with fallback masked the error.

## Heutagogical Checkpoint

1. **Learned:** `getattr` with defaults on protocol-typed objects hides attribute name mismatches — type checkers and tests can't catch this if the default is plausible
2. **Process change:** None — story was well-scoped
3. **Framework improvements:** None needed
4. **More capable of:** Offline fallback pattern (marker file + clear-on-success) is reusable

## Patterns

- **PAT-E-538:** getattr Gemba — verify actual attribute name against source class when accessing injected dependencies

## QR Fixes Applied

1. `getattr(self.local, "_path")` → `"path"` (critical — wrong attribute name)
2. `except (JSONDecodeError, Exception)` → `(JSONDecodeError, ValueError)` (narrower catch)
3. `tmp_path: object` → `tmp_path: Path` in tests (hygiene)
