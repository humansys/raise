## What Went Well

- Clean 3-task decomposition: model → integration → validation
- Optional `raise_dir` param preserved backward compat — zero changes to existing tests
- QR caught a real semantic bug (`_path` vs `path`) that would have silently broken marker content in production

## What to Improve

- Should have checked `FilesystemGraphBackend.path` attribute name during implementation, not relied on memory. The `getattr` with fallback masked the error.
- **E2E should have been in the original story design.** The story was called "Dogfood" but designed as a unit-testable feature. Cross-story integration bugs (auth header mismatch, asyncpg param limit) were invisible to mocks. Fixed by adding Integration gate to `/rai-story-design` and Integration checkpoint to `/rai-epic-plan` (PAT-E-539).

## Heutagogical Checkpoint

1. **Learned:** `getattr` with defaults on protocol-typed objects hides attribute name mismatches — type checkers and tests can't catch this if the default is plausible
2. **Process change:** Stories named "dogfood"/"E2E" must include real infra scenarios in AC
3. **Framework improvements:** Added Integration gate to rai-story-design + Integration checkpoint to rai-epic-plan
4. **More capable of:** Offline fallback pattern (marker file + clear-on-success) is reusable

## Patterns

- **PAT-E-538:** getattr Gemba — verify actual attribute name against source class when accessing injected dependencies
- **PAT-E-539:** E2E integration gate — multi-component epics need real infra validation, not just mocks

## QR Fixes Applied

1. `getattr(self.local, "_path")` → `"path"` (critical — wrong attribute name)
2. `except (JSONDecodeError, Exception)` → `(JSONDecodeError, ValueError)` (narrower catch)
3. `tmp_path: object` → `tmp_path: Path` in tests (hygiene)

## E2E Fixes Applied

1. `x-api-key` → `Authorization: Bearer` (auth header mismatch between S275.3 server and S275.6 client)
2. Skip nodes with `content=""` (server validates `min_length=1`)
3. Batch INSERT at 4000 rows (asyncpg 32767 param limit, 33k edges * 7 = 231k params)
4. Dev seed script for org + API key (`seed_dev.sql`)
5. Skills updated: rai-story-design + rai-epic-plan with integration gates
