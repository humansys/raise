# S275.2: PostgreSQL Schema + Alembic

**JIRA:** [RAISE-277](https://humansys.atlassian.net/browse/RAISE-277)
**Size:** S
**Epic:** E275 — Shared Memory Backend

## User Story

As a RaiSE platform developer,
I want a PostgreSQL schema with nodes/edges tables and Alembic migrations running in Docker Compose,
so that the rai-server has a tested persistence layer ready for the FastAPI endpoints.

## Acceptance Criteria

```gherkin
Scenario: rai-server package exists as uv workspace member
  Given the monorepo root pyproject.toml
  When I run `uv sync`
  Then rai-server is resolved as a workspace package
  And rai-server depends on rai-core

Scenario: SQLAlchemy models match research schema
  Given the rai_server.db.models module
  Then Organization model has id (UUID), name, slug, created_at
  And ApiKey model has id (UUID), org_id (FK), key_hash, prefix, is_active, created_at
  And GraphNode model has id (UUID), org_id (FK), repo_id, scope, node_type, content, properties (JSONB), created_at, updated_at
  And GraphEdge model has id (UUID), org_id (FK), repo_id, source_id (FK), target_id (FK), edge_type, weight, properties (JSONB), created_at

Scenario: Alembic initial migration creates tables
  Given a fresh PostgreSQL database
  When I run `alembic upgrade head`
  Then all 4 tables exist with correct columns and constraints
  And GIN indexes exist on JSONB properties columns

Scenario: Docker Compose provides dev environment
  Given the docker-compose.yml at repo root
  When I run `docker compose up -d`
  Then a PostgreSQL 16 container starts on port 5432
  And the database is accessible with configured credentials

Scenario: All quality gates pass
  When I run pytest, pyright, and ruff
  Then all pass with zero regressions
```
