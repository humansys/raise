# S275.4: Graph Sync + Query Endpoints

**JIRA:** [RAISE-279](https://humansys.atlassian.net/browse/RAISE-279)
**Size:** M
**Epic:** E275 — Shared Memory Backend

## User Story

As a RaiSE CLI user with a server connection,
I want to sync my local knowledge graph to the server and query the shared graph,
so that my team can share patterns and architectural knowledge across repositories.

## Acceptance Criteria

```gherkin
Scenario: Sync a full knowledge graph
  Given an authenticated API key for org "acme"
  When I POST /api/v1/graph/sync with a graph payload (nodes + edges)
  Then all nodes and edges are upserted for that org
  And existing nodes with same UUIDs are updated (idempotent)
  And the response includes counts of created/updated items

Scenario: Query the shared graph by keyword
  Given org "acme" has a synced graph with node "mod-memory" described as "Session and pattern memory"
  When I GET /api/v1/graph/query?q=memory&limit=10
  Then the response includes nodes matching "memory"
  And results are scoped to the authenticated org only
  And each result includes node_type, label, description, and relevance

Scenario: Sync is org-isolated
  Given org "acme" and org "beta" both have synced graphs
  When org "acme" queries the graph
  Then only org "acme" nodes and edges are returned

Scenario: Invalid graph payload rejected
  Given an authenticated request
  When I POST /api/v1/graph/sync with malformed data
  Then the response is 422 with validation errors

Scenario: Empty query returns helpful response
  Given an authenticated request
  When I GET /api/v1/graph/query without q parameter
  Then the response is 422 with a clear error message
```

## Specification by Example

### Sync payload
```json
{
  "project_id": "raise-commons",
  "nodes": [
    {"uuid": "550e8400-...", "node_type": "module", "label": "mod-memory", "description": "Session and pattern memory", "properties": {"language": "python"}}
  ],
  "edges": [
    {"uuid": "660e8400-...", "source_node_uuid": "550e8400-...", "target_node_uuid": "770e8400-...", "edge_type": "depends_on", "properties": {}}
  ]
}
```

### Sync response
```json
{
  "status": "ok",
  "project_id": "raise-commons",
  "nodes_created": 12,
  "nodes_updated": 3,
  "edges_created": 8,
  "edges_updated": 1
}
```

### Query response
```json
{
  "results": [
    {"uuid": "550e8400-...", "node_type": "module", "label": "mod-memory", "description": "Session and pattern memory", "score": 0.85}
  ],
  "total": 1,
  "query": "memory",
  "limit": 10
}
```
