# S275.3 Design: FastAPI Server Bootstrap

**Complexity:** Moderate (5 components, architecture resolved in epic design)

## What & Why

**Problem:** rai-server has a DB layer but no running application. No way to start, authenticate, or verify the server.

**Value:** `docker compose up` starts a real server. API keys validated, DB connectivity confirmed. Foundation for all domain endpoints (S275.4+).

## Approach

Build the FastAPI application shell following the 3-layer architecture from epic design:

| Component | Action | Layer |
|-----------|--------|-------|
| `app.py` | Create | Transport — app factory + lifespan |
| `config.py` | Create | Transport — pydantic-settings |
| `auth.py` | Create | Transport — API key dependency |
| `api/v1/health.py` | Create | Transport — health route |
| `Dockerfile` | Create | Infrastructure |

All components are new files. No existing code modified.

### Integration Decisions

1. **App factory pattern** — `create_app()` returns `FastAPI` instance. Lifespan creates engine + session factory, stores in `app.state`.
2. **Config** — `pydantic-settings` `ServerConfig` reads from env vars. Single source of truth.
3. **Auth** — FastAPI `Depends()` that extracts `Bearer rsk_...` header, hashes key, queries `api_keys` table, checks `is_active`, returns `OrgContext`.
4. **Health** — Public endpoint (no auth). Pings DB with `SELECT 1`. Returns server version.
5. **Dockerfile** — Multi-stage (builder + runtime). Uses `uv` for dependency resolution. Runs with `uvicorn`.

### App factory

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from rai_server.config import ServerConfig
from rai_server.db.session import create_engine, create_session_factory

@asynccontextmanager
async def lifespan(app: FastAPI):
    config = app.state.config
    engine = create_engine(config.database_url)
    app.state.engine = engine
    app.state.session_factory = create_session_factory(engine)
    yield
    await engine.dispose()

def create_app() -> FastAPI:
    config = ServerConfig()
    app = FastAPI(title="RaiSE Server", version="0.1.0", lifespan=lifespan)
    app.state.config = config
    app.include_router(health_router)
    return app
```

### Auth dependency

```python
from fastapi import Depends, Header, HTTPException, Request
from pydantic import BaseModel

class OrgContext(BaseModel):
    org_id: uuid.UUID
    org_name: str

async def verify_api_key(
    request: Request,
    authorization: str = Header(...),
) -> OrgContext:
    # Extract "Bearer rsk_..."
    if not authorization.startswith("Bearer rsk_"):
        raise HTTPException(status_code=401, detail="Invalid API key format")
    raw_key = authorization.removeprefix("Bearer ")
    key_hash = hashlib.sha256(raw_key.encode()).hexdigest()

    # Query DB
    session_factory = request.app.state.session_factory
    async with session_factory() as session:
        row = await session.execute(
            select(ApiKey, Organization)
            .join(Organization)
            .where(ApiKey.key_hash == key_hash, ApiKey.is_active == True)
        )
        result = row.first()
        if not result:
            raise HTTPException(status_code=401, detail="Invalid or inactive API key")
        api_key, org = result
        return OrgContext(org_id=org.id, org_name=org.name)
```

### Health endpoint

```python
@router.get("/health")
async def health(request: Request):
    engine = request.app.state.engine
    try:
        async with engine.connect() as conn:
            await conn.execute(text("SELECT 1"))
        db_status = "connected"
    except Exception:
        db_status = "disconnected"
    return {"status": "ok", "database": db_status, "version": "0.1.0"}
```

```bash
# Public — no auth needed
$ curl http://localhost:8000/health
{"status": "ok", "database": "connected", "version": "0.1.0"}

```

### Dockerfile

```dockerfile
# Stage 1: build
FROM python:3.12-slim AS builder
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv
WORKDIR /app
COPY pyproject.toml uv.lock ./
COPY packages/rai-core/ packages/rai-core/
COPY packages/rai-server/ packages/rai-server/
RUN uv sync --package rai-server --no-dev --frozen

# Stage 2: runtime
FROM python:3.12-slim
WORKDIR /app
COPY --from=builder /app/.venv /app/.venv
COPY packages/rai-server/alembic/ alembic/
COPY packages/rai-server/alembic.ini .
ENV PATH="/app/.venv/bin:$PATH"
EXPOSE 8000
CMD ["uvicorn", "rai_server.app:create_app", "--factory", "--host", "0.0.0.0", "--port", "8000"]
```

## Acceptance Criteria

### MUST
1. `GET /health` returns 200 with `{"status": "ok", "database": "connected"}` when DB is up
2. `GET /health` returns 200 with `{"database": "disconnected"}` when DB is down (server stays up)
3. Request without `Authorization` header to protected endpoint → 401
4. Request with invalid key → 401 with clear message
5. Request with valid active key → `OrgContext` available with correct org_id
6. `docker compose up` starts server + PG, health reachable at localhost:8000
7. All existing tests pass (2797+), pyright + ruff clean

### SHOULD
1. `ServerConfig` validates required fields at startup (fast-fail)
2. Structured JSON logging with configurable level

### MUST NOT
- Health endpoint MUST NOT require authentication
- Server MUST NOT start if DATABASE_URL is missing (fail fast, clear error)
- Auth MUST NOT store raw API keys anywhere (only hashes)
