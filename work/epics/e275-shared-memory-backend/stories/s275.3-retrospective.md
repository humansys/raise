## What Went Well

- **Research-driven architecture revision.** The user questioned the CRUD API design, which triggered 4 parallel research subagents (47 sources, ~4 minutes). This unanimously confirmed: domain-level intent operations, not CRUD. The epic design was revised before any implementation — saving significant rework.
- **Implementation was fast and clean.** Once design was solid, T1-T3 flowed smoothly. TDD caught nothing unexpected — a sign the design was well-specified.
- **Quality review earned its keep.** Found 6 issues in code that passed all automated gates: version duplication, circular import pattern, bare exception, muda test, monkeypatch-heavy test style, orphan config field.
- **Full skill cycle for a "bootstrap" story.** Would have been tempting to skip design for "just wiring." But the design phase is where the API ontology question surfaced.

## What To Improve

- **API consumer analysis belongs in epic design.** The CRUD→domain pivot happened during S275.3 but should have happened during `/rai-epic-design`. The framework should prompt: "Who consumes this API? What are their cognitive constraints?" → Pattern PAT-E-515.
- **deps.py from the start.** Circular imports appeared after T1 and were caught in QR. Next FastAPI project: create `deps.py` in T1 → Pattern PAT-E-510.
- **Docker multi-stage with uv needs `uv pip install`.** Discovered via container crash. Should have been known from docs → Pattern PAT-E-512.

## Heutagogical Checkpoint

1. **What did I learn?** Agentic consumers fundamentally change API design. The same backend needs different surfaces for different consumer types. Subgraph-on-demand (Option B) resolves the rai-core impedance mismatch.
2. **What would I change?** Research at epic design time, not story time. 4 parallel subagents as default for cross-cutting decisions.
3. **Framework improvements?** Add "API consumer analysis" to epic design skill. Validate QR remains non-optional.
4. **More capable of?** FastAPI factory pattern, Docker+uv builds, research-driven API ontology design.
