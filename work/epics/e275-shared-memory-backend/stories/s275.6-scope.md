# S275.6 — ApiGraphBackend + DualWrite — Scope

## In Scope

- `ApiGraphBackend` class implementing `KnowledgeGraphBackend` protocol
  - httpx client for HTTP calls to rai-server
  - `persist()` → POST /api/v1/graph/sync
  - `load()` → GET /api/v1/graph/query (all nodes/edges)
  - `health()` → GET /health
- `DualWriteBackend` class implementing `KnowledgeGraphBackend` protocol
  - Delegates to both `FilesystemGraphBackend` (local) and `ApiGraphBackend` (remote)
  - Local always succeeds; remote is best-effort (log warning on failure)
  - `load()` reads from local only (local is source of truth)
- Update `get_active_backend()` to return `DualWriteBackend` when `RAI_SERVER_URL` + `RAI_API_KEY` are set
- Env vars: `RAI_SERVER_URL`, `RAI_API_KEY`
- Graph serialization: convert Graph (NetworkX) → sync payload format (nodes + edges dicts)

## Out of Scope

- Offline queue / pending_sync → S275.7
- Sync-on-reconnect → S275.7
- Scope filtering (PERSONAL vs PROJECT) → parking lot
- Server-side changes (server endpoints already exist from S275.4)
- Auth token refresh / rotation
- Async persist (persist is synchronous in protocol)
