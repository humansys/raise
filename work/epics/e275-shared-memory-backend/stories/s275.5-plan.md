## Execution Order

```
T1 (models + migration) ──┐
                           ├→ T3 (queries) → T4 (services) → T5 (routes) → T6 (integration)
T2 (schemas) ─────────────┘
```

**T1 and T2 are parallel** — no dependency between models and schemas.
**T3 onwards is sequential** — each layer depends on the previous.

**Risk-first:** T1 (migration) has the most risk (Alembic config, FK to organizations). Do it first.
