# Story: Skills & Templates sync + serve

## User Story
As a Forge app developer (Fernando),
I want the server to store and serve RaiSE skills and templates via API,
so that Rovo agents can retrieve governance methodology and document templates to execute workflows.

## Acceptance Criteria

### Scenario: Sync skills from CLI
```gherkin
Given the CLI has 3 skills in .raise/skills/
When I call POST /api/v1/skills/sync with the skill payloads
Then the server stores all 3 skills for the org
And returns a count of synced skills
```

### Scenario: List available skills
```gherkin
Given the server has skills synced for my org
When I call GET /api/v1/skills
Then I receive a list with name and description for each skill
```

### Scenario: Get skill content
```gherkin
Given the server has a skill "lean-business-case" for my org
When I call GET /api/v1/skills/lean-business-case
Then I receive the full markdown content of the skill
```

### Scenario: Sync templates from CLI
```gherkin
Given the CLI has governance templates in .raise/templates/
When I call POST /api/v1/templates/sync with the template payloads
Then the server stores all templates for the org
And returns a count of synced templates
```

### Scenario: List available templates
```gherkin
Given the server has templates synced for my org
When I call GET /api/v1/templates
Then I receive a list with name and description for each template
```

### Scenario: Get template content
```gherkin
Given the server has a template "problem-brief" for my org
When I call GET /api/v1/templates/problem-brief
Then I receive the full markdown content of the template
```

### Scenario: Skill not found returns 404
```gherkin
Given no skill named "nonexistent"
When I call GET /api/v1/skills/nonexistent
Then I receive 404 with detail "Skill not found"
```

## Examples (Specification by Example)

| Endpoint | Input | Expected |
|----------|-------|----------|
| POST /skills/sync | [{name, description, content}] | {synced: 3} |
| GET /skills | auth header | [{name, description}] |
| GET /skills/lean-business-case | auth header | {name, description, content} |
| GET /skills/nonexistent | auth header | 404 |
| POST /templates/sync | [{name, description, content}] | {synced: 5} |
| GET /templates | auth header | [{name, description}] |
| GET /templates/problem-brief | auth header | {name, description, content} |
