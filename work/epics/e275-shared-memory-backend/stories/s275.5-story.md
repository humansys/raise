# Story: Agent telemetry + memory endpoints

## User Story
As a Rovo agent (via Forge tools),
I want to report my actions to the RaiSE server and update Rai's memory with learned patterns,
so that the system tracks what governance actions were taken and improves over time.

## Acceptance Criteria

### Scenario: Report agent event
```gherkin
Given a Rovo agent completed a governance skill
When it calls POST /api/v1/agent/events with event_type and payload
Then the server stores the event for the org
And returns the event id
```

### Scenario: Add learned pattern
```gherkin
Given a Rovo agent discovered a useful pattern during skill execution
When it calls POST /api/v1/memory/patterns with content and context
Then the server stores the pattern for the org
And returns the pattern id
```

### Scenario: List recent events
```gherkin
Given the org has telemetry events recorded
When I call GET /api/v1/agent/events?limit=10
Then I receive the 10 most recent events (newest first)
```

### Scenario: List patterns
```gherkin
Given the org has patterns stored
When I call GET /api/v1/memory/patterns
Then I receive all patterns for the org
```

## Examples (Specification by Example)

| Endpoint | Input | Expected |
|----------|-------|----------|
| POST /agent/events | {event_type: "skill_executed", payload: {skill: "lean-business-case", ...}} | {id: uuid, status: "ok"} |
| POST /memory/patterns | {content: "Always validate...", context: ["governance"]} | {id: uuid, status: "ok"} |
| GET /agent/events | auth + limit=5 | [{id, event_type, payload, created_at}, ...] |
| GET /memory/patterns | auth | [{id, content, context, created_at}, ...] |
