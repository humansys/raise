# Story: Trace, impact + dev constraints

## User Story
As a Rovo agent or Forge app consumer,
I want to traverse the knowledge graph for trace/impact analysis and query development constraints,
so that I can provide developers with dependency insight and guardrail enforcement.

## Acceptance Criteria

### Scenario: Trace upstream dependencies
```gherkin
Given a graph with nodes A → B → C (via edges)
When I call GET /api/v1/graph/trace?node_id={C_id}&direction=upstream
Then I receive [B, A] in traversal order
```

### Scenario: Impact downstream dependents
```gherkin
Given a graph with nodes A → B → C (via edges)
When I call GET /api/v1/graph/impact?node_id={A_id}&direction=downstream
Then I receive [B, C] in traversal order
```

### Scenario: Node not found returns 404
```gherkin
Given no node with id "nonexistent"
When I call GET /api/v1/graph/trace?node_id=nonexistent
Then I receive 404 with detail "Node not found"
```

### Scenario: Dev constraints lists guardrails
```gherkin
Given a graph with nodes of types [module, function, constraint, guardrail]
When I call GET /api/v1/dev/constraints
Then I receive only nodes where type in (constraint, guardrail)
```

### Scenario: Max depth limits traversal
```gherkin
Given a deep graph A → B → C → D → E
When I call GET /api/v1/graph/trace?node_id=E&max_depth=2
Then I receive at most 2 levels of ancestors [D, C]
```

## Examples (Specification by Example)

| Endpoint | Params | Expected |
|----------|--------|----------|
| GET /graph/trace | node_id=X, direction=upstream | BFS upstream from X |
| GET /graph/impact | node_id=X, direction=downstream | BFS downstream from X |
| GET /graph/trace | node_id=missing | 404 |
| GET /dev/constraints | org_id from auth | Nodes filtered by type ∈ {constraint, guardrail} |
