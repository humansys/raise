# S275.7 — Dogfood + Offline Fallback — Scope

## In Scope

- Dogfood DualWriteBackend with raise-commons real graph (`rai graph build`)
- `pending_sync` marker file (`.raise/pending_sync.json`) when remote write fails
- Marker contains: timestamp, graph_hash, error summary
- Sync-on-reconnect: clear pending on next successful remote write
- Integration test with real graph data (not just mocks)

## Out of Scope

- Automatic background retry / daemon process
- Partial sync (all-or-nothing per build)
- `rai graph sync-pending` CLI command (manual retry = just run `rai graph build` again)
- Server-side conflict resolution
- Notification/alert when offline

## Done When

- Real raise-commons graph syncs through DualWrite pipeline (manual verification)
- Failed remote writes produce `.raise/pending_sync.json` marker
- Successful remote writes clear any existing marker
- Tests cover: marker creation, marker clearing, marker content format
- All existing tests still pass
- Type checks + lint pass
