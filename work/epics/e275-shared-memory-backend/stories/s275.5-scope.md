# S275.5 — Agent telemetry + memory endpoints — Scope

## In Scope
- `POST /api/v1/agent/events` — record agent telemetry event (append-only)
- `GET /api/v1/agent/events` — list recent events (newest first, with limit)
- `POST /api/v1/memory/patterns` — add learned pattern
- `GET /api/v1/memory/patterns` — list patterns for org
- DB table(s) for events and patterns (per org)
- Alembic migration
- Pydantic schemas for request/response
- Service layer functions
- FastAPI route definitions with auth (OrgContext)
- Tests: schemas, db, services, API integration

## Out of Scope
- Event analytics / aggregation → future
- Pattern reinforcement / scoring → future (CLI has this, server doesn't yet)
- Pattern deduplication → future
- Event deletion / retention policy → future
- Skills/templates via API → Confluence native (see research)
- Trace/impact/constraints → parking lot

## Done When
- `POST /api/v1/agent/events` stores event, returns id
- `GET /api/v1/agent/events` lists events with pagination
- `POST /api/v1/memory/patterns` stores pattern, returns id
- `GET /api/v1/memory/patterns` lists patterns
- Auth required on all endpoints
- All tests pass (schemas, db, services, API)
- Type checks + lint pass
- Three-layer pattern followed
- Fernando has full OpenAPI spec (graph + telemetry + memory)
