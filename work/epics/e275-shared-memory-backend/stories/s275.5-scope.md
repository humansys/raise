# S275.5 — Trace, impact + dev constraints — Scope

## In Scope
- `GET /api/v1/graph/trace` — BFS traversal upstream/downstream from a node
- `GET /api/v1/graph/impact` — BFS traversal (alias with semantic distinction)
- `GET /api/v1/dev/constraints` — filter nodes by type (constraint, guardrail)
- Pydantic schemas for request params and responses
- DB query functions for edge traversal and node type filtering
- Service layer functions wrapping traversal logic
- FastAPI route definitions with auth (OrgContext)
- 404 for missing nodes (PAT-E-523)
- Max depth parameter for traversal (default sensible limit)
- Tests: schemas, db queries, services, API integration

## Out of Scope
- Advanced graph algorithms (PageRank, spreading activation) → parking lot
- Materialized views or precomputed traversals
- Caching layer
- Pagination on traversal results (graphs are small, <10K nodes)

## Done When
- `GET /api/v1/graph/trace` returns BFS traversal with depth limit
- `GET /api/v1/graph/impact` returns BFS traversal with depth limit
- `GET /api/v1/dev/constraints` returns filtered nodes
- 404 returned for nonexistent node_id
- All tests pass (schemas, db, services, API)
- Type checks + lint pass
- Three-layer pattern followed (routes → services → db/queries)
