## What Went Well

- **Pre-implementation AR** caught 4 structural issues (single gates.py, no run_gate wrapper, chunk not a gate, CLI import path) — all changed the implementation plan before writing code.
- **Research-informed design** — 56 new sources (R1 + R2) grounded every decision. "Pydantic + NetworkX = SHACL equivalent" came from research, not assumption.
- **Design evolved through conversation** — Emilio's challenges ("what does a new domain need?", "chunk isn't a gate", "domain.yaml as manifest") fundamentally improved the architecture. The initial design was "thin wrappers for ScaleUp"; the final design is "pluggable knowledge domain infrastructure."
- **RAISE-594 unblocked cleanly** — the CLI extension mechanism in raise-commons was ~8 lines, implemented by rai-dev in parallel, and integrated without friction.
- **QR caught 4 real issues** — exception visibility, empty directory warnings, CQ format documentation, status command path inference. All fixed before merge.

## What to Improve

- **Scope estimation was off** — S3.7 was scoped as "thin CLI wrappers" (S) but became "pluggable domain infrastructure" (M). The scope expansion was correct, but the story size should have been re-estimated after the domain-centric redesign.
- **Design iterations took longer than implementation** — 5 HITL decision rounds + research + AR before writing any code. The process was right (measure twice, cut once), but the estimate didn't account for design discovery.
- **pyright noise** — 17 errors in gates.py from dynamic typing pattern (GateConfig with `type[BaseModel]`). All are false positives but create noise. Consider a `# pyright: reportUnknownMemberType=false` at file level for gates.py.

## Heutagogical Checkpoint

1. **Learned:** Design is the product. domain.yaml manifest pattern. Pre-impl AR value. Exception catching visibility.
2. **Change:** Include "new domain workflow" as forcing function in design. Re-estimate after scope expansion.
3. **Framework:** Pre-implementation AR as documented option in story-run. Research prompts as dual-use assets.
4. **More capable:** Pluggable CLI infra via entry points. Domain abstractions with YAML manifests. Pre-impl AR.
