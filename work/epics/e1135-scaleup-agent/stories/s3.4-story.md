# Story: Graph Integration (Walking Skeleton)

## User Story
As the ScaleUp agent,
I want Eduardo's ontology YAML nodes mapped to RaiSE GraphNode/GraphEdge,
so that `rai graph query` can traverse ScaleUp concepts alongside other knowledge domains.

## Acceptance Criteria

### Scenario: YAML node maps to typed GraphNode subclass
```gherkin
Given a ScaleUp methodology YAML file with type "methodology"
When the graph builder processes it
Then a ScaleUpMethodologyNode is added to the graph with typed fields (decision_area, source_chapter)
And the node is discoverable via rai graph query --types scaleup.methodology
```

### Scenario: Relationships map to GraphEdge
```gherkin
Given a ScaleUp YAML node with relationships (e.g., part_of, depends_on)
When the graph builder processes it
Then GraphEdge instances are created with correct source, target, and type
And BFS traversal from the node returns its neighbors
```

### Scenario: Roundtrip serialization
```gherkin
Given ScaleUp nodes added to the graph
When the graph is serialized and deserialized
Then nodes reconstruct as their typed subclass (not generic GraphNode)
And all typed fields are preserved
```

### Scenario: Multi-domain interoperability
```gherkin
Given ScaleUp nodes and existing RaiSE nodes (patterns, epics) in the same graph
When querying with mixed types
Then both domains coexist and cross-domain edges work
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| `people/core_values.yaml` (methodology) | Build graph | `ScaleUpMethodologyNode(id="su-core-values", type="scaleup.methodology", decision_area="People")` |
| Node with `relationships: [{target: "su-topgrading", type: "depends_on"}]` | Build graph | `GraphEdge(source="su-core-values", target="su-topgrading", type="depends_on")` |
| `rai graph query "scaling up" --types scaleup.methodology` | Query | Returns ScaleUp methodology nodes sorted by relevance |
| Serialize → deserialize graph | Roundtrip | Nodes come back as `ScaleUpMethodologyNode`, not `GraphNode` |
