# S3.7 — Scope

## In Scope

- 5 CLI subcommands under `rai scaleup`: validate, reconcile, coverage, chunk, graph
- Each wraps an existing Python function (no new logic, thin wrappers)
- Generic interface: commands accept paths, not hardcoded ScaleUp locations
- Exit codes as gate signals: 0 = pass, 1 = fail
- Human-readable output + JSON output (`--format json`) for programmatic use
- Tests for each CLI command (invoke via click.testing or subprocess)

## Out of Scope

- Plugin system for multiple ontology schemas (YAGNI — one schema at a time)
- GUI or interactive mode for CLI tools
- Modifying existing extraction/validation/graph functions (wrap as-is)
- `rai scaleup extract` CLI command (extraction is inference, stays in skill)
- Integration with `rai gate` system (future, if needed)

## Done Criteria

1. `rai scaleup validate <dir>` validates YAML nodes, reports errors, exits 0/1
2. `rai scaleup reconcile <dir>` checks cross-refs, exits 0/1
3. `rai scaleup coverage <dir> --cqs <file>` checks CQ coverage, exits 0/1
4. `rai scaleup chunk <corpus> --output <dir>` produces chapters + manifest
5. `rai scaleup graph <dir>` builds graph and prints stats
6. All commands support `--format json` for programmatic consumption
7. Tests pass, pyright clean, ruff clean
