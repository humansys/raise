# S3.5 — Scope: Retrieval Engine

## In Scope

- Deterministic retrieval function: given query context → relevant ontology nodes
- Graph traversal strategies: direct match, neighborhood expansion, path-based
- Query model: structured input (decision_area, stage, keywords) → RetrievalResult
- Relevance scoring: distance-based (hops from match) + type weighting
- Works with existing ScaleUp nodes (Eduardo's curated ontology from S3.2-S3.4)
- Tests with real ScaleUp graph structure

## Out of Scope

- Vector similarity / embeddings / RAG — retrieval is symbolic only
- LLM calls in retrieval path — pure graph operations
- CLI command for retrieval (future — S3.6 or beyond)
- Coaching prompts or facilitation logic (E8 Eduardo)
- Multi-domain retrieval (only ScaleUp for now)
- Caching or performance optimization (premature for MVP)

## Done Criteria

1. `retrieve(graph, query)` returns relevant nodes for ScaleUp decision areas
2. Results include relevance score and traversal path
3. Empty queries return empty results (no fabrication)
4. Tests cover: direct match, neighborhood, no-match, edge cases
5. Pyright clean, tests pass
