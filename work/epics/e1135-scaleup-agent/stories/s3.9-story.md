# S3.9 — Knowledge Pipeline Skills (Interactive Extraction)

## User Story

**As** the knowledge pipeline operator (Emilio or future domain experts),
**I want** interactive skills that run each pipeline phase step-by-step with visible progress,
**so that** I can observe, validate, and intervene at each stage instead of running opaque batch scripts.

## Size: M

## Context

S3.8 revealed that running the extraction pipeline as a Python script is a black box — no progress visibility, no HITL checkpoints, and errors only surface after long waits. The pipeline has 5 phases (discover → extract → validate → diff → build graph), each requiring human oversight:

- **Discovery** produces a schema that needs expert review before extraction
- **Extraction** runs ~16 LLM calls — operator needs to see chapter-by-chapter progress
- **Validation** gates may fail — operator needs to see which gates and why
- **Diff** results need expert interpretation (which differences matter?)
- **Graph build** needs confirmation before integrating into the knowledge graph

These skills follow the same pattern as `/rai-story-run` — individual phase skills + optional orchestrator.

## Acceptance Criteria

### AC1: Schema discovery skill
**Given** a corpus path and domain name
**When** I run `/rai-knowledge-discover`
**Then** it calls `discover_schema()`, displays results interactively (types, fields, relationships), saves SchemaSpec to disk, and pauses for review

### AC2: Extraction skill
**Given** a corpus and domain config
**When** I run `/rai-knowledge-extract`
**Then** it runs `ExtractionPipeline` chapter-by-chapter, reporting progress ("Chapter 3/16: THE TEAM — 5 nodes extracted"), handles errors per-chapter (skip + warn, not crash), and writes extracted nodes to disk

### AC3: Validation skill
**Given** extracted nodes in a domain directory
**When** I run `/rai-knowledge-validate`
**Then** it runs `rai knowledge check {domain}` gates, displays results per gate (pass/fail + metrics), and pauses if any gate fails

### AC4: Diff skill
**Given** extracted and curated node directories
**When** I run `/rai-knowledge-diff`
**Then** it runs both `diff_schemas` and `diff_nodes`, displays comparison report (overlap, unique to each, by decision area), and saves reports to disk

### AC5: Orchestrator (optional)
**Given** all phase skills exist
**When** I run `/rai-knowledge-run`
**Then** it chains discover → extract → validate → diff, pausing at each phase for HITL review
