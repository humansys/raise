# S3.9 Scope — Knowledge Pipeline Skills

## In Scope

- 4 interactive skills: discover, extract, validate, diff
- Each skill: loads context, runs its phase, displays progress, saves output, pauses for HITL
- Skills use existing code from S3.2/S3.7/S3.8 — no new algorithms
- Extraction skill handles per-chapter progress and error resilience
- Skills work with any domain configured in `.raise/knowledge/{domain}/domain.yaml`
- Optional orchestrator skill that chains all phases

## Out of Scope

- New extraction algorithms or scoring logic
- Modifying existing pipeline code (only wrapping it in skills)
- CLI commands (these are Claude Code skills, not `rai` CLI subcommands)
- Automatic retry or self-healing on LLM failures
- Multi-domain concurrent extraction

## Done Criteria

1. Each skill runs interactively with visible progress per step
2. Extraction skill shows chapter-by-chapter progress ("Chapter 3/16: THE TEAM")
3. All skills save outputs to disk and report file paths
4. HITL pause after each phase (operator can review before continuing)
5. Skills work on ScaleUp domain as proof (the domain we have data for)
