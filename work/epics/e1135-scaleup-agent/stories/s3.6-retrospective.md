## What Was Learned

- **Worktree + uv = PYTHONPATH hack needed** — `uv run pytest` resolves to main repo's venv, requires `PYTHONPATH="$(pwd)/src:$PYTHONPATH"` override. Known issue with editable installs in worktrees.
- **EnterWorktree works** — Claude Code's native worktree support eliminates the exit/cd/relaunch friction
- **Knowledge without process is passive** — retrieval answers questions but doesn't facilitate action. The "domain cartridge" insight (knowledge + process) emerged from this gap.
- **Keyword-based retrieval has semantic gaps** — "hire A players" returns 0 results because the adapter does literal matching. The LLM interpretation layer compensates but the adapter could be improved with configurable synonyms.

## Patterns

- **PAT: SchemaRef as universal dynamic import** — the module+class_name pattern works for schema, adapter, AND builder. Generalizable to any pluggable component.
- **PAT: Comparison testing reveals product value** — running the same question with and without the tool made the value proposition concrete and measurable.
