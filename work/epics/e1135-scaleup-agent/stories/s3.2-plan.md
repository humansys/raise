- Create `src/rai_agent/scaleup/extraction/__init__.py`
- Create `src/rai_agent/scaleup/extraction/chunker.py` — `Chapter` model, `CorpusChunker` class

**TDD:**

Test edge case: chapter with no content. Test decision area mapping (chapters 3-5 → people, 6-8 → strategy, 9-11 → execution, 12-14 → cash).
- GREEN: Add Literal field. Implement chunker with H1 header detection + decision area assignment table.

**AC:** AC4 (chunking by chapter/decision area), part of AC2 (difficulty validates)

**Files:**
- Create `src/rai_agent/scaleup/extraction/reconciler.py` — `ReconciliationReport` model, `reconcile_nodes()` function

**TDD:**
- RED: Test phantom detection (node A references non-existent B). Test orphan detection (node with empty relationships). Test cross-decision edge detection (people node → cash node). Test clean graph (no phantoms, no orphans). Test reconciliation with real Eduardo node set (9 nodes).

- REFACTOR: Optimize if needed (unlikely — O(n) logic).

**Verify:** `uv run pytest tests/scaleup/extraction/test_reconciler.py -v && uv run pyright && uv run ruff check src/`

## Execution Order

```
T1 (foundation) → T2 (prompts + pipeline) ──→ T4 (integration + YAML)
                → T3 (reconciler) ──────────↗
```

T2 and T3 can run in parallel after T1 (no shared code surface), but with a single developer, sequential is fine. T4 requires both T2 and T3.

**Risk-first rationale:** T1 surfaces corpus parsing issues immediately. T2 surfaces instructor/LLM integration issues (highest technical risk). T3 is pure logic and lowest risk. T4 is integration — issues here indicate design gaps, not implementation bugs.

## Risks

| Risk | L/I | Mitigation |
|------|:---:|------------|
| Corpus H1 headers don't follow expected pattern | M/H | T1 tests against real corpus sample. Chunker logs skipped sections. |
| instructor structured output fails for complex OntologyNode | M/H | T2 uses mock responses for unit tests. Real LLM test in T4. Fallback: raw API + JSON parse. |
| LLM extracts inconsistent node IDs across runs | L/M | temperature=0, seed parameter, sorted output. AC7 tests structural equivalence. |
| Extraction misses >20% of corpus concepts | L/M | Coverage report in T4. Prompt iteration if needed. 80% is a SHOULD, not a MUST. |
| instructor dependency conflicts with existing packages | L/M | Check `pyproject.toml` compatibility in T2. instructor is lightweight. |
