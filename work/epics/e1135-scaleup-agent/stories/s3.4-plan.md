**Description:** Define 6 GraphNode subclasses with typed fields and auto-registration. Define edge type mapping from Eduardo's hyphenated names to RaiSE underscore convention.

**Files:**
- Create `src/rai_agent/scaleup/graph/__init__.py`

- Create `src/rai_agent/scaleup/graph/edges.py` — edge type constants + mapping

**TDD:**
- RED: Test each subclass auto-registers with correct `node_type`. Test `type` field is auto-set. Test typed fields exist and validate. Test edge mapping normalizes hyphens to underscores.

**AC:** AC1 (6 subclasses auto-registered)
**Verify:** `uv run pytest tests/scaleup/graph/test_nodes.py -v && uv run pyright`
**Depends on:** None

### T2: ScaleUpGraphBuilder (S)

**Description:** Builder that reads Eduardo's YAML files via `OntologyNode` (S3.1) and maps to GraphNode subclasses + GraphEdge instances. Adds them to a `Graph` instance.

**Files:**

**TDD:**

- REFACTOR: Simplify mapping dispatch (dict lookup vs if/elif).

**AC:** AC2 (builder reads YAML → Graph)
**Verify:** `uv run pytest tests/scaleup/graph/test_builder.py -v && uv run pyright`
**Depends on:** T1

## Execution Order

```
T1 (nodes + edges) → T2 (builder) → T3 (integration) → T4 (validation)
```

Linear sequence — each task builds on the previous. No parallelism (single developer, shared code surface).

**Risk-first:** T1 is riskiest (subclass registration across packages). If `GraphNode._registry` doesn't work across package boundaries, we discover it immediately.

## Risks

| Risk | L/I | Mitigation |
|------|:---:|------------|
| `GraphNode._registry` not shared across packages | L/H | T1 tests verify registration immediately. ClassVar on base class should work. |
| Eduardo's YAML has fields not in OntologyNode model | L/M | S3.1 already validated these 9 files. OntologyNode has `metadata: dict` catch-all. |
| `raise_core` import path differs in worktree | L/M | T1 will surface import errors early. Check pyproject.toml dependencies. |
