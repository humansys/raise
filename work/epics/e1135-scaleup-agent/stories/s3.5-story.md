# Story: Retrieval Engine — Deterministic Graph Traversal

## User Story
As the rai agent,
I want to retrieve relevant ontology nodes given a business context (decision area, stage, company situation),
so that I can provide contextually grounded guidance instead of generic book advice.

## Acceptance Criteria

### Scenario: Retrieve nodes for a specific decision area
```gherkin
Given a populated ScaleUp knowledge graph
When I query for decision_area="people" with stage="growth"
Then I receive ontology nodes related to People decisions
And each node includes its relationships to other relevant concepts
```

### Scenario: Retrieve nodes matching competency questions
```gherkin
Given a populated ScaleUp knowledge graph and competency questions
When I query "How do I build a high-performing leadership team?"
Then I receive nodes that address this question (e.g., Topgrading, Core Values, A-Players)
And nodes are ranked by relevance (direct match > related > transitive)
```

### Scenario: Empty or no-match query
```gherkin
Given a populated ScaleUp knowledge graph
When I query for a topic not covered by the ontology
Then I receive an empty result set with a clear "no matches" indication
And no hallucinated or fabricated nodes
```

## Examples (Specification by Example)

| Query Context | Expected Nodes (subset) | Relevance |
|--------------|------------------------|-----------|
| decision_area=people | Topgrading, A-Players, Core Values, FACe | direct |
| decision_area=strategy, stage=scaleup | BHAG, Brand Promise, One-Phrase Strategy | direct |
| "cash conversion cycle" | CCC, Power of One, Cash Acceleration | direct + related |
| "quantum physics" | [] (empty — not in domain) | none |

## Notes
- Retrieval is **symbolic** (graph traversal), NOT vector similarity
- Uses existing ScaleUp nodes from S3.2-S3.4 (Eduardo's curated ontology)
- No LLM calls in retrieval path — pure graph operations
- NetworkX is already a dependency (used in S3.7 gates)
