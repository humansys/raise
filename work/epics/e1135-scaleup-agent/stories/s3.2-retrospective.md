# S3.2 Retrospective: Extraction Pipeline

> **Story:** S3.2 | **Size:** M | **Date:** 2026-03-19
> **Estimated:** 110min | **Actual:** ~30min | **Velocity:** 3.67x

## Summary

Built the extraction pipeline for ScaleUp Agent: corpus chunker, LLM extraction via instructor+Pydantic, reconciler for graph consistency, and YAML output. 56 new tests added (147 total), all passing. QR/AR review applied with 2 critical fixes and 8 recommendations.

## What Went Well

1. **instructor + Pydantic** — Structured output extraction worked cleanly. The library handled schema validation, retries, and type coercion. Minimal integration friction.
2. **Natural chunking by chapters** — The corpus has clear H1/H2 structure; splitting by chapter gave each extraction pass focused context without artificial boundary problems.
3. **Reconciler as pure logic** — Separating graph consistency checks (phantoms, orphans, cross-decision edges) into a deterministic module made it trivial to test and reason about.
4. **QR caught real bugs** — Client resource leak (C1) and silent data drops in reconciler (C2) were invisible to happy-path tests. The review gate earned its cost.

## What Could Be Better

1. **Over-estimation** — 110min estimated vs ~30min actual (3.67x). The M sizing assumed more LLM integration friction than instructor actually introduces. For future stories with instructor, reduce estimate by ~50%.
2. **Test helpers duplication** — conftest.py test helper patterns (sample nodes, mock responses) keep recurring across stories. A shared test fixtures module would reduce boilerplate.
3. **AR proportionality questions unresolved** — Q1 (class vs function for pipeline) and Q2 (chunker complexity) remain open. The class approach works but may be heavier than needed for a single-use extraction pipeline.

## Decisions Made

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Use instructor library | Handles structured output, retries, validation — no custom parsing needed |
| D2 | Multi-pass by chapter | Natural document boundaries, focused LLM context per pass |
| D3 | DifficultyLevel as Literal | Consistent with existing ScaleUp model patterns, no enum overhead |
| D4 | Collect & reconcile pattern | Extract all nodes first, then validate graph consistency — clean separation |
