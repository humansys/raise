# S3.8 — Full-Cycle Extraction from Zero

## User Story

**As** the knowledge pipeline operator,
**I want** to run the full extraction cycle (discover → extract → validate) on a corpus without prior schema,
**so that** the pipeline can bootstrap ontologies for new domains without manual setup.

## Size: M

## Acceptance Criteria

### AC1: Schema discovery produces valid node types
**Given** a raw corpus (ScaleUp book)
**When** I run schema discovery without a pre-existing ontology schema
**Then** the pipeline discovers node types (concepts, tools, metrics, etc.) and their fields

### AC2: Extraction produces valid YAML nodes
**Given** discovered schema from AC1
**When** I run extraction against the corpus
**Then** YAML node files are produced that pass the S3.7 validation gate

### AC3: Gates validate extracted output
**Given** extracted nodes from AC2
**When** I run `rai knowledge check` gates (validate, reconcile, coverage, graph)
**Then** all gates pass or report specific, actionable failures

### AC4: Diff vs Eduardo's manual ontology
**Given** machine-extracted nodes AND Eduardo's manually curated nodes
**When** I compare them
**Then** a diff report shows: nodes found by both, nodes only in manual, nodes only in extracted — as epistemological cross-validation

## Dependencies

- S3.7 (CLI gates) — ✅ Done
- S3.2 (extraction pipeline) — ✅ Done
- S3.1 (schema validation) — ✅ Done
