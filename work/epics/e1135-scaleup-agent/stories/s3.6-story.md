# Story: Knowledge Query CLI — End-to-End Retrieval

## User Story
As the rai agent operator,
I want to query domain knowledge via `rai knowledge query`,
so that I can retrieve contextually relevant ontology nodes during working sessions
without needing custom scripts or Python imports.

## Acceptance Criteria

### Scenario: Query ScaleUp knowledge by topic
```gherkin
Given the ScaleUp domain has 447 extracted nodes
When I run `rai knowledge query scaleup "cash flow tools"`
Then I receive the top-K ranked nodes with scores
And each result shows node id, name, type, score, and summary
```

### Scenario: Query with domain auto-detection
```gherkin
Given only one domain is registered (scaleup)
When I run `rai knowledge query "leadership team"`
Then the domain is auto-detected from available domains
And results are returned from the scaleup domain
```

### Scenario: No matching results
```gherkin
Given the ScaleUp knowledge graph
When I query for a topic outside the domain (e.g., "kubernetes deployment")
Then I receive an empty result set
And a message indicating no relevant nodes were found
```

## Notes
- Retrieval uses existing `retrieve()` engine with `ScaleUpAdapter`
- Graph is built from YAML nodes on each query (no persistent index yet)
- Output format: human-readable by default, `--json` for machine consumption
- This is the capstone proof for E3 — validates the full extraction→retrieval chain
- Future epic will add pluggable domain packs with LLM prompting instructions
