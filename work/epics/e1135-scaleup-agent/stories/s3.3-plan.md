## T1: Models + Session Engine

**Description:** Build the curation data models (`CurationDecision`, `NodeDecision`, `CurationSessionState`, `CurationSummary`) and the `CurationSession` engine that loads extracted nodes, tracks decisions, persists state to YAML, and writes curated output.

**Files to create:**
- `src/rai_agent/scaleup/curation/__init__.py`
- `src/rai_agent/scaleup/curation/models.py`
- `src/rai_agent/scaleup/curation/session.py`
- `tests/scaleup/curation/__init__.py`
- `tests/scaleup/curation/test_models.py`
- `tests/scaleup/curation/test_session.py`

**TDD cycle:**

RED:
- `test_models.py`: Test `CurationDecision` enum values, `NodeDecision` construction with hash validation, `CurationSessionState` serialization round-trip, `CurationSummary` from session state.
- `test_session.py`: Test `CurationSession.load()` reads extracted YAML directory, `record_decision()` updates state and auto-saves, `write_curated()` writes accepted+edited nodes to curated dir, session resume from existing state file, cursor advancement, node ordering by decision area, skip functionality, drift detection via original_hash.

GREEN:
- Implement models in `models.py` (Pydantic, typed enums).
- Implement `CurationSession` in `session.py`:
  - `load(source_dir, output_dir, state_path)` — load extracted nodes, group by decision area, initialize or resume state.
  - `current_node()` — return the node at cursor position.
  - `record_decision(node_id, decision, reason?, edits?)` — validate edits against `OntologyNode` schema, update state, auto-save.
  - `write_curated()` — write accepted/edited nodes to output_dir using `node_to_yaml_dict`.
  - `summary()` — return `CurationSummary`.
  - State persistence: serialize `CurationSessionState` to YAML, deserialize on resume.

REFACTOR:
- Extract shared YAML I/O if duplication emerges with extraction utils.

**AC reference:** Scenarios: Node presentation, Accept, Reject, Edit, Session persistence, Session summary.

**Verification:**
```bash
uv run pytest tests/scaleup/curation/test_models.py tests/scaleup/curation/test_session.py -v
uv run ruff check src/ tests/
uv run pyright
```

**Size:** M — core logic, multiple model classes, file I/O, state management.

**Dependencies:** None (uses existing `OntologyNode` from `rai_agent.scaleup.validation.models` and `node_to_yaml_dict` from `rai_agent.scaleup.extraction.utils`).

## Execution Order

```
T1 (Models + Session) → T2 (Formatter) → T3 (Skill + Integration)
```

**Rationale:** Strictly sequential. T1 establishes the data model and engine that T2 formats and T3 orchestrates. No parallelism possible — each task builds on the prior.

**Risk-first:** T1 is the highest risk (state persistence, YAML I/O, edit validation) and is tackled first. T2 is low risk (pure formatting). T3 is verification.

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Edit validation rejects valid changes | Medium | Test edits against real OntologyNode schema with partial updates (model_copy) |
| YAML serialization drift between extracted and curated | Medium | Reuse `node_to_yaml_dict` from extraction utils for consistent serialization |
| Session state corruption on interrupted save | Low | Write to temp file then atomic rename |
| Large node count (~100) makes grouping slow | Low | Linear scan is fine for 100 nodes; no optimization needed |
