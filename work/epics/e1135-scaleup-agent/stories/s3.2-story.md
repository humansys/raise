# Story: Extraction Pipeline

## User Story
As Eduardo (Scaling Up coach),
I want to extract ontology nodes from the Scaling Up corpus automatically,
so that I can build a knowledge graph without manual YAML authoring.

## Acceptance Criteria

### Scenario: Corpus text extracts to valid OntologyNode
```gherkin
Given the Scaling Up corpus file (scaling_up_llamaparse.md)
When the extraction pipeline processes a section about a methodology
Then an OntologyNode is produced with required fields (id, name, type, decision_area)
And the node validates against the OntologyNode Pydantic schema from S3.1
```

### Scenario: Extraction covers corpus concepts
```gherkin
Given the full Scaling Up corpus
When the extraction pipeline runs end-to-end
Then at least 80% of known corpus concepts are covered as extracted nodes
And each node has a source_reference pointing to its origin in the corpus
```

### Scenario: Extracted nodes map to GraphNode subclasses
```gherkin
Given extracted OntologyNode instances
When mapped through the ScaleUpGraphBuilder from S3.4
Then each node produces a valid typed GraphNode subclass
And no validation errors occur at the builder boundary
```

### Scenario: Pipeline is reproducible
```gherkin
Given the same corpus and extraction configuration
When the pipeline runs twice with the same LLM settings
Then the output nodes are structurally equivalent (same IDs, types, relationships)
And differences are limited to LLM wording variations in descriptions
```

### Scenario: Extraction prompts are configurable
```gherkin
Given a custom extraction prompt template
When the pipeline runs with the custom template
Then the template is used for LLM calls
And the output still validates against OntologyNode schema
```

## Examples (Specification by Example)

| Input | Action | Expected Output |
|-------|--------|-----------------|
| Corpus section on "Core Values" (People decision) | Extract | `OntologyNode(id="su-core-values", type="methodology", decision_area="People", name="Core Values")` |
| Corpus section on "One-Page Strategic Plan" (Execution) | Extract | `OntologyNode(id="su-one-page-strategic-plan", type="worksheet", decision_area="Execution")` |
| Corpus section with cross-references to other concepts | Extract | Node with `relationships: [{target: "su-topgrading", type: "depends_on"}]` |
| Full corpus (~68 methodologies + ~34 worksheets) | Full pipeline | ≥80 valid OntologyNode instances covering all 4 decision areas |
| Same corpus, two runs | Reproducibility | Structurally equivalent output sets |
