# S3.8 Retrospective — Full-Cycle Extraction from Zero

> **Story:** S3.8 | **Branch:** story/s3.8/full-cycle-from-zero
> **Dates:** 2026-03-20
> **Estimated:** ~85min | **Actual:** ~90min (implementation ~40 + QR ~10 + inference migration ~25 + live run ~15)

## Summary

Schema discovery from corpus (zero knowledge LLM prompt), schema diff vs OntologyNode, extraction without few-shot examples, content diff extracted vs curated. Migrated inference from instructor+Anthropic API to Claude Agent SDK. Live discovery ran successfully — 11 types, 28 relationships discovered.

## What Went Well

- **PAT-E-001 in action** — data flow trace in plan, zero wiring bugs during implementation (vs 2 in S3.5)
- **HITL design decisions** — 3 clear choices (zero absoluto, discover→compare→extract, map to OntologyNode) eliminated ambiguity
- **Schema discovery worked** — LLM found 11 node types vs Eduardo's 6 (richer: company, person, habit, process, framework, strategy, book_resource)
- **Claude Agent SDK migration** — `invoke_structured()` wrapper works as inference bridge for pipelines
- **QR caught a real bug** — `.format()` with corpus content crashes on braces

## What to Improve

- **Pipeline needs interactive skills** — running extraction as a Python script is a black box. No progress visibility, no HITL checkpoints. Needs skills per pipeline phase (discover, extract, validate, diff) + orchestrator.
- **LLM doesn't respect Literal types** — generates relationship types outside the schema (e.g. `implemented-by`). Sanitization needed before Pydantic validation. `instructor` handled this; `invoke_structured` doesn't.
- **Tech debt: `diff_schemas` duck typing** — `reference_model: Any` needs a real adapter to work with OntologyNode.

## Heutagogical Checkpoint

1. **Learned:** (a) `.format()` with user content is injection-class bug. (b) Structured output without `instructor` requires explicit sanitization — the LLM will invent field values outside Literal constraints. (c) Nested Claude sessions blocked by `CLAUDECODE` env var — bypass works but indicates SDK wasn't designed for this.
2. **Process:** Pipeline execution needs to be skill-driven (interactive, step-by-step) not script-driven (batch, opaque). This is the key insight for the next story.
3. **Framework:** Need knowledge pipeline skills: `/rai-knowledge-discover`, `/rai-knowledge-extract`, `/rai-knowledge-validate`, `/rai-knowledge-diff`, potentially orchestrated by `/rai-knowledge-run`.
4. **New capability:** Claude Agent SDK integration for pipelines (`invoke_structured`). Zero-knowledge prompt design. Heterogeneous diff tools.

## Patterns

- **PAT-E-001** (reinforced, +1): Data flow trace prevented wiring bugs
