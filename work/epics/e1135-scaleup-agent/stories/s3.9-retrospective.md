# S3.9 Retrospective — Knowledge Pipeline Skills

> **Story:** S3.9 | **Dates:** 2026-03-20 → 2026-03-21
> **Estimated:** ~105min (6 tasks) | **Actual:** ~240min (expanded scope)

## Summary

Delivered 5 Claude Code skills wrapping the knowledge pipeline (discover, extract, validate, diff, run) plus significant improvements to extraction resilience and a new reconciliation step. End-to-end tested against the ScaleUp corpus: 447 nodes extracted from 35/35 chapters, all validation gates green.

## What Went Well

- **TDD caught the CLAUDECODE bypass gap** — test revealed that `_run_query` direct callers bypassed the env var suspension
- **Incremental testing approach** — 3 chapters first, then scaling to 35, revealed parsing issues early
- **Refinement pattern** — post-discovery comparison against OntologyNode recovered `worksheet` and `stage` types autonomously
- **User-driven scope expansion** — Emilio's questions about reconciliation and incremental processing led to real improvements, not scope creep

## What to Improve

- **Estimation didn't account for LLM integration tax** — unit tests with mocks pass instantly, but real LLM calls take 30-60s each and reveal format variance issues
- **Extraction writes all-at-end** — should write per-chapter incrementally for observability and resilience (identified but not implemented)
- **Schema diff (`diff_schemas`) doesn't work with OntologyNode** — it looks for `_node_types` / `_fields` class attrs that OntologyNode doesn't expose. The manual comparison was needed instead.

## Heutagogical Checkpoint

1. **Learned:** LLM output is non-deterministic at the format level. Same prompt + same content → different JSON structure each time. Defense in depth is mandatory, not optional.
2. **Process change:** Include "real LLM test" as an explicit task in plans for any story that wraps LLM calls. Mock-only testing is insufficient.
3. **Framework improvement:** New pattern PAT-E-003 (LLM defense-in-depth), PAT-E-004 (post-discovery refinement), PAT-E-005 (guard at lowest level).
4. **More capable:** Building resilient LLM extraction pipelines with progress visibility and autonomous gap-filling.
