**Key logic in `load_domain()`:**
```python
def load_domain(domain_dir: Path) -> tuple[DomainManifest, GateConfig]:
    manifest = DomainManifest.model_validate(yaml.safe_load(...))
    mod = importlib.import_module(manifest.schema.module)
    model_cls = getattr(mod, manifest.schema.class_name)
    if not (isinstance(model_cls, type) and issubclass(model_cls, BaseModel)):
        raise DomainConfigError(f"{manifest.schema.class_name} is not a Pydantic BaseModel")
    config = GateConfig(
        node_model=model_cls,
        cq_file=domain_dir / manifest.competency_questions if manifest.competency_questions else None,
        cq_threshold=manifest.thresholds.get("cq_coverage", 80.0),
        required_types=manifest.required_types,
        node_dir=domain_dir / "curated",  # prefer curated, fallback to extracted
    )
    return manifest, config
```

### T2: 4 gate functions (M)

**Description:** Implement the 4 gate functions in a single `gates.py` file. Each wraps existing ScaleUp functions generically via GateConfig.

**Files:**
- Create `src/rai_agent/knowledge/gates.py` — run_validate(), run_reconcile(), run_coverage(), run_graph()
- Create `tests/knowledge/test_gates.py`

**TDD cycle:**
- RED: Test each gate returns correct GateResult (passed/failed, metrics, errors, warnings, duration_ms). Test with valid nodes, invalid nodes, phantom refs, low coverage.
- GREEN: Implement gates wrapping existing functions via GateConfig.node_model.
- REFACTOR: Extract timing helper if repeated.

**Gate wrapping logic:**
- `run_validate(node_dir, config)`: iterate YAML files, `config.node_model.model_validate()` each, count valid/invalid, collect errors
- `run_reconcile(node_dir, config)`: load nodes, build id set + target set, detect phantoms/orphans/cross-edges. Uses same logic as `reconciler.py` but works generically with any node model that has `id`, `relationships`, and optional `decision` fields.
- `run_coverage(node_dir, config)`: load CQs from `config.cq_file`, load nodes, call coverage check logic, compare vs `config.cq_threshold`
- `run_graph(node_dir, config)`: load nodes, build NetworkX graph, report stats (node count, edge count, components, density, degree stats)

**Important:** The reconcile and coverage gates need to work with ANY node model, not just OntologyNode. The gate should check for attributes (`hasattr`) or use a protocol/structural typing approach. If the model doesn't have `relationships`, reconcile is a no-op (passes with warning).
