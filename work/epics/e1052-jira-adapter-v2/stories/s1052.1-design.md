# S1052.1: JiraClient Wrapper — Design

**Story:** S1052.1 | **Epic:** E1052 | **Complexity:** Moderate | **Date:** 2026-03-31

## What & Why

**What:** A thin Python wrapper (`JiraClient`) over `atlassian.Jira` providing auth resolution from environment variables, error normalization to a typed exception hierarchy, and multi-instance support via `from_config()` classmethod.

**Why:** The PythonApiJiraAdapter (S1052.3) needs a tested, reliable transport layer that isolates Jira API details. Mirroring the ConfluenceClient pattern (E1051) ensures consistency across the Atlassian integration surface.

### JiraClient Contract

```python
class JiraClient:
    def __init__(self, url: str, username: str, token: str) -> None: ...

    # Issue CRUD
    def get_issue(self, key: str) -> dict[str, Any]: ...
    def create_issue(self, fields: dict[str, Any]) -> dict[str, Any]: ...
    def update_issue(self, key: str, fields: dict[str, Any]) -> dict[str, Any]: ...
    def transition_issue(self, key: str, transition_id: str) -> None: ...
    def get_transitions(self, key: str) -> list[dict[str, Any]]: ...

    # Search
    def jql(self, query: str, limit: int = 50) -> list[dict[str, Any]]: ...

    # Relationships
    def create_link(self, source: str, target: str, link_type: str) -> None: ...
    def set_parent(self, child: str, parent: str) -> None: ...

    # Comments
    def add_comment(self, key: str, body: str) -> dict[str, Any]: ...
    def get_comments(self, key: str, limit: int = 10) -> list[dict[str, Any]]: ...

    # Health
    def server_info(self) -> dict[str, Any]: ...

    @classmethod
    def from_config(cls, config: Any, instance: str | None = None) -> JiraClient: ...
```

### Exception Hierarchy

```python
class JiraAdapterError(Exception):       # base
class JiraAuthError(JiraAdapterError):    # 401/403
class JiraNotFoundError(JiraAdapterError): # 404
class JiraApiError(JiraAdapterError):     # generic, with optional status_code
```

### Error Mapping (from atlassian.errors)

| atlassian exception | JiraClient exception |
|---------------------|---------------------|
| `ApiPermissionError` | `JiraAuthError` |
| `ApiNotFoundError` | `JiraNotFoundError` |
| `ApiError` | `JiraApiError` |
| Other `Exception` | `JiraApiError` (unexpected) |

### Auth Resolution

Environment variables with instance-specific override:

1. `JIRA_API_TOKEN_{INSTANCE}` > `JIRA_API_TOKEN`
2. `JIRA_USERNAME_{INSTANCE}` > `JIRA_USERNAME`
3. Config email as fallback for username

Missing token raises `JiraAuthError` at construction time (fail-fast).

## Design Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Mirror ConfluenceClient pattern exactly | Auto-decided, rationale from epic design D2 + D5. Same library, same auth model, consistency. |
| D2 | Constructor takes primitives; from_config() is factory | Auto-decided, rationale from epic design D3. Testable without config infra. |
| D3 | from_config accepts Any (duck-typed), not JiraConfig | Auto-decided, scope.md. JiraConfig is S1052.2; avoid circular dep. |
| D4 | Exceptions in separate file | Auto-decided, ConfluenceClient pattern. Importable without atlassian dep. |
| D5 | Methods return raw dicts | Auto-decided, scope.md. Parsing is adapter's job (S1052.3). |
| D6 | backoff_and_retry=True, 5 retries, factor 1.0 | Auto-decided, epic D5 + PAT-E-589. Built-in retry, no custom limiter. |
| D7 | cloud=True hardcoded | Auto-decided. RaiSE targets Jira Cloud only. |

## Examples

### Basic Usage

```python
client = JiraClient(
    url="https://humansys.atlassian.net",
    username="emilio@humansys.com",
    token="xxx",
)
issue = client.get_issue("RAISE-1052")
assert issue["fields"]["summary"] is not None
```

### Error Handling

```python
with pytest.raises(JiraNotFoundError, match="RAISE-99999"):
    client.get_issue("RAISE-99999")
```

### Factory from Config

```python
config = SimpleNamespace(
    default_instance="humansys",
    instances={"humansys": SimpleNamespace(site="humansys.atlassian.net", email="e@h.com")},
)
client = JiraClient.from_config(config, instance="humansys")
```

## Acceptance Criteria

### MUST (10)

- AC1: Constructor creates atlassian.Jira with `backoff_and_retry=True`
- AC2: `from_config()` resolves instance from config and env vars
- AC3: All 11 methods delegate to atlassian.Jira and return raw dicts
- AC4: `ApiPermissionError` maps to `JiraAuthError`
- AC5: `ApiNotFoundError` maps to `JiraNotFoundError`
- AC6: `ApiError` maps to `JiraApiError` with context string
- AC7: Missing `JIRA_API_TOKEN` raises `JiraAuthError` at construction
- AC8: `jira_exceptions.py` importable without atlassian-python-api
- AC9: Unit tests >= 80% coverage on jira_client.py
- AC10: Integration tests skip when `JIRA_API_TOKEN` not set

### SHOULD (3)

- SC1: Instance-specific env var override (`JIRA_API_TOKEN_{INSTANCE}`)
- SC2: Actionable ImportError message for missing atlassian dep
- SC3: Backoff params match ConfluenceClient (5 retries, factor 1.0)

### MUST NOT (4)

- MN1: Must NOT parse responses into Pydantic models
- MN2: Must NOT hardcode transition IDs or status names
- MN3: Must NOT import JiraConfig Pydantic model (S1052.2 scope)
- MN4: Must NOT build custom rate limiting
