# S1052.1: JiraClient Wrapper — Scope

## In Scope

- `JiraClient` class wrapping `atlassian.Jira`
- Auth resolution: url, username, token from env or config
- `from_config()` classmethod for multi-instance instantiation
- `backoff_and_retry=True` on constructor (PAT from E1051)
- Error normalization: `atlassian.errors` → typed exceptions
- `jira_exceptions.py`: JiraAdapterError, JiraApiError, JiraAuthError, JiraNotFoundError
- Methods (~12): get_issue, create_issue, update_issue, transition_issue, get_transitions, jql, create_link, set_parent, add_comment, get_comments, server_info
- Unit tests with mocked `atlassian.Jira`
- Integration tests (skip when JIRA_API_TOKEN not set)

## Out of Scope

- PythonApiJiraAdapter (S1052.3)
- JiraConfig Pydantic model (S1052.2 — accept raw config dict for now)
- Response parsing to Pydantic models (adapter's job, not client's)
- Backend discovery (RAISE-1130)
- Telemetry/logfire spans (SHOULD, add if time)

## Done Criteria

1. All ~12 methods work against live Jira (validated manually or via integration test)
2. Typed exceptions raised for auth errors, not-found, and API errors
3. Multi-instance: `from_config()` creates client for named instance
4. Unit tests with mocks: ≥80% coverage on new code
5. Integration tests skip gracefully when no credentials
6. `backoff_and_retry=True` enabled by default
