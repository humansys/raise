# S1052.2 Design: JiraConfig Pydantic

## Pattern: Mirror ConfluenceConfig (E1051)

Same structure as `confluence_config.py`: frozen Pydantic models + YAML loader +
module-level Path constant as test seam (PAT-E-589).

## Models

```python
class JiraInstance(BaseModel, frozen=True):
    site: str                    # e.g. "humansys.atlassian.net"
    email: str = ""              # optional, falls back to env
    projects: list[str] = []     # project keys on this instance (informational)

class JiraProject(BaseModel, frozen=True):
    instance: str                # instance key (must exist in instances dict)
    name: str = ""               # human-readable name
    category: str = ""           # e.g. "Development", "Testing"
    description: str = ""        # project description
    board_type: str = ""         # e.g. "scrum", "kanban"
    components: list[str] = []   # project components

class JiraConfig(BaseModel, frozen=True):
    default_instance: str
    instances: dict[str, JiraInstance]
    projects: dict[str, JiraProject] = {}
    model_config = ConfigDict(extra="allow")  # workflow, team, issue_types pass through

    @model_validator: default_instance must exist in instances
    @model_validator: project.instance must exist in instances (for each project)

    resolve_instance(project_key) -> JiraInstance
    resolve_site(project_key) -> str  # returns "https://{site}"
```

## Integration

JiraClient.from_config() already uses duck typing (config.default_instance,
config.instances). JiraConfig satisfies this interface. Add type annotation.

## Design Decisions

| # | Decision | Rationale |
|---|----------|-----------|
| D1 | `extra="allow"` on JiraConfig | workflow/team/issue_types consumed elsewhere; model ignores but preserves |
| D2 | projects field on JiraInstance kept as list[str] | Matches current YAML; informational, not routing |
| D3 | resolve_site returns full URL with https:// | Consistent with JiraClient.from_config() usage |
| D4 | No flat format normalization | jira.yaml is always multi-instance (unlike Confluence) |
