# S1052.6 Design — Integration Tests + Docs

## Overview

Adapter-level integration tests exercising PythonApiJiraAdapter against live Jira Cloud, plus architecture module documentation. Validates M1 (Walking Skeleton) completeness.

## Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D1 | Skip via `pytestmark` module-level when no `JIRA_API_TOKEN` | Same pattern as S1052.1 client tests — consistent, zero CI friction |
| D2 | Test adapter protocol surface, not client internals | Client integration tested in S1052.1; adapter tests verify parsing + protocol compliance |
| D3 | Architecture doc follows confluence-adapter.md pattern | Established module doc format with frontmatter, mermaid diagrams, component map |

## Integration Points

- **PythonApiJiraAdapter** — the SUT
- **JiraClient** — transport layer (already tested)
- **JiraConfig** — config loading (already tested)
- **.raise/jira.yaml** — configuration file
- **Jira Cloud API** — external dependency (skipped without creds)

## Test Strategy

All tests in one class `TestJiraAdapterIntegration`, skip-decorated at module level.
Helper method `_make_adapter()` constructs adapter from env vars.
Test issues created with `rai-test` label for cleanup identification.
