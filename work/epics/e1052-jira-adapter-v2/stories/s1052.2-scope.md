# S1052.2 Scope: JiraConfig Pydantic

## In Scope

- JiraInstance model (site, email)
- JiraProject model (instance, name, category, description, board_type, components)
- JiraConfig model (default_instance, instances, projects) with extra fields passthrough
- load_jira_config() function with Path constant test seam
- resolve_instance() and resolve_site() methods
- Integration: JiraClient.from_config() typed to accept JiraConfig
- Full TDD test suite

## Out of Scope

- Modeling workflow/team/issue_types sections (consumed by other code)
- Multi-format normalization (jira.yaml is always multi-instance)
- Migration tooling
