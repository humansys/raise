# S1052.1: JiraClient Wrapper

## User Story

As a RaiSE adapter developer,
I want a thin Python wrapper over `atlassian.Jira` with auth resolution, error normalization, and multi-instance support,
So that the PythonApiJiraAdapter (S1052.3) can delegate all Jira API calls to a tested, reliable client.

## Acceptance Criteria

**Given** valid environment variables (JIRA_URL, JIRA_USERNAME, JIRA_API_TOKEN)
**When** JiraClient is instantiated via `from_config()`
**Then** it connects to the correct Jira instance with `backoff_and_retry=True`

**Given** a JiraClient connected to Jira Cloud
**When** `get_issue("RAISE-1052")` is called
**Then** it returns the full issue dict with nested fields format

**Given** an invalid API token
**When** any method is called
**Then** it raises `JiraAuthError` (not a generic exception)

**Given** a non-existent issue key
**When** `get_issue("RAISE-99999")` is called
**Then** it raises `JiraNotFoundError`

**Given** a JiraConfig with multiple instances
**When** `from_config(config, instance="humansys")` is called
**Then** it creates a client pointing to `humansys.atlassian.net`

## SbE Examples

```python
# Auth from env
client = JiraClient(url="https://humansys.atlassian.net", username="e@h.com", token="xxx")
issue = client.get_issue("RAISE-1052")
assert issue["fields"]["summary"] is not None

# Error normalization
with pytest.raises(JiraNotFoundError):
    client.get_issue("RAISE-99999")

# Factory from config
client = JiraClient.from_config(config, instance="humansys")
assert "humansys" in client._url
```
