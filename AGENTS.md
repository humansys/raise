# RaiSE — Agent Instructions

> Context for AI agents working on this codebase.

## Quick Start

Run `/rai-session-start` to load full context (patterns, coaching, session continuity).

## Project Structure

- `src/rai_cli/` — CLI toolkit (Python, Typer-based)
- `packages/rai-core/` — Core library (graph, models)
- `.raise/` — Framework engine (memory, skills, templates, gates)
- `framework/` — Public methodology (concepts, reference)
- `governance/` — Project governance (architecture, vision)
- `docs/` — MkDocs documentation site (docs.raiseframework.ai)
- `work/` — Work artifacts (epics, stories, research)

## Branch Model

- `main` — stable releases (tags)
- `release/{version}` — release branches (e.g. release/2.4.0)
- `story/s{N}.{M}/{name}` — story branches (from release branch)
- Epics are logical containers, not branches

## Key Commands

```
rai init                    # Initialize project
rai session start --context # Start session with AI context
rai graph build             # Build knowledge graph
rai graph query "topic"     # Query the graph
rai skill list              # List available skills
rai gate check --all        # Run all quality gates
rai release check           # Pre-release quality gates
```

## Development Rules

- TDD always (RED-GREEN-REFACTOR)
- Type annotations on all code (Pyright strict)
- Pydantic models for all data structures
- Commit after each completed task
- Tests, types, and lint must pass before any commit

## Extension Points

- **Adapters**: `src/rai_cli/adapters/` — Protocol-based (Jira, Confluence, GitHub)
- **Skills**: `.claude/skills/` — Structured SKILL.md workflows
- **MCP Servers**: `.raise/mcp/*.yaml` — Model Context Protocol integrations
- **Hooks**: `.claude/hooks/` — Lifecycle event handlers
