## How It Actually Works

### The Structure

We use **repeatable processes** (we call them "skills" and "katas"):

1. **Design:** Create specs with concrete examples (15-20 min)
2. **Plan:** Break into atomic tasks with verification (10-15 min)
3. **Implement:** Execute with tests and quality gates (50-90 min)
4. **Review:** Extract learnings, update framework (15-20 min)

**Every. Single. Feature.**

This isn't bureaucracy. This is **collaboration infrastructure**.

### The Memory

AI has no memory between sessions. But we externalized memory:

- `memory.md` - Patterns discovered, anti-patterns avoided
- `calibration.md` - Velocity data (E1: 18x → 3.5x, E2: 3.5x → 2.1x stabilized)
- `session-index.md` - What happened when
- Skills/SOPs - Codified processes

**Result:** Continuity. Context doesn't reset. Learnings compound.

### The Feedback Loops

Retrospection at **every level**:

```
Session Close    → Daily learnings
     ↓
Feature Review   → Feature-level patterns
     ↓
Epic Close       → Cross-feature trends
     ↓
Systemic Review  → Cross-epic strategy
```

**"As above, so below" - fractal learning structure.**

Each level aggregates the one below. Patterns emerge through scale.

### The Meta-Cognition

The system **questions itself**:

- **During F2.3:** "Should retrospective improvements happen before commit?" → Yes, created protocol
- **After E2:** "Can session-close be progressive/idempotent?" → Exploring now
- **Comparing E1 to E2:** "Is velocity stabilizing?" → Yes, 2-3x is reproducible

**This is organizational intelligence.**

## For Anyone Building With AI

### What Actually Works

**1. Structure the collaboration**
- Don't just chat - use repeatable processes
- Design → Plan → Implement → Review
- Every time. No shortcuts.

**2. Externalize memory**
- Document patterns discovered
- Track velocity data
- Persist learnings in files
- AI has no session memory - build it externally

**3. Retrospect at every level**
- Session close (daily)
- Feature review (per feature)
- Epic close (per epic)
- Systemic review (quarterly)

**4. Welcome meta-questions**
- "Can this process be better?"
- "What pattern are we seeing?"
- Let AI question the process
- **System that questions itself improves itself**

**5. Give autonomy with boundaries**
- Let AI organize its own memory
- Trust execution within specs
- Provide course corrections when needed
- **Trust enables flow**

### What Doesn't Work

❌ **Ad-hoc prompting** - No structure = no compounding
❌ **Treating AI as tool** - Misses collaborative intelligence
❌ **Skipping retrospectives** - No learning = no improvement
❌ **Forgetting across sessions** - Context reset = starting over
❌ **Micromanaging** - Kills flow, wastes both partners' strengths
