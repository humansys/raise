## Learnings

### Technical Discoveries

1. **State machine guards prevent re-entry bugs**
   - Vision parser bug: header detection re-triggered on data rows containing "context"
   - Fix: Check `not in_outcomes_table` before entering table mode
   - Pattern applies to all state machine parsers

2. **Parser composition pattern**
   - All three parsers share structure: regex match → extract content → truncate → create concept
   - Made parallel implementation fast (Tasks 2-4 done in ~22 minutes)

3. **Integration tests high value**
   - Real governance file tests validated assumptions
   - Caught actual count: 24 concepts (vs 23 predicted in spike)
   - Proved end-to-end workflow works

### Process Insights

1. **Design-first with concrete examples = 3.5x velocity**
   - Design.md had copy-pasteable examples (CLI usage, Python API, data structures)
   - No ambiguity during implementation
   - Code matched spec exactly

2. **Spike validation de-risks implementation**
   - Had working proof-of-concept from `dev/experiments/concept_extraction_spike.py`
   - Knew patterns would work on real files
   - Gave confidence to proceed

3. **Task decomposition optimal at 7 tasks for 3 SP**
   - Not too granular, not too coarse
   - Clear dependencies enabled sequential/parallel execution

### Collaboration Notes

1. **Branch hygiene matters early**
   - Caught late that we were on `v2` instead of feature branch
   - **Learning:** Create branch FIRST (before any implementation)
   - Updated feature-implement skill mentally to include this

2. **Coverage confusion resolved**
   - Global `pyproject.toml` coverage fails when checking entire codebase
   - Module-specific coverage (91-100%) is what matters
   - **Learning:** Use `--cov=src/module --no-cov-on-fail` for module tests
