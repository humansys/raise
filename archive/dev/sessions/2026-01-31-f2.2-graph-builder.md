### Decisions Made

1. **Conservative relationship inference:** Use >3 shared keywords threshold for `related_to` to avoid false positives. Explicit references only for `governed_by` and `depends_on`.

2. **Simple BFS with visited set:** Sufficient for cycle handling. No need for complex graph algorithms.

3. **Accept Pyright false positives:** Pydantic `Field(default_factory=list)` triggers false positive in strict mode. Acceptable when Ruff passes and tests pass.

4. **Test with real data early:** Adjusted process to validate patterns/rules against real governance during Task 1, not just Task 6.

## Learnings

### Process

1. **Kata cycle velocity compounds:** Second use of design → plan → implement cycle was 2.3-3.2x faster. Process internalization reduces cognitive load and increases velocity.

2. **Conservative design wins:** Better to ship high-confidence features (few, accurate edges) than complex, noisy ones (many, questionable edges).

3. **Real data testing catches edge cases early:** Concept deduplication (24→23 nodes) and relationship pattern mismatches discovered because we tested with raise-commons governance, not just synthetic data.

### Technical

1. **BFS implementation pattern:** `visited` set + `deque` with (node, depth) tuples. Simple and performant (<100ms for 50 nodes).

2. **Pydantic JSON serialization is zero-effort:** `model_dump_json()` and `model_validate_json()` handle complex nested structures (dict of Concepts, list of Relationships) without custom serialization.

3. **Relationship inference confidence scoring:** Include `{"confidence": 0.8, "method": "keyword_match"}` metadata on edges for future filtering/ranking.

4. **Keyword extraction pattern:** Stopword filtering + length threshold (>3 chars) + lowercase normalization. Simple but effective for semantic similarity.
