# Getting Started with RaiSE

> Your first steps to reliable AI-assisted development

---

## Choose Your Path

### [Brownfield](./brownfield.md) — Existing Project

You have an existing codebase and want to adopt RaiSE incrementally.

**Best for:**
- Legacy projects needing governance
- Teams already using AI assistants
- Gradual adoption without disruption

### [Greenfield](./greenfield.md) — New Project

You're starting fresh and want RaiSE from day one.

**Best for:**
- New initiatives
- Teams wanting full methodology
- Maximum governance from the start

---

## Prerequisites

Before starting, ensure you have:

1. **Git repository** — RaiSE is Git-native
2. **AI coding assistant** — Claude Code, Cursor, Copilot, etc.
3. **Basic understanding** — Read the [Framework Overview](../README.md) first

## What You'll Set Up

Regardless of path, you'll end up with:

```
your-project/
├── .raise/              # RaiSE engine (katas, templates, gates)
├── governance/          # Approved artifacts (source of truth)
└── work/                # Work in progress (features, research)
```
