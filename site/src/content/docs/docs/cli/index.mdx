Complete reference for the `rai` command-line interface.

## Global Options

These options are available on all commands:

| Flag | Short | Description |
|------|-------|-------------|
| `--version` | `-V` | Show version and exit |
| `--format` | `-f` | Output format: `human`, `json`, or `table` |
| `--verbose` | `-v` | Increase verbosity (`-v`, `-vv`, `-vvv`) |
| `--quiet` | `-q` | Suppress non-error output |
| `--help` | | Show help and exit |

## Project

### `rai init`

Initialize a RaiSE project in the current directory. Detects project type (greenfield or brownfield), creates `.raise/manifest.yaml`, and sets up the project structure.

With `--detect`, also analyzes code conventions and generates guardrails.

| Flag | Short | Description |
|------|-------|-------------|
| `--name` | `-n` | Project name (defaults to directory name) |
| `--path` | `-p` | Project path (defaults to current directory) |
| `--detect` | `-d` | Detect conventions and generate `guardrails.md` |

```bash
# New project
rai init

# Named project
rai init --name my-api

# Existing project with convention detection
rai init --detect
```

## Session

### `rai session start`

Start a new working session. Increments the session counter and sets active session state. Checks for orphaned sessions (started but not closed) and warns if found.

With `--context`, outputs a token-optimized context bundle (~150 tokens) assembled from your developer profile, session state, and memory graph.

| Flag | Short | Description |
|------|-------|-------------|
| `--name` | `-n` | Your name (required for first-time setup) |
| `--project` | `-p` | Project path to associate with this session |
| `--agent` | | Agent identifier |
| `--context` | | Output a context bundle for AI consumption |

```bash
# First-time setup
rai session start --name "Alice" --project .

# Start session with context bundle
rai session start --project . --context

# Simple start
rai session start
```

### `rai session close`

End the current working session. With `--summary` or `--state-file`, performs a full structured close — records session, patterns, corrections, and updates state.

| Flag | Short | Description |
|------|-------|-------------|
| `--summary` | `-s` | Session summary |
| `--type` | `-t` | Session type (feature, research, maintenance, etc.) |
| `--pattern` | | Pattern description to record (repeatable) |
| `--state-file` | | YAML file with full structured session output |
| `--session` | | Session identifier |

```bash
# Simple close
rai session close

# Close with summary
rai session close --summary "Implemented auth module" --type feature

# Close with pattern learned
rai session close --summary "Refactored tests" --type maintenance \
  --pattern "Use fixtures for database setup"
```

### `rai session context`

Load specific context sections for AI consumption.

| Flag | Short | Description |
|------|-------|-------------|
| `--sections` | `-s` | Sections to include (governance, behavioral, coaching, deadlines, progress) |
| `--project` | `-p` | Project path |

```bash
rai session context --sections governance,progress --project .
```

### `rai session journal`

Incremental session memory for decisions, insights, and tasks.

```bash
# Add a decision to the journal
rai session journal add "Chose Pydantic over dataclasses" --type decision

# Show the journal
rai session journal show

# Compact view (for post-compaction restore)
rai session journal show --compact
```

## Graph

### `rai graph build`

Build the knowledge graph index from all sources: governance documents, memory (patterns, calibration, sessions), work tracking (epics, stories), skills, and components from discovery.

| Flag | Short | Description |
|------|-------|-------------|
| `--output` | `-o` | Path to save index JSON |
| `--no-diff` | | Skip diff output |

```bash
# Build index to default location
rai graph build

# Save to custom location
rai graph build --output custom_index.json
```

### `rai graph query`

Search the knowledge graph for relevant concepts. The graph contains all context sources: governance (principles, requirements, terms), memory (patterns, calibration, sessions), skills (workflow metadata), and work (epics, stories, decisions).

| Flag | Short | Description |
|------|-------|-------------|
| `--format` | `-f` | Output format: `human`, `json`, or `compact` |
| `--strategy` | `-s` | Query strategy: `keyword_search` or `concept_lookup` |
| `--types` | `-t` | Filter by types (comma-separated: `pattern`, `calibration`, `principle`, etc.) |
| `--limit` | `-l` | Maximum number of results (default: 10) |

```bash
# Search by keywords
rai graph query "planning estimation"

# Filter to patterns only
rai graph query "testing" --types pattern,calibration

# Lookup specific concept by ID
rai graph query "PAT-001" --strategy concept_lookup

# Output as JSON
rai graph query "velocity" --format json
```

### `rai graph context`

Show full architectural context for a module. Returns the module's bounded context (domain), architectural layer, applicable guardrails (constraints), and module dependencies.

| Flag | Short | Description |
|------|-------|-------------|
| `--format` | `-f` | Output format: `human` or `json` |

```bash
# Show context for memory module
rai graph context mod-memory

# JSON output
rai graph context mod-memory --format json
```

### `rai graph validate`

Validate graph index structure and relationships. Checks for cycles in `depends_on` relationships, valid relationship types, and that all edge targets exist as nodes.

```bash
rai graph validate
```

### `rai graph extract`

Extract concepts from governance markdown files.

```bash
# Extract from all governance files
rai graph extract

# Extract from specific file
rai graph extract governance/prd.md
```

### `rai graph list`

List concepts in the knowledge graph.

| Flag | Short | Description |
|------|-------|-------------|
| `--format` | `-f` | Output format: `human`, `json`, or `table` |
| `--memory-only` / `--all` | | Show only memory types or all |

```bash
# Show summary table
rai graph list

# Show only patterns/calibrations/sessions
rai graph list --memory-only
```

### `rai graph viz`

Generate an interactive HTML visualization of the knowledge graph.

```bash
rai graph viz
```

## Pattern

### `rai pattern add`

Add a new pattern to memory. Patterns capture learnings from development — process improvements, technical discoveries, architectural decisions.

| Flag | Short | Description |
|------|-------|-------------|
| `--context` | `-c` | Context keywords (comma-separated) |
| `--type` | `-t` | Pattern type: `codebase`, `process`, `architecture`, `technical` (default: `process`) |
| `--from` | `-f` | Story/session where learned |
| `--scope` | `-s` | Memory scope: `global`, `project`, `personal` (default: `project`) |

```bash
# Add a process pattern
rai pattern add "HITL before commits" -c "git,workflow"

# Add a technical pattern
rai pattern add "Use capsys for stdout tests" -t technical -c "pytest,testing"

# Add with source reference
rai pattern add "BFS reuse across modules" -t architecture --from S2.3
```

### `rai pattern reinforce`

Reinforce a pattern with a vote signal.

```bash
rai pattern reinforce PAT-042
```

### `rai pattern promote`

Promote a pattern from personal scope to project scope.

```bash
rai pattern promote PAT-042
```

## Signal

### `rai signal emit-work`

Emit a work lifecycle event for Lean flow analysis. Tracks work items (epics, stories) through normalized phases.

| Flag | Short | Description |
|------|-------|-------------|
| `--event` | `-e` | Event type: `start`, `complete`, `blocked`, `unblocked`, `abandoned` |
| `--phase` | `-p` | Phase: `design`, `plan`, `implement`, `review` |

```bash
# Story lifecycle
rai signal emit-work story S9.4 --event start --phase implement
rai signal emit-work story S9.4 -e complete -p implement
```

### `rai signal emit-session`

Emit a session event to telemetry.

| Flag | Short | Description |
|------|-------|-------------|
| `--type` | `-t` | Session type: `story`, `research`, `maintenance`, etc. |
| `--outcome` | `-o` | Session outcome: `success`, `partial`, `abandoned` |
| `--duration` | `-d` | Session duration in minutes |
| `--stories` | `-f` | Stories worked on (comma-separated) |

```bash
rai signal emit-session --type story --outcome success -d 45 --stories S9.1,S9.2
```

### `rai signal emit-calibration`

Emit a calibration event to telemetry. Records estimate vs. actual for velocity tracking.

| Flag | Short | Description |
|------|-------|-------------|
| `--size` | `-s` | T-shirt size: `XS`, `S`, `M`, `L` |
| `--estimated` | `-e` | Estimated duration in minutes |
| `--actual` | `-a` | Actual duration in minutes |

```bash
rai signal emit-calibration S9.4 --size S --estimated 30 --actual 15
```

## Discovery

### `rai discover scan`

Scan a directory and extract code symbols (classes, functions, methods, interfaces, module docstrings). Supports Python, TypeScript, and JavaScript.

| Flag | Short | Description |
|------|-------|-------------|
| `--language` | `-l` | Language: `python`, `typescript`, `javascript` |
| `--output` | `-o` | Output format: `human`, `json`, or `summary` |
| `--exclude` | `-e` | Patterns to exclude (repeatable) |

```bash
# Scan current directory
rai discover scan

# Scan Python files only
rai discover scan src/ --language python

# Exclude tests
rai discover scan . --exclude "**/test_*" --exclude "**/__tests__/**"
```

### `rai discover drift`

Check for architectural drift against baseline components.

```bash
rai discover drift
```

## Backlog

Manage backlog items via project management adapters (e.g., Jira).

### `rai backlog create`

Create a new backlog item. SUMMARY is positional, project is the `-p` flag.

```bash
rai backlog create "Add user authentication" -p MYPROJECT -t Story
```

### `rai backlog search`

Search backlog items. QUERY is positional (JQL for Jira).

```bash
rai backlog search "project = MYPROJECT AND status = 'In Progress'" -n 10
```

### `rai backlog get`

Get details for a single issue.

```bash
rai backlog get PROJ-123
```

### `rai backlog transition`

Transition an issue to a new status. Both arguments are positional.

```bash
rai backlog transition PROJ-123 "In Progress"
```

### `rai backlog comment`

Add a comment to an issue. Both arguments are positional.

```bash
rai backlog comment PROJ-123 "Implementation complete, ready for review"
```

### `rai backlog link`

Link two issues. All three arguments are positional.

```bash
rai backlog link PROJ-123 PROJ-456 blocks
```

## MCP

Manage and invoke MCP (Model Context Protocol) servers registered in `.raise/mcp/`.

### `rai mcp list`

List registered MCP servers.

```bash
rai mcp list
```

### `rai mcp health`

Check health of an MCP server. SERVER is positional.

```bash
rai mcp health my-server
```

### `rai mcp tools`

List tools available on an MCP server.

```bash
rai mcp tools my-server
```

### `rai mcp call`

Call a tool on an MCP server. SERVER and TOOL are positional.

```bash
rai mcp call my-server search --args '{"query": "test"}'
```

### `rai mcp install`

Install an MCP server package.

```bash
rai mcp install some-package --type uvx --name my-server
```

### `rai mcp scaffold`

Scaffold a new MCP server configuration.

```bash
rai mcp scaffold my-server --command "uvx some-tool"
```
